3. 캐시 메모리 (Cache Memory)

<공시순>

종류정의사례
공간 지역성(Spatial Locality)참조된 데이터인근에 있는 데이터가 곧 참조될 가능성이 높은 특성배열(Array) 순차 접근, 관련 데이터의 인접 배치
시간 지역성(Temporal Locality)최근에 참조된 데이터가가까운 미래에 다시 참조될 가능성이 높은 특성루프(Loop), 스택(Stack), 공통 변수(Counting)
순차 지역성(Sequential Locality)데이터가 메모리에 저장된순서대로 차례차례 참조되는 특성프로그램 코드의 실행 (분기/점프가 없는 경우)

<직연집>

매핑 방식특징장단점
직접 매핑 (Direct Mapped)메모리 블록이 특정 캐시 라인에만 매핑구현 간단, 충돌 미스 발생 가능
완전 연관 매핑 (Fully Associative)어느 캐시 라인에나 매핑 가능미스율 최소, 하드웨어 복잡·비용 높음
집합 연관 매핑 (Set Associative)캐시를 집합으로 나누어 집합 내 연관 매핑직접/완전 방식의 절충안, 실제 CPU 채택

[하위] 4-1. 메모리 인터리빙 (Memory Interleaving)

<상모하기>

방식설명특징
상위 인터리빙 (High-order)상위 비트로 모듈 선택, 하위 비트로 기억장소 선택 (C-Access)• 설계 단순성, 구현 용이성, 관리 효율성 • 모델 오류가 단일 모듈에만 영향 • 연속 접근 시 단일 뱅크에 집중(성능저하) • 병렬성 낮음
하위 인터리빙 (Low-order)하위 비트로 모듈 선택, 상위 비트로 기억장소 선택 (교차 배치, S-Access)• 연속 접근 시 여러 뱅크 분산 • 병렬성 높음(부하분산 효과 우수) • 회로 복잡, 설계 난이도 증가 • 모델 오류가 전체 메모리에 영향
혼합 인터리빙 (Mixed)상위·하위 방식 혼합유연한 메모리 배치 가능
