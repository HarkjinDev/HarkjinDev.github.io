4. 메모리 누수 (Memory Leak)

  • 정의: 프로그램이 할당한 메모리를 더 이상 사용하지 않음에도 해제하지 않아 가용 메모리가 점진적으로 감소하는 현상
  • 키워드: 참조 카운팅, 가비지 컬렉터, 순환 참조, 힙 메모리 고갈
  • 분류: 메모리 관리
  • 언어별 메모리 관리:
    언어메모리 관리 방식메모리 누수 대응
    C/C++수동 관리 (malloc/free, new/delete)스마트 포인터(unique_ptr, shared_ptr), RAII 패턴
    Java가비지 컬렉터(GC) 자동 관리정적 컬렉션의 순환 참조 주의, WeakReference 활용
    Python참조 카운팅 + GC순환 참조는 GC가 처리, gc.collect() 명시 호출
    Rust소유권(Ownership) 시스템컴파일 타임에 메모리 누수 방지 (GC 불필요)
  • 인사이트: 가비지 컬렉터가 있는 언어도 순환 참조나 정적 캐시 미관리로 논리적 메모리 누수가 발생할 수 있음. 장기 실행 서버에서 메모리 사용량 모니터링과 힙 덤프 분석은 필수 운영 역량임
