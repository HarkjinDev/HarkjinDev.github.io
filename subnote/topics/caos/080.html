4. 엣지 컴퓨팅 (Edge Computing)

  • 정의: 데이터 처리를 중앙 클라우드가 아닌 데이터 발생 지점(엣지)에 근접한 분산 노드에서 수행하여 지연을 최소화하는 컴퓨팅 패러다임
  • 키워드: 저지연, MEC(Multi-access Edge Computing), Fog Computing, IoT, 5G, 오프로딩, 분산처리
  • 분류: 고가용성 인프라
  • 클라우드 vs 엣지 비교:
    구분클라우드 컴퓨팅엣지 컴퓨팅
    처리 위치중앙 데이터센터데이터 발생 근처
    지연(Latency)수십~수백 ms1~10 ms
    대역폭 소비높음 (모든 데이터 전송)낮음 (현장 전처리 후 요약 전송)
    보안집중 관리 용이분산으로 공격 면적 증가
    적합 워크로드배치처리, 대용량 분석실시간 제어, 자율주행, AR/VR
  • 엣지 계층 구조:
    • 디바이스 엣지: 센서·IoT 기기 자체 처리 (TinyML)
    • 근접 엣지 (Near Edge): 게이트웨이·기지국 수준 (MEC 서버)
    • 원격 엣지 (Far Edge): 지역 소규모 DC, 클라우드 연동
  • 주요 표준 및 기술:
    • ETSI MEC: 5G 기지국 근처 엣지 서버에 컴퓨팅 배치 표준
    • Fog Computing: 엣지~클라우드 계층 간 중간 처리 계층 (Cisco 제안)
  • 인사이트: 엣지 컴퓨팅의 핵심 가치는 지연 감소 + 대역폭 절감 + 데이터 주권(Data Sovereignty) 확보이며, 5G MEC와 결합하여 자율주행·스마트팩토리·실시간 영상분석 등 초저지연 서비스를 가능하게 함을 서술.
