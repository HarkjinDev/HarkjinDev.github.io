[하위] 1-3. 프로세스 상태 전이 (Process State Transition)

  • 정의: 프로세스가 생성부터 종료까지 거치는 상태 변화 과정
  • 키워드: 생성, 준비, 실행, 대기, 종료, 디스패치, 선점, PCB
  • 분류: 프로세스 관리
  • 프로세스 상태 전이 절차:
    상태 전이전이 조건설명
    생성 → 준비프로세스 생성 완료메모리 할당 후 Ready Queue에 삽입
    준비 → 실행디스패치(Dispatch)스케줄러가 프로세스를 선택하여 CPU 할당
    실행 → 준비선점(Preemption) / 타임 퀀텀 만료높은 우선순위 프로세스가 CPU 선점
    실행 → 대기I/O 요청 / 이벤트 대기Block 상태로 전이, CPU 반납
    대기 → 준비I/O 완료 / 이벤트 발생Ready Queue로 복귀
    실행 → 종료정상/강제 종료자원 반납, PCB 제거
  • 인사이트: 디스패치는 스케줄러가 선택한 프로세스에 CPU를 할당하는 행위이며, 컨텍스트 스위칭(현재 프로세스 상태 저장 → 새 프로세스 상태 복원) 비용이 수반됨
