2. 스레드 (Thread)

정의: 프로세스 내에서 실행되는 경량 실행 단위. Code/Data/Heap 영역은 프로세스와 공유하고, Stack만 독립적으로 보유
키워드: TCB, 경량 프로세스, 멀티스레드, 1:1/N:1/M:N 모델, 동기화 필요
분류: 프로세스 관리
멀티스레딩 모델
1:1(커널 수준, 병렬성 우수), N:1(사용자 수준, 병렬성 낮음), M:N(혼합, 유연성 우수)
멀티스레딩 종류: <인블동칩>
모델
스레드 설명
Interleaved Multithreading
매 클록 사이클마다 스레드를 교체하여 실행하는 미세 단위(Fine-grain) 방식
Blocked Multithreading
캐시 미스 등 지연 발생 시에만 스레드를 교체하는 거대 단위(Coarse-grain) 방식
Simultaneous Multithreading (SMT)
슈퍼스칼라 구조를 활용해 여러 스레드의 명령어를 동시에 발송(Issue)하는 방식
Chip Multithreading
하나의 칩 안에 여러 개의 프로세서를 집적하여 스레드를 할당하는 형태
스레드 안전성(Thread Safety): 여러 스레드가 동시에 공유 자원에 접근해도 올바른 결과를 보장하는 코드 특성
인사이트: 스레드는 생성 비용이 낙고 컨텍스트 스위칭이 빠르지만, 오류가 전체 프로세스에 영향을 줄 수 있으므로 장애 격리가 중요한 경우 멀티프로세스 구조가 더 안전함
