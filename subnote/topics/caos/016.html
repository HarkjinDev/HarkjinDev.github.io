1. 프로세스 (Process)

  • 정의: 실행 중인 프로그램의 인스턴스. 독립적인 메모리 공간(Code/Data/Heap/Stack)과 PCB(Process Control Block)를 보유하는 실행 단위
  • 키워드: PCB, 메모리 격리, 컨텍스트 스위칭, IPC, 프로세스 생성/종료
  • 분류: 프로세스 관리
  • 프로세스와 스레드 비교:
    구분프로세스 (Process)스레드 (Thread)
    메모리독립적 주소 공간 (Code/Data/Heap/Stack)Code/Data/Heap 공유, Stack만 독립
    컨텍스트 스위칭비용 높음 (메모리 맵 전환)비용 낮음 (같은 프로세스 내)
    통신 방식IPC (파이프, 소켓, 공유메모리 등)공유 메모리 직접 접근
    장애 격리하나가 죽어도 다른 프로세스 무영향하나의 스레드 오류가 전체 프로세스에 영향
    생성 비용높음낮음
  • 인사이트: 멀티스레딩은 응답성 향상에 유리하지만 공유 자원 접근 시 동기화가 필수이며, Lock Contention이 성능 병목이 될 수 있음
