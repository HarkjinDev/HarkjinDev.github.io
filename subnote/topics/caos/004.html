[하위] 1-1. 폰 노이만 아키텍처 (Von Neumann Architecture)

  • 정의: 프로그램과 데이터를 동일한 메모리에 저장하고 CPU가 명령어를 순차적으로 인출·실행하는 컴퓨터 구조 모델
  • 키워드: SISD(Single Instruction Single Data), 폰 노이만 병목(Memory Wall), 하나의 명령
  • 분류: 컴퓨터 구조
  • 폰 노이만 병목 (Von Neumann Bottleneck): CPU와 메모리 간 단일 버스로 명령어와 데이터 전송이 순차 처리되어 발생하는 성능 병목
  • 폰 노이만 vs 하버드 아키텍처:
    구분폰 노이만 (Von Neumann)하버드 (Harvard)
    메모리 구조명령어·데이터 공유 메모리명령어·데이터 분리 메모리
    버스 구조단일 공유 버스분리된 명령어 버스 / 데이터 버스
    속도병목 발생 가능명령어·데이터 동시 접근 → 고속
    연산 구조CISC(가변)RISC(고정)
    적용 사례범용 CPU (x86, ARM)마이크로컨트롤러, DSP, L1 캐시 구조
  • 인사이트: 현대 CPU는 폰 노이만 기반이지만 L1 캐시를 명령어/데이터 캐시로 분리(수정 하버드 구조)하여 병목을 완화. 뉴로모픽 칩은 메모리-연산 통합으로 폰 노이만 병목을 근본적으로 극복하려는 접근임
