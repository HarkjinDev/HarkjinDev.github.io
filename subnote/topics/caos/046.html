3. 스레싱 (Thrashing)

  • 정의: 과도한 페이지 교체로 인해 실제 프로세스 실행보다 페이지 교환에 더 많은 시간을 소비하여 CPU 이용률이 급격히 저하되는 현상
  • 키워드: Page Fault 과다, 워킹셋(Working Set), PFF(Page Fault Frequency), 벨라디 변이
  • 분류: 메모리 관리
  • 발생 원인:
    발생 원인세부 원인설명
    리소스 부족저사양 CPUCPU 코어 수 및 연산 속도 리소스 부족
    메모리 용량 부족메모리 부족으로 인한 가상메모리 스왑 발생 증가
    부적절한 페이지 교체 정책Locality 미고려구역성(Locality)을 고려하지 않아 교체 효율성 저하
    페이지 빈도 미고려잦은 페이지 교체로 인한 심각한 지연 시간 발생
    과도한 멀티프로그래밍다중 프로세스 수용 초과멀티프로그래밍 정도 과다로 잦은 페이지 부재 발생
    페이지 교체 수용 초과수용 범위를 넘어서는 페이지 요구로 부재 빈도 급증
  • 해결 방법:
    방법설명핵심
    워킹셋 (Working Set)참조 페이지 집합에 충분한 프레임 보장페이지 집합 보장
    PFF (Page Fault Frequency)페이지 폴트 빈도로 프레임 수(상한/하한) 조절빈도 기반 조절
    다중 프로그래밍 정도 조절동시 프로세스 수 줄여 프레임 확보프로세스 수 감소
  • 밸라디 변이(Belady’s Anomaly):
    • 프로세스에 페이지 프레임을 더 할당하였음에도 페이지 부재율이 증가하는 현상
    • 원인: Locality를 고려하지 않은 FIFO의 한계로 인한 page fault 증가 발생
    • 영향: 페이지 부재율 증가로 인한 성능 저하 및 스레싱 발생
  • 인사이트: 스레싱은 단순히 메모리를 추가하는 것으로 해결되지 않으며, 프로세스별 워킹셋 크기를 분석하여 적절한 메모리 할당이 근본적 해결책. 현대 시스템에서는 OOM(Out-Of-Memory) Killer가 스레싱 상황에서 일부 프로세스를 강제 종료함
