2. 페이지 교체 알고리즘 (Page Replacement Algorithm)

  • 정의: 메모리 부족 시 교체할 페이지를 선택하여 스왑 공간으로 내보내는 알고리즘
  • 키워드: FIFO, LRU, LFU, 최적(Optimal), 클럭(Clock), 페이지 폴트, 벨라디 이상
  • 분류: 메모리 관리
  • 페이지 교체 알고리즘:
    알고리즘기준특징비고
    FIFO가장 먼저 들어온 페이지구현 간단벨라디 이상(Belady’s Anomaly) 발생 가능
    LRU가장 오래 전에 사용된 페이지시간적 지역성 활용, 성능 우수구현 비용 높음 (타임스탬프 필요)
    LFU사용 빈도가 가장 낮은 페이지빈도 기반초기 페이지에 불리, 최근 패턴 반영 어려움
    최적 (OPT)앞으로 가장 오래 사용되지 않을 페이지이론적 최적미래 예측 불가 → 성능 비교 기준(Benchmark)으로만 활용
    클럭 (Clock/NRU)참조 비트 기반 LRU 근사LRU 근사, 구현 효율적실제 OS에서 가장 많이 채택
  • 벨라디 이상(Belady’s Anomaly): FIFO에서 프레임 수를 늘려도 페이지 폴트가 증가하는 현상
  • 인사이트: Linux의 페이지 교체는 Active/Inactive 리스트 기반의 2-pass LRU 변형을 채택함
