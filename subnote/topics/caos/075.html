[하위] 1-2. Watchdog Timer (WDT)

  • 정의: CPU 응답이 일정시간 없으면 자동으로 리셋을 수행하는 감시타이머
  • 키워드: Clock, Kick, Timeout, Reset, NMI, Internal/External/Window WDT, Fault Tolerant
  • 분류: 고가용성 인프라
  • 구성요소·유형:
    구분세부설명
    구성요소ClockWDT와 CPU가 동작하는 기준시간
    KickWatchdog Timer를 일정시간 간격으로 Wakeup
    Timeout허용되는 최대 무응답 시간
    ResetTimeout까지 Kick 미수신 시 CPU Reset 수행
    유형Internal WDTProcessor 내부 내장 타이머, 구성 간단
    External WDT별도 외부Device로 구성, CPU전체 전원제어 가능해 신뢰성 높음
    Window WDT특정 시간윈도우 내에서만 유효Kick 허용, 윈도우 위반 시 Reset
  • 활용 분야:
    구분활용핵심
    구조Fault Tolerant Architecture하드웨어 안전장치
    임베디드 시스템구조자율 오류복구
    인터럽트·타이머제어NMI 강제인터럽트
    운영체제시스템복구·예외처리자동 복구·재부팅
    교착상태(Deadlock) 대응교착상태 강제해제
    실시간운영체제(RTOS)데드라인 감시
  • 인사이트: WDT는 하드웨어 타이머 기반의 최후 안전장치로, 임베디드·RTOS 환경에서 무인·실시간 시스템의 자율 복구를 보장하는 핵심 신뢰성 기술
