5. 세그멘테이션 오류 (Segmentation Fault)

  • 정의: 프로세스가 허용되지 않은 메모리 영역에 접근할 때 발생하는 런타임 오류 (SIGSEGV 신호)
  • 키워드: Core dumped, 잘못된 포인터 참조, 배열 인덱스 오류, NULL 포인터, 스택 오버플로우
  • 분류: 메모리 관리
  • 발생 원인 및 해결방안:
    측면발생 원인해결 방안
    프로그래밍포인터 초기화 누락, NULL 역참조, 배열 범위 초과 접근, Use-After-Free포인터 초기화 및 NULL 체크, 배열 경계 검증, 메모리 해제 후 포인터 NULL 설정
    메모리 관리메모리 누수, 이중 해제(Double Free), 미사용 동적 메모리 접근할당·해제 짝 맞추기, 스마트 포인터(C++) 사용, 해제 후 참조 금지
    운영체제/보안권한 없는 영역 접근, 스택/힙 경계 초과, 코드 실행 금지(NX) 영역 접근접근 제어 정책 강화, ASLR, NX bit, Stack Guard 등 보호 기법 활용
    디버깅/도구원인 파악 어려움, 숨겨진 포인터 오류, 실행 중 메모리 충돌Valgrind/ASan 동적 분석, gdb 위치 추적, 정적 분석 도구(cppcheck) 활용
  • 인사이트: Segmentation Fault는 메모리 보호 메커니즘이 제대로 동작한다는 신호이며, 오류 발생 즉시 프로세스를 종료하여 추가 피해를 막음. 보안 관점에서 Stack Canary, ASLR이 Segfault를 이용한 공격을 방어함
