[하위] 5-3. 스핀 락 (Spin Lock)

정의: 임계영역 진입 불가 시 루프(spinning)를 돌며 대기하는 동기화 기법
키워드: Read/Write Spin Lock, IRQL, 바쁜 대기(Busy Waiting), 인터럽트 비활성화, 멀티코어
분류: 프로세스 관리
특징:
CPU를 점유한 채 루프 대기 → 컨텍스트 스위칭 없음
임계영역이 매우 짧을 때 효율적 (컨텍스트 스위칭 비용 > 스핀 비용)
멀티코어 환경에서 유용, 단일 코어에서는 비효율
동작원리:
단계
절차
설명
1
Free spinlock 획득
CPU A가 가용 스핀락을 획득하고 공유 자원 사용 시작
2
자원 요청 Spinning
CPU B가 접근을 시도하나, 락 획득까지 무한 루프를 돌며 대기
3
Spinlock 획득
CPU A의 락 해제 후, 대기하던 CPU B가 스핀락을 획득하여 자원 사용
4
Release spin lock
CPU B의 작업 종료 후 스핀락을 해제하여 반납
인사이트: 대기 시간이 길어지면 CPU 낭비가 심각해지므로 커널 레벨의 짧은 임계영역 보호에만 사용해야 함
