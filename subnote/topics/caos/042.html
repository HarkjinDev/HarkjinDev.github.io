1. 가상 메모리 (Virtual Memory)

  • 정의: 주기억장치의 한계 극복하기 위해, 보조기억장치(디스크)를 주기억장치처럼 활용하는 메모리 확장 기법
  • 키워드: 페이징, 세그멘테이션, 구역성(Locality), 페이지 폴트, 스왑, MMU, TLB
  • 분류: 메모리 관리
  • 동작 원리: 가상 주소 공간을 물리 메모리와 분리 → MMU(Memory Management Unit)가 가상→물리 주소 변환 → 필요한 페이지만 메모리에 적재
  • 페이지 폴트(Page Fault) 처리: CPU가 가상 주소 참조 → 페이지가 메모리에 없음 탐지 → OS가 트랩 처리 → 디스크에서 메모리로 적재 → 페이지 테이블 갱신 → 명령어 재실행

  • MMU (Memory Management Unit): 가상 주소를 물리 주소로 변환하는 하드웨어 장치
    • 실제 메모리와 가상 메모리의 주소 변환 역할 담당
    • 또한 메모리 보호, 캐시 관리, 버스 중재 등의 역할을 담당
    • TLB(Translation Lookaside Buffer)를 내장하여 인기 엔트리를 캐싱해 변환 속도 향상
  • TLB (Translation Lookaside Buffer): 자주 사용되는 페이지 테이블 엔트리를 캐싱하여 주소 변환 속도 향상
  • 인사이트: 가상 메모리는 프로세스 간 메모리 격리/보호의 기반 기술이며, TLB 미스율이 성능에 큰 영향을 미치므로 메모리 접근 패턴 최적화가 중요함
