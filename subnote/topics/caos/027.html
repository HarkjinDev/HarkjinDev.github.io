[하위] 5-1. 세마포어 (Semaphore)

정의: P/V 연산으로 공유 자원의 접근을 제어하는 프로세스 동기화 도구
키워드: P(Wait, 진입 전 S–), V(Signal, 사용 후 S++), 이진, 카운팅, S>0(자원 사용 가능), S≤0(대기)
분류: 프로세스 관리
세마포어 종류:
종류
값 범위
용도
이진 세마포어 (Binary)
0 또는 1
상호배제 (뮤텍스와 유사)
카운팅 세마포어 (Count)
0 이상의 정수
자원 개수 관리 (N개 자원에 N개 프로세스 허용)
세마포어 연산:
연산
동작
의사코드
P 연산 (Wait/Down)
S > 0이면 S-- (자원 사용). S = 0이면 프로세스 대기 큐에 삽입후 블록
if S > 0: S--; else: wait();
V 연산 (Signal/Up)
S++ (자원 반납). 대기 중인 프로세스 있으면 깨움
S++; if (waiting): wakeup();
S 값 해석:
S > 0: 사용 가능한 자원 수
S = 0: 모든 자원 사용 중 (대기 필요)
S < 0: |S|는 현재 대기 중인 프로세스 수 (카운팅 세마포어)
인사이트: 세마포어는 강력하지만 P/V 연산 누락 시 교착 상태나 무한 대기가 발생할 수 있음. 이를 보완한 고수준 추상화가 모니터임
