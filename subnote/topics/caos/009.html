3. 캐시 메모리 (Cache Memory)

  • 정의: CPU와 주기억장치 사이에서 데이터를 임시 저장하여 접근 속도를 높이는 고속 메모리
  • 키워드: Write Through, Write Back, 캐시 적중률, 지역성(Locality), 직접/연관/집합연관 매핑
  • 분류: 컴퓨터 구조
  • 지역성(Locality) 원리: <공시순>
    종류정의사례
    공간 지역성 (Spatial Locality)참조된 데이터 인근에 있는 데이터가 곧 참조될 가능성이 높은 특성배열(Array) 순차 접근, 관련 데이터의 인접 배치
    시간 지역성 (Temporal Locality)최근에 참조된 데이터가 가까운 미래에 다시 참조될 가능성이 높은 특성루프(Loop), 스택(Stack), 공통 변수(Counting)
    순차 지역성 (Sequential Locality)데이터가 메모리에 저장된 순서대로 차례차례 참조되는 특성프로그램 코드의 실행 (분기/점프가 없는 경우)
  • 캐시 매핑 방식: <직연집> <태라단 / 태단 / 태세단>
    매핑 방식특징장단점
    직접 매핑 (Direct Mapped)메모리 블록이 특정 캐시 라인에만 매핑구현 간단, 충돌 미스 발생 가능
    완전 연관 매핑 (Fully Associative)어느 캐시 라인에나 매핑 가능미스율 최소, 하드웨어 복잡·비용 높음
    집합 연관 매핑 (Set Associative)캐시를 집합으로 나누어 집합 내 연관 매핑직접/완전 방식의 절충안, 실제 CPU 채택
  • 캐시 교체 알고리즘:
    교체정책내용특징
    LFU참조 횟수(Frequency)가 가장 적은 라인을 교체장기적 데이터 패턴 반영, 최근 적재된 데이터가 즉시 교체될 위험
    FIFO캐시에 가장 먼저 적재된 순서대로 교체구현 최단순, Belady's Anomaly(프레임 증가 시 성능 저하) 발생 가능
    Optimal향후 가장 오랫동안 참조되지 않을 라인을 교체이론적 최적값(Oracle), 실질적 구현 불가, 성능 비교 지표로 활용
    Random교체 대상을 임의로(Randomly) 선정프로세서 오버헤드 최소, 성능 예측 불가능, 저비용 하드웨어용
    MRU가장 최근에 사용된 데이터를 우선적으로 교체순환 참조(Cyclic Access)가 빈번한 특수 워크로드에서 성능 우위
    LRU가장 오랫동안 사용되지 않은 라인을 교체시간 지역성 기반, 적중률 우수, 구현 복잡도 높음
    TLRU (Time awareness)데이터의 남은 수명(Time to use)을 계산하여 교체실시간/임베디드 시스템, 유효 기간이 있는 데이터 관리에 특화
    Tree-PLRU (Pseudo-LRU)이진 트리(Binary Tree) 구조의 1비트 플래그를 따라가며 교체 대상 선정4-way, 8-way 등 세트 크기가 큰 캐시에 효율적, Intel/Apple 프로세서 채택
    bit-PLRU (Pseudo-LRU)각 라인에 MRU-bit를 부여, 액세스 시 1로 설정하고 0인 라인을 우선 교체하드웨어 구조 매우 단순, 고속 캐시에서 LRU 근사치 구현 시 유리
  • Cache Clean: Cache의 모든 Data를 Memory Store 후, V bit를 clear하는 기법
  • Cache Flush: Cache의 모든 Data를 Memory Store 하지 않고 V bit를 Clear하는 기법
  • 인사이트: 캐시 적중률이 1% 하락해도 전체 성능이 수십% 저하될 수 있음. L1/L2/L3 계층별 크기와 속도의 트레이드오프를 이해하는 것이 중요함
