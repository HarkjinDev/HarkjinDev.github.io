[하위] 5-2. 모니터 (Monitor)

정의: 상호배제를 자동으로 보장하고 조건 변수(Condition Variable)로 프로세스를 동기화하는 고수준 언어 구조체
키워드: 상호배제 자동 보장, 조건변수, 임계영역, 베타동기 큐, 조건동기 큐, Wait/Signal, Java synchronized
분류: 프로세스 관리
필요성: 타이밍 문제 해결, 언어 수준 편의성
기술요소:
구분
기술요소
설명
동기화 요소
임계영역
프로세스와 스레드가 접근하는 공유 자원 영역
배타동기 큐
한 번에 하나의 스레드만 임계영역에 접근하도록 제어하는 큐
조건동기 큐
특정 조건 충족 시까지 스레드를 대기시키거나 깨우는 큐
언어 요소 (JAVA 사례)
synchronized
스레드를 배타동기 큐로 진입시키는 동기화 키워드
wait()
스레드를 조건동기 큐로 이동시켜 일시 중단(잠금) 상태로 전환
notify(), notifyAll()
조건동기 큐의 스레드를 깨워 임계영역 재진입 기회 부여
인사이트: Java의
synchronized
메서드와
Object.wait()/notify()
가 모니터 패턴의 대표 구현체이며, 현대 언어에서 멀티스레딩의 표준 동기화 방식으로 자리잡음
