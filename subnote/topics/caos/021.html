3. 교착상태(Mutual Exclusion)

정의: 특정 프로세스가 공유자원을 사용하고 있을 경우 다른 프로세스가 해당 공유자원을 사용하지 못하게 제어하는 기법
키워드: race condtion, 임계영역, 교착상태, 기아상태, Test&Set 기법, Swap 기법, 인터럽트 금지, 데커 알고리즘, 피터슨 알고리즘, 램포트 알고리즘
분류: 프로세스 관리
개념도:
요구조건: <상진한>
구분
설명
관련사항
상호배제 조건
두 개 이상의 프로세스가 동시에 임계영역에 존재 불가
임계영역
진행 조건
임계영역 밖의 프로세스가 다른 프로세스의 진입을 방해 금지
교착상태
한계대기 조건
임계영역 진입이 무한정 연기되지 않도록 보장
기아상태
상대속도 조건
프로세스 간 상대적인 실행 속도에 대한 가정을 배제
기아상태
해결방안: <테스인/데피램>
구분
동기화 방안
설명
HW
Test&Set 기법
임계영역 진입 시 원자적으로 Lock을 설정하고 종료 후 해제하는 방식
Swap 기법
하드웨어적으로 두 메모리 워드의 내용을 원자적으로 교체하여 상호배제 구현
인터럽트 금지
임계영역 진입 전 인터럽트를 차단하여 공유 변수의 동시 변경을 방지
SW
데커(Dekker) 알고리즘
Flag와 Turn 변수를 활용하여 두 프로세스 간 상호배제를 보장하는 기법
피터슨(Peterson) 알고리즘
데커와 유사하나 타 프로세스에 진입 기회를 양보하는 로직이 추가된 방식
램포트(Lamport) 알고리즘
분산 시스템 환경에서 순번(Number)을 부여하여 상호배제를 구현하는 방식
세마포어(Semaphore)
P(Wait) 및 V(Signal) 연산을 이용한 가용 자원 기반의 동기화 기법
뮤텍스(Mutex)
공유 자원 독점 사용을 위해 Lock/Unlock 상태를 활용하는 스레드 제어 기법
모니터(Monitor)
공유 자원과 제어 메소드를 하나로 묶어 관리하는 고수준 추상 데이터 타입
