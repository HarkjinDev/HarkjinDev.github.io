1. CPU 스케줄링 (CPU Scheduling)

  • 정의: 준비 상태의 프로세스들에게 CPU를 효율적으로 할당하는 운영체제 기법
  • 키워드: 선점/비선점, RR, SRT, MLQ, MLFQ, FCFS, SJF, HRN, 기아현상, Convey Effect, 에이징
  • 분류: 프로세스 관리
  • 평가기준:
    구분요건설명
    성능처리량(Throughput)단위 시간당 처리하는 프로세스의 개수
    CPU 사용률(CPU Utilization)전체 시스템 가동 시간 중 CPU가 실제 작업한 시간의 비율
    공정성 보장모든 사용자에게 공평하게 CPU 및 I/O 시간 할당
    시간응답시간(Response time)요청 후 첫 번째 응답이 나올 때까지의 시간
    반환시간(Turnaround Time)프로세스 시작부터 종료까지 소요된 전체 시간
    대기시간(Waiting time)프로세스가 준비 큐(Ready Queue)에서 대기한 시간의 합
  • 선점/비선점:
    구분선점(Preemptive)비선점(Non-Preemptive)
    정의인터럽트를 통해 실행 중인 작업을 중단시키고 CPU 강제 할당실행 시작 시 프로세스가 자원을 반납할 때까지 CPU 독점
    장점 • 빠른 응답 시간 • 대화식 시분할 시스템에 적합 • 문맥 교환 오버헤드가 적음 • 응답 시간 예측이 용이
    단점잦은 문맥 교환(Context Switching)으로 인한 오버헤드 발생긴 작업이 CPU를 점유할 경우 짧은 작업들의 대기 시간 증가
    알고리즘RR, SRT, MLQ, MLFQ, RM, EDFFCFS, SJF, HRN, 우선순위, 기한부
  • 비선점 기법:
    기법핵심장점단점
    FCFS 도착 순서대로 처리구현 단순·공평성 보장호위 효과·평균 대기시간 김
    SJF 실행 시간 짧은 순서 처리평균 대기시간 최소화Starvation·실행시간 예측 어려움
    HRN 응답비율 높은 순서 처리 [(대기+실행)/실행]SJF 기아 문제 보완응답비율 계산 오버헤드
    Priority우선순위 높은 순서 처리중요 프로세스 우선 처리기아(Starvation) 발생 가능
    Deadline마감 시한 짧은 태스트 실행실시간 처리 기한 보장정확한 실행 시간 예측 필요
  • 선점 기법:
    기법핵심장점단점
    RR 시간 할당량(퀀텀)마다 순환 처리응답시간 균등·기아 없음퀀텀 크기에 따라 성능 편차
    SRTF남은 실행시간 짧은 순서 선점평균 대기시간 최소화기아·잦은 문맥 교환 비용
    우선순위 (선점)높은 우선순위 프로세스가 즉시 선점긴급 프로세스 즉각 처리기아(Starvation)·오버헤드
    MQ유형별 큐 분리·각 큐에 다른 정책유형별 차등 정책 적용큐 간 이동 불가·유연성 낮음
    MFQ실행 시간에 따라 큐 간 이동·우선순위 동적 조정기아 방지·적응적 처리구현 복잡·파라미터 설정 어려움
  • 기아(Starvation) 해결: 에이징(Aging) — 오래 대기한 프로세스의 우선순위를 점진적으로 높임
  • 인사이트: CPU 스케줄링은 시스템의 처리량과 응답 시간 사이의 상반관계를 조율하는 핵심 메커니즘이며, 기아 현상 방지를 위한 에이징 기법과 컨텍스트 스위칭 오버헤드 최소화를 통해 자원 가동률을 극대화하는 설계 역량이 중요함.
