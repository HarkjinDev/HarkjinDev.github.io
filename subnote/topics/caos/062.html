7. PNM (Processing Near Memory)

정의: 메모리 소자 외부의 근접 위치(메모리 컨트롤러, 버퍼 칩 등)에 연산 유닛을 배치하여 CPU-메모리 간 데이터 이동 비용을 절감하는 메모리 중심 연산 아키텍처
키워드: Near-Memory Computing, 메모리 컨트롤러, 데이터 이동 최소화, 메모리 대역폭, HBM, HMC
분류: AI 연산 하드웨어
등장 배경:
배경
내용
폰 노이만 병목
CPU와 메모리 사이의 데이터 이동이 성능·전력의 주요 병목으로 작용
메모리 대역폭 한계
AI/빅데이터 워크로드의 데이터 집약성 증가로 기존 버스 대역폭 부족
전력 효율 요구
데이터 이동 에너지가 연산 에너지보다 수십 배 큰 문제 대두
기술/구성요소:
항목
설명
CXL 인터페이스
CPU와 메모리 간 대역폭을 획기적으로 늘리는 인터페이스 기술
DRAM
각 비트 정보를 커패시터에 저장하는 임의 접근 기억 장치
Memory Die
회로가 집적된 반도체 물질의 작은 사각형 조각
연산기
산술 및 논리 연산을 수행하는 디지털 회로
장단점:
구분
내용
장점
기존 DRAM 제조 공정과 호환 가능, 설계 유연성 높음 메모리 컨트롤러 교체만으로 기존 시스템에 적용 가능
단점
연산 유닛이 메모리 셀과 물리적으로 분리되어 PIM 대비 대역폭 이득 제한 버퍼 다이 면적·발열 증가로 패키지 복잡도 상승
인사이트: PNM은 기존 메모리 공정과의 호환성을 유지하면서 데이터 이동 비용을 줄이는 현실적 접근으로, 상용 HBM 기반 AI 가속기(예: Samsung HBM-PIM)에 적용되고 있음을 답안에서 강조할 것.
