[하위] 5-3. 모니터 (Monitor)

  • 정의: 상호배제를 자동으로 보장하고 조건 변수(Condition Variable)로 프로세스를 동기화하는 고수준 언어 구조체
  • 키워드: 상호배제 자동 보장, 조건변수, 임계영역, 베타동기 큐, 조건동기 큐, Wait/Signal, Java synchronized
  • 분류: 프로세스 관리
  • 필요성: 타이밍 문제 해결, 언어 수준 편의성
  • 특징: 캡슐화, 자동 상호배제, 조건변수, 고수준 동기화
  • 모니터(Monitor) 구성요소:
    구성요소설명핵심
    지역변수모니터 내부에서만 접근 가능외부 직접접근 차단
    조건변수동기화의 부수적 메커니즘 제공프로세스 대기·재개
    cwait호출 프로세스를 조건 c에서 일시 중지조건 대기
    csignalcwait에 의해 중지된 프로세스 재개대기 프로세스 깨움
    Next Queue다른 프로세스가 모니터 내부에 있어서 잠시 대기하는 큐모니터 진입 대기
  • 기술요소:
    구분기술요소설명
    동기화 요소임계영역프로세스와 스레드가 접근하는 공유 자원 영역
    배타동기 큐한 번에 하나의 스레드만 임계영역에 접근하도록 제어하는 큐
    조건동기 큐특정 조건 충족 시까지 스레드를 대기시키거나 깨우는 큐
    언어 요소 (JAVA 사례)synchronized스레드를 배타동기 큐로 진입시키는 동기화 키워드
    wait()스레드를 조건동기 큐로 이동시켜 일시 중단(잠금) 상태로 전환
    notify(), notifyAll()조건동기 큐의 스레드를 깨워 임계영역 재진입 기회 부여
  • 인사이트: Java의 synchronized 메서드와 Object.wait()/notify()가 모니터 패턴의 대표 구현체이며, 현대 언어에서 멀티스레딩의 표준 동기화 방식으로 자리잡음
