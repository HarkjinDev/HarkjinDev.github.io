[하위] 3-2. MESI 프로토콜 (MESI Protocol)

  • 정의: 멀티프로세서에서 각 CPU 캐시의 블록 상태를 4가지로 관리하여 캐시 일관성을 보장하는 프로토콜
  • 키워드: Modified, Exclusive, Shared, Invalid, 스누피 버스, 디렉토리 기반, False Sharing, Padding
  • 분류: 컴퓨터 구조
  • MESI 상태:
    상태설명메모리 일치타 캐시 보유
    Modified(수정)해당 캐시만 수정·메모리 불일치 (Dirty)XX
    Exclusive(베타)해당 캐시만 보유·메모리 일치OX
    Shared(공유)여러 캐시 공유·메모리 일치OO
    Invalid(무효)타 캐시 수정으로 블록 무효화--
  • 상태 전이:
    현재 상태동작 (Action)전이 후 상태핵심 설명
    M, E, S읽기 (Read Hit)현 상태 유지이미 가진 데이터를 그대로 사용
    I읽기 (Read Miss)E 또는 S남들이 안 갖고 있으면 E, 갖고 있으면 S로 다시 읽어옴
    E쓰기 (Write Hit)M나만 갖고 있으므로 즉시 수정
    S쓰기 (Write Hit)M내 데이터를 수정하고, 남들의 복사본은 무효화(I) 시킴
    I쓰기 (Write Miss)M데이터를 새로 읽어와서 수정하고, 남들은 무효화(I) 시킴
  • False Sharing: 다른 변수가 같은 캐시 라인에 있어 불필요한 일관성 트래픽 발생 → 패딩(Padding)으로 완화
  • 인사이트: MESI는 스누피 프로토콜의 표준이며, AMD MOESI/Intel MESIF등으로 확장됨. False Sharing은 멀티코어 프로그래밍 성능 튜닝의 가장 합정 이슈임
