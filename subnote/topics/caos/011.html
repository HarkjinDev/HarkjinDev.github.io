[하위] 3-1. 캐시 일관성 (Cache Coherence)

  • 정의: 멀티프로세서 환경에서 각 프로세서의 캐시에 있는 데이터가 동일한 값을 유지하도록 보장하는 기법
  • 키워드: 스누피 프로토콜, 디렉토리 프로토콜, MESI 프로토콜, False Sharing, 패딩
  • 분류: 컴퓨터 구조
  • 일관성 유지를 위한 캐시 쓰기 정책:
    정책설명특징
    Write Through (즉시 쓰기)캐시와 메모리에 동시 쓰기일관성 보장, 메모리 트래픽 증가
    Write Back (나중 쓰기)캐시에만 쓰고, 교체 시 메모리에 반영성능 우수, 일관성 문제 발생 가능
    Write-Around쓰기 연산 시, 메모리에만 기록캐시오염 방지, 캐시 적중률 저하
  • 일관성 유지 방법:
    구분해결방법세부 항목요약 내용
    SW공유 캐시 사용-단일 캐시 공유 (충돌 심함)
    공유변수 캐시 미사용-공유변수 미저장 (성능 저하)
    HW버스감시 메커니즘Snoopy Controller메모리 동작 감시 및 상태 제어
    쓰기갱신 (Write Update)갱신 시 모든 캐시에 정보 전송
    쓰기무효 (Write Invalid)독점 쓰기 (타 캐시 데이터 무효화)
    디렉토리 기반Full Map Directory주기억장치에 전체 상태 중앙 관리
    Limit DirectoryFull Map 크기를 작게 제한
    Chained DirectoryLinked List 형태의 분산 관리
    프로토콜MESI 프로토콜수정, 베타, 공유, 무효
    MEI, MSI, MOESI, MESIFOwned, Forwarding
  • 인사이트: False Sharing(다른 변수가 같은 캐시 라인에 있어 불필요한 일관성 트래픽 발생) 문제는 패딩(Padding)으로 완화 가능하며, 멀티코어 프로그래밍 성능 튜닝의 핵심 이슈임
