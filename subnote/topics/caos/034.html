7. IPC (Inter-Process Communication)

  • 정의: 프로세스 간 데이터 교환과 동기화를 위한 통신 메커니즘
  • 키워드: 파이프, 공유메모리, 메시지큐, 메모리맵, 소켓, RPC, FIFO
  • 분류: 프로세스 관리
  • IPC(프로세스 간 통신) 유형:
    구분기법설명핵심
    파일 기반파일 공유 파일 읽기·쓰기로 프로세스 간 데이터 교환영속성·느림
    FIFO 이름 있는 파이프·비관련 프로세스 간 통신단방향·영속
    Pipe 부모·자식 프로세스 간 단방향 익명 통신단방향·임시
    메모리 기반공유 메모리 동일 메모리 영역 공유·가장 빠른 IPC동기화 필수
    메시지 큐 커널 큐에 메시지 저장·비동기 송수신비동기·순서 보장
    메모리 맵 파일을 메모리에 매핑·파일처럼 접근대용량 데이터
    네트워크 기반소켓 IP·Port 기반 프로세스 간 양방향 통신원격·범용
    RPC 원격 함수 호출·네트워크 투명성 제공분산 시스템
  • 인사이트: 공유 메모리는 가장 빠른 IPC 방식이지만 동기화 없이 사용하면 Race Condition이 발생하므로 세마포어/뮤텍스와 반드시 함께 사용해야 함. 마이크로서비스 아키텍처에서는 소켓/RPC 기반 IPC가 표준
