[하위] 1-1. 기아현상(Starvation Effect) / 호위현상(Convey Effect)

  • 기아현상: 낮은 우선순위 프로세스가 자원을 할당받지 못하고 무한정 대기하는 현상
    구분세부 항목설명
    원인우선순위 스케줄링고정 우선순위 환경에서 하위 프로세스의 실행 기회가 박탈됨
    자원 경합 및 점유특정 프로세스에만 자원이 집중되거나 비선점형으로 자원을 독점함
    실행 시간 편차긴 작업 우선 처리 시 실행 시간이 짧은 프로세스들이 장시간 대기함
    대응방안에이징(Aging)대기 시간에 비례하여 프로세스의 우선순위를 점진적으로 높이는 기법
    타임 슬라이스 제한특정 프로세스의 독점을 방지하기 위해 CPU 사용 시간을 강제로 제한
  • 호위현상: 긴 작업의 CPU 독점으로 짧은 작업들이 뒤따라 대기하며 시스템 성능이 저하되는 현상
    구분세부 항목설명
    원인FCFS 방식 사용선착순 방식의 비선점형 스케줄링 알고리즘을 사용할 때 발생
    프로세스 편차CPU 사용 시간이 매우 긴 프로세스가 짧은 프로세스보다 먼저 도착할 경우 발생
    비선점형 특성이미 CPU를 점유한 긴 작업을 중간에 멈추거나 뺏어올 수 없기 때문에 발생
    대응방안선점형 스케줄링긴 작업 중에도 짧은 작업이 CPU를 선점할 수 있도록 인터럽트 허용
    SJF실행 시간이 짧은 프로세스를 우선 배치하여 평균 대기 시간 최소화
    라운드 로빈(RR)할당 시간(Time Slice)을 부여해 특정 작업의 독점을 방지하고 순환 처리
    멀티레벨 큐(MLQ)작업의 우선순위나 특성에 따라 큐를 분리하여 짧은 작업을 먼저 처리
