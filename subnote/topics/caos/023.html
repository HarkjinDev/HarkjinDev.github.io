[하위] 4-1. 은행가 알고리즘 (Banker’s Algorithm)

정의: 자원 할당 전 시스템이 안전 상태인지 확인하여 교착 상태를 회피하는 알고리즘
키워드: 안전 상태, 불안전 상태(교착상태), MAX, Allocated, Available, Need, 안전 순서열
분류: 프로세스 관리
은행가 알고리즘 연산:
자료명
설명
수식
Available
사용 가능한 자원의 수
Available[j] = K
Max
프로세스별 최대 자원 요구량
Max[i, j] = K
Allocation
현재 프로세스별 할당된 자원 수
Allocation[i, j] = K
Need
프로세스별 추가로 필요한 자원 수
Need[i, j] = Max - Allocation
Request
프로세스가 현재 요청 중인 자원 수
Request[i, j] = K
안전 상태 판단: Need ≤ Available인 프로세스를 순차 실행 가능 → 안전 순서열 존재 → 안전 상태
안전여부 확인하는데 (m∗n2(m*n^2(m∗n2) 개의 연산이 필요 → 프로세스수(N)에 따른 오버헤드 큼
동작: 준비 → 자원 할당 요청 → 안정상태 확인 → (안정 상태) 자원할당 → (불안정 상태) 할당 거부
인사이트: 사전에 최대 자원 수요를 알아야 하는 제약이 있어 실용성이 제한됨. 시험에서 MAX/Allocated/Need/Available 행렬 계산 문제가 핵심 출제 포인트
