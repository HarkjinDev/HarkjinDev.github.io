1. CPU (Central Processing Unit)

정의: 명령어 해석과 연산을 수행하는 컴퓨터의 핵심 처리 장치
키워드: ALU, 레지스터(PC, IR, AC, MAR, MBR), CU, 시스템버스(주소/데이터/제어), 명령어 사이클
분류: 컴퓨터 구조
구성요소:
구분
세부 항목
내용
ALU (Arithmetic and Logic Unit)
산술 연산 및 논리 연산을 수행하는 핵심 모듈
레지스터 (Register)
CPU 내부의 초고속 기억장치
프로그램 카운터 (PC) (Program Counter)
다음에 실행할 명령어의 주소 보관
명령어 레지스터 (IR) (Instruction Register)
가장 최근에 인출된 명령어 코드 저장
누산기 (AC) (Accumulator)
연산 데이터 및 결과를 일시적으로 저장
MAR (Memory Address Register)
메모리 주소를 버스로 출력하기 전 일시 저장
MBR (Memory Buffer Register)
메모리와 주고받는 데이터를 일시 저장하는 버퍼
제어장치 (Control Unit)
명령어 해석 및 시스템 제어 신호 순차 발생
CPU 내부버스
시스템 버스
ALU, 레지스터, 제어장치 간의 내부 통로
주소 버스
외부로 전송할 주소 정보의 집합
데이터 버스
메모리나 I/O 장치와 데이터를 주고받는 통로
제어 버스
각 요소의 동작을 제어하는 신호 전송 통로
명령어 처리 사이클:
명령어 인출(IF) → 명령어 해독(ID) → 데이터 인출(OF) → 데이터 처리(EX) → 데이터 저장(WB)
인사이트: CPU 성능 향상의 핵심은 IPC(Instruction Per Cycle) 극대화이며, 파이프라인·슈퍼스칼라·투기적 실행 등을 통해 달성함. 투기적 실행은 성능을 높이지만 Spectre/Meltdown 취약점의 원인이 됨
