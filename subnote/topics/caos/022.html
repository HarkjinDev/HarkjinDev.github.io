4. 교착 상태 (Deadlock)

  • 정의: 둘 이상의 프로세스가 서로 상대방이 점유한 자원을 무한히 대기하여 진행이 불가능한 상태
  • 키워드: 코프만 조건, 상호배제, 점유대기, 비선점, 환형대기, 예방, 회피, 탐지, 복구
  • 분류: 프로세스 관리
  • 발생 필요 조건 (코프만 조건 4가지): <상점비환>
    • 상호배제(Mutual Exclusion): 자원을 한 번에 하나의 프로세스만 사용 가능
    • 점유 대기(Hold & Wait): 자원을 점유한 채 다른 자원을 대기
    • 비선점(No Preemption): 다른 프로세스의 자원을 강제로 빼앗을 수 없음
    • 환형 대기(Circular Wait): 프로세스들이 자원을 원형으로 대기
  • 대처 방법: <예회발복>
    방법기법핵심
    예방 (Prevention)상호 배제 조건 제거 — 공유 가능 자원으로 전환조건 원천 제거
    점유 대기 조건 제거 — 필요 자원 한번에 모두 요청일괄 자원 요청
    비선점 조건 제거 — 대기 시 보유 자원 강제 반납강제 자원 반납
    순환 대기 조건 제거 — 자원에 순서 부여, 순서대로만 요청자원 순서화
    회피 (Avoidance)은행가 알고리즘 — 안전 상태 유지 가능한 경우만 자원 할당안전 상태 유지
    사전 최대 자원 선언 — 프로세스 시작 전 최대 요구량 명시최대 요구 선언
    발견 (Detection)자원 할당 그래프 — 사이클 탐지로 교착 상태 식별그래프 사이클 탐지
    탐지 알고리즘 — 주기적 실행으로 교착 상태 프로세스 식별주기적 탐지
    복구 (Recovery)프로세스 종료 — 희생 프로세스 선택·강제 종료로 자원 해제프로세스 강제 종료
    자원 선점 — 프로세스에서 자원 강제 회수·재할당자원 강제 선점
    롤백 — 교착 상태 이전 안전 상태로 프로세스 복귀체크포인트 복귀
  • 인사이트: 현대 OS는 교착 상태 예방보다 탐지+복구 방식을 선호하며, 데이터베이스 시스템에서는 Wait-For Graph를 통한 교착 상태 탐지가 표준 방식임
