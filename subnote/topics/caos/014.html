[하위] 4-2. 메모리 영역 (코드/데이터/힙/스택)

정의: 프로세스 실행 시 운영체제가 코드, 데이터, 힙, 스택으로 분할하는 메모리 구조
키워드: 코드, 데이터, BSS, 힙, 스택, 프로세스 메모리 구조, 정적/동적 할당
분류: 컴퓨터 구조
메모리 영역:
영역
내용
특징
코드(Text) 영역
실행 가능한 프로그램 기계어 코드
읽기 전용(Read-Only), 공유 가능
데이터(Data) 영역
초기화된 전역 변수, 정적 변수
프로그램 시작 시 할당, 종료 시 해제
BSS 영역
초기화되지 않은 전역/정적 변수
실행 시 0으로 초기화
힙(Heap) 영역
동적으로 할당되는 메모리 (malloc/new)
런타임 결정, 낮은 주소→높은 주소 방향 성장
스택(Stack) 영역
지역 변수, 함수 매개변수, 복귀 주소
LIFO 구조, 높은 주소→낮은 주소 방향 성장
인사이트: 힙과 스택은 서로를 향해 성장하므로, 힙/스택 오버플로우가 심각한 충돌을 유발할 수 있음. Stack Canary, ASLR이 스택 기반 공격을 방어하는 보안 메커니즘임
