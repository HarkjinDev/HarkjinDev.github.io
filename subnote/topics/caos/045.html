3. 스레싱 (Thrashing)

정의: 과도한 페이지 교체로 인해 실제 프로세스 실행보다 페이지 교환에 더 많은 시간을 소비하여 CPU 이용률이 급격히 저하되는 현상
키워드: Page Fault 과다, 워킹셋(Working Set), PFF(Page Fault Frequency), 벨라디 변이
분류: 메모리 관리

밸라디 변이(Belady’s Anomaly):
프로세스에 페이지 프레임을 더 할당하였음에도 페이지 부재율이 증가하는 현상
원인: Locality를 고려하지 않은 FIFO의 한계로 인한 page fault 증가 발생
영향: 페이지 부재율 증가로 인한 성능 저하 및 스레싱 발생
인사이트: 스레싱은 단순히 메모리를 추가하는 것으로 해결되지 않으며, 프로세스별 워킹셋 크기를 분석하여 적절한 메모리 할당이 근본적 해결책. 현대 시스템에서는 OOM(Out-Of-Memory) Killer가 스레싱 상황에서 일부 프로세스를 강제 종료함
