4. 동적 프로그래밍 (Dynamic Programming, DP)

  • 정의: 복잡한 문제를 중복되는 부분 문제로 분해하고, 이미 계산한 결과를 저장·재사용하여 불필요한 반복 연산을 제거하는 최적화 알고리즘 설계 패러다임
  • 키워드: 메모이제이션, 타뷸레이션, 최적 부분 구조, 중복 부분 문제, 점화식, 상태 정의, 피보나치, 배낭 문제, LCS
  • 분류: 알고리즘 기초
  • 두 가지 구현 방식:
    항목메모이제이션 (Top-Down)타뷸레이션 (Bottom-Up)
    접근 방향큰 문제 → 작은 문제 재귀작은 문제 → 큰 문제 반복
    구현 방식재귀 + 캐시 저장반복문 + 테이블 채우기
    메모리필요한 부분만 계산전체 테이블 사전 할당
    스택 위험재귀 깊이에 따른 위험없음
    직관성점화식에 충실, 직관적순서 설계 필요
  • 핵심 설계 단계:
    단계설명핵심
    1. 상태 정의DP 테이블의 각 셀이 의미하는 바를 명확히 정의문제 추상화
    2. 점화식 도출현재 상태와 이전 상태의 관계식 수립관계식 정의
    3. 초기값 설정기저 조건(Base Case) 값 설정계산 시작점
    4. 계산 순서 결정부분 문제가 먼저 계산되도록 순서 설정의존성 해결
  • 대표 문제 유형:
    항목설명핵심
    피보나치 수열DP의 가장 기본 예제, O(n)으로 최적화기본 예제
    배낭 문제 (Knapsack)무게 제한 내 최대 가치 선택 최적화선택 최적화
    LCS두 문자열의 최장 공통 부분 수열 탐색문자열 비교
    LIS수열에서 가장 긴 증가하는 부분 수열 탐색수열 최적화
    최단 경로 (Floyd-Warshall)모든 정점 쌍 간 최단 경로 동시 계산전체 최단 경로
    행렬 체인 곱셈행렬 곱셈 순서 최적화로 연산 비용 최소화연산 최적화
  • 타 패러다임과 비교:
    항목동적 프로그래밍분할정복탐욕 알고리즘
    부분 문제 중복✅ 있음❌ 없음❌ 없음
    최적해 보장✅ 항상✅ 항상△ 조건 성립 시
    결과 저장✅ 필수❌ 불필요❌ 불필요
    속도중간빠름가장 빠름
    적용 범위광범위제한적제한적
  • 인사이트: 동적 프로그래밍은 알고리즘 설계 패러다임 중 가장 광범위하게 적용되며, "상태를 어떻게 정의하느냐" 가 곧 문제 해결의 핵심이자 시험 답안의 핵심 포인트.분할정복과 구조는 유사하나 부분 문제의 중복 여부가 두 패러다임을 가르는 핵심 기준이며, 이 차이를 명확히 설명하는 것이 시험 답안 차별화의 핵심
