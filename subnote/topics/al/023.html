[하위] 4-1. B-트리 (B-Tree)

  • 정의: 다진 탐색 트리의 균형을 유지하는 자체 균형 탐색 트리
  • 키워드: M-원 탐색 트리, 차수(M), 균형, 디스크 I/O 최소화, B+트리, 페이지 분할
  • 분류: 비선형 자료구조
  • B-트리 vs B+트리 비교:
    구분B-트리B+트리
    데이터 저장 위치내부 노드 + 리프 노드리프 노드만
    리프 노드 연결없음연결 리스트로 연결
    순차 탐색트리 전체 순회 필요리프 연결로 O(n) 순회
    범위 검색비효율효율적 (BETWEEN, ORDER BY)
    주요 활용파일 시스템 (HFS+)DBMS 인덱스 (InnoDB)
  • 삽입 알고리즘: 리프 노드에 키 삽입 → 오버플로 시 중간 키를 부모로 올리며 분할(Split) → 루트까지 전파
  • 인사이트: DBMS의 인덱스는 대부분 B+트리로 구현되며, 리프 노드의 연결 리스트 구조가 범위 검색의 효율을 극대화함; M차 B-트리에서 노드당 최대 M-1개의 키와 M개의 자식을 갖는다는 제약 조건도 서술
