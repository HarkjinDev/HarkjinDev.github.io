7. AVL 트리 / 레드-블랙 트리 (Balanced BST)

  • 정의: 이진 탐색 트리(BST)의 편향 문제를 해결하기 위해 삽입·삭제 시 자동으로 균형을 유지하는 자가 균형 이진 탐색 트리
  • 키워드: 균형 인수(Balance Factor), 회전(Rotation), 높이 균형, 레드-블랙 속성, 자가 균형, O(log n), BST 편향
  • 분류: 비선형 자료구조
  • AVL 트리:
    항목설명핵심
    균형 조건모든 노드의 좌·우 서브트리 높이 차이가 1 이하엄격한 균형
    균형 인수BF = 왼쪽 높이 - 오른쪽 높이 (-1, 0, 1만 허용)불균형 탐지
    회전 연산불균형 발생 시 LL·RR·LR·RL 4가지 회전으로 복구균형 복구
    탐색 성능엄격한 균형으로 탐색 O(log n) 항상 보장탐색 최적
    단점삽입·삭제 시 빈번한 회전으로 갱신 비용 높음갱신 비용
  • 레드-블랙 트리:
    항목설명핵심
    색상 속성모든 노드는 Red 또는 Black으로 표시색상 기반 균형
    루트 속성루트 노드는 항상 Black루트 규칙
    Red 속성Red 노드의 자식은 반드시 Black (연속 Red 불가)연속 Red 금지
    Black 속성모든 경로의 Black 노드 수는 동일Black 높이 균형
    균형 수준AVL보다 느슨한 균형, 회전 횟수 적음갱신 효율
  • AVL vs 레드-블랙 트리 비교:
    항목AVL 트리레드-블랙 트리
    균형 엄격도엄격 (BF ≤ 1)느슨 (Black 높이 동일)
    탐색 성능✅ 더 빠름△ 약간 느림
    삽입·삭제△ 회전 많음✅ 회전 적음
    구현 복잡도중간높음
    주요 활용탐색 빈번한 경우삽입·삭제 빈번한 경우
    대표 사례DB 인덱스Linux 커널, Java TreeMap
  • 활용 분야: MySQL InnoDB의 B-Tree 내부 구현 기반,가상 메모리 영역 관리, Linux 커널 CFS 스케줄러
  • 인사이트: AVL과 레드-블랙 트리는 "얼마나 엄격하게 균형을 유지하느냐"의 트레이드오프로 구분되며, 탐색이 주된 경우 AVL, 삽입·삭제가 빈번한 경우 레드-블랙 트리가 유리함. 실제 운영체제·언어 표준 라이브러리에서는 갱신 효율이 우수한 레드-블랙 트리가 사실상 표준으로 채택되어 있으며, 시험에서는 두 트리의 균형 조건과 회전 원리의 차이를 명확히 설명하는 것이 핵심 포인트
