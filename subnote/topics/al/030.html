8. 라우팅 테이블 탐색 알고리즘 (Routing Table Lookup Algorithm)

  • 정의: 라우터가 수신 패킷의 목적지 IP 주소를 라우팅 테이블에서 가장 긴 프리픽스(Longest Prefix Match)로 탐색하여 최적 출력 인터페이스를 결정하는 알고리즘
  • 키워드: LPM, 트라이, TCAM, 이진 트라이, 경로 압축, Lulea, FIB, 고속 패킷 포워딩
  • 분류: 비선형 자료구조
  • 핵심 개념:
    항목설명핵심
    LPM (Longest Prefix Match)목적지 IP와 가장 길게 일치하는 프리픽스 선택최장 일치 우선
    FIB (Forwarding Information Base)라우팅 테이블의 포워딩 최적화 테이블하드웨어 탐색 기반
    트라이 (Trie)비트/문자 트리구조, IP 프리픽스 저장·탐색접두사 트리
    TCAM삼진 논리(0·1·X)로 병렬 IP 매칭 수행하드웨어 고속 탐색
  • 라우팅 테이블 탐색 알고리즘 유형:
    구분알고리즘설명핵심
    SW이진 트라이 (Binary Trie)IP 주소 비트를 0·1 단위로 검사하며 자식 노드로 내려가는 방식비트별 순차 탐색
    경로 압축 트라이 이진 트라이에서 자식 노드가 하나인 중간 노드를 병합·체인 축소노드 수 감소
    다중 비트 트라이 1비트씩 검사 대신 k비트씩 묶어 한 번에 검사하는 방식탐색 깊이 감소
    Lulea 기법다중 비트 트라이를 비트맵 압축으로 메모리 획기적 절감압축 트라이
    HWTCAM (Ternary CAM)삼진 논리(0·1·X)로 모든 엔트리 병렬 동시 매칭 수행나노초 고속 탐색
    해시해시 테이블IP 주소 해시 함수로 O(1) 탐색·충돌 처리 필요정확 매칭에 적합
  • 트라이 구조 특성:
    항목설명핵심
    노드 구성각 노드가 비트값(0 또는 1) 표현이진 분기
    탐색 방향Root→Leaf 방향으로 비트 단위 하향 탐색상위 비트 우선
    LPM 처리탐색 중 마지막으로 매칭된 프리픽스 선택최장 일치 기록
    IPv4 최대 깊이32비트→최대 32레벨 탐색32 홉 최대
    IPv6 확장128비트로 깊이 증가→다중 비트 트라이 필수128 홉 문제
  • TCAM 동작 원리:
    항목설명핵심
    삼진 논리0·1·X(Don't Care) 3가지 상태 저장와일드카드 매칭
    병렬 탐색모든 엔트리 동시 비교·우선순위 적용O(1) 탐색
    우선순위가장 긴 프리픽스 엔트리 최상위 배치LPM 구현
    단점고비용·고전력·용량 제한(수십만 엔트리)DRAM 대비 비용
    적용코어 라우터·L3 스위치 하드웨어 포워딩라인속도 처리
  • 인사이트: 라우팅 테이블 탐색은 LPM을 얼마나 빠르게 수행하느냐가 핵심으로, 소프트웨어는 트라이 압축·다중 비트화로 탐색 깊이를 줄이고, 하드웨어는 TCAM 병렬 탐색으로 라인속도(Line-rate) 포워딩을 실현하는 이중 전략이 현대 라우터 설계의 표준
