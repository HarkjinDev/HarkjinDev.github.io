4. 이진 탐색 트리 (Binary Search Tree)

  • 정의: 왼쪽 자식이 작고 오른쪽 자식이 큰 이진트리 기반 탐색 자료 구조
  • 키워드: 왼쪽 < 루트 < 오른쪽, 중위 순회, 편향 트리, AVL 트리, 레드-블랙 트리
  • 분류: 비선형 자료구조
  • 연산 복잡도:
    연산평균최악 (편향 트리)
    탐색O(log n)O(n)
    삽입O(log n)O(n)
    삭제O(log n)O(n)
  • 삭제 케이스:
    케이스처리 방법
    자식 없음 (리프)노드 직접 삭제
    자식 1개자식 노드로 대체
    자식 2개중위 후계자(오른쪽 부트리 최솟값) 또는 중위 전위 노드로 대체
  • 인사이트: BST 최악 O(n) 문제는 AVL 트리(회전 연산으로 균형 유지, O(log n) 보장)나 레드-블랙 트리로 해결하며, 이것이 디스크 기반 B-트리로 확장됨을 언급
