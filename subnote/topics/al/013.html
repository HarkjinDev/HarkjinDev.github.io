3. 스택 (Stack)

  • 정의: 후입선출(LIFO) 원칙으로 삽입과 삭제가 한쪽 끝에서 이루어지는 자료 구조
  • 키워드: LIFO, Push, Pop, Top, Overflow, Underflow, 재귀 호출, 콜스택
  • 분류: 선형 자료구조
  • 주요 연산 및 예외 상태:
    연산/상태설명
    PushTop에 데이터 삽입; Overflow: Top = MAX-1일 때 추가 시도
    PopTop에서 데이터 삭제; Underflow: Top = -1일 때 삭제 시도
    PeekTop 데이터 조회 (삭제 없음)
  • 적용: 재귀 알고리즘, 함수 호출 스택(콜스택), 수식 변환(중위→후위), 괄호 검사, DFS 탐색, 브라우저 뒤로가기
  • Push/Pop 알고리즘 요약:
    • Push: if top == MAX-1 → overflow; else stack[++top] = data
    • Pop: if top == -1 → underflow; else return stack[top--]
  • 인사이트: 재귀 알고리즘의 내부 구현이 스택이며, 컴파일러의 수식 파싱(중위→후위 변환)과 실행 시 콜스택 관리에 핵심 역할을 한다는 점이 시험 차별화 포인트
