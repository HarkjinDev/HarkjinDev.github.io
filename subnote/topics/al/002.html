1. 알고리즘 복잡도 (Algorithm Complexity)

정의: 알고리즘의 실행 시간과 메모리 사용량을 입력 크기 함수로 표현한 성능 척도
키워드: 시간 복잡도, 공간 복잡도, 수행시간, Big-O, Big-Ω, Big-Θ, 점근 표기법, 최선/평균/최악
분류: 알고리즘 기초
성능 분석:
시간 복잡도 : 알고리즘에 사용되는 연산횟수의 총량(연산의 횟수)
공간 복잡도 : 알고리즘에 사용되는 메모리 공간의 총량(고정영역(프로그램 사이즈) + 가변영역(데이터 사이즈))
성능 측정: 수행시간 측정
점근 표기법 비교:
표기법
의미
설명
O(g(n))- 빅오
점근적 상한 (최악)
f(n) ≤ c·g(n)을 만족하는 c, n₀ 존재
Ω(g(n)) - 오메가
점근적 하한 (최선)
f(n) ≥ c·g(n)을 만족하는 c, n₀ 존재
Θ(g(n)) - 세타
점근적 정확한 경계 (평균)
O이면서 Ω를 동시에 만족
인사이트: 알고리즘 설계 답안에서 반드시 시간·공간 복잡도를 최선/평균/최악 케이스로 구분하여 분석해야 하며, 상수 계수와 하위 차수 항을 제거한 최고차항만 표기하는 점근 표기법의 수학적 의미를 명확히 서술하면 차별화됨
