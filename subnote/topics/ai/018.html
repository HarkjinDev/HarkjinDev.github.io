[하위] 3-1. 배깅 (Bagging, Bootstrap Aggregating)

  • 정의: 원본 데이터에서 복원 추출(Bootstrap)로 여러 개의 독립적으로 학습한 모델의 예측 결과를 투표(Voting) 또는 평균으로 최종 결정하는 앙상블 기법
  • 키워드: 복원 추출(Bootstrap), 병렬 학습, 분류(하드보팅), 회귀(평균), 랜덤 포레스트, 37% OOB(Out-of-Bag) 검증
  • 분류: 지도학습
  • 상세 프로세스: <분하회평>
    프로세스설명핵심
    1. Bootstrap 샘플링중복 허용하여 데이터셋 생성무작위 중복 추출
    2. 병렬 학습독립적인 학습기 동시 학습모델 간 독립성 유지
    3. 집계 (Aggregation)투표나 평균으로 결과 도출최종 예측값 결정 • 분류: Hard Voting • 회귀: 평균
  • 특징 및 대표 모델:
    • 효과: 모델 간 예측이 독립적이므로 분산(Variance)을 줄여 과적합 방지에 탁월
    • 대표 모델:
      • 랜덤 포레스트(Random Forest): 배깅에 특징(Feature) 무작위 선택까지 추가하여 트리 간 다양성을 극대화한 모델
    • 한계: 모든 모델이 병렬로 학습되므로 편향(Bias)이 높은 경우 개선 효과가 제한적임
  • 인사이트: 복원 추출 시 선택되지 않는 약 37%의 데이터(OOB 샘플)를 별도 검증 데이터로 활용할 수 있어 교차 검증 없이도 일반화 성능을 추정할 수 있는 장점이 있음
