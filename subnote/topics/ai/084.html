5. 연합학습 (Federated Learning)

  • 정의: 데이터를 중앙 서버로 수집하지 않고, 개별 기기(로컬)에서 모델을 학습시킨 후 학습된 '가중치'값만 서버에 공유하여 개인정보를 보호하며 학습하는 분산 학습 기법
  • 키워드: 개인정보 보호, 데이터 비식별화, 가중치 평균, 분산 학습, 프라이버시 보존, 가중치 집계, FedSGD, FedAvg, 차등 프라이버시, 통신 효율, Non-IID, 온디바이스 학습
  • 분류: AI 응용/트렌드
  • 특징: 데이터 주권 확보, 프라이버시 보호, 통신 비용 절감
  • 동작 원리:
    항목설명핵심
    1. 모델 배포서버가 전역 모델을 각 클라이언트에 배포초기 배포
    2. 로컬 학습클라이언트가 자신의 로컬 데이터로 모델 학습분산 학습
    3. 가중치 업로드데이터 대신 학습된 가중치(기울기)만 서버에 전송프라이버시 보존
    4. 집계 (Aggregation)서버가 각 클라이언트 가중치를 FedAvg 등으로 통합전역 모델 갱신
    5. 반복갱신된 전역 모델을 재배포하며 위 과정 반복수렴까지 반복
  • 주요 유형:
    항목설명핵심
    수평 연합학습클라이언트 간 동일한 특징, 다른 샘플 데이터 보유샘플 분산
    수직 연합학습클라이언트 간 동일한 샘플, 다른 특징 데이터 보유특징 분산
    연합 전이학습데이터·특징 모두 다를 때 전이학습과 결합이종 도메인
  • 핵심 기술:
    항목설명핵심
    FedSGD클라이언트 가중치를 서버가 집계하여 전역모델 갱신기본 연합학습 방식
    FedAvg클라이언트 가중치를 데이터 비율로 가중 평균 집계표준 집계 알고리즘
    차등 프라이버시 (DP)가중치에 노이즈를 추가하여 개인정보 역추론 방지프라이버시 강화
    보안 집계 (Secure Aggregation)암호화 기반으로 개별 가중치 노출 없이 집계 수행암호화 집계
    통신 압축가중치 양자화·희소화로 통신 비용 절감통신 효율화
  • 주요 문제점:
    항목설명핵심
    Non-IID 문제클라이언트 간 데이터 분포 불균일로 모델 수렴 저하데이터 이질성
    통신 비용반복적인 가중치 송수신으로 네트워크 부하 발생통신 병목
    무임승차 공격일부 클라이언트가 학습 기여 없이 전역 모델만 수신신뢰성 문제
    적대적 공격악의적 클라이언트가 오염된 가중치로 모델 훼손보안 취약성
  • 인사이트: 연합학습은 데이터 프라이버시 규제가 강화되는 환경에서 '데이터는 모으지 않고 지식만 모으는' 현실적 대안으로 부상함. 특히 의료·금융 등 데이터 민감도가 높은 산업에서 AI 도입의 핵심 enabler로 주목받고 있으며, 차등 프라이버시·보안 집계와의 결합으로 프라이버시 보존 수준이 지속적으로 강화되는 추세
