1. 머신러닝 모델링 & 파이프라인

  • 정의: 머신러닝의 전 과정을 체계화하여 지속적으로 반복 가능하게 만든 엔드투엔드(End-to-End) 시스템 절차
  • 키워드: ML Pipeline, 데이터 리니지, 자동화, CI/CD/CT, MLops, Modelops, DSML
  • 분류: AI 운영/실무
  • 단계:
    • Data Ingestion → Validation → Preparation → Training → Model Evaluation → Serving
  • RLHF 및 RAFT 기반의 학습 파이프라인 비교:
    구분RLHFRAFT
    핵심 목적인간 선호 기반 정렬 (Alignment)검색 기반 응답 정확성 강화
    중심 데이터인간 피드백 (Human Feedback)검색 문서 (Retrieved Context)
    학습 방식강화학습 (PPO) 기반 최적화Fine-Tuning 기반 최적화
    주요 초점응답 품질·안전성·윤리성근거 기반 생성·환각 감소
    핵심 기술Reward Model, PPO, Reinforcement LearningRetrieval, Vector DB, Retrieval-aware Fine-Tuning
    활용 기술Human Feedback, Preference LearningEmbedding, Semantic Search, RAG
    장점인간 친화적 응답 생성 가능최신 정보 및 검색 정확성 향상
    한계피드백 비용과 학습 비용 증가검색 품질 의존성 존재
    적합 환경AI Assistant, Chatbot, Alignment기업 문서 검색, QA, RAG 환경
    대표 효과유해 응답 감소 및 응답 자연성 향상Grounded Response 및 최신성 강화
  • 인사이트: 기술 부채 해결을 위한 MLOps 도입의 필요성, 모델 재학습(Retraining) 트리거 설계
