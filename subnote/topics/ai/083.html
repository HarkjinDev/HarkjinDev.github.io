4. 컨텍스트 엔지니어링 (Context Engineering)

  • 정의: LLM·AI 에이전트가 최적의 응답을 생성하도록 컨텍스트 윈도우에 제공되는 정보의 구조·선택·흐름을 체계적으로 설계·관리하는 엔지니어링 방법론
  • 키워드: 컨텍스트 윈도우, 메모리 관리, RAG, 정보 아키텍처, 에이전트, 토큰 최적화, 컨텍스트 오버로드, 프롬프트 엔지니어링 진화
  • 분류: 생성형 AI / LLM / 에이전트 시스템
  • 컨텍스트 구성 요소:
    구분항목설명핵심
    지시시스템 프롬프트에이전트 역할·규칙·제약 정의행동 기준 설정
    태스크 지시현재 수행할 작업 명확화목표 명시
    메모리단기 메모리현재 대화·세션 내 컨텍스트 유지대화 흐름
    장기 메모리외부 DB에 영구 저장·필요 시 검색지속 기억
    에피소딕 메모리과거 상호작용 이력·결과 저장경험 활용
    지식RAG 검색 결과외부 지식 베이스 실시간 검색·주입최신 지식
    도메인 데이터업무 특화 문서·DB·API 결과도메인 맥락
    도구툴 정의·결과호출 가능한 툴 스펙·실행 결과행동 확장
    에이전트 상태이전 단계 추론·행동·관찰 이력추론 연속성
  • 핵심 4대 전략:
    항목설명핵심
    Write (저장)중요 정보를 외부에 영구 저장하여 컨텍스트 의존 탈피외부 메모리화
    Select (선택)현재 태스크에 관련성 높은 정보만 선별하여 주입관련 정보 선별
    Compress (압축)토큰 효율 위해 요약·필터링으로 정보 압축토큰 최적화
    Isolate (격리)서브에이전트·모듈로 컨텍스트를 분리하여 오염 방지컨텍스트 격리
  • 에이전트 시스템에서의 역할:
    항목설명핵심
    단계별 컨텍스트 구성에이전트 루프 매 단계마다 최적 컨텍스트 동적 구성동적 컨텍스트
    멀티에이전트 컨텍스트 분리에이전트별 독립 컨텍스트로 오염·간섭 방지에이전트 격리
    메모리 계층 설계단기·장기·에피소딕 메모리 계층적 통합 설계메모리 아키텍처
    동적 RAG 연계태스크 맥락에 따라 검색 쿼리·범위 동적 조정적응형 검색
  • 인사이트: 컨텍스트 엔지니어링은 2025년 중반 프롬프트 엔지니어링의 진화적 후계자로 부상했으며, 대부분의 에이전트 실패는 모델 자체의 문제가 아닌 컨텍스트 설계 실패에서 비롯된다는 인식이 확산되면서 AI 에이전트 개발자의 핵심 역량으로 자리잡고 있음. 프롬프트 한 줄이 아닌 정보 아키텍처 전체를 설계하는 역량이 프로덕션 AI 시스템의 품질을 결정하는 시대로 전환되고 있음
