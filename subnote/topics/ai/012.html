7. 교차 검증 (Cross Validation)

  • 정의: 고정된 테스트 데이터로 인한 성능 왜곡을 막기 위해 데이터를 여러 부분으로 나누어 학습과 검증을 반복하는 기법
  • 키워드: 데이터 홀드아웃(학습/검증/평가), K-폴드, LOO, Stratified K-폴드
  • 분류: 학습/최적화
  • 데이터 홀드아웃(Hold-out): 모델 일반화 성능 평가시 전체 데이터를 Train, Validation, Test 세 집합으로 단순 분리 및 학습, 모델 선택, 최종 성능 평가에 활용하는 데이터 분할 기법(7:3, 8:2 등 비율 조절)
    데이터 세트설명목적
    Training (학습)모델 학습에 직접 사용패턴 습득 및 가중치 결정
    Validation (검증)학습 중 성능 중간 점검하이퍼파라미터 최적화
    Test (평가)최종 성능 확인용 데이터모델의 실제 일반화 성능 측정
  • Hold-out 기법 한계점:
    구분한계점설명핵심
    평가신뢰성분할 의존성 분할 방식에 따라 평가지표 크게 달라짐분할 운에 좌우
    높은 평가 분산단일 분할로 성능 과대·과소 추정 가능신뢰구간 불안정
    데이터특성 미반영클래스 불균형 미반영무작위 분할 시 클래스 비율 원본과 차이소수클래스 신뢰저하
    시계열 데이터 부적합미래 데이터 학습 혼입·배포 성능 괴리데이터 누수 위험
  • 유형:
    항목설명핵심
    K-Fold데이터를 K개로 분할 및 반복 검증데이터 활용 극대화, 클래스 불균형에 취약
    Leave-One-Out (LOOCV)데이터 1개만 제외하고 N번 반복 검사소규모 데이터셋 유리, 매우 높은 연산 비용
    Stratified K-Fold원본 클래스 비율을 유지하며 분할불균형 데이터 평가 표준, 높은 신뢰성 확보
  • 인사이트: 데이터 부족 상황에서 모델의 신뢰성 검증을 위한 필수 절차이며, 일반적으로 K=5 또는 K=10이 연산 비용과 성능 신뢰도 간의 균형점으로 권장됨
