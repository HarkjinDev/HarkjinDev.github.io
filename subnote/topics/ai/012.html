7. 교차 검증 (Cross Validation)

정의: 고정된 테스트 데이터로 인한 성능 왜곡을 막기 위해 데이터를 여러 부분으로 나누어 학습과 검증을 반복하는 기법
키워드: 데이터 홀드아웃(학습/검증/평가), K-폴드, LOO, Stratified K-폴드
분류: 학습/최적화
데이터 홀드아웃(Hold-out): 7:3, 8:2 등 비율 조절
데이터 세트
설명
목적
Training (학습)
모델 학습에 직접 사용
패턴 습득 및 가중치 결정
Validation (검증)
학습 중 성능 중간 점검
하이퍼파라미터 최적화
Test (평가)
최종 성능 확인용 데이터
모델의 실제 일반화 성능 측정
유형:
항목

핵심
K-Fold
데이터를 K개로 분할 및 반복 검증
데이터 활용 극대화, 클래스 불균형에 취약
Leave-One-Out (LOO)
데이터 1개만 제외하고 N번 반복 검사
소규모 데이터셋 유리, 매우 높은 연산 비용
Stratified K-Fold
원본 클래스 비율을 유지하며 분할
불균형 데이터 평가 표준, 높은 신뢰성 확보
인사이트: 데이터 부족 상황에서 모델의 신뢰성 검증을 위한 필수 절차이며, 일반적으로 K=5 또는 K=10이 연산 비용과 성능 신뢰도 간의 균형점으로 권장됨
