4. 클래스 불균형 (Class Imbalance)

정의: 분류 문제에서 특정 클래스의 데이터 비율이 다른 클래스에 비해 압도적으로 많거나 적어, 모델이 다수 클래스(Majority Class)에 편향되게 학습되는 현상
키워드: 편향, 정확도의 역설(Accuracy Paradox), 재현율 저하, F1-Score, ROC-AUC, 언더샘플링, 오버샘플링, SMOTE, 비용 민감 학습(Cost-Sensitive)
분류: 학습/최적화
클래스 불균형의 주요 문제 및 평가 지표:
구분
내용
정확도의 역설
모두 다수 클래스로 예측해도 높은 정확도(Accuracy)가 나오는 착시 현상
재현율(Recall) 저하
실제 찾고자 하는 소수 클래스(예: 불량, 사기)를 제대로 예측하지 못함
F1-Score
정밀도(Precision)와 재현율(Recall)의 조화평균으로 불균형 데이터의 핵심 평가지표
ROC-AUC
임계값 변화에 따른 모델의 전반적인 클래스 분류 성능을 평가하는 지표
클래스 불균형 해결 기법:
항목
설명
핵심
언더샘플링 (Undersampling)
다수 클래스 데이터 삭제
학습시간 단축, 정보 손실 위험
오버샘플링 (Oversampling)
소수 클래스 데이터 복제
정보 손실 방지, 과적합 발생 위험
SMOTE (합성 데이터 생성)
가상 소수 데이터 생성
KNN 기반 데이터 합성
비용 민감 학습(Cost-Sensitive)
소수 클래스 가중치 부여
오분류 페널티 강화
인사이트: 클래스 불균형 상황에서는 단순 정확도(Accuracy)에 의존하는 것을 극도로 경계해야 하며, 풀고자 하는 비즈니스 문제(예: 암 진단은 재현율 중시, 스팸 필터는 정밀도 중시)에 맞춰 적절한 평가 지표(F1-Score, AUC)를 재설정하는 것이 샘플링 기법 적용만큼이나 필수적 전제 조건임.
