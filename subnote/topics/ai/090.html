2. MLOps (Machine Learning Operations)

  • 정의: 머신러닝 모델의 개발(Dev)과 운영(Ops)을 통합하여, 모델의 학습, 배포, 모니터링을 자동화하고 지속적으로 관리하는 체계
  • 키워드: 파이프라인 자동화, 피쳐 스토어, 모델 서빙(Serving), 데이터 드리프트, 기술 부채 해소
  • 분류: AI 운영/실무
  • 핵심 기능:
    • CI/CD/CT: 지속적 통합, 배포, 그리고 AI만의 특징인 지속적 학습(Continuous Training)
    • Model Monitoring: 모델의 성능 저하(Drift) 감지 및 재학습 트리거
  • 구성요소:
    항목설명핵심
    데이터 파이프라인데이터 수집·전처리·검증을 자동화하는 워크플로우데이터 자동화
    피처 스토어 (Feature Store)학습·서빙에 사용되는 피처를 중앙 저장·재사용 관리피처 재사용
    실험 관리 (Experiment Tracking)하이퍼파라미터·메트릭·결과를 버전별로 기록·비교재현성 확보
    모델 레지스트리 (Model Registry)학습된 모델을 버전별로 등록·관리·승인하는 저장소모델 버전 관리
    CI/CD 파이프라인코드·데이터·모델 변경 시 자동 학습·테스트·배포 수행배포 자동화
    모델 서빙 (Model Serving)학습된 모델을 API 형태로 실시간·배치 추론에 제공추론 제공
    모델 모니터링운영 중 모델의 성능·데이터 드리프트를 지속 감시품질 유지
    데이터 드리프트 감지학습·운영 데이터 분포 변화를 탐지하여 재학습 트리거분포 변화 감지
    오케스트레이션파이프라인 전체 워크플로우 스케줄링·실행 관리워크플로우 관리
    인프라 관리학습·서빙 환경의 컨테이너·클러스터 자원 관리자원 관리
  • 인사이트: MLOps의 구성요소는 데이터→학습→배포→모니터링의 전체 ML 라이프사이클을 자동화·체계화하는 데 초점을 두며, 단순 배포를 넘어 데이터의 변화(Concept Drift)를 실시간 감지하여 모델의 생명주기(Lifecycle)읲 모니터링이 운영 안정성의 핵심 축
