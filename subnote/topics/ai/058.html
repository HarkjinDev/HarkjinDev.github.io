5. 프롬프트 엔지니어링 (Prompt Engineering)

  • 정의: LLM으로부터 원하는 최적의 결과를 얻기 위해 입력 메시지(Prompt)를 정교하게 설계하고 최적화하는 기술
  • 키워드: 페르소나 설정, 명령-맥락-출력 형식, Zero-shot, Few-shot, CoT, ReAct, ToT, APE, 시스템/유저/어시스턴트 프롬프트, 온도(Temperature), 할루시네이션 제어, 프롬프트 인젝션
  • 분류: 생성형 AI / LLM
  • 프롬프트의 구성 요소: <역지맥예출>
    구성 요소설명예시
    역할 (Role/Persona)모델이 수행할 역할 지정"너는 10년 경력의 보안 전문가야"
    지시 (Instruction)수행할 작업을 명확히 기술"아래 코드의 취약점을 분석해줘"
    맥락 (Context)배경 정보 및 제약 조건 제공"대상은 Python 3.10 환경이야"
    예시 (Examples)입출력 형태를 예시로 제시Few-shot 예시 포함
    출력 형식 (Format)결과물의 형태 명시"JSON 형식으로 출력해줘"
  • 주요 기법:
    기법설명특징
    Zero-shot예시 없이 명령만으로 태스크 수행범용성 높음, 복잡한 태스크에 한계
    Few-shot입출력 예시를 몇 가지 제공하여 성능 향상예시 품질이 결과를 좌우
    CoT (Chain of Thought)"단계별로 생각해봐"로 추론 과정 유도수학·논리 문제에 효과적
    Self-Consistency동일 질문에 여러 추론 경로 생성 후 다수결CoT의 신뢰도 향상 기법
    ReAct추론(Reasoning)과 행동(Action)을 교차 수행에이전트·툴 사용에 핵심 기법
    ToT (Tree of Thought)여러 추론 경로를 트리 구조로 탐색·평가복잡한 다단계 문제 해결에 적합
    APE (Auto Prompt Engineer)LLM이 스스로 최적 프롬프트를 생성·평가자동화된 프롬프트 최적화
  • 프롬프트 유형:
    구분역할특징
    시스템 프롬프트모델의 전반적 행동·역할·제약 정의세션 전체에 적용, 우선순위 높음
    유저 프롬프트개별 요청 및 질문 입력매 턴마다 변경 가능
    어시스턴트 프롬프트이전 모델 응답을 맥락으로 포함멀티턴 대화 흐름 제어
  • 주요 리스크:
    리스크설명대응 방법
    프롬프트 인젝션악의적 입력으로 시스템 프롬프트 우회입력값 검증, 샌드박스 처리
    탈옥 (Jailbreak)안전 제약을 우회하도록 유도하는 프롬프트가드레일 설정, 출력 필터링
    과도한 의존프롬프트만으로 해결 불가한 구조적 한계 무시Fine-tuning·RAG 병행 검토
    프롬프트 누출시스템 프롬프트 내용이 외부에 노출민감 정보 미포함, 접근 제어
  • 인사이트: 모델 재학습 없이 성능을 끌어올리는 가장 가성비 높은 방법이며, 에이전틱 AI의 핵심 제어 수단임. CoT → ToT → ReAct로 이어지는 기법의 진화는 단순 질의응답을 넘어 LLM이 스스로 추론·행동·검증하는 에이전트 패러다임의 기반이 됨. 프롬프트 인젝션·탈옥 등 보안 리스크가 증가함에 따라, 프롬프트 설계는 단순 성능 최적화를 넘어 보안 설계의 영역으로 확장되고 있음
