13. MoE (Mixture of Experts, 전문가 혼합 모델)

  • 정의: 전체 모델 파라미터 중 일부 전문가(Expert) 네트워크만 선택적으로 활성화하여 추론하는 희소 활성화 기반 AI 모델 아키텍처
  • 키워드: 희소 활성화(Sparse Activation), Expert, Gate Network, Sparse Activation, Top-K 라우팅, 토큰 라우팅, 로드 밸런싱
  • 분류: AI·ML / LLM 아키텍처 / 모델 효율화
  • 핵심 구성요소:
    구성요소설명핵심
    Expert (전문가 네트워크)특정 유형의 입력을 처리하는 전문화된 FFN 서브네트워크전문화 연산 단위
    Gate Network (라우터)입력 토큰을 어떤 Expert에 할당할지 결정하는 라우팅 네트워크전문가 선택 결정
    Top-K 라우팅각 토큰에 대해 상위 K개의 Expert만 선택·활성화희소 활성화 핵심
    로드 밸런싱특정 Expert에 토큰 편중 방지·균등 분배 보조 손실전문가 균등 활용
    토큰 라우팅각 토큰이 독립적으로 Expert를 선택하는 방식토큰별 전문가 배정
  • MoE 동작 원리:
    단계설명핵심
    1. 토큰 입력입력 시퀀스가 토큰 단위로 분리토큰 단위 처리
    2. Gate 연산Gate Network가 각 토큰의 Expert 적합도 점수 계산라우팅 점수 산출
    3. Top-K 선택점수 상위 K개 Expert 선택 (보통 K=1~2)희소 Expert 선택
    4. Expert 연산선택된 Expert가 해당 토큰 처리·출력 생성전문화 연산
    5. 출력 집계선택된 Expert 출력을 가중 합산전문가 출력 결합
  • MoE 유형:
    유형설명핵심
    Sparse MoETop-K 선택으로 일부 Expert만 활성화희소 활성화
    Dense MoE모든 Expert를 활성화·출력 가중 합산전체 활성화
    Hierarchical MoEExpert를 계층적으로 구성·다단계 라우팅계층 구조
    Soft MoE토큰을 Expert에 소프트하게 분배·혼합연속적 분배
    Token-Choice MoE각 토큰이 Expert 선택 (일반적 방식)토큰 주도 선택
    Expert-Choice MoE각 Expert가 처리할 토큰 선택Expert 주도 선택
  • MoE 장단점:
    구분항목설명핵심
    장점파라미터 효율대규모 파라미터 보유·실제 연산은 일부만 수행연산 효율 극대화
    전문화Expert별 특정 도메인·유형 데이터 특화 학습전문 능력 강화
    확장성Expert 수 증가로 성능 향상·연산량 증가 최소효율적 스케일업
    병렬 처리다수 Expert 동시 병렬 연산 가능분산 처리 용이
    단점메모리 부담전체 Expert 파라미터 메모리 상주 필요높은 메모리 요구
    로드 불균형특정 Expert 편중·나머지 미활용 위험균등 활용 어려움
    학습 복잡도Gate Network 학습·로드 밸런싱 설계 복잡학습 난이도 높음
    통신 오버헤드분산 환경에서 Expert 간 토큰 전송 비용분산 통신 비용
  • 인사이트: MoE는 "모든 파라미터를 항상 사용하지 않아도 된다"는 조건부 연산 개념으로 LLM 스케일링의 새로운 패러다임을 열었으며, GPT-4·Gemini·DeepSeek 등 최신 최고 성능 모델의 핵심 아키텍처로 자리잡고 있음. Dense 모델 대비 동일 연산량으로 훨씬 많은 파라미터를 효율적으로 활용할 수 있어 성능·비용 균형의 핵심 기술이며, 로드 밸런싱·메모리 관리·분산 학습 설계가 MoE 실용화의 핵심 선결 과제임
