17. 엣지 컴퓨팅 (Edge Computing)

  • 정의: 데이터 발생 지점(디바이스·현장)에 가까운 네트워크 엣지에서 데이터를 처리·분석하여 클라우드 전송 없이 실시간 저지연 응답을 실현하는 분산 컴퓨팅 패러다임
  • 키워드: 저지연, 실시간 처리, 엣지 노드, 온디바이스 AI, 5G·MEC, 대역폭 절감, 데이터 주권, 오프로딩
  • 분류: 클라우드 컴퓨팅
  • 특징: 실시간성(저지연), 분산, 보안/프라이버시 강화
  • 클라우드 vs 엣지 컴퓨팅 비교:
    항목클라우드 컴퓨팅엣지 컴퓨팅
    처리 위치중앙 데이터센터데이터 발생 지점 인근
    지연높음 (수십~수백ms)낮음 (수ms 이하)
    대역폭대용량 전송 필요현장 처리로 절감
    실시간성제한적높음
    보안·프라이버시중앙 집중현장 처리로 유출 최소화
    가용성네트워크 의존오프라인 동작 가능
  • 엣지 컴퓨팅 계층 구조:
    계층설명핵심
    디바이스 계층IoT 센서·카메라·스마트폰 등 데이터 발생 단말데이터 수집
    엣지 계층엣지 서버·게이트웨이·MEC에서 실시간 처리현장 처리
    클라우드 계층대규모 학습·장기 저장·글로벌 분석 수행중앙 분석
  • 핵심 구성요소:
    구분항목설명핵심
    HW엣지 서버현장 인근 소형 서버·고성능 처리 노드현장 서버
    엣지 게이트웨이디바이스↔엣지 간 데이터 수집·전처리·전달데이터 중계
    AI 가속칩 (NPU·GPU)엣지 환경의 AI 추론 전용 저전력 가속현장 AI 연산
    SW컨테이너·오케스트레이션엣지 환경 앱 배포·관리 (K3s·KubeEdge)경량 오케스트레이션
    엣지 AI 프레임워크TensorFlow Lite·ONNX Runtime 경량 추론경량 AI 실행
    데이터 오프로딩처리 완료 후 필요 데이터만 클라우드 전송대역폭 절감
    NW5G·MEC이동통신사 기지국 인접 엣지 서버 활용초저지연 연결
    SDN·NFV소프트웨어 기반 엣지 네트워크 동적 제어네트워크 유연성
  • MEC (Mobile Edge Computing):
    항목설명핵심
    정의이동통신 기지국 인접 서버에서 모바일 데이터 처리기지국 엣지
    5G 연계5G 초저지연·대용량과 MEC 결합으로 실시간 서비스5G 시너지
    활용자율주행·AR·스마트팩토리·실시간 영상 분석실시간 서비스
  • 인사이트: 엣지 컴퓨팅은 AI·IoT·5G의 융합으로 클라우드 중심에서 분산 지능형 인프라로의 패러다임 전환을 이끄는 핵심 기술로, 자율주행·스마트팩토리·의료 등 초저지연이 요구되는 산업의 필수 인프라로 자리잡고 있음. 온디바이스 AI·MEC·클라우드의 3계층 협업 아키텍처 설계가 엣지 컴퓨팅 최적화의 핵심이며, 분산 보안 관리와 이기종 플랫폼 표준화가 대규모 확산의 핵심 선결 과제임
