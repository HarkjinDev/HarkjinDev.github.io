4. 컨테이너 (Container)

  • 정의: 호스트 OS의 커널을 공유하면서 네임스페이스와 cgroups로 애플리케이션을 프로세스 수준에서 격리 실행하는 경량 가상화 기술
  • 키워드: 네임스페이스, cgroups, Docker, 이미지, Dockerfile, 레이어, OCI 표준
  • 분류: 가상화/컨테이너
  • 구성요소/가상화 방식:
    구분항목설명핵심
    구성요소컨테이너 엔진독립적 운영환경 제공운영환경 고립
    컨테이너격리된 앱 실행 공간앱 실행 공간
    호스트 OS실제 서버 설치 OS기반 운영체제
    가상화 방식LXC컨테이너 관리 모듈관리 모듈
    libcontainerDocker 가상화 방식도커 가상화
    cgroups프로세스 그룹 자원 관리그룹 자원 관리
  • 기술요소:
    구분항목설명핵심
    실행 환경컨테이너 엔진컨테이너 생성 및 관리 수행수명주기 관리
    런타임 (runc)컨테이너를 실제 실행하는 도구실행 엔진
    배포 관리이미지앱 실행에 필요한 불변 패키지표준화 패키지
    레지스트리이미지 저장 및 배포용 저장소이미지 저장소
    핵심 기술Namespace시스템 자원의 논리적 격리독립 공간 격리
    Cgroups하드웨어 자원 사용량 제어자원 할당 관리
    Overlay FS레이어 기반의 파일 시스템계층형 저장
    네트워크가상 브리지컨테이너 간 통신 경로 제공내부 망 연결
  • 장점: 빠른 기동, 이식성(Build Once, Run Anywhere), 자원 효율, CI/CD 파이프라인 통합
  • 인사이트: 컨테이너는 MSA와 DevOps의 실현 도구로, 불변 인프라(Immutable Infrastructure) 개념과 결합하여 배포 안정성과 롤백 용이성을 제공함을 강조.
