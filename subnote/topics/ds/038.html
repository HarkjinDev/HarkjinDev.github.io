[하위] 1-6. API Gateway

  • 정의: 클라이언트와 백엔드 서비스 사이에서 API 요청을 단일 진입점으로 수신하여 라우팅·인증·제한·변환 등을 처리하는 미들웨어
  • 키워드: 단일 진입점, 라우팅, Rate Limiting, 인증/인가, 로드밸런싱, MSA, BFF 패턴
  • 분류: API/웹 기술
  • API Gateway 주요 기능:
    기능설명
    라우팅요청 URL·메서드 기반 서비스 라우팅
    인증/인가JWT·OAuth2 토큰 검증
    Rate Limiting호출 횟수 제한으로 서비스 보호
    로드밸런싱트래픽 분산
    프로토콜 변환REST↔︎SOAP, HTTP↔︎gRPC
    로깅/모니터링API 호출 추적·분석
    캐싱응답 캐시로 백엔드 부하 감소
  • 인사이트: API Gateway는 MSA에서 “공통 횡단 관심사(인증·로깅·제한)”를 집중 처리하여 개별 서비스의 복잡성을 줄이는 필수 인프라이며, Kong·AWS API Gateway·Nginx가 대표적.
