[하위] 1-4. SOAP (Simple Object Access Protocol)

정의: XML 기반 메시지를 HTTP·SMTP 등의 프로토콜로 교환하는 웹 서비스 통신 프로토콜
키워드: XML, WSDL, UDDI, 엔벨로프(Envelope), 헤더, 바디, WS-Security, 엔터프라이즈
분류: API/웹 기술
WSDL/UDDI:
WSDL(Web Services Description Language):
웹서비스가 기술된 정의 파일의 총칭으로 XML을 사용해 기술
UDDI(Universal Description, Discover and Integration):
필요한 서비스를 찾을 수 있는 웹 서비스 레지스트리
SOAP 메시지 구성요소:
구성요소
필수
핵심 내용 및 기능
Envelope (봉투)
필수
◦ SOAP 메시지의 최상위 루트 요소 ◦ 해당 XML 문서가 SOAP 메시지임을 정의하고 네임스페이스 선언 포함
Header (헤더)
선택
◦ 메시지 본문과는 별도로 처리해야 할 메타데이터 포함 ◦ 인증(Authentication), 트랜잭션 관리, 암호화 정보 등 기술
Body (바디)
필수
◦ 실제 전송하고자 하는 비즈니스 데이터(Payload) 포함 ◦ 호출할 서비스 메서드의 이름과 매개변수 정보 등이 기술됨
Fault (폴트)
선택
◦ 메시지 처리 중 발생한 오류 상태 및 상세 정보 기술 ◦ 반드시 Body 요소 내부에 포함되어야 함

인사이트: SOAP은 트랜잭션·보안이 표준화되어 금융·공공 시스템에서 신뢰성이 높으나, REST 대비 복잡성·성능 단점으로 신규 시스템에서는 REST API가 주류.
