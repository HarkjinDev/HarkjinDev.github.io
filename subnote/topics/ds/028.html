15. 코드형 인프라 (IaC, Infrastructure as Code)

  • 정의: 인프라 구성(서버·네트워크·스토리지)을 선언형 코드로 정의하고 버전 관리하여 자동화된 프로비저닝과 일관된 환경을 실현하는 방법론
  • 키워드: Terraform, Ansible, CloudFormation, 선언형, 불변 인프라, GitOps, 드리프트 방지
  • 분류: 클라우드 컴퓨팅
  • 도구 비교:
    도구방식특징
    Terraform선언형멀티클라우드, 상태 파일 관리
    Ansible절차형에이전트리스, 구성 관리 강점
    CloudFormation선언형AWS 전용, 깊은 AWS 통합
  • 인사이트: IaC는 인프라를 개발 코드처럼 관리(Git 버전 관리, PR 리뷰, 자동 테스트)하여 환경 드리프트를 방지하고 재현 가능한 인프라를 구현하는 DevOps 핵심 실천임.
