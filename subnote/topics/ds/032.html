1. API (Application Programming Interface)

  • 정의: 소프트웨어 컴포넌트 간 상호작용을 위해 기능 호출 방식과 데이터 교환 형식을 규정된 인터페이스 명세
  • 키워드: 인터페이스, 엔드포인트, 요청/응답, 계약(Contract), SDK, 표준화
  • 분류: API/웹 기술
  • API 유형 분류:
    유형설명예시
    Open API공개된 외부 개발자 대상 API카카오 지도 API
    Partner API특정 파트너사에 제한 공개B2B 연동 API
    Internal API조직 내부용 비공개 API사내 마이크로서비스
    Composite API여러 API를 조합한 단일 호출BFF 패턴
  • 웹 API 종류 비교:
    종류특징
    REST APIHTTP 메서드 기반, 경량, JSON 응답, 무상태
    SOAPXML 기반, 엄격한 계약, 보안·트랜잭션 강점
    GraphQL클라이언트 주도 쿼리, 오버패칭 방지
    gRPCProtobuf 기반, 고성능 바이너리 통신
  • 인사이트: API는 소프트웨어 간 계약(Contract)으로, 인터페이스가 안정적일수록 독립적 개발·배포가 가능하며 마이크로서비스 아키텍처의 핵심 통신 수단.
