[하위] 1-2. OAS (OpenAPI Specification)

정의: REST API의 구조·엔드포인트·파라미터·응답 형식을 기계 가독 가능한 YAML/JSON으로 기술하는 표준 명세
키워드: Swagger, YAML/JSON, 엔드포인트, 스키마, API 문서 자동화, OpenAPI 3.x
분류: API/웹 기술
OAS 주요 구성 요소:
섹션
설명
info
API 제목·버전·설명
servers
API 서버 URL
paths
엔드포인트별 HTTP 메서드·파라미터·응답 정의
components
재사용 가능한 스키마·파라미터·응답 정의
security
인증 방식 (API Key, OAuth2, JWT 등)
인사이트: OAS는 API 설계→문서화→테스트→코드 생성을 자동화하는 표준으로, Swagger UI를 통해 실시간 API 테스트 환경을 제공하여 개발자 경험(DX)을 향상.
