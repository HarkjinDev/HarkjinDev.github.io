12. 클라우드 네이티브 (Cloud Native)

  • 정의: 클라우드 환경의 탄력성·확장성·자동화를 최대한 활용하도록 MSA·컨테이너·CI/CD·DevOps를 기반으로 설계·개발되는 애플리케이션 방식
  • 키워드: MSA, 컨테이너, CI/CD, DevOps, 선언적 API, 불변 인프라, CNCF, 12 Factor App
  • 분류: 클라우드 컴퓨팅
  • 핵심 원칙 (12 Factor App 기반):
    • 환경 분리(Config), 무상태 프로세스, 로그 스트림, 포트 바인딩, 수평 확장
  • CNCF 핵심 기술 스택: Kubernetes, Prometheus, Envoy, Helm, Argo CD
  • 특성:
    특성설명핵심
    확장성 (Scalability)부하 변화에 따라 자원을 자동으로 수평 확장·축소오토 스케일링
    탄력성 (Elasticity)수요에 따라 즉시 할당·해제하는 유연한 자원 관리동적 자원 조정
    복원력 (Resiliency)장애 발생 시 자동 복구·서비스 연속성 유지자가 치유
    관측가능성 (Observability)메트릭·로그·트레이싱 시스템 상태 실시간 파악시스템 가시성
    자동화 (Automation)빌드·테스트·배포·운영을 코드 기반으로 자동화CI/CD·IaC
    이식성 (Portability)어떤 클라우드·환경에서도 동일하게 실행 가능벤더 종속 탈피
    느슨한 결합 (Loose Coupling)서비스 간 독립성 유지·변경 영향 최소화독립 배포 가능
    선언적 API (Declarative API)목표 상태를 선언하면 시스템이 자동으로 상태 유지상태 기반 관리
    불변 인프라 (Immutable Infrastructure)배포 후 변경 없이 새 이미지로 교체 운영 방식교체 기반 운영
    보안 내재화 (Security by Design)설계 단계부터 보안을 내재화·DevSecOps 적용보안 시프트 레프트
  • 기술요소:
    구분구성요소설명핵심
    프로비저닝컨테이너 (Docker)앱·의존성 패키징한 경량 실행 단위이식성·일관성
    오케스트레이션쿠버네티스 컨테이너 배포·확장·복구 자동화컨테이너 자동 관리
    아키텍처MSA서비스를 독립 배포 가능 단위로 분리독립 배포·확장
    네트워킹API 게이트웨이외부 요청 단일 진입점·라우팅·인증단일 진입점
    서비스 메시 (Istio·Linkerd)서비스 간 통신·보안·트래픽 제어서비스 통신 관리
    CI/CDCI/CD 파이프라인코드 변경부터 배포까지 자동화배포 자동화
    GitOps/ArgoCDGit 기반 인프라·배포 상태 관리Git 기반 운영
    인프라IaC 인프라를 코드로 선언·버전 관리코드 기반 인프라
    오토 스케일링부하 기반 자원 자동 조정자동 자원 조정
    관측가능성Observability(MELT) 메트릭·로그·트레이싱 통합 모니터링시스템 가시성
    보안DevSecOps개발·운영 전 단계 보안 내재화보안 자동화
    제로 트러스트모든 접근 지속 검증·최소 권한신뢰 최소화
  • 인사이트: 클라우드 네이티브는 단순한 클라우드 이전이 아닌 아키텍처 혁신으로, 기존 모놀리식 앱의 MSA 전환 시 데이터 일관성·서비스 간 통신 복잡성을 체계적으로 관리해야 함.
