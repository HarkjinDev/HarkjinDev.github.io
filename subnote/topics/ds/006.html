5. 도커 (Docker)

  • 정의: 애플리케이션과 그 의존성을 컨테이너 이미지로 패키징하여 어떤 환경에서도 동일하게 실행 가능한 오픈소스 컨테이너 플랫폼
  • 키워드: Dockerfile, 이미지, 컨테이너 레지스트리, Docker Hub, 레이어, OCI, BuildKit
  • 분류: 가상화/컨테이너
  • 핵심 구성:
    구성요소설명세부 기술
    Client데몬과 상호작용하는 Binary 파일CLI·Remote API
    이미지 (Images)프로그램·라이브러리 설치한 파일 패키지aufs·btrfs·devicemapper
    컨테이너 (Container)이미지 실행 상태·다수 생성 가능LXC·namespace·cgroups·SELinux
    데몬 (Daemon)호스트에서 컨테이너 관리하는 프로세스service
    레지스트리 (Registry)이미지 저장·공유·배포 저장소docker hub·docker registry
    네트워크호스트 간 컨테이너 데이터 전송Network overlay
    오케스트레이션멀티 컨테이너 조직화·Failover·LBswarm·mesos·kubernetes
    모니터링컨테이너·앱·로그 중앙 모니터링cAdvisor·Sensu·Prometheus
  • 이점: 환경 일관성(Dev=Staging=Prod), 빠른 배포, 마이크로서비스 분리
  • 인사이트: Docker는 DevOps 파이프라인의 표준 패키징 단위로, "환경 차이로 인한 버그" 문제를 근본적으로 해결하며 Kubernetes와 결합하면 전체 컨테이너 수명주기 자동화가 가능함.
