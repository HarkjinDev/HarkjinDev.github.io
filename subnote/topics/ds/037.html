[하위] 1-5. REST API (REpresentational State Transfer API)

  • 정의: 자원을 URI로 식별하고 HTTP 메서드로 CRUD 연산을 수행하는 REST 아키텍처 기반 HTTP API
  • 키워드: HTTP 메서드(GET/POST/PUT/DELETE/PATCH), 상태코드, URI 설계, JSON
  • 분류: API/웹 기술
  • HTTP 메서드와 CRUD 매핑:
    HTTP 메서드CRUD특성예시
    GETRead안전·멱등GET /users/1
    POSTCreate비멱등POST /users
    PUTUpdate (전체)멱등PUT /users/1
    PATCHUpdate (부분)비멱등PATCH /users/1
    DELETEDelete멱등DELETE /users/1
  • Richardson 성숙도 모델:
    레벨설명
    Level 0단일 URI, 단일 메서드
    Level 1자원별 URI 분리
    Level 2HTTP 메서드 적절히 활용
    Level 3HATEOAS 적용 (진정한 REST)
  • 인사이트: 실무에서는 Level 2가 일반적이며, 멱등성(Idempotency)이 API 신뢰성의 핵심으로 PUT·DELETE는 반복 호출 시 동일 결과를 보장해야 함.
