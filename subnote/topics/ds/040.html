3. 웹 성능 최적화 (Web Performance Optimization)

  • 정의: 웹 페이지의 로딩 속도와 사용자 체감 성능을 개선하기 위한 기술적 전략과 기법의 총칭
  • 키워드: Core Web Vitals, LCP, FID, CLS, CDN, 레이지 로딩, 코드 분할, 캐싱, SEO
  • 분류: API/웹 기술
  • Core Web Vitals (Google 핵심 지표):
    지표의미기준
    LCP (Largest Contentful Paint)주요 콘텐츠 렌더링 시간2.5초 이하
    FID (First Input Delay)첫 사용자 입력 응답 시간100ms 이하
    CLS (Cumulative Layout Shift)레이아웃 시각적 안정성0.1 이하
  • 주요 최적화 기법:
    구분항목설명핵심
    이미지 최적화WebP 변환고효율 포맷으로 용량 축소용량 절감
    레이지 로딩시각적 영역만 우선 로드지연 로딩
    코드 최적화코드 분할필요 시점에 필요 코드 로드모듈 최적화
    전송 최적화CDN 활용근접 서버에서 콘텐츠 제공전송 속도↑
    캐싱 전략HTTP 캐싱브라우저에 자원 임시 저장재사용성
    서비스 워커오프라인 지원 및 캐싱 제어PWA 구현
    번들링·압축Gzip/Brotli파일 압축으로 전송량 감소전송량 최소화
    트리 쉐이킹미사용 코드 자동 제거코드 경량화
  • SEO(Search Engine Optimization, 검색 엔진 최적화): 웹사이트가 광고가 아닌 일반 검색 결과(Organic Search) 상단에 노출되도록 관리하는 모든 기술적, 전략적 활동을 의미
    영역핵심 활동비유
    기술적 SEO (Technical)사이트 속도 최적화, 모바일 대응, 보안(HTTPS)식당의 쾌적한 시설/서비스
    콘텐츠 SEO (On-Page)고품질 정보 제공, 키워드 배치, 제목 태그 최적화메뉴의 맛과 영양가
    오프페이지 SEO (Off-Page)외부 사이트로부터의 인용(백링크), 브랜드 언급단골 손님의 입소문과 추천
  • 인사이트: 웹 성능은 사용자 이탈률과 전환율에 직접 영향을 미치며, Google의 Core Web Vitals가 검색 순위 알고리즘에 반영되어 SEO와 성능 최적화가 연계된 전략 필요.
