3. ELK 스택 (Elasticsearch / Logstash / Kibana)

  • 정의: 로그 수집(Logstash)·저장·검색(Elasticsearch)·시각화(Kibana)를 통합한 오픈소스 로그 분석 플랫폼
  • 키워드: 로그 수집, 분산 검색, 시각화 대시보드, Beats, 역색인(Inverted Index), ELK, Elastic Stack, 관찰가능성
  • 분류: 미들웨어/아키텍처
  • ELK 스택 구성 요소:
    구성 요소역할특징
    Elasticsearch분산 검색·저장 엔진역색인 기반 빠른 전문 검색, RESTful API
    Logstash데이터 수집·변환·파이프라인다양한 소스 입력, 필터·출력 플러그인
    Kibana데이터 시각화·대시보드실시간 차트·지도·대시보드
    Beats경량 데이터 수집 에이전트Filebeat(로그)·Metricbeat(메트릭)
  • ELK 데이터 흐름:
    • 서버/앱 로그 → Beats(에이전트) → Logstash(수집·변환) → Elasticsearch(저장·색인) → Kibana(시각화)
  • Elasticsearch 역색인 원리:
    • 일반 색인: 문서 → 단어 목록
    • 역색인: 단어 → 해당 단어가 포함된 문서 목록 → 전문 검색 고속화
  • 인사이트: ELK는 DevOps·MSA 환경의 중앙화 로그 관리(Centralized Logging) 표준 스택으로, 최근 Elastic사의 라이선스 변경(SSPL)으로 AWS의 OpenSearch가 대안으로 부상.
