[하위] 1-3. REST (Representational State Transfer)

  • 정의: 자원(Resource)을 URI로 식별하고 HTTP 메서드로 상태를 전이시키는 분산 하이퍼미디어 시스템의 아키텍처 스타일
  • 키워드: 자원(URI), HTTP 메서드(상태), 무상태(Stateless), 균일한 인터페이스, 캐시 가능, 6대 제약
  • 분류: API/웹 기술
  • REST 6대 제약 조건:
    제약설명
    클라이언트-서버UI와 데이터 저장 분리
    무상태 (Stateless)각 요청은 독립적, 세션 미보관
    캐시 가능 (Cacheable)응답에 캐시 가능 여부 표시
    계층화 시스템중간 계층(프록시·게이트웨이) 투명
    균일한 인터페이스자원 식별·자기 서술·HATEOAS
    코드 온 디맨드 (선택)서버가 실행 코드 전송 가능
  • 인사이트: REST는 아키텍처 스타일이지 프로토콜이 아님을 명확히 하며, 6대 제약을 모두 준수할 때 진정한 RESTful로 인정되고 특히 무상태와 균일 인터페이스가 핵심.
