4. 버전관리 시스템 (Version Control System)

  • 정의: 소스코드와 파일의 변경 이력을 추적·관리하고 협업을 지원하는 시스템
  • 키워드: Git, SVN, 분산형/중앙집중형, 브랜치, 머지, 커밋, 형상관리
  • 분류: API/웹 기술
  • 분산형 vs 중앙집중형 비교:
    구분분산형 (Git)중앙집중형 (SVN, CVS)
    저장소로컬+원격 (모두 전체 이력)중앙 서버만
    오프라인 작업가능불가
    브랜칭가볍고 빠름상대적으로 무거움
    네트워크 의존낮음높음
    복구로컬 복사본 활용서버 장애 시 위험
  • Git 브랜치 전략:
    구분Git FlowGitHub FlowTrunk-based
    주요 브랜치Main, Develop, Feature, Release, HotfixMain, Topic (Feature)Main (Trunk)
    복잡도매우 높음낮음중간
    배포 주기정기적 배포 (배포 단위)수시로 (지속적 배포)실시간 (매우 빠름)
    적합한 팀대규모, 엄격한 버전 관리 필요소규모 팀, 빠른 웹 서비스숙련된 팀, CI/CD 자동화 완성
    안정성매우 높음 (다단계 검증)중간 (PR 리뷰 중심)높음 (강력한 자동 테스트 전제)
  • 인사이트: Git은 현재 사실상 표준(De facto Standard)으로, 브랜치 전략은 팀 규모·배포 주기에 따라 선택하며 CI/CD 파이프라인과 연계하여 자동화 효과를 극대화.
