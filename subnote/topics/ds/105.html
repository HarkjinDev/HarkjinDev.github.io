1. EDA (Event Driven Architecture)

  • 정의: 시스템의 상태 변화를 이벤트로 표현하고, 이벤트의 생산·전달·소비를 기반으로 서비스 간 비동기 통신을 구성하는 아키텍처 패턴
  • 키워드: 이벤트(Event), 프로듀서(Producer), 컨슈머(Consumer), 이벤트 브로커, 비동기, 느슨한 결합, Kafka, CQRS, 이벤트 소싱
  • 분류: 미들웨어/아키텍처
  • EDA 핵심 구성 요소:
    구성 요소역할
    이벤트 프로듀서상태 변화 시 이벤트 생성·발행
    이벤트 브로커이벤트 수신·저장·라우팅 (Kafka, RabbitMQ)
    이벤트 컨슈머이벤트 구독·처리
    이벤트 스토어이벤트 영속적 저장 (이벤트 소싱 시)
  • EDA 패턴 유형:
    패턴설명
    Pub/Sub주제(Topic) 기반 다중 컨슈머 구독
    Event Streaming이벤트 스트림 순서 보장·재처리 (Kafka)
    Event Sourcing상태 변화를 이벤트 시퀀스로 저장
    CQRS + EDA커맨드 이벤트로 읽기·쓰기 모델 동기화
  • EDA 장단점:
    구분내용
    장점서비스 간 느슨한 결합, 확장성, 비동기 처리
    단점이벤트 순서·중복 처리, 분산 추적 복잡, 최종 일관성
  • 인사이트: EDA는 MSA와 자연스럽게 결합되어 서비스 간 직접 호출을 이벤트 기반 비동기로 대체하며, 최종 일관성(Eventual Consistency)을 수용하는 설계 철학이 필수 전제.
