2. 관측가능성 (Observability)

  • 정의: 시스템의 문제 해결 및 자동화를 위해 외부 출력값인 메트릭(Metric), 추적(Trace), 로그(Log) 등을 분석하여 시스템의 내부 상태를 이해하고 예측할 수 있는 속성
  • 키워드: MELT-메트릭(Metric), 이벤트(Event), 추적(Trace), 로그(Log), SRE, SLO, 외부 출력값, 내부 상태 파악, AIOps
  • 분류: 신기술/동향
  • 관측가능성과 모니터링의 비교:
    비교 항목관측가능성 (Observability)모니터링 (Monitoring)
    사용 목적복잡한 오류를 추적발생한 오류를 발견
    측정 값 활용시스템 오작동 원인을 확인시스템 영향도를 식별
    데이터 상관관계데이터의 상관관계 파악이 용이데이터의 상관관계 파악이 어려움
    분석 대상여러 시스템을 연관 분석독립형 시스템을 분석
    분석 결과원인 및 과정을 도출시점 및 종류를 도출
  • 관측가능성 핵심 구성 요소(MLT/MELT):
    구성 요소역할 및 메커니즘 (요약)주요 출력 형태
    메트릭 (Metric)가용성 신호의 수치화 및 집계CPU/메모리 사용량, 메시지 개수
    이벤트 (Event)특정 시점에 발생한 상태 변화·행위 기록배포 이벤트·알람 발생·사용자 클릭
    로그 (Log)실행 시 발생하는 텍스트/JSON 기록에러, 경고, 디버깅 정보
    추적 (Tracing)트랜잭션의 이동 경로 및 지연 시간 분석시스템 이동 경로, 처리 대기 시간
  • 기반 기술 및 세부 구현 기술:
    기반 기술세부 기술핵심 메커니즘 (요약)
    트래픽 관리eBPF커널 내 샌드박스 실행 및 Hook 활용 모니터링
    복원성 패턴서비스 메시·서킷브레이커 기반 수평 확장 배포
    오토 스케일링메트릭 측정 요청 수, 요청 기간, 동시 요청 등
    HPA메트릭 선정, 스래싱 방지, 지연 반응
    운영 프로세스관측가능성 운영메트릭 → 추적 ↔ 로그 순서의 연계 분석
    장애 프로세스장애 대응장애 발생 → 알람 전송 → 추적 분석 → 로그 분석
    마이크로서비스CQRS읽기/쓰기 모델 분리를 통한 독립적 최적화
    파티셔닝샤딩키 기반 데이터 분산 및 특정 샤드 부하 해소
  • 인사이트: 관측가능성은 외부 출력값을 분석하여 시스템 내부 상태를 파악한다는 점에서 단순 모니터링과 차별화되며, SRE를 통해 SLO를 정의하고 보호하는 효율적인 운영을 실현함. 향후 IT 운영 문제를 자동 파악하여 해결하는 AIOps를 적용함으로써 운영 자동화 체계로 발전할 것으로 예상됨.
