2. SDN (Software Defined Networking)

  • 정의: 네트워크 장비의 제어 평면(Control Plane)을 데이터 평면(Data Plane)에서 분리하여 중앙 집중식 소프트웨어 컨트롤러로 네트워크를 프로그래밍 방식으로 제어하는 기술
  • 키워드: ONF, 제어 평면 분리, 오픈플로우, 컨트롤러, 사우스바운드/노스바운드 API, 프로그래밍
  • 분류: 네트워크 가상화/SDN
  • SDN 3계층 구조:
    계층구성요소인터페이스
    응용 계층네트워크 앱(방화벽, LB, 모니터링)노스바운드 API(REST)
    제어 계층SDN 컨트롤러(ONOS, OpenDaylight)사우스바운드 API
    인프라 계층네트워크 스위치/라우터(데이터 평면)OpenFlow
  • 기존 네트워크 vs SDN 비교:
    구분기존SDN
    제어 위치분산(장비별)중앙 집중(컨트롤러)
    설정 방식CLI, SNMPAPI, 프로그래밍
    유연성낮음높음
    가시성제한적전체 토폴로지
  • 인사이트: SDN의 핵심은 지능(제어)과 전달(포워딩)의 분리로, 네트워크를 소프트웨어로 추상화하여 자동화와 민첩성을 확보. 단일 컨트롤러 장애 시 SPOF(단일 장애점) 위험이 있으므로 컨트롤러 HA 구성 필수.
