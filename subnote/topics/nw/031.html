3. OSPF (Open Shortest Path First)

  • 정의: 링크 상태 정보를 AS 내 모든 라우터에 플러딩하여 각 라우터가 SPF 알고리즘으로 최단 경로를 계산하는 링크 상태 기반 IGP 프로토콜
  • 키워드: RFC 2328, LSA, LSDB, SPF·Dijkstra, Area, DR·BDR, Cost, Hello, ABR, ASBR, OSPFv3
  • 분류: 라우팅 / 주소체계
  • 동작 절차:
    단계설명핵심
    1. 인접 관계 수립Hello 패킷으로 인접 라우터 발견·Neighbor 관계 형성Hello 프로토콜
    2. DR·BDR 선출멀티액세스 네트워크에서 대표 라우터 선출 * DR(지정 라우터), BDR(백업 DR)플러딩 최소화
    3. LSA 교환Link State Advertisement 교환으로 LSDB 동기화토폴로지 공유
    4. SPF 계산Dijkstra 알고리즘으로 최단 경로 트리 계산최단 경로 산출
    5. 라우팅 테이블 갱신계산된 최단 경로를 라우팅 테이블에 반영경로 적용
  • LSA 유형: Type 1(Router LSA), Type 2(Network LSA), Type 3(Summary LSA), Type 4(ASBR Summary LSA), Type 5(External LSA)
  • Cost 계산:
    항목설명핵심
    공식Cost = 10⁸ / 링크 속도(bps)속도 반비례
    FastEthernetCost = 10⁸ / 10⁸ = 1최저 Cost
    T1 (1.544Mbps)Cost = 10⁸ / 1,544,000 ≈ 64높은 Cost
    경로 선택누적 Cost 합산 최솟값 경로 선택최소 Cost 우선
  • 인사이트: OSPF의 Area 설계는 네트워크 확장성의 핵심으로, Area 0은 반드시 존재해야 하며 모든 Area는 Area 0에 직·간접 연결되어야 함. Cost = 10⁸/링크속도로 고속 링크를 선호하며, DR·BDR 선출로 멀티액세스 환경의 LSA 플러딩을 최소화하는 설계가 대규모 네트워크 안정성의 핵심 포인트임
