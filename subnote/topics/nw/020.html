[하위] 6-4. CRC (Cyclic Redundancy Check)

  • 정의: 데이터를 생성 다항식으로 나눈 나머지(체크섬)를 프레임에 부가하여 전송 오류를 검출하는 순환 중복 검사 기법
  • 키워드: 생성 다항식, 나머지, FCS, 코드워드, GF(2) 나눗셈, 버스트 오류
  • 분류: 전송 프로토콜
  • CRC 계산 절차:
    • ① 데이터 비트열 뒤에 생성 다항식 차수만큼 0 추가
    • ② GF(2)(XOR) 나눗셈 수행
    • ③ 나머지 = CRC 체크섬 → FCS 필드에 첨부
    • ④ 수신측: 동일 나눗셈 후 나머지 = 0이면 정상
  • 주요 표준:
    표준다항식적용
    CRC-16x¹⁶+x¹⁵+x²+1HDLC, USB
    CRC-3232차 다항식Ethernet, ZIP
    CRC-CCITTx¹⁶+x¹²+x⁵+1SDLC, X.25
  • 인사이트: CRC는 버스트 오류 검출에 탁월하지만 오류 정정은 불가. Ethernet 프레임의 FCS가 CRC-32를 사용하며, CRC는 검출만 가능하므로 오류 정정은 ARQ/FEC 필요.
