4. TCP 3-way Handshake

정의: TCP 연결 설정 시 SYN→SYN-ACK→ACK 3단계 메시지 교환으로 양방향 연결을 수립하는 절차
키워드: SYN, SYN-ACK, ACK, ISN, 연결 상태, LISTEN, SYN_SENT, ESTABLISHED
분류: 전송 프로토콜
절차: //그림 추가
보안 취약점: SYN Flooding 공격 — 대량 SYN으로 서버 backlog 소진 → SYN Cookie로 대응
인사이트: 3-way Handshake는 상호 ISN 교환을 통해 양방향 순서번호를 동기화하며, SYN Flooding DDoS 공격의 타깃이 됨. SYN Cookie는 메모리 없이 상태를 검증하는 핵심 대응 기법.
