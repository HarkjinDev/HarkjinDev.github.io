[하위] 6-3. FEC (Forward Error Correction, 전진 오류 정정)

  • 정의: 송신측이 데이터에 잉여 비트(Redundancy)를 추가하여 전송하고 수신측이 별도 재전송 요청 없이 스스로 오류를 검출·정정하는 채널 부호화 기술
  • 키워드: 채널 부호화, 잉여 비트, 부호율, 해밍 코드, 터보 코드, LDPC, Polar Code, BCH, 리드-솔로몬
  • 분류: 네트워크 / 통신 이론
  • 핵심 개념:
    항목설명핵심
    부호율 (Code Rate)정보 비트 수 / 전체 전송 비트 수 (낮을수록 강한 오류 정정)정정 능력 조절
    해밍 거리두 코드워드 간 다른 비트 수, 클수록 오류 정정 능력 향상정정 능력 기준
    잉여 비트 (Redundancy)오류 검출·정정을 위해 추가하는 패리티·체크 비트오류 정정 재료
    코드워드정보 비트 + 잉여 비트로 구성된 전송 단위전송 단위
  • FEC 주요 알고리즘:
    구분알고리즘설명핵심
    블록 코드해밍 코드 (Hamming)1비트 정정 및 2비트 검출단순 1비트 정정
    BCH 코드다중 비트 오류 정정 가능다중 비트 정정
    리드-솔로몬 (RS)버스트 오류 정정 특화 부호버스트 오류 정정
    컨볼루션 코드비터비 알고리즘컨볼루션 부호 최적 복호법최적 복호
    터보 코드터보 코드 (Turbo)샤논 한계 도달 반복 복호반복 복호
    LDPCLDPC 코드희소 행렬 기반 5G 부호5G 표준 부호
    Polar 코드Polar Code채널 분극 이론 기반 5G 부호5G 제어 채널
  • FEC vs BEC 비교표:
    항목FEC (전진 오류 정정)BEC (후진 오류 정정)
    정정 방식수신측 자체 정정재전송 요청 (ARQ)
    피드백 채널❌ 불필요✅ 필요
    지연없음재전송 RTT 발생
    오버헤드잉여 비트 추가재전송 트래픽
    오류율 대응높은 오류율 적합낮은 오류율 적합
    적합 환경위성·방송·무선유선·저지연 환경
    대표 기술터보코드·LDPC·PolarStop-and-Wait·GBN·SR
    결합 방식—HARQ (FEC + BEC 혼합)
  • 인사이트: FEC는 재전송 없이 오류를 자체 정정하므로 위성·방송·5G처럼 피드백 채널이 없거나 지연이 허용되지 않는 환경의 필수 기술임. 터보 코드→LDPC→Polar Code로 이어지는 발전은 Shannon 정보 이론의 한계에 점점 근접하는 과정이며, 5G에서 데이터 채널(LDPC)과 제어 채널(Polar)을 구분 적용하는 설계가 이론적 최적 성능 실현의 핵심임
