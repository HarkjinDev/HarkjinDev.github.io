4. Tor (The Onion Router)

  • 정의: 데이터를 다중 암호화해 여러 중계 노드를 경유시켜 발신자를 은닉하는 익명 통신 네트워크
  • 키워드: 양파 라우팅, 회로, Guard/Middle/Exit 릴레이, Hidden Service, .onion, 익명성
  • 분류: VPN
  • Tor vs VPN 비교:
    구분VPNTor
    목적보안 터널 + 우회익명성 극대화
    신뢰 요구VPN 제공자 신뢰 필요신뢰 노드 불필요
    속도빠름느림(다중 홉)
    추적 가능성VPN 로그 존재 가능매우 어려움
    합법성대부분 합법국가별 차이
  • 익명성 한계: Exit 노드가 평문 트래픽 감청 가능, 트래픽 분석 공격(상관관계 분석)에 취약
  • 인사이트: Tor는 완전한 익명성을 제공하지 않으며, Exit 노드 공격과 타이밍 분석에 취약. .onion 주소의 Hidden Service는 서버와 클라이언트 모두 익명성 보장으로 다크웹에 활용.

4. Tor (The Onion Router)

  • 정의: 데이터를 다중 암호화해 여러 중계 노드를 경유시켜 발신자를 은닉하는 익명통신망
  • 키워드: 양파 라우팅, 회로, Guard/Middle/Exit 릴레이, Hidden Service, .onion, 트래픽 분석
  • 분류: VPN
  • Tor 네트워크 구성요소 (3-Hop 릴레이):
    구분릴레이설명핵심
    1홉Guard Relay클라이언트 직접 연결 노드IP 노출 지점
    2홉Middle Relay중간 데이터 중계 노드경로 은닉
    3홉Exit Relay목적지로 최종 전달평문 노출 가능
  • 양파 라우팅 동작 원리:
    단계설명핵심
    1. 회로 구성3개 노드 선택해 경로 설정회로 생성
    2. 다중 암호화공개키로 계층별 순차 암호화양파 껍질 구조
    3. 순차 복호화각 노드가 한겹씩 복호화경로 정보 분산
    4. 데이터 전달Exit가 최종 복호화 전송평문 전달
    5. 회로 갱신약 10분마다 회로 재구성추적 방지
  • Hidden Service (.onion):
    항목설명핵심
    개념실제 IP 없이 서비스 제공서버 익명성
    접속 방식양측 모두 Tor 회로 경유양방향 익명
    주소 체계공개키 기반 자체 생성DNS 불필요
    활용다크웹·검열우회 서비스익명 서비스
  • Tor vs VPN 비교:
    구분VPNTor
    목적보안 터널 + 우회익명성 극대화
    신뢰 요구VPN 제공자 신뢰 필요신뢰 노드 불필요
    경유 노드 수1개 (VPN 서버)3개 (Guard·Middle·Exit)
    운영 주체단일 사업자전세계 자원봉사 릴레이
    속도빠름느림 (다중 홉)
    추적 가능성VPN 로그 존재 가능매우 어려움
    합법성대부분 합법국가별 차이
  • 익명성 한계 및 공격 유형:
    공격 유형설명핵심
    Exit 노드 감청평문 트래픽 감청·조작HTTPS 필수
    트래픽 분석입출력 패턴 상관분석국가급 공격 취약
    Guard 노드 식별고정 진입점 장기 노출첫 홉 IP 노출
    악성 Exit 노드공격자 운영 노드 존재데이터 탈취
    브라우저 핑거프린팅설정·플러그인 기반 식별탈익명화 위험
    DoS 공격릴레이 과부하로 저하가용성 저하
  • 인사이트: Tor는 완전한 익명성을 제공하지 않으며 Exit 노드 감청과 트래픽 상관관계 분석에 구조적으로 취약하므로 HTTPS 병행이 필수이고, Hidden Service(.onion)는 서버·클라이언트 양방향 익명성을 보장해 다크웹·검열우회에 활용되나 국가급 공격자에게는 탈익명화 가능성이 존재함
