10. IoT 프로토콜 (XMPP/CoAP/MQTT)

  • 정의: IoT 기기의 제한된 자원과 불안정한 네트워크 환경에서 효율적인 메시지 통신을 지원하는 경량 응용 계층 프로토콜들
  • 키워드: MQTT, CoAP, XMPP, Pub/Sub, REST, QoS, 브로커, 경량, M2M
  • 분류: 이동통신/무선
  • 3가지 프로토콜 비교:
    구분MQTTCoAPXMPP
    전송TCPUDPTCP
    패턴Pub/Sub요청-응답(REST)IM 스타일
    브로커필요불필요서버 필요
    오버헤드매우 낮음매우 낮음중간
    QoS0/1/2 레벨확인/비확인없음
    주요 용도센서, IoT 플랫폼스마트 에너지실시간 메시징
  • 인사이트: MQTT는 IoT 플랫폼(AWS IoT, Azure IoT)의 사실상 표준으로 Pub/Sub 모델과 낮은 오버헤드가 장점. CoAP는 HTTP와 유사한 REST 방식으로 웹 생태계와 통합이 용이.
