3. Switch (스위치)

  • 정의: 수신 프레임·패킷을 목적지 주소 기반으로 해당 포트로 전달하는 네트워크 장비
  • 키워드: L2/L4 스위치, MAC Table, VLAN, Load Balancing, QoS, cut-through, SDN
  • 분류: LAN
  • L2 스위치 기능:
    기능설명핵심
    learning출발지 MAC 없으면 학습 저장MAC 학습
    flooding목적지 없으면 전체 포트 전달전체 포트 전송
    forwarding목적지 있으면 해당 포트만 전달단일 포트 전달
    filtering동일 세그먼트면 전달 차단세그먼트 차단
    aging일정시간 지나면 주소 삭제주소 자동 삭제
  • L2 전송방식:
    전송방식설명특징
    cut-through목적지 주소만 확인 후 즉시 전달속도 빠름, 에러감지 어려움
    store-and-forward전체 프레임 수신·검사 후 전달속도 느림, 에러감지 용이
    fragment free앞 64바이트만 확인 후 전달속도·에러감지 절충
  • L2 부가기능:
    구분구성요소핵심 기술
    VLAN포트를 가상 LAN으로 그룹핑802.1q
    Link Aggregation포트 여러 개 묶어 대역폭 확장802.3ad
    STP루프 방지, 단일 경로만 사용802.1d
    Port Mirroring특정 포트 트래픽 복사 분석SPAN
  • L4 스위치 기능:
    기능설명특징
    부하분산서버 트래픽을 여러 대에 분산서버 가상화
    캐싱 서비스투명한 캐싱 서비스 제공사용자 설정 불필요
    서비스 연결트래픽 종류별 다른 서비스 연결포트 기반 인식
    QoS우선순위에 따라 대역폭 차등 할당서비스 포트 이용
  • L4 서버 감시 방식:
    구분감시방식핵심
    ICMP 이용Ping Request/Response 확인가장 단순
    TCP Connection 이용SYN 전송 후 SYN-ACK 확인ICMP보다 정확
    HTTP Request 이용Request 후 Response 상태 확인가장 확실
  • L4 스위치 부하분산 알고리즘:
    구분항목설명핵심
    정적기반Round Robin요청을 순서대로 균등 분배순차 균등 분배
    HashingIP 조합 해시값으로 노드 선택해시값 기반 선택
    Fixed소스 IP 주소로 서버 결정소스IP 고정
    동적기반Weighted Round Robin가중치 부여해 트래픽 차등 할당가중치 차등 분배
    Least Connection연결 수 최소 서버로 연결최소 연결 우선
    Weighted Least Connection가중치와 연결수 고려해 분산가중치+연결수
  • 인사이트: L2는 MAC 기반 프레임 스위칭, L4는 포트·세션 기반 부하분산·QoS로 계층별 역할이 분리됨
