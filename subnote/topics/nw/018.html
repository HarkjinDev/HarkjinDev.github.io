[하위] 6-2. HARQ (Hybrid Automatic Repeat reQuest)

  • 정의: ARQ의 재전송 기법과 FEC의 전진 오류 정정을 결합하여 오류 패킷을 재전송 시 이전 수신 신호와 합산·정정함으로써 전송 효율과 신뢰성을 동시에 향상시키는 오류 제어 기술
  • 키워드: ARQ, FEC, 소프트 컴바이닝, Chase Combining, IR, 터보코드, LDPC, ACK/NACK, 재전송
  • 분류: 네트워크 / 이동통신
  • 동작 원리:
    단계설명핵심
    1. 초기 전송송신측이 FEC 부호화된 패킷 전송부호화 전송
    2. 수신·복호수신측이 FEC로 오류 정정 시도자체 정정
    3. ACK·NACK정정 성공 시 ACK, 실패 시 NACK 전송결과 피드백
    4. 재전송NACK 수신 시 동일 또는 추가 패킷 재전송재전송 수행
    5. 소프트 컴바이닝이전 수신 신호와 재전송 신호를 합산하여 복호신호 합산 복호
  • HARQ 유형:
    유형설명핵심
    Type I HARQ재전송 시 동일 패킷 반복 전송·합산 복호단순 재전송
    Type II HARQ (CC)Chase Combining, 동일 패킷 재전송 후 소프트 합산동일 패킷 합산
    Type III HARQ (IR)Incremental Redundancy, 재전송마다 추가 잉여 비트 전송점진적 잉여 추가
  • HARQ 프로세스 구조:
    항목설명핵심
    다중 HARQ 프로세스여러 HARQ 프로세스 병렬 운용으로 RTT 지연 보상병렬 전송
    Stop-and-Wait각 프로세스는 ACK 수신 전까지 대기단순 제어
    최대 재전송 횟수보통 4회로 제한, 초과 시 상위 계층 ARQ 처리재전송 한계
    RTT (왕복 지연)LTE 8ms, 5G NR 최소 1ms 수준응답 지연
  • 인사이트: HARQ는 재전송 패킷을 버리지 않고 누적 합산하는 소프트 컴바이닝이 핵심으로, Chase Combining보다 Incremental Redundancy가 스펙트럼 효율이 높아 LTE·5G의 표준 방식으로 채택됨. 5G에서는 CBG 단위 부분 재전송·비동기 HARQ로 지연을 최소화하여 URLLC(초저지연·고신뢰) 서비스의 핵심 기반 기술로 자리잡고 있음
