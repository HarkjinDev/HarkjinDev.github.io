4. BGP (Border Gateway Protocol)

  • 정의: 다양한 속성의 정책을 통한 AS 간 경로 정보 전달하는 인터넷 표준 경로벡터 기반 EGP 프로토콜
  • 키워드: AS, eBGP·iBGP, 경로속성, AS-Path, BGP 테이블, 정책기반 라우팅, TCP 179
  • 분류: 라우팅 / 주소체계
  • BGP 핵심 특징:
    특징설명핵심
    경로벡터 방식거리(홉수) 대신 AS 경로 정보로 판단AS-Path 기반
    신뢰성 전송TCP 179번 포트 사용연결지향 전송
    정책기반 라우팅다양한 속성으로 세밀한 정책 적용정책 중심
    루프 방지자신의 AS번호 포함 시 경로 폐기AS-Path 검사
    증분 업데이트변경분만 전송·주기적 갱신 없음트리거 업데이트
    대규모 확장성인터넷 전체 라우팅 테이블 처리90만+ 경로 처리
  • BGP 동작 방식:
    항목설명핵심
    eBGP서로 다른 AS 간 BGP 세션외부 AS 연결
    iBGP동일 AS 내부 BGP 세션내부 경로 전파
    피어링 (Peering)BGP 라우터 간 TCP 179 포트로 세션 수립세션 기반 연결
    NLRI도달 가능한 네트워크 정보를 포함한 경로 광고경로 정보 교환
  • BGP 경로 속성:
    속성설명핵심
    AS-PATH경로가 통과한 AS 번호 목록, 루프 방지에 활용루프 방지
    NEXT-HOP다음 라우터의 IP 주소다음 홉 지정
    LOCAL-PREFAS 내부에서 선호 경로 결정 (높을수록 우선)내부 선호도
    MED외부 AS에 선호 진입 경로 전달 (낮을수록 우선)외부 선호도
    COMMUNITY경로 태깅으로 그룹 정책 적용정책 그룹화
    ORIGIN경로 정보 출처 (IGP·EGP·Incomplete)경로 출처
  • 인사이트: BGP는 홉수 대신 AS-Path·Local Preference·MED 등 다양한 경로속성으로 정책기반 라우팅을 수행하는 유일한 EGP로, eBGP는 AS 간, iBGP는 AS 내부 전파를 담당하며 Route Reflector로 Full-Mesh 확장성 한계를 극복하는 것이 핵심
