8. 혼잡 제어 (Congestion Control)

정의: 네트워크 혼잡 발생 시 전송률을 동적으로 조절하여 패킷 손실과 지연을 최소화하는 트래픽 관리 기법
키워드: 혼잡 윈도우(cwnd), Slow Start, AIMD, 혼잡 회피, Fast Retransmit, ECN
분류: 전송 프로토콜
TCP 혼잡 제어 단계:
단계
동작
설명
Slow Start
cwnd 지수 증가
1→2→4→8… (매 RTT 2배)
혼잡 회피
cwnd 선형 증가
ssthresh 도달 후 +1/RTT
Fast Retransmit
3 Dup ACK → 즉시 재전송
타임아웃 대기 없이 빠른 복구
Fast Recovery
ssthresh 재설정 후 혼잡 회피 진입
cwnd = ssthresh + 3
인사이트: AIMD(Additive Increase Multiplicative Decrease)는 공정성을 보장하는 핵심 원리로, 손실 감지 시 cwnd를 절반으로 줄여 빠른 회복을 유도. QUIC/BBR 등 신규 혼잡 제어는 RTT 기반으로 동작하여 대역폭 활용률을 개선.
