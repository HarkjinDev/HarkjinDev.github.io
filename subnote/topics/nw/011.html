1. 소켓 통신 (Socket Communication)

정의: IP 주소와 포트 번호를 결합한 소켓을 통해 프로세스 간 양방향 데이터를 교환하는 네트워크 통신 방식
키워드: IP+Port, 소켓 API, BSD 소켓, 스트림 소켓, 데이터그램 소켓, 전이중
분류: 전송 프로토콜
소켓 유형 비교:
구분
스트림 소켓(TCP)
데이터그램 소켓(UDP)
웹소켓
연결 방식
연결 지향
비연결
HTTP 업그레이드 후 연결 유지
신뢰성
보장
비보장
보장(TCP 기반)
주요 용도
파일 전송, HTTP
스트리밍, DNS
실시간 웹 앱
소켓 통신 흐름: //그림 표현 필요
서버: socket()→bind()→listen()→accept()
클라이언트: socket()→connect()
통신: send()/recv()
종료: close().
HTTP vs 웹소켓: HTTP는 요청-응답 단방향 반이중, 웹소켓은 핸드셰이크 후 전이중 유지
인사이트: 소켓은 OS 커널이 제공하는 추상화 인터페이스로, 애플리케이션이 네트워크 세부 사항을 몰라도 통신 가능. 실시간 서비스에서는 WebSocket, IoT에서는 MQTT over TCP가 활용됨.
