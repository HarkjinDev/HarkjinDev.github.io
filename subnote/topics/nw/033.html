4. FTP/FTPS/SFTP

  • 정의: 파일 전송을 위한 프로토콜 및 보안이 강화된 확장 버전들의 집합
  • 키워드: RFC 959, 포트 21/22, 능동/수동 모드, TLS, SSH, 제어 채널, 데이터 채널
  • 분류: 프로토콜
  • 3가지 프로토콜 비교:
    구분FTPFTPSSFTP
    보안없음(평문)SSL/TLS 암호화SSH 기반 암호화
    제어 포트2121(또는 990)22
    채널 구조제어+데이터 분리제어+데이터 분리단일 채널
    기반 프로토콜TCPTCP + TLSSSH
    방화벽 통과어려움(수동모드)어려움용이(포트 22)
  • FTP 모드:
    항목능동 모드 (Active)수동 모드 (Passive)
    제어 연결클라이언트 → 서버 21번 포트클라이언트 → 서버 21번 포트
    데이터 연결서버(20번) → 클라이언트 임시 포트클라이언트 → 서버 임시 포트
    연결 주체서버가 클라이언트로 데이터 연결 요청클라이언트가 서버 데이터 포트로 연결
    방화벽 문제❌ 클라이언트 방화벽 차단 위험✅ 방화벽 통과 용이
    주요 사용내부 네트워크 환경NAT·방화벽 환경
  • 인사이트: 보안 요구 환경에서는 FTP 사용을 금지하고 SFTP(SSH 기반)를 권장. FTPS는 기존 FTP 인프라를 유지하며 TLS를 추가하는 방식. SFTP는 파일 전송 외 원격 파일 관리도 지원.
