4. IntServ (Integrated Services)

정의: RSVP 프로토콜로 각 데이터 흐름에 필요한 네트워크 자원을 종단 간 사전 예약하여 엄격한 QoS를 보장하는 모델
키워드: RFC 2210, RSVP, 자원 예약, per-flow, Guaranteed Service, CL Service, 상태 유지
분류: QoS

RSVP 동작: 수신측이 PATH 메시지 수신 → RESV 메시지로 역방향 경로 자원 예약 → 중간 라우터 상태 저장
단점: 라우터당 per-flow 상태 유지 → 대규모 망 확장성 한계
인사이트: IntServ는 엄격한 보장이 필요한 소규모 망(기업 내부, 실시간 제어망)에 적합하며, 인터넷 전체 적용은 DiffServ가 대안. 두 모델의 비교는 시험 단골 주제.
