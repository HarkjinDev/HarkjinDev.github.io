[하위] 5-1. 슬라이딩 윈도우 (Sliding Window)

  • 정의: ACK 대기 없이 윈도우 크기만큼 프레임을 연속 전송해 채널효율을 높이는 흐름제어 기법
  • 키워드: 윈도우 크기, ACK, 흐름 제어, 버퍼, 수신 윈도우(rwnd), 송신 윈도우
  • 분류: 네트워크 기초
  • 동작 원리:
    단계설명핵심
    1. 윈도우 설정수신측 rwnd 기반으로 송신 윈도우 크기 결정전송 허용 범위 설정
    2. 연속 전송ACK 대기 없이 윈도우 크기만큼 프레임 연속 전송연속 전송
    3. ACK 수신수신측이 정상 수신 확인 후 ACK + 최신 rwnd 전송수신 확인·갱신
    4. 윈도우 슬라이드ACK 수신 시 윈도우를 오른쪽으로 이동·새 프레임 전송 허용윈도우 이동
    5. 윈도우 축소수신 버퍼 감소 시 rwnd 축소·송신 속도 제한속도 조절
    6. Zero Windowrwnd=0 수신 시 전송 중단·Window Probe 주기 전송교착 방지
    7. 전송 재개수신 버퍼 여유 생기면 rwnd 갱신 ACK로 전송 재개전송 복구
  • 윈도우 크기 결정 요소:
    요소설명
    수신 윈도우(rwnd)수신 버퍼 여유 공간
    혼잡 윈도우(cwnd)네트워크 혼잡 기반 제한
    실효 윈도우min(rwnd, cwnd)
  • 인사이트: 슬라이딩 윈도우는 흐름 제어의 핵심 메커니즘이며, TCP에서는 수신 윈도우(rwnd)와 혼잡 윈도우(cwnd)의 최솟값으로 실효 윈도우를 결정. Zero Window Probe로 rwnd=0 교착 상태 해소.
