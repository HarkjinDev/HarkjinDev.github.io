[하위] 3-5. 낙관적 검증 (Optimistic Validation / OCC)

  • 정의: COMMIT 직전 검증 단계에서 충돌 여부를 확인하여 직렬 가능성을 보장하는 비잠금 기반 병행제어 기법
  • 키워드: 판독/검증/기록 3단계, Read Set, Write Set, 비잠금, 충돌 드문 환경
  • 분류: 트랜잭션/병행제어
  • 3단계 처리 구조: <판검기>
    • 판독 단계(Read Phase): 데이터를 로컬 복사본에 읽어 연산 수행. 실제 DB에 반영 없음
    • 검증 단계(Validation Phase): 다른 트랜잭션과의 충돌 여부 확인. 충돌 시 롤백
    • 기록 단계(Write Phase): 검증 통과 시 로컬 결과를 실제 DB에 반영
  • 충돌 검증 기준: 검증 중인 트랜잭션의 Read Set과 병행 트랜잭션의 Write Set 교집합 확인
  • 장점: Lock 오버헤드 없음, 충돌이 드문 환경에서 높은 처리량
  • 단점: 충돌이 잦은 환경에서는 반복 롤백으로 성능 저하, 기아 가능성
  • 인사이트: 낙관적 검증은 충돌 가능성이 낮은 읽기 집약적 또는 단기 트랜잭션 환경에 적합하며, 충돌 빈도가 높은 환경에서는 잦은 롤백으로 비관적 잠금(Pessimistic Locking)보다 오히려 성능이 저하될 수 있음
