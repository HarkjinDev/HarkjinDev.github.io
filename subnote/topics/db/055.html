[하위] 1-1. NoSQL 모델링

정의: NoSQL 특성에 맞게 쿼리 패턴 중심으로 데이터 구조를 설계하는 방법론
키워드: 비정규화, 집계, 쿼리 기반 설계, Embedded/Reference/Bucket 패턴
분류: NoSQL
모델링 특징:
쿼리 우선 설계(Query-First Design): 예상 질의 패턴을 먼저 정의하고 그에 맞게 구조 설계
비정규화: 조인 대신 중복 허용으로 단일 조회 완결성 확보
집계 지향(Aggregate Orientation): 관련 데이터를 하나의 집계 단위로 묶어 관리
모델링 패턴: <비집프원인>
모델링 절자: <도쿼패/기후디>
절자
절자 별 상세 설명
도메인 모델 파악
데이터 개체 및 관계 분석 후 ERD 도식화
쿼리결과 디자인
도메인 모델 기반으로 쿼리 결과값 정하여 출력 내용 기반 디자인
패턴을 이용한 데이터 모델링
RDBMS 기능 배제, Put/Get 형태로 NoSQL 테이블 재정의
기능 최적화
Secondary Index를 이용하여 기능 최적화
후보 NoSQL을 선정 및 테스트
부하·안정성·확정성 테스트 후 최적 솔루션 선택
데이터 모델 최적화 및 하드웨어 디자인
데이터 모델 최적화 + 애플리케이션 인터페이스 설계 + 하드웨어 디자인 실시
인사이트: 관계형 DB 모델링 방식을 NoSQL에 그대로 적용하면 성능 문제가 발생하므로, 사용 패턴(읽기/쓰기 비율, 트랜잭션 범위)을 먼저 정의하는 것이 핵심
