1. 데이터베이스 (Database)

  • 정의: 데이터를 체계적으로 저장·관리하여 여러 사용자와 응용 프로그램이 공유할 수 있도록 통합된 데이터 집합체
  • 키워드: DBMS, 관계형, 스키마, 트랜잭션, 데이터 독립성, 무결성
  • 분류: DB 기초
  • 비교 (파일 vs 데이터베이스 vs 블록체인 저장):
    구분파일 (File)데이터베이스 (Database)블록체인 저장 (Blockchain Storage)
    개념레코드의 집합을 물리적 저장매체에 기록하는 단위DBMS를 통해 데이터를 통합 관리하는 저장 구조분산 원장 기반으로 해시 체인을 통해 불변성을 보장하는 분산 저장 방식
    접근 방식순차/직접/색인 접근SQL 기반 질의트랜잭션 기반 분산 합의
    독립성/공유성낮음높음 (다중 사용자 지원)높음 (탈중앙화)
    무결성낮음높음 (ACID 보장)매우 높음 (위·변조 방지)
    성능빠름중간낮음 (합의 비용 큼)
    적합 용도단순 파일 관리금융·ERP 등 다중 사용자 업무 시스템신뢰성이 핵심인 분산 환경
  • 핵심 요소:
    • DBMS: 데이터베이스를 생성·관리하는 소프트웨어 시스템 (Oracle, MySQL, PostgreSQL 등)
    • 스키마(Schema): 데이터베이스의 구조와 제약조건을 정의하는 메타데이터
    • 트랜잭션(Transaction): 데이터베이스의 상태를 변환시키는 논리적 작업 단위
  • 인사이트: 데이터 저장 방식 선택 시 무결성 요구 수준과 분산 환경 여부를 먼저 고려해야 하며, 블록체인은 신뢰성은 높으나 성능 비용이 크다는 트레이드오프를 답안에 반드시 언급 권장
