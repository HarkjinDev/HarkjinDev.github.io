2. 역정규화/반정규화 (De-normalization)

정의: 성능 향상을 위해 정규화된 데이터 모델의 일부 중복을 허용하고 구조를 재조정하는 과정
키워드: 테이블 병합/분할/추가, 컬럼 반정규화, 관계 반정규화, 성능 vs 무결성 트레이드오프
분류: 정규화
수행 절차: <대다적/범대통조/뷰클인응>
단계
절차
상세 내용
1단계
역정규화 대상 조사
범위처리 빈도수 조사, 대량의 범위 처리 조사, 통계성 프로세스 조사, 테이블 조인 개수
2단계
다른 방법 유도 검토
뷰(View) 테이블, 클러스터링 적용, 인덱스 조정, 응용 어플리케이션
3단계
역정규화 적용
테이블 역정규화, 속성 역정규화, 관계 역정규화
반정규화 유형: <테컬관/병분추/중파이피/중>
테이블 반정규화: 테이블 병합(1:1, 1:M), 분할(수직/수평), 추가(통계/중복)
컬럼(속성) 반정규화: 중복 컬럼 추가, 파생 컬럼 추가, 이력 컬럼 추가, PK 컬럼 추가
관계 반정규화: 중복 관계 추가 (조인 최소화)
적용 고려사항: 데이터 무결성 저하 가능성, 유지보수 비용 증가
인사이트: 반정규화는 조회 성능을 높이지만 DML(Insert/Update/Delete) 비용이 증가하므로, 읽기 빈도가 쓰기보다 압도적으로 높은 경우에만 적용을 권장함
