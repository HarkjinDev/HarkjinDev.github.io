3. 데이터베이스 튜닝 (Database Tuning)

  • 정의: 데이터베이스의 성능을 향상시키기 위해 SQL, 인덱스, 스키마 구조, 메모리 파라미터 등을 최적화하는 전반적인 활동
  • 키워드: 반정규화, 파티셔닝, 인덱스설계, 캐시활용, 메모리 파라미터, 서브쿼리, 힌트
  • 분류: 인덱스/튜닝
  • 튜닝 절차:
    튜닝단계설명사례
    1단계: DB 설계 튜닝설계 및 모델링 성능 고려반정규화, 분산파일배치
    2단계: DBMS 튜닝H/W 자원 및 환경 최적화Buffer 크기, Cache 크기
    3단계: SQL 튜닝SQL 및 실행 계획 최적화Hash / Join
  • 튜닝 단계별 기법:
    튜닝 영역기법상세
    DB 설계 튜닝테이블 분할 및 통합파티션 기능, 테이블 수평/수직 분할
    식별자 지정/Key 설정본질/인조 식별자 정의, 클러스터링
    적정 인덱스 설정인덱스 분포도 고려 10~15% (손익분기점)
    정규화/반정규화테이블, 컨럼, 관계 정규화/반정규화
    적절한 데이터 타입 선정조인시 연결되는 데이터 타입의 일치
    성능 데이터 모델링수퍼/서브타입, PK 조정, 이력테이블, 파티셔닝, 데이터 통합
    DBMS 튜닝I/O 최소화실제 필요한 데이터만 Read, Query off-Loading
    Buffer Pool 튜닝지역성 관점 데이터 관리, 오라클 Keep Buffer Cache
    Commit/Check PointCheck Point 수행주기 조절 통해 Commit 주기 조정
    Thread/ReuseMiddleware의 기능과 연동
    SQL 튜닝Undo Segment 설정Undo 영역 크기 조정
    옵티마이저RBO, CBO 이해, 통계정보 최신화
    힌트(HINT) 사용지원되는 힌트 통해 원하는 실행계획 유도
    부분범위 처리전체가 아닌 일부분만 Access, 옵티마이저 정보제공
    인덱스 활용인덱스 통한 조회 속도 향상, Sort 연산 대체
    조인방식/조인순서실행계획 (PLAN) 확인 후 조정
    동적 SQL 지양파싱(Parsing) 부하 감소위해 Static SQL 사용
    다중처리한번의 DBMS 호출로 여러 건을 동시에 처리
    병렬처리하나의 SQL을 여러 개의 CPU가 병렬로 분할 처리
    SORT 튜닝 수행인덱스 이용 통해서 MIN, MAX 구하기, TOP-N 쿼리
  • 인사이트: SQL 튜닝이 가장 높은 ROI를 보이는 튜닝 영역이며, 실행계획 분석(EXPLAIN PLAN)이 모든 튜닝의 출발점임을 답안에 명시 권장, S/W처리가 불가능하거나 효율적이지 않는 경우 H/W 방식의 시스템 데이터베이스 튜닝도 필요
