[하위] 1-2. 논클러스터드 인덱스 (Non-Clustered Index)

정의: 데이터의 물리적 순서와 독립적으로 별도의 인덱스 구조를 생성하며, 리프 노드에 실제 데이터 대신 포인터(RID 또는 클러스터드 인덱스 키)를 저장하는 인덱스
키워드: 포인터(RID), 다수 생성 가능, 별도 인덱스 구조, Lookup, 커버링 인덱스
분류: 인덱스/튜닝
특징:
테이블당 다수 생성 가능 (SQL Server 기준 최대 999개)
인덱스 구조와 데이터 파일이 분리 — 리프 노드는 포인터 보유
조회 시 포인터를 통한 추가 탐색(Lookup) 발생 → 클러스터드보다 I/O 증가 가능
DML 시 물리적 재정렬 불필요 → 삽입·삭제 비용 상대적으로 낮음
구조: B-Tree 기반 — 루트 → 브랜치 → 리프(포인터) → 실제 데이터 순 탐색
커버링 인덱스: 인덱스만으로 쿼리 결과 반환이 가능하도록 필요 컬럼을 포함시켜 Lookup 제거
인사이트: 논클러스터드 인덱스의 Lookup 비용을 제거하는 커버링 인덱스(Covering Index) 설계가 조회 성능 최적화의 핵심이며, 조회 컬럼을 INCLUDE 절에 포함시키는 방식으로 구현 가능
