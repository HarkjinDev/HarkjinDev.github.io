[하위] 2-2. 릴레이션 무결성 제약 (Relation Integrity Constraint)

정의: 릴레이션의 상태와 상태 전이에 대한 무결성을 보장하는 제약 조건 체계
키워드: 상태/과도/집합 제약, 즉시/지연 제약, 튜플 무결성
분류: 무결성
제약 유형: <상과집튜즉지>
구분
유형
설명
예시
상태변환
상태제약 (state constraint)
데이터 정적 일관성 유지
특정 시점에 만족
과도제약 (transition constraint)
상태 변환 전후 동적 비교
변환 직전/후 비교
집합여부
집합제약 (set constraint)
튜플 집합 전체 대상 제약
CHECK 절 전체 명시
튜플제약 (tuple constraint)
개별 튜플 단위 적용 제약
CHECK 절 단일 명시
시간관계
즉시제약 (immediate constraint)
연산 수행 즉시 적용 제약
AFTER 사용
지연제약 (deferred constraint)
트랜잭션 종료 후 적용
WHEN COMMIT 사용
인사이트: 즉시 제약은 DML 실행마다 검증하여 성능 영향이 있고, 지연 제약은 COMMIT 시점에 일괄 검증하여 복잡한 트랜잭션 처리에 유리함
