2. 옵티마이저 (Optimizer)

정의: SQL 쿼리를 분석하여 최적의 실행계획(Execution Plan)을 생성하는 DBMS의 핵심 엔진
키워드: RBO, CBO, 실행계획, 통계정보, 비용 추정, 카디널리티, 선택도
분류: 인덱스/튜닝
핵심 기능: 실행 계획 탐색, 비용 산정
RBO vs CBO 비교:
RBO(Rule Based Optimizer):
미리 정의된 규칙의 우선순위에 따라 실행계획 결정
인덱스 우선 처리 원칙 적용. 통계정보 불필요하나 최적 실행계획 보장 어려움
CBO(Cost Based Optimizer):
테이블 통계정보(행 수, 블록 수, 카디널리티 등)를 기반으로 최소 비용 경로 선택
현재 대부분의 RDBMS 표준 방식
실행계획 구성 요소: 조인 순서, 조인 방법(NL/Hash/Sort-Merge), 인덱스 사용 여부
인사이트: CBO는 통계정보가 최신으로 유지될 때 정확한 실행계획을 생성하므로, 대량 DML 후 통계 재수집(ANALYZE/DBMS_STATS)이 성능 유지의 핵심임
