[하위] 1-1. 클러스터드 인덱스 (Clustered Index)

정의: 인덱스 키 순서에 따라 데이터가 물리적으로 정렬·저장되는 인덱스로, 테이블 자체가 인덱스 구조로 구성된 인덱스
키워드: 물리적 정렬, 테이블당 1개, 리프=실제데이터, Range Scan, B-Tree
분류: 인덱스/튜닝
특징:
테이블당 1개만 생성 가능 (기본키 설정 시 자동 생성)
리프 노드(Leaf Node)에 실제 데이터 페이지가 저장됨
범위 검색(Range Scan)에 유리 — 물리적 인접성으로 연속 I/O 발생
삽입·삭제 시 물리적 재정렬이 발생하여 DML 성능 저하 가능
구조: B-Tree 기반 — 루트 → 브랜치 → 리프(실제 데이터) 순 탐색
적합한 상황:
범위 조건(BETWEEN, >=, <=) 조회가 빈번한 컬럼
단조 증가하는 값(ID, 날짜)이 기본키인 경우
인사이트: 클러스터드 인덱스 키가 무작위 값(UUID 등)이면 페이지 분할(Page Split)이 빈번히 발생하여 성능이 저하되므로, 순차 증가 키를 클러스터드 인덱스로 설정하는 것을 권장
