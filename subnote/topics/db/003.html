2. 데이터베이스 Key

정의: 릴레이션에서 튜플(행)을 유일하게 식별하기 위한 속성(Attribute) 또는 속성의 집합
키워드: 후보키, 기본키, 외래키, 슈퍼키, 본질적/내재적 제약, 유일성, 최소성
분류: DB 기초
키의 특징: 유일성, 최소성, 대표성 <유최대>
Key 종류 및 계층 관계: <슈후기대외>
슈퍼키(Super Key): 유일성을 만족하는 속성 집합 (최상위)
후보키(Candidate Key): 유일성 + 최소성을 만족하는 키 (슈퍼키의 부분집합)
기본키(Primary Key): 후보키 중 선택된 대표키. NULL 불가, 중복 불가
대체키(Alternate Key): 기본키로 선택되지 못한 나머지 후보키
외래키(Foreign Key): 다른 릴레이션의 기본키를 참조하는 속성 (참조 무결성 구현)
제약조건:
본질적 제약(Inherent Constraint): 관계형 모델 자체에서 요구하는 제약 (고유성 등)
내재적 제약(Implied Constraint): 특정 스키마에서 정의되는 제약 (외래키 참조 등)
키 도출 절차: <결슈후기/유최대>
순서
절차
내용
1
유일성 검증
함수적 종속성 확인을 통해서 유일성 검증 → 슈퍼키 선정
2
최소성 검증
최소성 확인 후 키 선정 (후보키가 선정되는 것임)
3
엔티티 대표성
후보키 중에서 엔티티 대표성을 기준으로 기본 키 확정
인사이트: 기본키 선정 시 불변성(변경 가능성이 낮은 속성)을 최우선으로 고려하며, 복합키보다 단일 대리키(Surrogate Key) 사용이 성능 측면에서 유리함을 답안에 언급 권장
