[하위] 1-1. 참조 무결성 (Referential Integrity)

  • 정의: 외래키(FK)가 참조하는 기본키(PK) 값이 항상 유효하게 존재함을 보장하는 데이터베이스 제약 조건
  • 키워드: 외래키, 부모-자식 관계, CASCADE, RESTRICT, SET NULL
  • 분류: 무결성
  • 원리: 부모 테이블(참조되는 테이블)과 자식 테이블(참조하는 테이블) 간의 데이터 일관성을 유지하기 위한 논리적 연결 고리
  • 참조 무결성 옵션:
    • CASCADE: 부모 변경 시 자식 데이터도 자동 변경/삭제
    • RESTRICT/NO ACTION: 자식 데이터가 있으면 부모 변경/삭제 차단
    • SET NULL: 부모 삭제 시 자식 FK를 NULL로 설정
    • SET DEFAULT: 부모 삭제 시 자식 FK를 기본값으로 설정
  • 인사이트: CASCADE 옵션은 편리하지만 의도치 않은 대량 삭제를 유발할 수 있어, 운영 DB에서는 RESTRICT 사용 후 명시적 삭제 로직 구현을 권장
