4. DA & DBA 역할

  • 정의:
    • DA(Data Architect): 데이터 구조, 표준, 정책을 설계하고 전사 데이터 전략을 수립하는 전문가
    • DBA(Database Administrator): 데이터베이스의 설치, 운영, 성능관리, 보안, 백업을 담당하는 관리자
  • 키워드: DA, DBA, 데이터 표준, 모델링, 데이터 전략, DB 운영, 성능관리, 보안
  • 분류: 데이터 거버넌스/품질
  • 역할 비교:
    구분DADBA
    주요 역할데이터 표준화, 모델링, 전략DB 설치/운영, 백업, 성능관리
    관점비즈니스/논리적기술/물리적
    산출물ERD, 데이터 표준DBA 스크립트, 백업 정책
  • 인사이트: 대규모 조직에서는 DA와 DBA 역할이 분리되지만, 중소 규모에서는 DBA가 두 역할을 겸하는 경우가 많으며, 디지털 전환 시대에 데이터 거버넌스를 이끄는 CDO(Chief Data Officer) 역할이 부상하고 있음
