[하위] 1-1. BCNF (Boyce-Codd Normal Form, 보이스-코드 정규형)

  • 정의: 모든 결정자(Determinant)가 후보키(Candidate Key)여야 한다는 조건을 만족하는 정규형
  • 키워드: 결정자, 후보키, 함수 종속성, 이상 현상, 분해, 3NF 차이, 정규화
  • 분류: 정규화
  • BCNF 조건:
    항목설명핵심
    결정자 (Determinant)X → Y에서 X가 Y를 결정하는 속성 집합함수 종속 좌변
    후보키 (Candidate Key)튜플을 유일하게 식별하는 최소 속성 집합유일 식별자
    BCNF 조건모든 비자명 함수 종속 X → Y에서 X는 반드시 후보키결정자 = 후보키
  • BCNF 위반 사례:
    • 수강 테이블: (학생ID, 과목명, 교수명)
      학생ID과목명교수명
      S001DB김교수
      S001OS이교수
      S002DB박교수
    • 후보키: (학생ID, 과목명), (학생ID, 교수명)
    • 함수 종속: 교수명 → 과목명 (교수는 하나의 과목만 담당)
    • 문제: 교수명이 결정자이지만 후보키가 아님 → BCNF 위반
  • BCNF 분해:
    • 위반 테이블 → 2개 테이블로 분해: 교수명이 후보키인 테이블로 분리 → BCNF 만족
