3. 병행제어 (Concurrency Control)

  • 정의: 다수의 트랜잭션이 동시에 실행될 때 데이터 일관성과 무결성을 보장하는 제어 기법
  • 키워드: Locking, 2PL, Timestamp, MVCC, 낙관적 검증, 갱신분실/현황파악오류/모순읽기/연쇄복귀
  • 분류: 트랜잭션/병행제어
  • 주요 문제점 (병행 오류): <갱현모연>
    • 갱신 분실(Lost Update): 두 트랜잭션이 동시에 같은 데이터를 갱신하여 하나가 손실
    • 현황 파악 오류(Dirty Read): 아직 완료되지 않은 트랜잭션의 값을 읽음
    • 모순 읽기(Inconsistent Read): 한 트랜잭션 내에서 같은 데이터가 다르게 읽힘
    • 연쇄 복귀(Cascading Rollback): 한 트랜잭션 취소가 연관 트랜잭션 취소로 연쇄
  • 제어 기법:
    • Locking: 데이터 접근 시 잠금(Lock)을 설정하여 다른 트랜잭션 접근 제한
    • 2PL (2-Phase Locking): 확장 단계(Lock 설정)와 축소 단계(Lock 해제)로 구분
    • Timestamp: 트랜잭션 시작 시간 기준으로 직렬화 순서 결정
    • MVCC(Multi-Version Concurrency Control): 데이터의 여러 버전 유지 — Oracle(USA), PostgreSQL(MGA)
    • 낙관적 검증(Optimistic Validation): 실행 후 검증 단계에서 충돌 확인
  • 인사이트: MVCC는 읽기 작업 시 잠금을 사용하지 않아 동시성이 높으나 저장 공간이 추가 필요하며, 현재 대부분의 현대 RDBMS(Oracle, PostgreSQL)의 표준 방식으로 자리잡음
