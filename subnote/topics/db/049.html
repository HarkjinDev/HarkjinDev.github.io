1. 데이터베이스 분할 (Partitioning)

  • 정의: 대용량 테이블을 관리 가능한 단위로 분할하여 성능과 관리 효율성을 향상시키는 기법
  • 키워드: 수평/수직 분할, 범위/해시/리스트/콤포지트, 파티션 프루닝
  • 분류: 분산DB/파티셔닝
  • 분할 목적: 성능 향상, 데이터 지역성 확보, 확장성 확보, 가용성 및 장애 격리, 보안-규제 대응
  • 분할 방식:
  • 수평 분할 (Horizontal Partitioning):
    • 개념: 하나의 테이블을 행(Row, 튜플) 기준으로 여러 조각으로 분리하는 방식
    • 분할 기준:
      분할 기법설명
      범위(Range) 분할숫자, 날짜 등 값의 범위를 기준으로 분할 (예: 월별 매출)
      해시(Hash) 분할해시 함수를 적용해 데이터를 균등하게 배분
      리스트(List) 분할특정 목록 값을 기준으로 분할 (예: 지역별 코드)
      조건(Condition) 분할컬럼 기준 분리
      복합(Composite) 분할위 기법들을 혼합하여 분할하는 기법
  • 수직 분할 (Vertical Partitioning):
    • 개념: 테이블의 컬럼(Column)을 기준으로 나누어 별도의 테이블로 분리하고, 기본키(PK)를 통해 관계를 유지하는 방식
    • 분할 기준:
      분할 기법특징설명
      접근 빈도 기반조회 패턴 중심 분리빈도별 분리로 I/O 성능 최적화
      기능(업무) 기반업무 기능별 분리업무 단위별 컬럼 그룹 분리
      민감정보 기반보안 중심 분리중요 정보 별도 관리로 보안 강화
  • 인사이트: 파티셔닝은 파티션 프루닝(Partition Pruning)을 통해 스캔 범위를 줄이는 것이 핵심이며, 파티션 키 선택이 성능을 좌우함
