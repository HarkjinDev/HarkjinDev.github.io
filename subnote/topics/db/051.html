3. 분산 데이터베이스 (Distributed Database)

정의: 여러 물리적 사이트에 데이터가 분산 저장되어 있으나 사용자에게는 하나의 논리적 데이터베이스로 보이는 시스템
키워드: 위치/복제/병행/분할/장애 투명성, 분할/복제/할당 전략
분류: 분산DB/파티셔닝
분산 DB 투명성 (5가지): <위복병분장>
위치 투명성(Location): 데이터 위치를 몰라도 접근 가능
복제 투명성(Replication): 복제 사실을 모르고 사용 가능
병행 투명성(Concurrency): 동시 접근 관리를 인식하지 못함
분할 투명성(Fragmentation): 데이터 분할 사실을 모르고 사용 가능
장애 투명성(Failure): 일부 사이트 장애 시에도 작동
설계 전략: 분할(Fragmentation), 복제(Replication), 할당(Allocation)
인사이트: 분산 DB의 핵심 가치는 투명성이며, CAP 이론에 따라 분산 환경에서 일관성과 가용성을 동시에 보장하는 것은 불가능함을 함께 언급 권장
