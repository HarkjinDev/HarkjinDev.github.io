5. 카디널리티 (Cardinality)

  • 정의: 엔터티 간 관계에서 하나의 인스턴스가 다른 엔터티의 몇 개 인스턴스와 관계를 맺는지를 나타내는 수적 대응 관계
  • 키워드: 1:1, 1:N, M:N, 선택/필수 참여, 교차 엔터티, 바커 표기법
  • 분류: 데이터 모델링
  • 유형:
    • 1:1 관계: 각 인스턴스가 상대 엔터티의 정확히 1개와 대응
    • 1:N 관계: 한 인스턴스가 상대 엔터티의 여러 인스턴스와 대응 (가장 일반적)
    • M:N 관계: 양쪽 모두 여러 인스턴스와 대응 → 물리 구현 시 교차 테이블로 분해 필요
  • 참여도: 선택적 참여(Optional) vs 필수 참여(Mandatory)
  • 인사이트: M:N 관계는 관계형 DB에서 직접 구현 불가능하므로 반드시 분해해야 하며, 참여도(선택/필수)는 NULL 허용 여부와 직결되어 물리 설계에 영향을 줌
