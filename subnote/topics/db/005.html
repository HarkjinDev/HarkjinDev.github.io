1. 데이터 모델링 (Data Modeling)

정의: 현실 세계의 데이터를 개념/논리/물리의 단계적 추상화를 통해 데이터베이스 구조로 변환하는 설계 과정
키워드: 요구분석, 개념모델링, 논리모델링, 물리모델링, 식별, 비식별, ERD, Inside-Out 전략
분류: 데이터 모델링
3단계 모델링 프로세스: <요개논물, 주후핵관, 속엔이, 물논반>
단계
세부 단계
설명
주요 고려사항
1. 요구분석
요구사항 도출
사용자 요구 도출
목적 및 요구 명확화
2. 개념모델링
1. 주제 영역 정의
최상위 집합 구분
중복 최소화·확장성
2. 후보 엔티티 선정
후보 개체 수집
제외 금지·현업 참여
3. 핵심 엔티티 정의
핵심 엔티티 선별
관리 및 동질성 판단
4. 관계 정의
엔티티 간 관계 설정
양방향 관점 고려
3. 논리모델링
1. 속성 정의
세부 속성 정의
최소 단위 분할
2. 엔티티 상세화
식별자 지정·정규화
단일값·무결성 유지
3. 이력 관리 정의
이력 관리 방식 설계
선분 이력 및 성능
4. 물리모델링
1. 물리 환경 조사
시스템 환경 분석
관리·백업·보안 고려
2. 논리모델 변환
물리 요소로 전환
PK 최적화 및 통합
3. 반정규화
성능 및 운영 효율화
최소화·SQL 최적화
식별/비식별 관계:
식별 관계: 부모 PK가 자식의 PK에 포함되는 강한 의존 관계
비식별 관계: 부모 PK가 자식의 일반 FK로만 존재하는 독립적 관계
Inside-Out 전략:
핵심 엔터티에서 시작하여 관계를 확장해 나가는 데이터 모델링 전략
중심 업무 객체를 먼저 정의하고 주변 엔터티를 연결하는 방식
데이터베이스 독립성:
인사이트: 논리 모델링 단계에서 식별/비식별 관계 설계가 이후 물리 구현과 성능에 직접 영향을 미치므로, 업무 의존도 분석을 반드시 선행해야 함
