1. 데이터 마이그레이션 (Data Migration)

  • 정의: 기존 시스템의 데이터를 신규 시스템으로 안전하게 이전하는 전환 작업
  • 키워드: ETL, 데이터 전환, 검증, 매핑, 정합성, 빅뱅 전환, 점진적 전환
  • 분류: 마이그레이션
  • 수행 단계: 분석(원천/목적지 파악) → 설계(매핑 규칙, 변환 로직) → 개발(ETL 스크립트) → 테스트(검증, 정합성 확인) → 전환(빅뱅 or 점진적) → 검증(마이그레이션 후 품질 검증)
  • 핵심 도구 및 방법:
    • ETL(Extract-Transform-Load): 추출 → 변환 → 적재의 표준 데이터 이전 방법
    • 데이터 매핑: 원천 필드와 목적지 필드의 변환 규칙 정의
    • 데이터 정합성 검증: Row Count, Sum Check, Hash Check 등으로 이전 전후 데이터 검증
  • 인사이트: 마이그레이션에서 가장 큰 리스크는 데이터 손실과 무결성 위반이므로, Rollback 계획을 사전에 수립하고 병행 운영(Parallel Run) 기간을 반드시 설정해야 함
