6. 수퍼타입 & 서브타입 모델링 (Supertype & Subtype Modeling)

  • 정의: 공통 속성을 가진 상위 엔터티(수퍼타입/통합화)와 각각의 고유 속성을 가진 하위 엔터티(서브타입/세분화)로 계층을 구성하여 데이터 중복을 줄이고 확장성을 높이는 데이터 모델링 기법
  • 키워드: 공통속성, 상속, 고유 속성, 배타적 관계, 포괄적 관계, 단일 테이블, 서브타입별, 타입별 분리
  • 분류: 데이터 모델링
  • 개념:
    • 수퍼타입(Supertype): 공통 속성과 관계를 정의하는 상위 엔터티(통합화) (예: 직원)
    • 서브타입(Subtype): 수퍼타입을 상속하면서 고유 속성을 추가로 갖는 하위 엔터티(세분화) (예: 정규직, 계약직, 파견직)
  • 표기법:
    • 배타적 관계(Exclusive): 하나의 인스턴스가 하나의 서브타입에만 속함
    • 포괄적 관계(Inclusive): 하나의 인스턴스가 여러 서브타입에 동시에 속할 수 있음
  • 구현방식: <단서타>
    • 단일 테이블: 수퍼타입과 모든 서브타입을 하나의 테이블로 통합. 조회 성능 우수, NULL 컬럼 다수 발생
    • 서브타입별 테이블: 서브타입마다 별도 테이블 생성, 수퍼타입 속성 포함. 중복 컬럼 발생
    • 타입별 분리: 수퍼타입 테이블 + 서브타입 테이블 분리. 정규화 수준 높으나 JOIN 비용 증가
  • 구현방식 비교:
    구분단일 테이블 (Single)서브타입별 테이블 (Individual)타입별 분리 (Super+Sub)
    테이블 수1개서브타입 수수퍼+서브타입 수
    NULL 발생많음없음없음
    JOIN 비용없음낮음높음
    적합한 상황조회 빈번 시속성 차이 클 때정규화 중요 시
  • 인사이트: 물리 변환 전략 선택 시 조회 패턴과 서브타입 수를 먼저 고려해야 하며, 서브타입별 조회가 빈번하면 타입별 분리, 전체 통합 조회가 많으면 단일 테이블 전략이 유리함. 시험에서는 세 가지 변환 전략의 장단점 비교가 핵심 출제 포인트임
