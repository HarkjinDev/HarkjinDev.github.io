2. SQL JOIN

  • 정의: 두 개 이상의 테이블을 특정 조건으로 연결하여 데이터를 조회하는 SQL 연산
  • 키워드: INNER, OUTER, NATURAL, EQUAL, CROSS, SEMI, NESTED LOOP, SORT MERGE, HASH, CARTESIAN, INDEX
  • 분류: SQL
  • JOIN 유형:
    구분종류/방식설명핵심
    논리적 종류Inner Join키 일치 행 검색교집합 추출
    Outer Join미대응 행 NULL 추출NULL 포함 결합
    Natural Join공통 속성 기반 결합동일 컬럼 매칭
    Equal Join공통 정보 단순 조인등호(=) 조인
    Cross Join모든 조합 카테시안 곱모든 경우의 수
    Semi Join조건 부합 시 연산 중단존재 여부 확인
    물리적 방식Nested loops순차적 루프 수행 방식순차 루프 실행
    Sort Merge정렬 후 병합 조인 방식정렬 후 병합
    Hash Join해시 함수 활용 조인해시 맵 활용
    Cartesian Join전체 조합 조인 방식대량 데이터 부하
    Index Join인덱스 간 해시 조인인덱스 최적화
  • 성능 향상 기법: 조인 컬럼에 인덱스 생성, 결과셋이 작은 테이블을 드라이빙 테이블로 설정
  • 인사이트: OUTER JOIN은 NULL 처리 로직이 필요하며, 대용량 테이블 JOIN 시 실행계획을 반드시 확인하여 Full Table Scan 여부를 검토해야 함
