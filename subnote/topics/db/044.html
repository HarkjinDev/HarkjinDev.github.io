[하위] 2-1. CBO (Cost Based Optimizer)

  • 정의: 테이블 통계정보(행 수, 블록 수, 카디널리티 등)를 기반으로 비용(Cost)을 계산하여 최적의 실행계획을 선택하는 옵티마이저
  • 키워드: 질의 변환기, 비용 산정기, 실행 계획 생성기, 선택도, 카디널리티, 히스토그램, 통계정보, Dictionary
  • 분류: 인덱스/튜닝
  • CBO 구성요소: <질비실>
    구성요소역할사례
    질의 변환기 (Query Transformer)효율적 형태로 SQL 모양 변환뷰 병합, 서브쿼리 비내포화
    비용 산정기 (Estimator)최적 실행계획 탐색 알고리즘선택도, 카디널리티, 비용
    실행 계획 생성기 (Plan Generator)최소 비용 실행계획 최종 선택적응적·경험적 탐색
  • CBO 기반 SQL 최적화 수행 과정: <파최행실>
    수행 과정구성요소수행 역할
    1. SQL 파싱SQL Parser구문 분석 및 오류 확인
    2. 최적화 (Optimization)Query Transformer쿼리 표준 형태 변환
    Estimator비용 산출 및 통계 계산
    Plan Generator최적 실행 계획 선정
    3. 행 소스 생성Row-Source Generator실행 가능 코드로 변환
    4. 실행 (Execution)SQL Engine계획 실행 및 결과 제공
  • CBO 참조 통계정보 유형: <테인컬시>
    통계정보 유형세부 통계 항목설명
    테이블 통계전체 레코드 수테이블의 전체 행의 수
    총 블록 수테이블이 차지하고 있는 전체 블록의 수
    평균 행 크기테이블의 행들이 갖는 평균 길이
    인덱스 통계리프 블록 수데이터를 보관하는 Leaf Block의 수
    인덱스 Level인덱스 트리의 높이 정보
    클러스터링 팩터접근하고자 하는 데이터가 모여 있는 밀집도
    컬럼 통계값의 수컬럼 내 값의 종류 수
    최저/최고 값컬럼 값의 범위
    컬럼 히스토그램컬럼 내 데이터의 분포 추정치
    시스템 통계CPU 속도CPU 성능 및 사용률
    평균 I/O 속도I/O 수행 성능
    초당 I/O 처리량I/O 사용률
  • 인사이트: CBO의 행동에 영향을 미치는 요소로 SQL의 형태, DBMS 제약 설정, 클러스터링/파티셔닝 등 다양한 요소가 존재하며, 이 중 통계정보에 대한 의존성이 절대적임. 대량 DML 이후 반드시 통계 재수집(ANALYZE/DBMS_STATS)을 수행해야 최적 실행계획 유지 가능
