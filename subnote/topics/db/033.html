[하위] 3-1. Locking

  • 정의: 트랜잭션이 데이터에 접근하기 전 잠금(Lock)을 설정하여 다른 트랜잭션의 동시 접근을 제한하는 병행제어 기법
  • 키워드: 공유 락, 배타 락, Locking 단위, Lock 호환성, 교착 상태, Wait-for Graph, 타임아웃, 선점
  • 분류: 트랜잭션/병행제어
  • Lock 유형:
    • 공유 락(Shared Lock, S-Lock): 읽기만 가능. 다른 트랜잭션의 읽기는 허용, 쓰기는 차단
    • 배타 락(Exclusive Lock, X-Lock): 읽기/쓰기 가능. 다른 트랜잭션의 모든 접근 차단
  • Locking 단위:
    • 행 ↔ 페이지 ↔ 테이블 ↔ DB (왼쪽으로 갈수록 단위가 크고, 오른쪽으로 갈수록 단위가 작음)
    • Locking 단위가 작을수록 오버헤드 증가, 병행성 향상
    • Lokcing 단위가 클수록 오버헤드 감소, 병행성 저하
  • Lock 호환성:
    • S-Lock ↔ S-Lock: 호환 (동시 읽기 가능)
    • S-Lock ↔ X-Lock: 비호환
    • X-Lock ↔ X-Lock: 비호환
  • 문제점: 교착 상태(Deadlock) — 두 트랜잭션이 서로의 Lock 해제를 무한 대기하는 상황
  • 교착 상태 해결: 대기 그래프(Wait-for Graph) 탐지, 타임아웃, 선점(Preemption)
  • 인사이트: Locking은 직관적이지만 Lock 경합이 심할수록 처리량이 저하되며, 교착 상태 감지·해소 오버헤드가 발생하므로 Lock 범위(Row vs Table)를 최소화하는 것이 성능 유지의 핵심임
