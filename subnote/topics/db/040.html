1. 인덱스 (Index)

정의: 테이블의 검색 속도를 향상시키기 위해 특정 컬럼의 값과 해당 데이터의 물리적 위치를 매핑한 별도의 데이터 구조
키워드: B-Tree, 해시, 비트맵, 함수, 클러스터드, 논클러스터드, Range Scan
분류: 인덱스/튜닝
특징: 성능향상, 알고리즘, 독립성, Trade-off
인덱스 유형: <트해비/함조도/정동/클논밀희고비>
분류 기준
인덱스 종류
설명
형태
트리 기반 (Tree-based)
B-Tree, B+Tree 구조 활용
해시 기반 (Hash-based)
해시 테이블 기반 동등 비교
비트맵 (Bitmap)
비트맵 변환, 저카디널리티용
목적
함수 기반 (Function-based)
함수/표현식 결과 인덱스화
조인 (Join)
조인 성능 향상을 위한 생성
도메인 (Domain)
사용자 정의 특화 인덱스
구조
정적 (Static)
데이터 변경 시에도 구조 고정
동적 (Dynamic)
데이터 따라 구조 자동 변경
사용자/운영자
클러스터드 (Clustered)
실제 데이터와 인덱스 순서 동일
논클러스터드 (Non-Clustered)
물리적 순서와 인덱스 다름
밀집 (Dense)
모든 레코드 대상 인덱스 생성
희소 (Sparse)
일부 레코드 대상 인덱싱 수행
고유 (Unique)
인덱스 컬럼 내 중복 불허
비고유 (Non-Unique)
중복 허용 보조 인덱스 활용
스캔 방식: <렌풀유스디패>
인사이트: 인덱스는 조회 성능을 높이지만 DML(Insert/Update/Delete) 시 인덱스도 함께 갱신되므로 쓰기 빈도가 높은 테이블에서는 과도한 인덱스 생성을 지양해야 함
