1. 데이터 무결성 (Data Integrity)

정의: 데이터의 정확성과 일관성을 보장하기 위해 DB에 설정되는 제약 조건의 집합
키워드: 개체/참조/속성/사용자/키/도메인 무결성, 제약조건, 일관성, 정확성
분류: 무결성
무결성 유형: <개참속/사키도>
종류
설명
예시
개체 무결성 (Entity Integrity)
기본키 존재 및 NULL 불가
Primary Key, NOT NULL
참조 무결성 (Referential Integrity)
외래키 기반 데이터 일관성
Foreign Key
속성 무결성 (Domain Integrity)
필드 속성 및 제약 정의
DATE, CHAR, NUMBER
사용자정의 무결성 (User Defined Integrity)
비즈니스 규칙 기반 정합성
Trigger, CHECK, DEFAULT
키 무결성 (Key Integrity)
키의 성질을 활용한 무결성
PK, Unique Index
도메인 무결성 (Domain Integrity)
정의된 도메인 범위 준수
Check, Foreign Key
데이터 무결성 확보방법
DBMS 무결성 요소 : Primary key, Unique, Foreign Key, Data Type, Check, Default
절차적 제약조건 : Trigger, Stored procedure, Application
인사이트: 무결성과 정합성의 차이를 명확히 구분해야 함. 무결성은 '단일 데이터의 정확성', 정합성은 '복수 데이터 간 논리적 일치성'을 의미함
