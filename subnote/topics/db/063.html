5. 데이터 레이크하우스 (Data Lakehouse)

  • 정의: 레이크의 유연성과 웨어하우스의 관리기능을 결합한 통합 아키텍처
  • 키워드: ACID Transaction, Delta Lake, Open Table Format, Schema Enforcement, 통합 거버넌스
  • 분류: 특수 DB
  • 핵심 개념·원리:
    항목설명핵심
    오픈 테이블 포맷스토리지 위에 트랜잭션 계층 추가테이블 포맷 계층
    ACID 트랜잭션동시성·정합성을 보장하는 처리데이터 정합성
    스키마 강제(Enforcement)쓰기 시점에 스키마 검증 적용데이터 품질확보
    통합 접근계층BI·ML 도구가 동일 데이터 접근단일소스 접근
  • 유형·분류 (오픈 테이블 포맷):
    항목설명핵심
    Delta LakeDatabricks 주도 오픈 포맷Spark 최적화
    Apache IcebergNetflix 기원 오픈 테이블포맷벤더중립 표준
    Apache HudiUber 기원, 실시간 upsert지원증분처리 특화
  • 구성요소·기술요소:
    구분구성요소핵심 기술
    저장계층클라우드 오브젝트 스토리지S3·ADLS·GCS
    메타데이터계층트랜잭션 로그·카탈로그Delta Log·Iceberg메타
    처리계층분산 처리엔진Spark·Trino·Flink
    거버넌스계층통합 접근제어·감사Unity Catalog 등
  • 인사이트: 레이크하우스는 레이크의 저비용·확장성과 웨어하우스의 신뢰성·성능을 통합해 데이터 이원화 구조의 한계를 해소한다.
