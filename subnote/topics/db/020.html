[하위] 1-2. 제4정규형 (4NF)

  • 정의: BCNF를 만족하면서 다치 종속성(MVD, Multi-Valued Dependency)을 제거한 정규화 단계
  • 키워드: 다치종속 제거, MVD, 4NF, BCNF
  • 분류: 정규화
  • 다치 종속(MVD): 하나의 속성 값에 다른 두 속성이 각각 독립적으로 다중 값을 가지는 종속성 (A →→ B)
    항목설명핵심
    다치 종속 (MVD)X →→ Y: X 값에 대해 Y 값의 집합이 독립적으로 결정됨독립적 다중 값
    함수 종속과 차이함수 종속은 1:1, 다치 종속은 1:多 독립 관계1:多 독립 종속
    발생 조건하나의 테이블에 독립적인 두 다치 속성이 공존할 때독립 다치 공존
    자명한 MVDX →→ Y이고 Y ⊆ X 이거나 X∪Y = 전체 속성인 경우위반 아님
  • 4NF 위반 사례:
    • 교수-과목-교재 테이블: (교수명, 과목명, 교재명)
      교수명과목명교재명
      김교수DB교재A
      김교수DB교재B
      김교수OS교재A
      김교수OS교재B
    • 다치 종속: 교수명 →→ 과목명, 교수명 →→ 교재명
    • 과목명과 교재명은 서로 독립적 (과목과 교재는 무관)
    • 문제: 교수명이 슈퍼키가 아님에도 두 개의 독립적 다치 종속 존재 → 4NF 위반
    • 위반 시 이상 현상:
      유형설명핵심
      삽입 이상신규 교재 추가 시 모든 과목 조합 행 삽입 필요불필요한 삽입
      삭제 이상교재 삭제 시 연관 과목 정보도 함께 소멸의도치 않은 삭제
      갱신 이상과목 변경 시 교재 개수만큼 모든 행 수정 필요중복 수정 부담
      데이터 중복과목·교재 조합의 수만큼 중복 행 발생저장 공간 낭비
  • 4NF 분해:
    • 위반 테이블 → 2개 테이블로 분해: → 독립적인 다치 종속을 각각 별도 테이블로 분리 → 4NF 만족
      교수-과목 테이블교수-교재 테이블
      교수명과목명교수명교재명
      김교수DB김교수교재A
      김교수OS김교수교재B
  • 인사이트: 4NF 이상은 실무에서 적용 빈도가 낮으나 시험에서 5NF까지 단계별 차이를 명확히 구분하는 문제로 출제되므로 개념 정의 암기 필요
