1. NoSQL (Not Only SQL)

정의: 관계형 모델의 엄격한 스키마를 탈피하여 유연한 데이터 구조와 수평 확장을 지원하는 비관계형 데이터베이스
키워드: Key-Value, Column Family, Document, Graph, BASE, 수평 확장, 스키마리스
분류: NoSQL
특징: Schemaless, In-Memory, Scale out
NoSQL 주요 모델:
NoSQL 특성 (BASE):
Basically Available: 항상 가용성 제공
Soft State: 상태가 외부 요인으로 변할 수 있음
Eventual Consistency: 최종적으로 일관성 달성
인사이트: NoSQL 선택 시 데이터 특성(관계 복잡도, 접근 패턴, 볼륨)에 따라 DB 유형을 결정해야 하며, RDBMS와 NoSQL의 혼합 사용(Polyglot Persistence)이 현실적 대안
