3. 그래프 데이터베이스 (Graph Database)

  • 정의: 점(Node)과 선(Edge)으로 이루어진 그래프 형태로 저장하고 탐색하는 비관계형(NoSQL) 데이터베이스
  • 키워드: 노드, 엣지, 속성, 레이블, 관계, Cypher, GQL, Neo4j, 지식 그래프, 트래버설
  • 분류: DB / NoSQL / 그래프
  • 특징:
    • 일관성 보장: ACID 제공, 관계 통한 무결성 제공
    • 직관성 제공: RESTFul API, 그래프 모델
  • 핵심 구성요소:
    구성요소설명핵심
    노드 (Node)엔티티를 나타내는 기본 단위·레이블로 분류객체 표현
    엣지 (Edge·관계)두 노드 간의 방향성 있는 관계 표현관계 표현
    속성 (Property)노드·엣지에 부여되는 키-값 형태의 데이터속성 데이터
    레이블 (Label)노드의 유형·카테고리 분류 태그노드 분류
    관계 유형 (Type)엣지의 종류·방향을 나타내는 명칭관계 분류
  • 그래프 DB 유형:
    유형설명대표 제품핵심
    프로퍼티 그래프노드·엣지에 속성 부여 가능한 범용 그래프Neo4j·Amazon Neptune범용 그래프
    RDF 그래프주어-술어-목적어 트리플 기반 시맨틱 웹GraphDB·Stardog시맨틱 웹
    하이퍼그래프하나의 엣지가 2개 이상 노드 연결 가능HyperGraphDB다중 관계
    지식 그래프엔티티·관계·온톨로지 통합 지식 표현GoogleKG·Wikidata지식 표현
  • 그래프 DB vs 관계형 DB 통합 비교표
    구분그래프 DB관계형 DB
    데이터 모델Property Graph2차원 테이블
    스키마 방식Schema-on-Read (지연 바인딩)Schema-on-Write
    저장 방식Index-Free AdjacencyB-Tree 인덱스
    주소 지정포인터 기반 직접 참조키 기반 간접 참조
    확장 방식수평적 확장 (Scale-Out)수직적 확장 (Scale-Up)
    구성 요소노드·간선·속성행·열·제약조건
    무결성 관리그래프 제약 (Graph Constraints)참조 무결성 (Referential Integrity)
    관계 표현 객체간선 (Edge)외래키 (Foreign Key)
    관계 취급 방식독립 데이터 객체논리적 제약조건
    관계 탐색 방식그래프 순회 (Graph Traversal)조인 연산 (JOIN)
    질의 언어Cypher · Gremlin · SPARQLSQL
    다단계 탐색 성능O(1) 수준 일정 성능O(Nk) 기하급수적 증가
    다대다 관계 처리간선 직접 모델링매핑 테이블 (Mapping Table)
    관계 속성 부여간선에 직접 속성 할당별도 관계 테이블 생성
  • 그래프 DB 한계:
    항목설명핵심
    집계 쿼리 약점SUM·COUNT 등 집계는 RDBMS 대비 비효율집계 쿼리 부적합
    표준화 미성숙GQL 표준화 진행 중·벤더별 쿼리 언어 상이호환성 문제
    대규모 분산 한계수평 확장(샤딩)이 RDBMS·문서 DB 대비 복잡분산 확장 어려움
    학습 곡선그래프 사고방식·Cypher 쿼리 학습 필요진입 장벽
  • 인사이트: 그래프 데이터베이스는 관계 자체가 핵심 데이터인 도메인에서 RDBMS의 다중 조인 성능 한계를 극복하는 최적 솔루션으로, 노드 수가 아닌 관계의 깊이에 따른 탐색 성능이 일정하다는 인덱스-프리 인접성(Index-Free Adjacency)이 핵심 경쟁력임. LLM·RAG 시대에 지식 그래프와 그래프 DB를 결합한 GraphRAG가 AI의 추론·관계 탐색 역량을 대폭 향상시키는 핵심 인프라로 부상하고 있음
