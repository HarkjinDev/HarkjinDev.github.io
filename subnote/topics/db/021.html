[하위] 1-3. 제5정규형 (5NF)

  • 정의: 4NF를 만족하면서 조인 종속성(JD, Join Dependency)을 제거하여 무손실 분해를 보장하는 최고 단계의 정규형
  • 키워드: 조인종속 제거, 무손실 분해, 5NF, PJ/NF(투영-조인, Projection-Join Normal Form)
  • 분류: 정규화
  • 조인 종속(JD): 릴레이션을 세 개 이상의 릴레이션으로 분해했을 때, 다시 조인하면 원래 릴레이션이 복원되는 종속성
    항목설명핵심
    조인 종속 (JD)테이블 R이 여러 프로젝션으로 분해 후 다시 조인 시 원본과 동일한 제약분해 후 복원 가능
    표현 방식*(R1, R2, R3): R = R1 ⋈ R2 ⋈ R3무손실 조인
    다치 종속과 차이MVD는 2개 분해·JD는 3개 이상 분해3방향 이상 종속
    자명한 JD분해가 후보키에 의해 함의될 때위반 아님
  • 5NF 위반 사례
    • 교수-학생-과목 테이블: (교수명, 학생명, 과목명)
      교수명학생명과목명
      김교수철수DB
      김교수영희DB
      이교수철수OS
      이교수민준OS
    • 제약 조건: 김교수가 철수를 가르치고(교수-학생), 철수가 DB를 수강하고(학생-과목), 김교수가 DB를 담당하면(교수-과목) → (김교수, 철수, DB) 반드시 존재
    • 조인 종속: *(교수-학생, 학생-과목, 교수-과목) → 후보키에 의해 함의되지 않음 → 5NF 위반
    • 위반 시 이상 현상
      유형설명핵심
      삽입 이상이교수가 철수에게 DB 수업 추가 시 교수-과목 관계도 동시 성립해야 삽입 가능연쇄 삽입 조건
      삭제 이상민준이 수강 취소 시 이교수-OS 담당 관계 정보 함께 소멸의미 정보 손실
      갱신 이상교수 담당 과목 변경 시 관련 학생 수만큼 모든 행 수정 필요연쇄 수정 부담
      중복 데이터교수-학생-과목 3자 조합 수만큼 중복 행 발생저장 공간 낭비
  • 5NF 분해 (3개 테이블)
    • 위반 테이블 → 3개 테이블로 분해: → 3개 테이블 자연 조인 시 원본 복원 → 5NF 만족
      교수-학생학생-과목교수-과목
      교수명학생명학생명과목명교수명과목명
      김교수철수철수DB김교수DB
      김교수영희영희DB이교수OS
      이교수철수철수OS
      이교수민준민준OS
  • 인사이트: 5NF(=PJ/NF)는 이론적 완전성을 의미하며, 실무 적용 시 지나친 분해로 오히려 성능이 저하될 수 있어 3NF~BCNF 수준이 현실적 목표
