1. 정규화 (Normalization)

정의: 데이터 중복과 이상현상(Anomaly)을 제거하기 위해 릴레이션을 함수적 종속성 이론에 따라 분해하는 과정
키워드: 정보의 무손실, 데이터 중복성 감소, 분리의 원칙, 함수적 종속
분류: 정규화
원리 : <무중분>
정보의 무손실: 무손실 분해(Lossless Decomposition)를 통한 원본 데이터 복원 가능성 보장
데이터 중복성 감소: 중복 제거를 통한 데이터 일관성 유지 및 이상현상 제거
분리의 원칙: 독립적 개체 표현 및 단일 주제 중심의 릴레이션 구성
이상현상 3가지: <삽삭갱>
삽입 이상(Insertion Anomaly): 원치 않는 데이터를 함께 삽입해야 하는 문제
삭제 이상(Deletion Anomaly): 데이터 삭제 시 연관된 다른 정보까지 손실되는 문제
갱신 이상(Update Anomaly): 중복 데이터 일부만 수정 시 불일치가 발생하는 문제
단계별 정규형: <도부이결다조>
1NF: 원자값(Atomic Value)만 허용 — [X → Y]
2NF: 부분 함수 종속 제거 — [X(A+B) → Y]
3NF: 이행 함수 종속 제거 — [X → A → Y]
BCNF: 결정자가 모두 후보키 — [결정자=키]
4NF: 다치 종속(MVD) 제거 [X → Y, X → Z]
5NF: 조인 종속(JD) 제거 — [X - Y - Z]
인사이트: 정규화는 데이터 무결성을 높이지만 JOIN 증가로 성능이 저하될 수 있어, 실무에서는 반정규화(Denormalization)와의 균형이 필요함. 참고로 6NF(제6정규형)는 5NF를 넘어 모든 조인 종속을 제거하여 릴레이션을 더 이상 분해할 수 없는 수준까지 분해한 이론적 최고 단계로, 시간적 데이터(Temporal Data) 처리에 활용됨 — 단, 지나친 분해로 조인 비용이 극도로 증가하므로 실무 적용은 거의 없고 시험에서도 개념 수준의 언급으로 충분
