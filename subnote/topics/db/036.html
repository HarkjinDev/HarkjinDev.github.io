[하위] 3-4. MVCC (Multi-Version Concurrency Control)

정의: 데이터 항목의 여러 버전을 동시에 유지하여 읽기 연산 시 잠금 없이 특정 시점의 스냅샷을 제공함으로써 읽기-쓰기 간 충돌을 제거하는 병행제어 기법
키워드: 다중 버전, 스냅샷 읽기, 읽기 무잠금, Undo Log, Vacuum, USA(Oracle), MGA(PostgreSQL)
분류: 트랜잭션/병행제어
동작 원리:
쓰기 트랜잭션은 새 버전 생성, 읽기 트랜잭션은 자신의 TS 이전의 최신 버전을 조회
읽기는 잠금 없이 수행 → 높은 동시성
구현 방식별 비교:
Oracle: Undo 세그먼트 활용 (USA, Undo Segment Area)
PostgreSQL: 힙 내 다중 버전 직접 저장 (MGA, Multi-Generation Architecture)
MySQL InnoDB: Undo Log + Redo Log 기반
장점: 읽기와 쓰기 간 차단 없음, 높은 동시성, 일관된 스냅샷 읽기 지원
단점: 버전 관리로 인한 스토리지 증가, 오래된 버전 정리(Vacuum/Purge) 오버헤드
인사이트: MVCC는 현대 RDBMS의 표준 동시성 제어 방식으로, 읽기 집약적 워크로드에서 탁월하나 장기 미정리 버전(Long-running Transaction)이 Undo 공간 부족과 성능 저하를 유발하므로 트랜잭션 수명 관리가 중요함
