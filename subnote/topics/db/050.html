2. 샤딩 (Sharding)

  • 정의: 데이터를 여러 독립적인 노드(Shard)에 물리적으로 수평 분산하여 저장하는 스케일 아웃(Scale-out) 확장 기법
  • 키워드: 샤드키, 메타데이터, Range/Hash/Directory/Vertical, 수평 확장, 프록시
  • 분류: 분산DB/파티셔닝
  • 파티셔닝 vs 샤딩 비교:
    • 파티셔닝: 단일 DBMS 내에서 물리/논리적 분할. 투명성 제공
    • 샤딩: 여러 서버에 걸쳐 데이터를 물리적으로 분산. 각 샤드는 독립적 DBMS
  • 구성 요소:
    • 샤드키: 샤드를 선택하기 위한 식별자 컬럼
    • 샤드 메타데이터: 샤드의 동작을 위한 설정 정보
    • 샤드ID: 샤드 DB 식별하기 위한 식별자
    • 프록시: 샤드 메타데이터를 이용, 샤드 DB로 요청 전달
  • 적용 시 고려사항: 크로스 샤드 조인 어려움, 트랜잭션 관리 복잡, 리샤딩 비용
  • 인사이트: 샤딩은 데이터 볼륨 문제를 해결하지만 크로스 샤드 트랜잭션과 조인이 어려워 애플리케이션 레이어에서의 복잡도가 증가하므로, 단순 파티셔닝이나 레플리케이션으로 해결 가능한지 먼저 검토해야 함
