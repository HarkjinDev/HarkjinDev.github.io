[하위] 4-1. 데이터 늪(Data Swamp)

  • 정의: 메타데이터 관리 부재로 데이터 레이크가 활용 불가능한 상태로 전락한 현상
  • 발생 원인
    영역발생 원인설명
    데이터 소스데이터 품질 저하결측·오류·중복으로 신뢰도 저하
    데이터 의미 불명확정의·기준 없어 해석 오류 발생
    데이터 수집 계층무분별한 데이터 적재기준 없이 데이터 우선 저장
    스키마·포맷 불일치구조·포맷 달라 통합 불가
    데이터 관리·운영 계층메타데이터·계보 부재이력·흐름 추적 기능 부재
    보안·거버넌스 미흡정책 부재로 통제 안 되는 저장소
  • 방지 방안:
    방지 방안발생 원인 제거설명
    데이터 존 설계무분별한 적재·품질 혼재Raw·Silver·Gold 계층별 품질 관리
    데이터 등록 표준화의미 불명확·소유자 부재소유자·목적 등 필수 정보 등록
    메타데이터 카탈로그 구축발견 어려움·계보 부재스키마·계보 정보 중앙 관리
    데이터 품질 게이트결측·오류·중복 문제단계별 품질 규칙 자동 검증
    거버넌스·보안 통제권한 과다·민감정보 혼재접근권한·암호화·감사로그 통제
    수명주기·운영 모니터링데이터 방치·비용 증가보존·폐기·파이프라인 상태 점검
