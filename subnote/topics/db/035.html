[하위] 3-3. 타임스탬프 (Timestamp Ordering)

  • 정의: 트랜잭션 시작 시 부여된 타임스탬프를 기준으로 트랜잭션 간 접근 순서를 결정하여 직렬 가능성을 보장하는 비잠금 기반 병행제어 기법
  • 키워드: 타임스탬프, 논리적 계수, 시스템 시계, Read-TS, Write-TS, 롤백 재시작, 교착 상태 없음, 기아 발생
  • 분류: 트랜잭션/병행제어
  • 구성요소:
    • 논리적 계수(Logical Counter): 하나씩 증가시켜 타임스탬프값으로 부여
    • 시스템 시계(System Clock): 시스템 시계의 값을 타임스탬프값으로 부여
    • read_TS(x): 판독 타임스탬프 중에 가장 큰 타임스탬프
    • write_TS(x): 쓰기 타임스탬프 중에 가장 큰 타임스탬프
  • 동작 원리:
    • 각 데이터 항목에 Read-TS(최근 읽기 타임스탬프)와 Write-TS(최근 쓰기 타임스탬프) 기록
    • 트랜잭션의 TS가 데이터의 TS보다 작으면(과거 트랜잭션) → 롤백 후 재시작
  • 규칙:
    • 읽기 규칙:
      • TS(T) ≥ Write-TS(X): 큰 값을 Read-TS(X)에 저장
      • TS(T) < Write-TS(X): 롤백 (이미 더 최신 트랜잭션이 쓴 데이터)
    • 쓰기 규칙:
      • TS(T) ≥ Read-TS(X) AND Write-TS(X): Write-TS(X)에 저장
      • TS(T) < Read-TS(X) OR TS(T) < Write-TS(X): 롤백
  • 장점: Lock 불필요 → 교착 상태 발생 없음
  • 단점: 잦은 롤백 및 재시작으로 기아(Starvation) 발생 가능
  • 인사이트: 타임스탬프 기법은 교착 상태를 원천 차단하지만, 충돌이 잦은 환경에서는 반복 롤백으로 인한 처리량 저하가 심각해 낙관적 검증이나 MVCC와 함께 비교 검토가 필요함
