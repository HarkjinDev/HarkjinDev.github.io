2. 트랜잭션 격리 수준 (Transaction Isolation Level)

  • 정의: 동시에 실행되는 여러 트랜잭션 간의 간섭 정도를 단계별로 정의한 격리 수준
  • 키워드: ReadUncommitted, ReadCommitted, RepeatableRead, Serializable, Dirty Read, Phantom Read
  • 분류: 트랜잭션/병행제어
  • 4가지 격리 수준 (낮음 → 높음):
    • (LEVEL 0) Read Uncommitted: 미완료 트랜잭션의 데이터 읽기 허용 → Dirty Read 발생
    • (LEVEL 1) Read Committed: 커밋된 데이터만 읽기 허용 → Non-Repeatable Read 발생
    • (LEVEL 2) Repeatable Read: 동일 쿼리 반복 시 동일 결과 보장 → Phantom Read 발생
    • (LEVEL 3) Serializable: 완전한 격리 보장 → 가장 안전, 성능 가장 낮음
  • 이상 현상:
    • Dirty Read: 미완료 데이터 읽기
    • Non-Repeatable Read: 동일 행 재조회 시 결과 변경
    • Phantom Read: 동일 조건 재조회 시 행 수 변경
  • 인사이트: 격리 수준이 높을수록 동시성이 저하되므로, 대부분의 RDBMS는 Read Committed(Oracle) 또는 Repeatable Read(MySQL)를 기본값으로 채택함
