4. 데이터 웨어하우스 & 데이터 레이크

  • 정의:
    • 데이터 웨어하우스(DW): ETL을 통해 정제된 데이터를 의사결정 지원을 위해 통합·저장하는 분석 지향 데이터 저장소
    • 데이터 레이크(Data Lake): 원시데이터(정형·비정형)를 원본 그대로 저장하는 중앙집중형 저장소
  • 키워드: ETL, 주제지향, 통합, 시변성, 비휘발성, 스키마온리드, Data Swamp, LDW
  • 분류: 특수 DB
  • 데이터 관련 변화:
  • 구성요소:
    구분구성요소핵심 기술
    수집계층(Ingestion)배치 수집기ETL/ELT 배치 적재
    실시간 수집기Kafka·Flume 스트리밍
    저장계층(Storage Zone)Raw Zone(원본영역)원본 데이터 그대로 저장
    Staging Zone(정제영역)정제·표준화 처리
    Curated Zone(신뢰영역)분석용 고품질 데이터
    Sandbox Zone(탐색영역)실험·탐색적 분석용
    처리계층(Processing)배치 처리엔진Spark·Hadoop MapReduce
    스트림 처리엔진Spark Streaming·Flink
    거버넌스계층(Governance)메타데이터 카탈로그데이터 검색·계보관리
    접근제어·보안IAM·암호화·마스킹
    데이터 품질관리검증·정합성 체크
    활용계층(Consumption)BI·분석 도구셀프서비스 리포팅
    ML/AI 플랫폼모델 학습용 데이터 소비
  • 비교:
    구분데이터 웨어하우스데이터 레이크
    데이터 유형정형 데이터정형/비정형 모두
    스키마스키마 온 라이트스키마 온 리드
    처리 방식ETLELT
    목적BI/보고AI/ML, 탐색적 분석
  • 로지컬 데이터웨어하우스(LDW): 기존 DW와 빅데이터를 가상 통합하여 단일 분석 레이어 제공
  • 인사이트: 데이터 레이크의 유연성과 DW의 구조화된 분석 능력을 결합한 데이터 레이크하우스(Lakehouse) 아키텍처가 최신 트렌드로, Delta Lake·Apache Iceberg가 대표 기술임
