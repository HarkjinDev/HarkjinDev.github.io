[하위] 1-1. 오픈소스 DBMS 전환

  • 정의: 비용 절감 및 벤더 종속성 탈피를 위해 상용 DBMS를 오픈소스 DBMS로 전환하는 이전 프로젝트
  • 키워드: 상용→오픈소스, SQL 호환성, 저장 프로시저 변환, HA 아키텍처, 벤더 종속성 탈피
  • 분류: 마이그레이션
  • 전환 시 주요 제약사항:
    • SQL 호환성: 상용 DB 전용 문법(힌트, 내장 함수) 변환 필요
    • 저장 프로시저: PL/SQL → PL/pgSQL 등 언어 변환 작업
    • HA 아키텍처: Active-Active 또는 Active-Standby 구성 재설계
  • 전환 절차: 현황 분석 → 호환성 검토 → 파일럿 구축 → 마이그레이션 → 성능 검증 → 전환
  • 인사이트: Oracle → PostgreSQL 전환 시 DATE 타입 차이, ROWNUM vs LIMIT, 시퀀스 방식 등 세부 문법 차이가 예상보다 많으므로, 전환 전 충분한 호환성 테스트와 자동화 변환 도구(Ora2Pg 등) 활용을 권장함
