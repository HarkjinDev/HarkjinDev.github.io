8. 국가 데이터 카탈로그 (National Data Catalog)

  • 정의: 공공·민간이 보유한 데이터셋의 소재·품질·접근조건·라이선스 등 메타데이터를 표준화하여 등록·검색·연계할 수 있는 범국가 데이터 목록 체계
  • 키워드: DCAT, 메타데이터, ODRL, 데이터 스페이스, 탐색성, DQS, IDS Connector, One-윈도우
  • 분류: 데이터 거버넌스/품질
  • 개요:
    구분내용
    목적데이터 탐색성(Findability) 향상·중복 수집 방지·공공데이터 개방 촉진·데이터 스페이스 참여 기반 제공
    법적 근거데이터산업진흥법 제19조(국가데이터맵 구축)·공공데이터법 제17조(공공데이터 목록 제공 의무)
    국내 현황과기정통부 주관 국가 데이터 카탈로그 표준 v1.0(2024) 제정·공공 52개 기관 1차 연계·One-윈도우 포털(data.go.kr) 고도화 추진
    핵심 특성데이터 자체가 아닌 메타데이터를 표준화·등록하여 데이터 발견·접근·활용 원활화
  • 국가 데이터 카탈로그 구성요소:
    구성요소역할설명
    메타데이터 표준 (DCAT 기반)데이터셋 설명 표준화W3C DCAT 3.0 준용
    데이터 등록 체계데이터 목록 수집·등록데이터 등록 체계 품질 검증 4단계
    검색·탐색 엔진사용자 데이터 발견 지원전문 검색(Full-text)·분류 검색·태그 검색 지원·연관 데이터셋 추천(유사도 기반)·API 제공
    접근정책 관리 (ODRL)데이터 이용조건 표준화라이선스·이용조건·금지사항 기계가독형 표현·자동화된 정책 협상·계약 체결 지원
    데이터 품질 관리카탈로그 신뢰성 확보메타데이터 완전성·정확성·최신성 자동 검증·품질 점수(DQS) 부여·부적합 데이터 반려·개선 요청
    연계·상호운용 계층이종 카탈로그 연결기관 카탈로그·지자체·민간 플랫폼과 API 연계·IDS Connector·GAIA-X 상호운용 프로토콜 지원
  • 데이터 등록 체계 품질 검증 4단계:
    단계검증 내용핵심
    등록 검증필수 메타데이터 항목 누락 여부 확인필수 항목 완전성
    표준 검증DCAT 표준 형식·어휘 준수 여부 확인표준 준수
    중복 검증기존 등록 데이터셋과 중복 여부 탐지중복 방지
    형식 검증접근 URL·파일 형식·인코딩 유효성 확인접근 가능성
  • 데이터 이용자 유형별 접근:
    이용자접근 방식핵심
    일반 국민검색·탐색·시각화·미리보기데이터 발견
    기업·연구자정책 기반 접근 요청·API 활용연구·분석
    공무원통제된 데이터 접근·업무 활용정책 기반 접근
    개발자·시스템API 호출·데이터 활용·연계시스템 연동
  • 인사이트: 국가 데이터 카탈로그는 DCAT 기반 메타데이터 표준·등록 체계·검색 엔진·ODRL 정책 관리·품질 보증·연계 계층의 6대 구성요소가 유기적으로 작동하여 데이터 탐색성과 상호운용성을 실현하는 데이터 스페이스 구현의 핵심 전제 인프라
