[하위] 2-2. 소프트웨어 품질보증 (SQA, Software Quality Assurance)

  • 정의: 소프트웨어 개발 프로세스 전반에 걸쳐 품질 표준과 절차를 계획·감시·검증하는 체계적 활동
  • 키워드: 프로세스 중심, 감사(Audit), 검토(Review), V&V, 품질 계획, Walk-through, Technical Review, Inspection
  • 분류: SW공학 개론
  • 주요 SQA 활동:
    • 품질 계획(Quality Plan) 수립, 공식 기술 검토(FTR, Formal Technical Review), 소프트웨어 테스트 감독, 표준·절차 준수 감사, 변경 관리 통제
  • QA vs QC 비교
    구분QA (Quality Assurance)QC (Quality Control)
    초점프로세스제품(결과물)
    시점전 주기(예방적)완료 후(발견적)
    활동감사, 프로세스 개선테스트, 검사
    성격능동적(Proactive)반응적(Reactive)
    산출물프로세스 개선검증된 오류수정
  • 동료 검토 기법 비교:
    구분워크스루 (Walk-through)기술 검토 (Technical Review)인스펙션 (Inspection)
    공식성가장 낮음 (비공식적)보통 (공식/비공식 혼합)가장 높음 (매우 공식적)
    주관자저자 (Author)기술 전문가 또는 팀장중재자 (Moderator)
    주요 목적지식 공유, 교육, 이해도 증진기술적 문제 해결, 의사결정결함 발견, 품질 측정
    사전 준비필요 없음 (또는 최소화)필요 (자료 검토)필수 (개인 사전 검토)
    체크리스트사용 안 함선택적 사용필수 사용
    결과물검토 메모, 교육 자료기술 결정 보고서검토 결과 보고서, 결함 로그
    특징저자가 진행하며 편안한 분위기에서 지식 전파전문가 집단이 모여 기술적 대안을 평가Michael Fagan이 제안한 6단계 절차 엄수
  • 인사이트: SQA는 “올바른 프로세스를 통해 올바른 제품이 나온다”는 철학 기반으로, 테스트(QC)와 달리 개발 전 단계에 걸쳐 예방 중심으로 운영됨을 답안에 명시.
