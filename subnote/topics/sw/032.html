[하위] 5-6. SAFe (Scaled Agile Framework, 확장형 애자일 프레임워크)

  • 정의: 대규모 조직에서 애자일·린·DevOps 원칙을 전사적으로 확장·적용하여 다수의 팀이 정렬된 방식으로 가치를 신속하게 전달하는 스케일드 애자일 프레임워크
  • 키워드: PI Planning, ART(Agile Release Train), RTE, 스프린트, Epic, Feature, Story, 린 애자일, DevOps, Value Stream, 릴리즈 트레인
  • 분류: 개발 방법론
  • 애자일 vs SAFe 비교:
    항목애자일 (스크럼)SAFe
    적용 규모소규모 단일 팀대규모 다팀·전사
    계획 주기스프린트 (2~4주)PI (8~12주)
    조정 단위팀 단위ART·Solution Train
    전략 연계제한적포트폴리오~팀 완전 연계
    출시 단위팀별 릴리즈통합 릴리즈 트레인
  • 4대 구성 계층:
    계층설명핵심
    Team (팀)스크럼·칸반 기반 2주 스프린트 단위 개발기본 실행 단위
    Program (프로그램)ART 중심 PI Planning으로 다팀 정렬·조율ART 실행
    Large Solution (대형 솔루션)복수 ART를 조율하는 Solution Train 운영복잡 솔루션 조율
    Portfolio (포트폴리오)전략·투자·Lean Governance로 전사 방향 설정전사 전략 정렬
  • 핵심 구성요소:
    구분항목설명핵심
    조직ART (Agile Release Train)가치 흐름 기반 팀 연합체핵심 실행 조직
    RTE (Release Train Engineer)ART 조율 담당 서번트 리더ART 오케스트레이터
    Product Manager제품 백로그 및 우선순위 관리제품 방향 관리
    System Architect기술 아키텍처 및 방향 가이드기술 방향 설정
    계획·실행PI Planning8~12주 목표 수립 대면 계획SAFe 핵심 이벤트
    Program Increment(PI)8~12주 단위 가치 전달 주기전달 주기 단위
    System Demo통합 솔루션 시연 및 검증통합 검증
    Inspect & Adapt회고를 통한 개선 사항 도출지속 개선
    백로그Epic포트폴리오 수준 대형 전략전사 이니셔티브
    Feature프로그램 수준의 기능 단위기능 단위
    Story팀 수준의 작은 가치 단위구현 단위
  • PI Planning 절차:
    단계설명핵심
    1. 비전 공유경영진·PM이 비즈니스 컨텍스트·비전 발표방향 정렬
    2. 아키텍처 비전시스템 아키텍처·기술 방향 공유기술 정렬
    3. 팀 브레이크아웃각 팀이 PI 목표·스프린트 계획·의존성 식별팀 계획 수립
    4. 리스크·의존성 조율팀 간 의존성·리스크를 Program Board로 시각화의존성 관리
    5. PI 목표 확정팀·ART 수준 PI 목표 확정·경영진 검토목표 확정
  • SAFe 구성 비교 (규모별):
    구성적용 규모계층 수특징
    Essential SAFe중소 규모2계층Team+Program 기본 구성
    Large Solution SAFe대형 솔루션3계층Solution Train 추가
    Portfolio SAFe전사 포트폴리오3계층포트폴리오 거버넌스 추가
    Full SAFe대기업·전사4계층전체 계층 완전 구성
  • 인사이트: SAFe는 단순 애자일 방법론을 넘어 팀→프로그램→포트폴리오를 전략적으로 연결하는 전사 운영 모델로, PI Planning을 통한 다팀 정렬이 대규모 조직의 애자일 전환 성공의 핵심 요소임. 단 도입 초기에 형식적 이벤트·문서화가 과중해지는 오버헤드 위험이 있으므로, Essential SAFe부터 단계적으로 도입하고 ART 구성에 충분한 조직 설계가 선행되어야 함
