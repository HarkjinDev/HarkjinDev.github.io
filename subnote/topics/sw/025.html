4. 폭포수 모델 (Waterfall Model)

정의: 소프트웨어 개발을 요구분석→설계→구현→테스트→배포 순서로 단계별로 순차 진행하는 고전적 생명주기 모델
키워드: 순차적, 피드백 없음, 문서 중심, 대형 프로젝트
분류: 개발 방법론
개발 단계: <요설구테유>
단계
주요 산출물
요구분석
SRS (요구사항 명세서)
설계
아키텍처·상세설계서
구현
소스코드
테스트
테스트 결과서
배포 및 유지보수
운영 문서
장단점
구분
내용
장점
단계별 검토 명확, 문서화 충실, 관리 용이
단점
요구사항 변경 어려움, 후반부 결함 발견 시 비용 급증, 고객 피드백 지연
인사이트: 폭포수 모델은 요구사항이 명확하고 변경이 적은 대형 공공 프로젝트에 적합하며, 변화 대응 한계 보완을 위해 반복적·점진적 모델(애자일)이 등장한 배경이 됨.
