[하위] 2-2. 화이트박스 테스트 (White Box Test) - 구조 기반

  • 정의: 소프트웨어 내부 구조·코드·논리를 분석하여 실행 경로와 제어 흐름을 검증하는 테스트 기법
  • 키워드: 구조 기반, 커버리지, 제어 흐름 그래프(CFG), 기본 경로 테스트, 조건 테스트, 루프 테스트
  • 분류: 테스트
  • 구조기반 테스트(ISO/IEEE 29119-4): <구분결/분건합변/정계선사/정사>
    구분기법핵심 요약특징
    문장·분기구문 테스팅코드 라인 실행프로그램의 모든 문장이 최소 한 번은 실행되는지 확인
    분기 테스팅제어 흐름 분기모든 분기점(if, switch 등)의 각 방향이 실행되는지 검증
    결정 테스팅전체 결정문 결과결정문 전체의 결과가 참/거짓이 되는 케이스 확보
    조건분기조건 테스팅개별 조건식 결과결정문 내의 개별 조건식들이 최소 한 번은 T/F가 되도록 수행
    분기조건조합 테스팅모든 조건 조합한 결정문 안의 모든 개별 조건의 조합을 테스트하여 엄격하게 검증
    변경 조건 결정(MC/DC)독립적 영향력 검증각 조건이 결과에 독립적으로 영향을 주는지 확인
    데이터 흐름모든 정의 테스팅변수 정의(Def) 커버각 변수의 정의 지점이 최소 한 번은 사용 지점으로 도달하는지 검증
    모든 계산 사용 테스팅C-Use(Calculation)변수의 정의가 모든 계산적 사용(C-use) 지점에 도달하는지 테스트
    모든 선언 사용 테스팅P-Use(Predicate)변수의 정의가 모든 조건/결정 사용(P-use) 지점에 도달하는지 테스트
    모든 사용 테스팅All-Uses모든 변수의 정의가 모든 계산 사용 및 조건 사용 지점에 도달하는지 검증
    모든 정의-사용 경로 테스팅All-du-Paths변수의 정의에서 사용까지의 모든 유효한 경로를 테스트 (가장 강력함)
  • 인사이트: 화이트박스는 코드 커버리지 확보가 목적이지만, 100% 커버리지가 결함 부재를 보장하지 않으며 명세 누락 결함은 블랙박스로 보완 필요.
