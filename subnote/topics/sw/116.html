[하위] 2-3. 데이터옵스 (DataOps)

  • 정의: DevOps 원칙을 데이터 파이프라인에 적용하여 데이터 분석·엔지니어링의 품질·속도·협업을 향상시키는 방법론
  • 키워드: 데이터 파이프라인, 데이터 품질, 자동화, 애자일 데이터, 오케스트레이션, Airflow
  • 분류: 형상관리/배포
  • DevOps vs DataOps 비교:
    구분DevOpsDataOps
    대상소프트웨어 코드데이터 파이프라인
    산출물애플리케이션 릴리즈데이터 분석 결과·모델
    테스트단위·통합 테스트데이터 품질 테스트
    핵심 도구Jenkins, GitHub ActionsAirflow, dbt, Great Expectations
  • DataOps 핵심 원칙:
    • 데이터 파이프라인 버전 관리
    • 데이터 품질 자동 검증
    • 지속적 모니터링·알림
    • 실험 관리 및 반복 가능성
  • 인사이트: DataOps는 AI/ML 파이프라인 운영(MLOps)의 기반으로, 데이터 품질 이슈를 파이프라인 초기에 자동 탐지하는 데이터 계약(Data Contract) 개념이 최근 주목.
