10. 뮤테이션 테스트 (Mutation Test)

  • 정의: 소스코드에 의도적인 결함(Mutant)을 삽입하고, 기존 테스트 케이스가 해당 변이를 탐지(Kill)하는지 평가하여 테스트 케이스의 품질을 측정하는 기법
  • 키워드: Mutant, Kill/Survive, 뮤테이션 점수(Mutation Score), 대치, 변경, 삭제
  • 분류: 테스트
  • 뮤테이션 테스트 절차:
    • 원본 코드에 변이 연산자(Mutation Operator) 적용 → Mutant 생성
    • 기존 테스트 스위트로 Mutant 실행
    • 테스트 실패 = Mutant Kill (탐지 성공)
    • 테스트 통과 = Mutant Survive (테스트 케이스 보강 필요)
    • 뮤테이션 점수 = (Kill 수 / 전체 Mutant 수) × 100%
      • 100% 의미: T는 뮤테이션 적합성 만족, 뮤턴트 Detect 가능
  • 뮤테이션 연산자:
    연산자연산자 사례코드 기반 사례연산자 설명
    대치상수 대치, 변수 대치, 상수변수 대치int a = c; → int a = b;변수, 상수, 배열 등을 다른 값으로 변경
    변경(변환)입력값 변경, 서비스 순서 변경serviceA(); serviceB(); → serviceB(); serviceA();입력/출력의 순서 변경, 속성 등의 변경
    삭제Cast 연산자 삭제, 오버로드 함수 삭제c = (int)b; → c = b;특정 함수, 변수, 연산자 등을 삭제한 돌연변이
  • 인사이트: 뮤테이션 테스트는 테스트 케이스 자체의 결함 탐지 능력을 측정하는 메타 테스트로, 커버리지 지표만으로 포착 불가능한 약한 테스트 케이스를 식별.
