[하위] 4-1. AA (Application Architect)

  • 정의: 특정 애플리케이션의 내부 구조·컴포넌트·인터페이스·기술 패턴을 설계하고 개발팀의 기술 방향을 이끄는 역할
  • 키워드: 컴포넌트 설계, 레이어 아키텍처, 프레임워크 선정, 코드 품질, 설계 패턴 적용
  • 분류: 소프트웨어 아키텍처
  • AA 주요 산출물:
    산출물설명
    애플리케이션 아키텍처 문서레이어 구조·컴포넌트·인터페이스 명세
    기술 스택 명세프레임워크·라이브러리·도구 선정 근거
    코딩 표준·가이드팀 내 일관된 개발 규약
    컴포넌트 다이어그램모듈 간 의존 관계 시각화
  • 인사이트: AA는 TA 방향성 안에서 애플리케이션 레벨의 설계 일관성을 확보하는 역할로, 코드 리뷰·페어프로그래밍으로 아키텍처 원칙이 실제 코드에 반영되도록 지속 지원.
