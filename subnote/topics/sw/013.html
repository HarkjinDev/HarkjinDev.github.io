2. 객체지향 프로그래밍 (OOP, Object Oriented Programming)

정의: 현실 세계를 객체(Object)로 모델링하고, 객체 간 메시지 교환을 통해 프로그램을 구성하는 프로그래밍 패러다임
키워드: 클래스, 객체, 메소드, 인스턴스, 캡슐화, 상속, 다형성, 추상화, 메시지 전달
분류: 프로그래밍
OOP 4대 특성: <캡추다정상>
특성
설명
주요 효과 및 키워드
캡슐화 (Encapsulation)
데이터·기능을 하나의 단위로 묶음
응집도 향상·결합도 감소·재사용성
추상화 (Abstraction)
본질적 특징만 추출·세부 제거
인터페이스·추상 클래스·복잡도 관리
다형성 (Polymorphism)
상황에 따라 다양한 형태로 동작
동작 바인딩, 오버로딩·오버라이딩
정보은닉 (Information Hiding)
내부 데이터 직접 접근 차단
접근 제어자·보안성·무결성
상속 (Inheritance)
부모 클래스 특성을 자식이 계승
코드 재사용·계층 구조·유지보수
객체 구성요소: <클객메인>
구분
핵심 설명
비고
클래스 (Class)
객체를 생성하기 위한 설계도 또는 틀
추상적 개념
객체 (Object)
클래스의 설계대로 만들어질 구현 대상
개념적 실체
메소드 (Method)
객체가 수행할 수 있는 기능이나 동작
객체의 행위
인스턴스 (Instance)
설계도(클래스)를 바탕으로 메모리에 실제 할당된 실체
구체적 실체
인사이트: OOP는 재사용성, 유지보수성, 확장성 향상이 핵심 장점이며, 4대 특성(캡슐화·상속·다형성·추상화)을 명확히 서술하는 것이 답안 기본.
