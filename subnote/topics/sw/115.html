[하위] 2-2. DevSecOps (Development, Security and Operations)

  • 정의: DevOps 파이프라인 전 단계에 보안을 내재화하여 개발 속도를 저하시키지 않으면서 보안을 지속적으로 확보하는 방법론
  • 키워드: Shift Left Security, SAST, DAST, SCA, IaC 보안, 보안 게이트, SBOM
  • 분류: 형상관리/배포
  • DevSecOps 보안 활동 (파이프라인 단계별):
    단계보안 활동도구 예시
    계획위협 모델링, 보안 요구사항 정의STRIDE, OWASP
    코드시큐어 코딩, SAST (정적 분석)SonarQube, Checkmarx
    빌드SCA (오픈소스 취약점 분석)Snyk, OWASP Dependency-Check
    테스트DAST (동적 분석), 퍼징OWASP ZAP, Burp Suite
    배포IaC 보안 검사, 컨테이너 이미지 스캔Trivy, Checkov
    운영RASP, 모니터링, 인시던트 대응WAF, SIEM
  • 인사이트: DevSecOps의 핵심은 “Shift Left”로 보안을 개발 초기부터 적용하여 취약점 수정 비용을 최소화하는 것이며, SBOM(소프트웨어 구성 명세)이 공급망 보안의 새로운 표준으로 부상.
