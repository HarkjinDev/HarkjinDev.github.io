7. 소프트웨어 유지보수 (Software Maintenance)

  • 정의: 소프트웨어 인도(Delivery) 후 결함 수정, 성능 향상, 환경 변화 적응 등을 위해 소프트웨어를 수정하는 활동
  • 키워드: IEEE 14764, 수정형, 적응형, 완전형, 예방형, 유지보수 비용
  • 분류: SW공학 개론
  • 유지보수 비용 특성
    • 전체 소프트웨어 생명주기 비용의 60~80% 차지
    • 완전형 유지보수가 가장 높은 비율 (약 50%)
  • 유지보수 유형 (IEEE 14764): <수적예완>
    구분유형원인예시
    Reactive수정형 (Corrective)결함 수정버그 픽스, 오류 수정
    적응형 (Adaptive)환경 변화 대응OS 업그레이드, DB 마이그레이션
    Proactive예방형 (Preventive)미래 장애 예방리팩토링, 문서화
    완전형 (Perfective)기능 향상 요구성능 개선, 새 기능 추가
  • 유지보수 비용 영향 요인:
    구분항목설명핵심
    기술적SW 복잡도복잡한 구조·로직은 유지보수 비용 증가복잡도 비례
    문서화 수준문서 부실 시 분석·수정 시간 증가문서 품질
    테스트 커버리지테스트 부족 시 회귀 결함 증가검증 수준
    관리적유지보수 팀 역량개발팀과 유지보수팀 분리 시 지식 손실도메인 지식
    변경 관리 체계형상관리·변경 통제 미흡 시 혼란 가중변경 통제
  • 유지보수 프로세스 (IEEE 14764): <프문수/유이폐>
    단계설명핵심
    1. 프로세스 구현유지보수 계획·절차·환경 수립유지보수 기반 수립
    2. 문제·수정 분석변경 요청 분석·영향도 평가·타당성 검토변경 요청 분석
    3. 수정 구현설계·코딩·단위 테스트·통합 테스트 수행변경 개발·검증
    4. 유지보수 검토·승인변경 내용 완전성·정확성·일관성 검토변경 품질 검증
    5. 이전 (Migration)구 환경에서 신 환경으로 SW·데이터 이전환경 전환
    6. 폐기 (Retirement)SW 운영 종료·데이터 보존·대체 시스템 전환SW 생명주기 종료
  • 유지보수 지원 기술:
    항목설명핵심
    형상관리 (SCM)변경 이력·버전 관리로 유지보수 추적성 확보변경 이력 관리
    역공학 (Reverse Engineering)기존 코드에서 설계·요구사항 역추출레거시 분석
    재공학 (Re-Engineering)기존 SW를 분석 후 재설계·재구현현대화
    회귀 테스트 (Regression Test)수정 후 기존 기능 영향 여부 재검증변경 영향 검증
    프로파일링성능 병목·메모리 누수 탐지·분석성능 개선
  • 인사이트: 소프트웨어 유지보수는 전체 SW 생명주기 비용의 약 60%를 차지하는 핵심 활동으로, 개발보다 유지보수 비용이 더 크다는 사실이 SW 설계 단계에서부터 유지보수성을 내재화해야 하는 근거임. IEEE 14764의 4대 유형 중 완전 유지보수가 50% 이상으로 가장 높은 비중을 차지하며, 이는 유지보수가 단순 결함 수정이 아닌 지속적 품질·기능 향상 활동임을 의미함. 유지보수 비용 절감의 핵심은 개발 단계에서의 모듈화·문서화·테스트 자동화 투자임
