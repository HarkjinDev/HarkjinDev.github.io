[하위] 1-3. 소프트웨어 아키텍처 분석

  • 정의: 시스템의 비즈니스 목표 달성을 위해 아키텍처 설계가 품질 속성을 만족하는지 체계적으로 평가하고, 상충 관계(Trade-off) 분석 및 잠재적 리스크를 식별하는 활동
  • 키워드: 시나리오기반 평가, 설계 기반, 트레이드오프 식별, ATAM, CBAM, 민감점, 위험 결정, 비위험 결정
  • 분류: 소프트웨어 아키텍처
  • 평가모델:
    분류평가모델내용
    시나리오 기반SAAM* Software Architecture Analysis Method * 수정 용이성과 기능 분석 중심의 최초 아키텍처 평가 방법 * 다양한 수정 가능성 관점에서 아키텍처 분석 수행
    ATAM* Architecture Tradeoff Analysis Method * 품질 속성(Quality Attributes) 간의 Trade-off 관계까지 포함하여 평가 수행
    CBAM* Cost Benefit Analysis Method * ATAM에 경제성 평가를 보강하여 비용과 일정 관계를 파악 * 아키텍처 의사결정에 따른 전략적 비용 및 이익 고려
    EATAM* Extending Architecture Tradeoff Analysis Method * 개별 평가모델의 확장판으로, 스테이지 기반 모델을 통한 PL(Product Line) 아키텍처 평가 수행
    설계/혼합 기반ARID* Architecture Review for intermediate Design * 부분 아키텍처를 설계 초기 단계에 평가하는 방법 * 시나리오 중심(ATAM, SAAM)과 설계 검토(ARD) 방식을 혼합하여 위험 감소
    ADR* Active Design Review * 설계 기반으로 아키텍처 구성 요소 간의 응집도 평가
  • 인사이트: ATAM은 품질 속성 간 트레이드오프(예: 성능↑ vs 보안↑)를 명시적으로 파악하는 것이 핵심이며, 아키텍처 확정 전 리스크를 조기에 제거하는 예방적 분석 기법.
