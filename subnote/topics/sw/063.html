[하위] 1-1. 소프트웨어 아키텍처 모델

  • 정의: 소프트웨어 시스템의 구조·동작·특성을 추상화하여 표현하는 공식적 또는 반공식적 표현 방식
  • 키워드: 4+1 뷰, C4 모델, ADL(Architecture Description Language), 컴포넌트·커넥터
  • 분류: 소프트웨어 아키텍처
  • 4+1 뷰 모델 (Kruchten): <논개프물유>
    뷰관점주요 UML 다이어그램
    논리 뷰 (Logical)기능·설계 구조클래스, 순차 다이어그램
    개발 뷰 (Development)코드·모듈 구조컴포넌트, 패키지 다이어그램
    프로세스 뷰 (Process)런타임 동작·동시성활동, 상태 다이어그램
    물리 뷰 (Physical)배포·하드웨어 구성배치 다이어그램
    Usecase(시나리오) (+1)유즈케이스로 뷰 통합 검증유즈케이스 다이어그램
  • C4 모델 (Simon Brown): <텍테포코>
    레벨표현 대상대상 독자
    Context (시스템 컨텍스트)시스템과 사용자·외부 시스템 관계모든 이해관계자
    Container (컨테이너)애플리케이션·데이터 저장소 배치기술 책임자
    Component (컴포넌트)컨테이너 내부 주요 구성 요소개발자·아키텍트
    Code (코드)클래스 수준 UML개발자
  • 인사이트: 4+1 뷰는 UML 기반 다중 관점으로 전통적 아키텍처 문서화에 강점이 있고, C4 모델은 계층별 줌인(Zoom-in) 방식으로 현대 시스템의 가시화에 적합.
