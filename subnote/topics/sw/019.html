4. AOP (Aspect Oriented Programming)

정의: 횡단 관심사(Cross-cutting Concern)를 핵심 비즈니스 로직과 분리하여 모듈화하는 프로그래밍 패러다임
키워드: 횡단 관심사, Aspect, Advice, Pointcut, JoinPoint, Weaving, Spring AOP
분류: 프로그래밍
AOP 핵심 개념
용어
설명
Aspect
횡단 관심사를 모듈화한 단위
Advice
Aspect가 실행하는 동작 (Before/After/Around 등)
JoinPoint
Advice가 적용될 수 있는 실행 지점
Pointcut
JoinPoint 중 실제 Advice를 적용할 지점 선택 표현식
Weaving
Aspect를 핵심 코드에 적용하는 과정 (컴파일/런타임/로드타임)
횡단 관심사 예시: 로깅 (Logging), 보안 (Security), 트랜잭션 관리, 성능 모니터링, 예외 처리
인사이트: AOP는 OOP의 한계(횡단 관심사 코드 중복)를 보완하는 패러다임으로, Spring AOP는 런타임 위빙 방식을 사용하며 코드 응집도 향상과 중복 제거가 핵심 이점.
