[하위] 1-5. STPA (System Theoretic Process Analysis)

  • 정의: 사고를 단순 고장의 연쇄가 아닌 제어 구조의 부적절한 제약에서 발생하는 것으로 보고, 시스템 이론 기반으로 위험을 분석하는 현대적 안전성 분석 기법
  • 키워드: STAMP(System-Theoretic Accident Model), 제어 구조, UCAs(Unsafe Control Actions), 손실 시나리오
  • 분류: 안전성 분석
  • UCAs(Unsafe Control Action 4가지 유형):
    • Unsafe Control Action : 시스템의 위험을 유발할 수 있는 Control Action의 불안전한 형태
      1. Control Action의 부재
      2. 부적절한 Control Action
      3. Control Action 제공 시간, 순서
      4. Control Action 지속시간
  • STPA vs 전통 기법 비교:
    구분전통 기법 (FTA·FMEA)STPA
    기반 이론신뢰성 이론 (부품 고장)시스템 이론 (제어 구조)
    사고 원인 관점컴포넌트 고장 연쇄부적절한 제어 상호작용
    소프트웨어 적용제한적적합 (SW 로직 분석)
    창발적 위험 처리어려움가능
  • 인사이트: STPA는 소프트웨어·자율주행·AI 시스템처럼 복잡한 상호작용이 많은 현대 시스템에 특히 적합하며, 컴포넌트 자체가 정상이어도 제어 흐름 오류로 사고가 발생함을 설명.
