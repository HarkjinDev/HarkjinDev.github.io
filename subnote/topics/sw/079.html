1. 소프트웨어 테스트 원리

  • 정의: 효과적인 소프트웨어 테스트 수행을 위한 보편적 기본 원칙들의 집합
  • 키워드: 결함 존재 증명, 완벽한 테스트 불가, 조기 테스트, 결함 집중, 살충제 역설, 정황 의존, 오류 부재의 궤변
  • 분류: 테스트
  • 소프트웨어 테스트 7대 원리 (ISTQB): <결완초집/살정오>
    원리설명
    결함 존재 증명테스트는 결함이 있음을 증명, 없음은 증명 불가
    완벽한 테스트 불가입력 조합 무한대, 완전 테스트는 불가능
    초기 테스트조기 시작할수록 결함 수정 비용 감소
    결함 집중결함의 80%는 20%의 모듈에 집중 (파레토 법칙)
    살충제 역설동일 테스트 반복 시 새 결함 발견 불가 (내성)
    정황 의존테스트는 개발 컨텍스트에 따라 다르게 수행
    오류 부재의 궤변결함 없어도 요구사항 불충족 시 품질 미달
  • 소프트웨어 테스트 구성도:
  • ISO 29119 구성요소: <데프도테키>
    구분설명주요 항목
    Part 1. 개념과 정의 (Concepts/Definitions)전체 시리즈에 대한 가이드 및 용어, 테스팅 개념 정의 (8개 단락 구성)소프트웨어 테스팅 소개, SDLC 모델 내 테스팅 프로세스, 위험 기반 테스팅, 테스트 자동화 및 결함관리
    Part 2. 테스트 프로세스 (Test Process)조직, 테스트 관리, 동적 테스트 수준의 다계층 프로세스 모델 설명조직의 테스트 프로세스, 테스트 관리 프로세스(전략, 모니터링), 동적 테스트 프로세스
    Part 3. 테스트 문서화 (Test Document)테스트 단계별 산출 문서의 견본, 작성 방법 및 포함 내용 제공조직의 테스트 프로세스 문서, 테스트 관리 프로세스 문서, 동적 테스트 프로세스 문서
    Part 4. 테스트 설계기법 (Test Techniques)설계 및 구현 단계에서 활용 가능한 구체적인 테스트 설계 기법 제공테스트 설계 기법 (명세/구조/경험 기반), 테스트 커버리지 측정
    Part 5. 키워드 주도 테스팅 (Keyword-Driven)키워드 기반의 테스트 자동화 프레임워크 및 접근 방법 소개키워드 주도 테스팅 활용 및 프레임워크, 기본 키워드 사례
  • 테스트 프로세스(ISO/IEEE 29119-2): <계모설실완>
    구분프로세스상세 설명
    테스트 관리1. 테스트 계획- 테스트 목적, 범위 및 대상 정의, 리스크 식별 및 분석 - 테스트에 필요한 자원 할당, 테스트 일정 계획 수립
    2. 테스트 모니터링 및 제어- 진행 상황 모니터링, 테스트 계획 대비 실행 수준 점검 - 테스트 진행 중 발생하는 이슈 식별 및 제어
    동적 테스트3. 테스트 설계 및 구현- TC(Test Case) 설계 및 작성, 테스트 데이터 준비 - 테스트 환경 구축, 테스트 실행 순서(Procedure) 구현
    4. 테스트 실행- 상세 TC(Test Case) 및 테스트 코드 실행, 테스트 수행 기록 - 테스트 수행 결과 분석, 오류 수정 및 회귀 테스트 수행
    테스트 관리 5. 테스트 완료- 테스트 완료 조건(Exit Criteria) 만족 여부 확인 - 테스트 프로세스 평가 및 회고, 테스트 결과 문서화
  • Test Case 구성요소(IEEE 829-1998): <식항입출/환특의>
    항목영문 명칭설명비고
    식별자Test Case Specification Identifier테스트케이스를 식별하기 위한 유일한 식별자추적성
    테스트 항목Test Items테스트 할 모듈이나 기능테스트 대상
    입력 명세Input Specifications테스트에 필요한 입력값 또는 조건입력값
    출력 명세Output Specifications테스트케이스 실행 시 기대되는 결과출력값
    환경 설정Experimental Needs테스트 수행 시 필요한 HW/SW 환경테스트베드
    특수절차 요구Special Procedural Requirements테스트 수행 시 특별히 요구되는 절차프로세스
    의존성 기술Inter-case Dependencies테스트케이스 간의 의존성 및 선후 관계선후 관계
  • 인사이트: 살충제 역설은 회귀 테스트 케이스의 정기적 갱신 필요성의 근거이며, 결함 집중 원리는 위험 기반 테스트(Risk-Based Testing) 전략의 이론적 기반.
