1. 구조적 방법론 (Structured Methodology)

정의: 업무를 기능 단위로 분해하고 절차적 흐름으로 분석·설계하는 전통적 소프트웨어 개발 방법론
키워드: DFD(Data Flow Diagram), 구조도, 하향식 설계, 분할 정복, 프로세스 중심
분류: 개발 방법론
주요 산출물
산출물
설명
DFD (Data Flow Diagram)
데이터 흐름 시각화 (프로세스·데이터 저장소·외부 엔티티)
구조도 (Structure Chart)
모듈 간 계층적 호출 관계 표현
자료 사전 (Data Dictionary)
DFD에 등장하는 데이터 정의
소단위 명세서
최하위 프로세스 상세 명세
인사이트: 구조적 방법론은 하향식(Top-down) 분해가 특징으로, 데이터 흐름 중심 분석(DFD)과 기능 분해 설계(구조도)의 연계가 핵심이며 대규모·공공 시스템에 적합.
