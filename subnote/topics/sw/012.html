1. 절차지향 프로그래밍 (Procedure Oriented Programming)

  • 정의: 프로그램을 순차적으로 실행되는 함수(프로시저)들의 집합으로 구성하는 프로그래밍 패러다임
  • 키워드: 함수, 프로시저, 순차 실행, 전역변수, C언어, 구조적 프로그래밍
  • 분류: 프로그래밍
  • OOP와 비교
    구분절차지향객체지향
    중심 개념함수(프로시저)객체(데이터+메서드)
    데이터 처리전역변수 중심캡슐화된 속성
    재사용함수 재호출상속·다형성
    유지보수데이터-함수 결합 높음독립적 모듈화
    대표 언어C, Pascal, FortranJava, C++, Python
  • 인사이트: 절차지향은 처리 흐름이 명확하고 성능이 높은 장점이 있어 임베디드·시스템 프로그래밍에 여전히 활용되며, OOP와의 장단점 비교가 주요 출제 포인트.
