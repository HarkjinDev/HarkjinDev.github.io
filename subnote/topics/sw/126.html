7. 백필 (Backfill)

  • 정의: 과거 기간의 누락되거나 잘못된 데이터를 소급하여 재처리·재적재하는 작업
  • 키워드: 재처리, 소급 적용, 데이터 파이프라인, Airflow backfill, 멱등성(Idempotency), 전체 재처리, 증분, 병렬, 섀도우
  • 분류: 형상관리/배포
  • 백필 필요 상황:
    항목설명핵심
    파이프라인 장애장애로 특정 기간 데이터 누락 발생누락 데이터 복구
    비즈니스 로직 변경계산 로직 변경으로 과거 데이터 재계산 필요소급 재계산
    신규 데이터 소스 도입새 소스 연동 시 과거 기간 소급 적재히스토리 확보
    데이터 오류 정정오류 발견 후 해당 기간 데이터 재처리품질 정정
    스키마 변경컬럼 추가·타입 변경 후 과거 데이터 재적재구조 변경 대응
  • 백필 방식:
    방식설명핵심
    전체 재처리전체 기간 데이터를 일괄 삭제 후 재적재단순·시간 소요
    증분 백필영향 받은 특정 기간만 선택적 재처리효율적·정밀
    병렬 백필기간을 분할하여 병렬로 동시 처리속도 향상
    섀도우 백필기존 데이터 유지하며 신규 테이블에 병행 처리무중단 검증
  • 백필 설계 시 고려사항:
    고려사항설명핵심
    멱등성 (Idempotency)동일 기간 재실행 시 항상 동일 결과 보장중복 실행 안전
    부분 실패 처리특정 날짜 실패 시 해당 구간만 재시도구간별 재시도
    의존성 순서업스트림 완료 후 다운스트림 실행 보장실행 순서 제어
    프로덕션 영향 최소화리소스 분리 또는 야간·주말 실행서비스 영향 차단
    데이터 중복 방지UPSERT·덮어쓰기로 중복 적재 방지적재 무결성
    백필 범위 최소화영향 받은 기간만 선별하여 범위 제한불필요 처리 방지
  • 인사이트: 백필은 멱등성(Idempotency) 설계가 전제되어야 안전하게 반복 실행 가능하며, 대규모 백필은 프로덕션 파이프라인과 리소스 격리하여 서비스 영향을 방지.
