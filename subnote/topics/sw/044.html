4. 요구사항 추적표 (RTM, Requirements Traceability Matrix)

정의: 요구사항과 설계·구현·테스트 산출물 간의 연결 관계를 매핑하여 전 주기 추적성을 확보하는 표
키워드: 영향도 분석, 추적성, 순방향/역방향 추적, 변경 관리, 커버리지
분류: 요구공학
RTM 추적 방향:
구분
순방향 추적 (Forward)
역방향 추적 (Backward)
정의
요구사항에서 최종 산출물까지의 흐름을 추적
최종 산출물에서 요구사항 근거를 역으로 추적
추적 흐름
요구사항 → 설계 → 코드 → 테스트 케이스
테스트 케이스 → 코드 → 설계 → 요구사항
핵심 목적
누락 방지: 모든 요구사항이 반영되었는가?
근거 확인: 구현된 기능의 원천이 무엇인가?
주요 활용
요구사항 변경 시 영향도 분석, 진행사항 모니터링
불필요한 기능(Gold Plating) 및 중복/불필요 코드 제거, 테스트 실패 시 역 검증
기대 효과
요구사항 충족 및 범위 관리
소프트웨어 정제 및 유지보수 효율성 향상
관리 도구
RTM (Requirement Traceability Matrix)
RTM, 형상관리 도구, 소스코드 정적 분석
