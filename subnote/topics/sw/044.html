4. 요구사항 추적표 (RTM, Requirements Traceability Matrix)

  • 정의: 요구사항과 설계·구현·테스트 산출물 간의 연결 관계를 매핑하여 전 주기 추적성을 확보하는 표
  • 키워드: 영향도 분석, 추적성, 순방향/역방향 추적, 변경 관리, 커버리지
  • 분류: 요구공학
  • RTM 추적 방향:
    구분순방향 추적 (Forward)역방향 추적 (Backward)
    정의요구사항에서 최종 산출물까지의 흐름을 추적최종 산출물에서 요구사항 근거를 역으로 추적
    추적 흐름요구사항 → 설계 → 코드 → 테스트 케이스테스트 케이스 → 코드 → 설계 → 요구사항
    핵심 목적누락 방지: 모든 요구사항이 반영되었는가?근거 확인: 구현된 기능의 원천이 무엇인가?
    주요 활용요구사항 변경 시 영향도 분석, 진행사항 모니터링불필요한 기능(Gold Plating) 및 중복/불필요 코드 제거, 테스트 실패 시 역 검증
    기대 효과요구사항 충족 및 범위 관리소프트웨어 정제 및 유지보수 효율성 향상
    관리 도구RTM (Requirement Traceability Matrix)RTM, 형상관리 도구, 소스코드 정적 분석
  • 주요 항목:
    항목설명예시
    요구사항 ID각 요구사항을 식별할 수 있는 고유 IDRQ-001, RQ-002
    요구사항 설명해당 요구사항의 기능적 또는 비기능적 설명사용자는 아이디와 비밀번호를 입력하여 로그인할 수 있어야 함
    요구사항 유형기능적 요구사항(FR) 또는 비기능적 요구사항(NFR)기능적 요구사항 (FR)
    우선 순위요구사항 중요도 및 긴급도(High/Medium/Low)High
    연관 시스템/모듈요구사항이 적용될 시스템 또는 모듈사용자 인증 모듈 (AuthModule)
    설계 문서 참조해당 요구사항이 반영된 설계 문서 링크 또는 참조 ID설계 문서 ID: DS-001

    테스트 케이스 ID요구사항 검증을 위한 테스트 케이스 IDTC-001
    테스트 결과요구사항 검증 결과 (Pass/Fail)Pass
    변경 이력요구사항 변경 내역 및 수정 일자2024-01-15: 비밀번호 최소 길이 변경 (8→10)
    검토자요구사항을 검토 및 승인한 담당자홍길동(PM)
  • RTM 구성 예시:
    요구사항 ID요구사항 설명설계 산출물소스 모듈테스트 케이스 ID
