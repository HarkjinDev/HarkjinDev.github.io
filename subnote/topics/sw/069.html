[하위] 1-7. 아키텍처 스타일 (Architecture Style)

  • 정의: 소프트웨어 시스템을 구성하는 컴포넌트·커넥터·제약의 재사용 가능한 조직화 패턴
  • 키워드: Pipe-and-Filter, Layered, Client-Server, Event-Driven, Microservices, Repository
  • 분류: 소프트웨어 아키텍처
  • 주요 아키텍처 스타일:
    스타일특징적합 시스템
    Layered (계층형)상위→하위 단방향 의존웹 애플리케이션, ERP
    Pipe-and-Filter데이터 변환 필터 직렬 연결컴파일러, ETL, 신호처리
    Client-Server클라이언트 요청·서버 응답웹서비스, DB 시스템
    Event-Driven이벤트 생산자·소비자 비동기IoT, 금융 거래, EDA
    Microservices독립 배포 가능한 소형 서비스대규모 웹 플랫폼
    Repository공유 데이터 저장소 중심IDE, CASE 도구
    Blackboard전문가 지식 협업 문제 해결AI·음성인식 시스템
  • 인사이트: 아키텍처 스타일은 시스템 품질 속성(성능·유지보수성·확장성)에 따라 선택하며, 단일 스타일보다 혼합(Hybrid) 적용이 실제 시스템에서 일반적.
