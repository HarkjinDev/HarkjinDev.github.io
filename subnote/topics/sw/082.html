[하위] 2-1. 블랙박스 테스트 (Black Box Test) - 명세 기반

  • 정의: 내부 구현을 모른 채 입력과 기대 출력만으로 소프트웨어의 기능을 검증하는 테스트 기법
  • 키워드: 명세 기반, 동등 분할, 경계값 분석, 원인-결과 그래프, 상태 전이, 유즈케이스 테스트
  • 분류: 테스트
  • 명세기반 테스트(ISO/IEEE 29119-4): <동분경구랜/모페이베/결원상시>
    구분기법핵심특징
    입력동등 분할영역별 대표값입력 도메인을 등가 영역으로 테스트 케이스 최적화
    분류 트리 기법계층적 트리 구조입력 항목을 트리 구조로 시각화하여 테스트 케이스 설계
    경계값 분석경계 지점 값결함 발생 가능성이 높은 경계값 집중 검증
    구문 규칙 테스팅입력 문법 규칙입력 데이터 형식·프로토콜 문법 규칙 검증
    랜덤 테스팅무작위 데이터 입력무작위 입력으로 예기치 못한 결함 탐색
    조합모든 조합 기법전체 경우의 수모든 입력 파라미터의 가능한 전체 조합 테스트
    페어와이즈 테스팅2개 요소 간 조합파라미터 간 모든 쌍을 최소 1회 조합하여 효율 극대화
    이치 초이스 테스팅각 값의 1회 노출각 파라미터 값이 최소 1회 테스트에 포함되도록 구성
    베이스 초이스 테스팅기준 값 중심 변경기준 조합 정하고 파라미터 값만 하나씩 변경 테스트
    명세결정 테이블 테스팅논리적 조합 표복잡한 조건·행위를 표로 명세화하여 검증
    원인-결과 그래프인과 관계 도식화입력·출력의 논리적 관계를 그래프로 분석
    상태 전이 테스팅시스템 상태 변화이벤트에 따른 상태 전이 경로·유효 동작 확인
    시나리오 테스팅업무 프로세스 흐름유스케이스·비즈니스 흐름 기반 실사용자 환경 검증
  • 인사이트: 블랙박스는 기능 명세서가 기준이므로 명세 오류 발견에 한계가 있으며, 화이트박스와 조합하여 기능·구조 양면 검증이 완전한 테스트 전략.
