6. 워크플로우 관리 (Workflow Management)

  • 정의: 업무 프로세스의 단계·조건·참여자·자동화 규칙을 정의하고 실행·모니터링하는 시스템
  • 키워드: DAG(Directed Acyclic Graph), 오케스트레이션, Apache Airflow, 태스크 의존성, 스케줄링, 재시도
  • 분류: 형상관리/배포
  • 워크플로우 관리 핵심 개념:
    개념설명
    DAG태스크 간 방향성 있는 비순환 의존 그래프
    오케스트레이션중앙 조율자가 태스크 실행 순서·조건 제어
    코레오그래피각 서비스가 이벤트로 자율적으로 반응
    스케줄링시간·이벤트 기반 워크플로우 자동 트리거
  • 주요 워크플로우 관리 도구:
    도구특징
    Apache AirflowPython DAG 기반, 데이터 파이프라인
    Prefect현대적 Python 워크플로우, 동적 구성
    Argo WorkflowsKubernetes 네이티브 컨테이너 워크플로우
    AWS Step Functions서버리스 시각적 워크플로우
  • 인사이트: 오케스트레이션과 코레오그래피는 MSA 내 서비스 협력 방식의 두 축으로, 오케스트레이션은 중앙 제어로 가시성이 높고 코레오그래피는 분산 자율로 결합도가 낮음.
