5. 로드밸런싱 (Load Balancing)

  • 정의: 서버 클러스터에 유입되는 트래픽을 여러 서버에 분산하여 가용성과 성능을 향상시키는 기법
  • 키워드: L4/L7 로드밸런서, 라운드 로빈, 가중치, 최소 연결, IP 해시, 세션 어피니티, 헬스체크
  • 분류: 형상관리/배포
  • 로드밸런싱 알고리즘:
    구분알고리즘설명적합 상황
    RR라운드 로빈 (Round Robin)순서대로 균등 분산서버 동일 사양
    가중 라운드 로빈서버 성능 비율로 분산서버 이기종 환경
    최소최소 연결 현재 연결 수 가장 적은 서버 선택요청 처리 시간 편차 큼
    최소 응답시간응답 빠른 서버 우선 선택응답시간 편차 큼
    고정IP 해시 (IP Hash)클라이언트 IP 기반 고정 서버세션 유지 필요
  • L4 vs L7 로드밸런서 비교:
    구분L4 (전송 계층)L7 (응용 계층)
    기준IP·포트URL·헤더·쿠키·콘텐츠
    속도빠름상대적으로 느림
    스마트 라우팅불가가능 (컨텐츠 기반)
    SSL 종료불가가능
  • 인사이트: L7 로드밸런서는 API Gateway 기능과 중첩되는 부분이 있으며, MSA에서는 서비스 경로·버전 기반 스마트 라우팅이 필요하므로 L7 또는 서비스 메쉬(Envoy)를 함께 활용.
