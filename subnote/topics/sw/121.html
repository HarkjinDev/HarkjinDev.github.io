4. 리버스 프록시 (Reverse Proxy)

  • 정의: 클라이언트와 백엔드 서버 사이에서 클라이언트 요청을 대신 받아 내부 서버로 전달하고 응답을 반환하는 중간 서버
  • 키워드: 포워드 프록시, Nginx, HAProxy, 캐싱, SSL 종료, 로드밸런싱, 보안
  • 분류: 형상관리/배포
  • 포워드 프록시 vs 리버스 프록시 비교:
    구분포워드 프록시리버스 프록시
    위치클라이언트 앞서버 앞
    대리 대상클라이언트서버
    주요 용도클라이언트 익명화, 우회서버 보호, 로드밸런싱
    서버 인지서버가 클라이언트 실IP 모름클라이언트가 서버 구성 모름
  • 리버스 프록시 주요 기능:
    • SSL/TLS 종료(Termination): 암복호화를 리버스 프록시에서 처리
    • 로드밸런싱: 백엔드 서버 트래픽 분산
    • 캐싱: 정적 콘텐츠 캐시로 백엔드 부하 감소
    • 압축: Gzip 응답 압축
    • 보안: DDoS 방어, IP 차단, WAF 연동
  • 인사이트: 리버스 프록시는 백엔드 서버 구조를 클라이언트에 완전히 숨기는 추상화 계층으로, MSA에서 API Gateway와 함께 외부 노출 지점을 단일화하는 핵심 인프라.
