[하위] 2-1. 캡슐화 (Encapsulation)

정의: 객체의 데이터(속성)와 이를 처리하는 메서드(행위)를 하나의 클래스로 묶고, 내부 구현을 외부로부터 감추는 OOP 원칙
키워드: 재사용성 극대화, 접근 제어자(public/private/protected), 게터/세터, 정보은닉
분류: 프로그래밍

핵심 개념
'묶기': 데이터와 함수를 하나로 묶음
'숨기기': 내부 구현을 외부에 감춤
목적
응집도 향상, 재사용성 증대
결합도 감소, 무결성 보호, 보안 강화
수단
클래스(Class), 인터페이스, 접근제어자
접근제어자(private, public 등)
관계
정보은닉을 포함하는 상위 개념
캡슐화 안에서 실현되는 하위 개념
인사이트: 캡슐화는 객체의 독립성을 높여 재사용성을 극대화하고, 정보은닉은 객체 간의 결합도를 낮추어(Loose Coupling) 변경에 유연한 아키텍처를 가능케 한다. 캡슐화가 정보은닉을 구현하는 메커니즘이라는 포함 관계를 시험에서 정확히 서술하는 것이 중요.
