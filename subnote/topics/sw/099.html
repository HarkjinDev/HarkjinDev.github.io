15. 사용성 테스트(UT, Usability Testing)

  • 정의: 사용자가 제품을 직접 사용해 보면서 직관성, 학습 용이성, 효율성 등을 측정하고, 사용 과정의 문제점을 파악하여 UX 품질을 개선하는 테스트 기법
  • 키워드: UI/UX, 페이퍼 목업, 카드 소팅, 생각소리내기(Thinking Aloud), 스크린 목업
  • 분류: 테스트
  • 프로세스:
    단계주요 활동 내용산출물
    Planning평가 목적 분석, 사용자 집단 정의, 태스크 분석사용성 테스트 계획서
    Design테스트 디자인, 참가자 선정, 과업 선정, 질문지 작성사용성 테스트 설계서
    Execution진행 스크립트 작성, 사전 테스트, 본 테스트 진행질적/양적 Raw data
    Report결과 분석(3P 분석 등), 보고서 작성, 개선 우선순위 선정사용성 테스트 결과 보고서
  • 유형:
    구분시기목적방법
    탐색 테스트초반디자인 컨셉 유효성 평가페이퍼 목업, 화면 디자인
    평가 테스트초중반컨셉의 효율성 평가정량적 자료, 과업 수행
    확인 테스트후반부사용성 보증, 표준 부합여부실행척도(속도, 정확도), 선호도
    비교 테스트초,중,후반대안 평가인터페이스 스타일, 요소 평가
  • 방법:
    방법설명핵심
    카드 소팅 (Card Sorting)사용자가 메뉴·콘텐츠를 직접 분류정보 구조 설계
    페이퍼 목업 (Paper Mock-up)종이 프로토타입으로 초기 구조 검증저비용 초기 검증
    스크린 목업 (Screen Mock-up)실제 화면 유사 프로토타입으로 테스트네비게이션 오류 측정
    사고 발화법 (Think Aloud)사용 중 생각을 소리 내어 말하게 유도내면 행동 파악
    휴리스틱 평가 (Heuristic Evaluation)전문가가 10대 원칙 기준으로 평가전문가 원칙 검토
    인지적 워크스루 (Cognitive Walkthrough)전문가가 사용자 관점으로 태스크 수행학습 용이성 평가
    아이 트래킹 (Eye Tracking)시선 추적으로 주목 영역·패턴 분석시선 흐름 분석
    A/B 테스트두 버전 비교로 사용성 우위 측정버전 비교 검증
    원격 사용성 테스트원격 환경에서 실제 사용자 행동 관찰비대면 실사용 관찰
    설문·인터뷰사용 후 만족도·불편 사항 수집정성적 피드백
    태스크 기반 테스트특정 과업 수행 후 완료율·시간 측정태스크 완료율 측정
    5초 테스트화면 5초 노출 후 첫인상·목적 확인직관적 인지 측정
  • 인사이트: 사용자 경험(UX) 중심의 품질 보증 전략이 필수적이며, 개발 후반부의 수정 비용을 줄이기 위해 초기에 카드 소팅 및 페이퍼 목업을 적극 활용하는 Shift-Left 테스팅 구현이 핵심 경쟁력
