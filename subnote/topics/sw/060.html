6. MVC 패턴 (Model View Controller)

  • 정의: 애플리케이션을 Model·View·Controller로 분리하여 관심사를 독립적으로 관리하는 아키텍처 패턴
  • 키워드: 관심사 분리, Model, View, Controller, MVP, MVVM, Spring MVC
  • 분류: 설계 원리/패턴
  • MVC 구성 역할:
    구성 요소역할예시
    Model데이터·비즈니스 로직 처리DB 연동, 도메인 객체
    View사용자 화면(UI) 표시HTML, JSP, Template
    Controller사용자 입력 처리 → Model·View 연결Servlet, @Controller
  • MVC 변형 패턴 비교:
    패턴특징
    MVCController가 Model·View 중재
    MVPPresenter가 View를 능동적으로 갱신
    MVVMViewModel이 데이터바인딩으로 View 자동 갱신
  • 인사이트: MVC는 유지보수성·테스트 용이성 향상이 핵심 장점이며, 웹에서는 Spring MVC, 프론트엔드에서는 MVVM(Vue·Angular)이 주류로, 패턴 선택 기준은 UI 업데이트 복잡도.
