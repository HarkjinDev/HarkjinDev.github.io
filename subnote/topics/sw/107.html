21. 패키지 소프트웨어 테스트

  • 정의: 상용 패키지(ERP·CRM·그룹웨어 등)를 도입 환경에 맞게 설치·구성한 후 요구사항 충족 및 커스터마이징 정합성을 검증하는 테스트
  • 키워드: 인수 테스트(UAT), 설치 테스트, 구성 테스트, 호환성 테스트, ISO 25051
  • 분류: 테스트
  • 패키지 SW 테스트 주요 유형:
    유형설명
    설치 테스트설치·업그레이드·제거 정상 동작 확인
    구성 테스트다양한 구성 옵션에서의 정상 동작
    인수 테스트 (UAT)실제 사용자가 업무 시나리오로 검증
    회귀 테스트커스터마이징 후 기본 기능 영향 확인
    호환성 테스트기존 시스템·데이터와의 연동 검증
  • 인사이트: 패키지 SW는 소스 수정이 제한적이므로 도입 전 GAP 분석(기능 갭 파악)이 필수이며, UAT에서 실 업무 담당자가 직접 시나리오를 검증하는 것이 핵심.
