[하위] 2-4. 추상클래스/인터페이스 (Abstract Class/Interface)

  • 정의: 추상클래스는 일부 구현을 포함한 미완성 클래스이고, 인터페이스는 구현 없이 메서드 시그니처만을 정의하는 계약(Contract)
  • 키워드: abstract, interface, 다중 상속, 계약, 강제 구현, 설계 원칙
  • 분류: 프로그래밍
  • 추상클래스 vs 인터페이스 비교
    구분추상클래스인터페이스
    구현 포함가능 (일부 구현 허용)불가 (Java 8+: default 메서드 예외)
    다중 상속불가 (단일 상속)가능 (다중 구현)
    생성자있음없음
    목적공통 기능 공유행위 계약 강제
    키워드abstract classinterface / implements
  • 인사이트: 인터페이스는 DIP(의존성 역전 원칙)와 연계되어 구체 클래스 의존을 제거하는 핵심 수단이며, 추상클래스는 공통 로직을 상속으로 공유할 때 선택.
