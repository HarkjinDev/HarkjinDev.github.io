[하위] 3-1. 무중단 배포 (Zero Downtime Deployment)

  • 정의: 사용자가 배포 중 서비스 중단을 경험하지 않도록 서비스 가용성을 유지하면서 새 버전으로 교체하는 배포 방식
  • 키워드: 가용성, 헬스체크(Health Check), 트래픽 드레이닝(Draining), 로드밸런서, Graceful Shutdown, Expand-Contract 패턴
  • 분류: 형상관리/배포
  • 무중단 배포 구현 핵심 요소:
    요소설명핵심
    헬스체크신규 인스턴스 정상 확인 후 트래픽 전환정상 확인 후 전환
    트래픽 드레이닝구버전 처리 중 요청 완료 대기요청 완료 대기
    Graceful Shutdown진행 중 요청 완료 후 인스턴스 종료안전 종료
    로드밸런서 연동트래픽 라우팅 자동 제어자동 라우팅
    DB 하위 호환성구·신버전 공존 시 스키마 호환 유지스키마 호환 유지
    리버스 프록시보안 강화·부하 분산 역할 수행보안·부하 분산
  • 인사이트: 무중단 배포의 실제 장애 원인은 코드보다 DB 스키마 변경인 경우가 많아, Expand-Contract 패턴(스키마 확장→마이그레이션→구칼럼 제거)이 필수 실천.
