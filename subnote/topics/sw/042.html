2. 요구사항 도출 (Requirements Elicitation)

정의: 이해관계자로부터 시스템에 대한 요구사항을 수집·발견하는 활동
키워드: 인터뷰, 설문, 워크숍, 프로토타이핑, 관찰(Ethnography), 유즈케이스, JAD(공동 애플리케이션 설계)
분류: 요구공학
도출 기법: <인설워포/유컨벤브/관프사자>
구분
도출 기법
핵심
직접 소통
인터뷰 (Interview)
이해관계자 직접 면담·질의
설문 (Questionnaire)
다수 대상 서면 응답 수집
워크숍 (Workshop)
이해관계자 공동 토론·합의
포커스 그룹 (Focus Group)
소규모 집단 토론·의견 수렴

컨텍스트 다이어그램
시스템 경계·외부 인터페이스 시각화

타 시스템·사례 비교 분석

시제품으로 요구사항 조기 검증
사용자 스토리 (User Story)
사용자 관점 기능 요구 기술
JAD (공동 요구사항 정의)
개발자·사용자 공동 정의 워크숍
인사이트: 단일 기법보다 복합 기법(인터뷰 + 프로토타이핑) 적용이 요구사항 누락을 최소화하며, 특히 암묵지(Tacit Knowledge)는 관찰 기법이 효과적.
