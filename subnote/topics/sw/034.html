7. 제품계열 방법론 (Product Line Engineering, PLE)

  • 정의: 공통 아키텍처와 재사용 가능한 자산(Core Asset)을 기반으로 유사한 제품 군(Product Family)을 체계적으로 개발하는 소프트웨어 공학 방법론
  • 키워드: Core Asset, 가변성(Variability), 공통성(Commonality), Feature Model, 도메인 공학, 응용 공학, Mass Customization, SPL
  • 분류: 개발 방법론
  • 핵심 구성요소:
    구성요소설명핵심
    Core Asset Base공통 컴포넌트·아키텍처·문서·테스트 자산 저장소재사용 자산 허브
    도메인 공학 재사용 가능한 코어 자산 개발·분석·관리자산 생산 활동
    응용 공학 코어 자산 조합·커스터마이징으로 개인 제품 개발자산 소비 활동
    가변성 관리 Feature Model로 제품 간 차이점·공통점 체계 관리 (공통피처, 선택피처, 대안피처, 변동점)제품 차이 관리
    프로세스 정의도메인·응용 공학 간 연계 프로세스·역할 정의개발 체계화
  • 적용 절차:
    단계설명핵심
    1. 스코핑제품군 범위·공통성·가변성 분석제품군 경계 정의
    2. 도메인 분석Feature Model 작성·공통·가변 요구 식별요구 구조화
    3. 코어 자산 개발공통 아키텍처·컴포넌트·테스트 자산 생성재사용 자산 구축
    4. 제품 개발코어 자산 조합·가변점 구현으로 제품 파생제품 신속 생성
    5. 자산 진화신제품 요구 기반 코어 자산 지속 갱신자산 고도화
  • 인사이트: 제품계열 방법론은 “소프트웨어 대량 맞춤 생산(Mass Customization)” 개념으로, 초기 코어 자산 투자 비용이 크나 제품 수가 많을수록 ROI가 급격히 상승.
