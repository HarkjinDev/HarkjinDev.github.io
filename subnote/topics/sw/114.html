[하위] 2-1. CI/CD (Continuous Integration / Continuous Deployment)

  • 정의: CI는 개발자가 코드를 자주 통합하고 자동 빌드·테스트로 검증하는 관행이며, CD는 검증된 코드를 자동으로 스테이징 또는 프로덕션 환경에 배포하는 관행
  • 키워드: 지속적 통합, 지속적 전달(Delivery), 지속적 배포(Deployment), 파이프라인, Jenkins, GitHub Actions, ArgoCD
  • 분류: 형상관리/배포
  • CD 용어 구분:
    • Continuous Delivery: 프로덕션 배포 직전까지 자동화, 최종 배포는 수동 승인
    • Continuous Deployment: 프로덕션까지 완전 자동 배포
  • CI/CD 기술요소:
    구분기술요소설명관련 도구
    CI버전관리 저장소소스 코드의 중앙 관리 및 팀원 간 동기화Git, Subversion(SVN)
    통합 서버빌드, 테스트, 배포 전 과정을 자동화 제어Jenkins, GitHub Actions
    빌드 스크립트컴파일 및 패키징 절차를 자동 실행하는 파일Maven, Gradle, Shell Script
    자동화 테스트코드 통합 시 결함 여부를 스스로 검증JUnit, SonarQube
    CD인프라 자동화코드를 통한 서버 및 네트워크 자원 자동 생성Terraform, Ansible
    배포 자동화빌드된 결과물을 대상 환경에 자동 반영ArgoCD, Spinnaker
    설정 관리환경별 변수 및 보안 정보를 코드와 분리 관리K8s ConfigMap, Vault
    배포 전략 제어무중단, Blue-green, Canary 배포Istio, AWS CodeDeploy
    공통모니터링전체 파이프라인 및 서비스 상태 실시간 추적Prometheus, Grafana
  • CI/CD 파이프라인 단계:
    단계활동
    코드 커밋개발자가 VCS에 코드 푸시
    빌드소스코드 컴파일·패키징
    단위 테스트자동화 단위·통합 테스트 실행
    코드 분석정적 분석·보안 스캐닝 (SAST)
    아티팩트 저장빌드 결과물 레지스트리 저장
    스테이징 배포검증 환경 자동 배포
    인수 테스트E2E·성능 테스트 자동화
    프로덕션 배포자동(CD) 또는 수동 승인 후 배포
  • 인사이트: CI/CD는 DevOps 자동화의 핵심 구현체로, “Continuous Delivery”와 “Continuous Deployment”의 차이(수동 승인 유무)는 명확히 구분 필요.
