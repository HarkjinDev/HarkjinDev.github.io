[하위] 1-5. ATAM (Architecture Trade-off Analysis Method)

  • 정의: 품질 속성 시나리오를 기반으로 아키텍처 결정이 여러 품질 속성에 미치는 트레이드오프를 분석하는 아키텍처 평가 방법
  • 키워드: 트레이드오프 포인트, 민감점(Sensitivity Point), Phase0~3, 유틸리티 트리, 위험 결정(Risk), 비위험 결정, 품질 속성 유틸리티 트리
  • 분류: 소프트웨어 아키텍처
  • ATATM 프로세스 단계:
    과정참여자설명
    Phase 0 (협력과 준비)평가팀 리더·프로젝트 주요 결정권자킥오프·평가 가이드라인·역할 공유
    Phase 1 (기본 평가)평가팀·프로젝트 결정권자Top-Down 아키텍처 소개·분석
    Phase 2 (상세 평가)평가팀·결정권자·이해관계자Bottom-Up 분석·결과 검증
    Phase 3 (후속조치)평가팀·평가 의뢰인최종 보고서·Postmortem 피드백
  • ATAM 평가 단계:
    단계세부단계활동
    소개1. ATAM 소개방법·평가팀·목적 설명
    2. 비즈니스 드라이버 제시기능·제약·품질 목표 설명
    3. 아키텍처 제시아키텍처 설명
    조사/분석4. 아키텍처 접근법 식별사용된 아키텍처 패턴·전술 분석
    5. 유틸리티 트리 생성품질 속성 시나리오 우선순위화
    6. 아키텍처 접근법 분석민감점·트레이드오프 식별
    테스트7. 브레인스토밍·우선순위 결정시나리오 확장
    8. 아키텍처 접근법 재분석추가 시나리오 반영
    보고9. 결과 발표위험·비위험·민감점·트레이드오프 보고
  • ATAM 산출물:
    • 위험 결정 (Risk) / 비위험 결정 (Non-risk) / 민감점 (Sensitivity Point) / 트레이드오프 포인트
  • 인사이트: ATAM에서 민감점은 단일 품질 속성에 영향, 트레이드오프 포인트는 복수 품질 속성에 동시 영향을 미치는 아키텍처 결정으로, 이 두 개념의 차이가 핵심 출제 포인트.
