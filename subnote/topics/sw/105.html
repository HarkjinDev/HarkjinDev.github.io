19. 소프트웨어 테스트 자동화(Test Automation)

  • 정의: 테스트 케이스 실행·결과 검증·리포팅을 자동화 도구와 스크립트로 처리하여 반복 테스트의 효율성을 높이는 활동
  • 키워드: 테스트 자동화 도구, Selenium, JUnit, CI/CD 연동, ROI, 자동화 피라미드
  • 분류: 테스트
  • 주요 테스트 자동화 도구:
    계층 (Layer)분류대표 도구 (Tools)
    UI / E2EWeb / MobileSelenium, Cypress, Playwright, Puppeteer, Appium, Espresso, XCUITest
    Integration / APIAPI / ContractPostman, REST Assured, SoapUI, Newman, Pact (CDC)
    Unit (단위)Unit FrameworkJUnit, TestNG, PyTest, Unittest, GoogleTest, Jest, Mocha
    성능부하 / 스트레스JMeter, nGrinder, LoadRunner, k6, Gatling, Locust, WebTune
  • 인사이트: 자동화 피라미드는 단위 테스트 비중을 높이고 E2E 비중을 낮추는 구조를 지향하며, UI 테스트 과의존은 유지보수 비용 급증을 야기하는 안티패턴(아이스크림 콘).
