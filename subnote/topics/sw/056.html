[하위] 4-1. 코드스멜 (Code Smell)

  • 정의: 코드에 잠재적 문제가 있음을 시사하는 표면적 징후로, 리팩토링이 필요함을 나타내는 신호
  • 키워드: 기술 부채, 리팩토링 트리거, 잠재적 위험, 나쁜 코드 패턴, 스파게티 코드, 에일리언 코드
  • 분류: 설계 원리/패턴
  • 주요 코드스멜 유형:
    유형설명
    긴 메서드 (Long Method)지나치게 긴 메서드, 이해·테스트 어려움
    거대한 클래스 (Large Class)너무 많은 필드·메서드, SRP 위반
    중복 코드 (Duplicated Code)동일 코드 반복, DRY 원칙 위반
    긴 매개변수 목록 (Long Parameter List)파라미터 과다, 가독성 저하
    데이터 덩어리 (Data Clumps)항상 함께 등장하는 데이터 그룹
    기능 편애 (Feature Envy)다른 클래스 데이터를 과도하게 참조
    주석 과다 (Comments)이해하기 어려운 코드를 주석으로 설명
  • 인사이트: 코드스멜은 즉각적 오류가 아닌 잠재적 품질 위험 신호로, 주석 과다는 역설적으로 코드 자체가 불명확하다는 스멜이며, 이름이 좋은 코드는 주석이 불필요.
