[하위] 5-2. 테스트 드라이버 (Test Driver)

  • 정의: 상향식 통합 테스트 시 아직 개발되지 않은 상위 모듈을 대체하여 하위 모듈을 호출하고 결과를 검증하는 임시 제어 프로그램
  • 키워드: 상향식 통합, 상위 모듈 대체, 테스트 하네스, JUnit, 테스트 더블
  • 분류: 테스트
  • 드라이버 역할:
    • 테스트 입력 데이터 생성·전달
    • 테스트 대상 모듈 호출
    • 반환 결과 검증 및 리포팅
  • 인사이트: JUnit·TestNG 같은 단위 테스트 프레임워크가 테스트 드라이버의 역할을 자동화 방식으로 수행하며, 상향식 통합의 핵심 도구.
