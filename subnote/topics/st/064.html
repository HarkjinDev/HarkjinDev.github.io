4. 빅데이터 플랫폼 아키텍처

정의: 빅데이터 수집부터 시각화까지의 전체 처리 흐름을 설계하는 시스템 구조
키워드: 수집, 저장, 처리, 분석, 시각화, 람다/카파 아키텍처
분류: 시각화/빅데이터
5계층 구조:
수집 계층: Kafka, Flume, Sqoop
저장 계층: HDFS, HBase, Cassandra
처리 계층: MapReduce, Spark, Flink
분석 계층: Hive, Pig, Spark SQL
시각화 계층: Tableau, Kibana, Superset
인사이트: 빅데이터 플랫폼은 수집→저장→처리→분석→시각화의 5계층 구조로 이해. 람다/카파 아키텍처는 배치/실시간 처리 방식의 차이를 나타낸다.
