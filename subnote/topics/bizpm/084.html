[하위] 1-1. 기능 점수(FP, Function Point)

  • 정의: 소프트웨어 개발 규모를 산출하기 위해, 기능을 정량화하고 계수적 측정을 통해 나타낸 수치 또는 기법
  • 키워드: 데이터 기능, 트랜잭션 기능, 보정계수, 간이법(평균복잡도), 정통법 데이터기능(DET/RET), 정통법 트랜잭션기능(DET/FTR)
  • 기능점수 구성요소: <논연/입출조/규연성운보>
    구분측정 요소설명
    데이터 기능내부 논리파일(ILF)시스템 내부 저장 데이터 파일
    외부 연계파일(EIF)외부 시스템 참조 데이터 파일
    트랜잭션 기능외부 입력(EI)사용자 입력 데이터 처리
    외부 출력(EO)사용자 요청 데이터 출력
    외부 조회(EQ)사용자 요청 데이터 조회
    보정 계수규모시스템 전체 크기와 복잡도
    연계 복잡성외부 시스템과의 연계 복잡도
    성능데이터 처리 속도와 응답 시간
    운영 환경운영되는 하드웨어 및 소프트웨어 환경
    보안데이터 보호와 접근 제어 정책
  • 기능점수 측정절차: <유범데트/미보후>
  • 산정 기법:
    • 정통법: 기능별 복잡도 매트릭(Low, Avg, High)
      • 데이터 기능 측정 : DET(데이터요소 유형), RET(레코드참조 유형)
      • 트랜잭션 기능 측정 : DET(데이터요소 유형), FTR(참조파일 유형)
    • 간이법: 데이터 기능, 트랙잭션 기능의 평균 복잡도 산출
  • 정통법 vs 간이법 비교:
    구분정통법 (Detailed)간이법 (Estimated)
    개념기능 도출 후 각 유형별 실제 복잡도를 고려하여 정확하게 산출기능 복잡도 판단이 어려울 때 평균 복잡도를 적용하여 신속하게 산출
    장점규모 측정의 정확도 및 신뢰도가 상대적으로 높음측정 소요 시간 및 측정 지식 습득 기간이 짧음
    단점많은 소요 시간과 높은 수준의 측정 전문 지식이 필요함정밀도가 낮아 정통법 대비 오차가 발생할 수 있음
    적용시점상세 설계 완료 이후 (설계공정 이후~폐기)사업 초기 단계 (기획, 발주 단계부터 전 과정)
  • 인사이트: 기능점수 산정은 프로젝트의 생애주기에 따라 전략적으로 선택해야 하며, 사업 초기에는 간이법을 통해 신속하게 예산을 확보하고 프로젝트가 구체화된 설계 이후에는 정통법을 통해 정밀한 규모 검증 및 정산을 진행하는 것이 합리적임.
