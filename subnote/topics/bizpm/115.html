3. DRS (Disaster Recovery System)

  • 정의: 재해 발생 시 정보시스템과 데이터를 신속하게 복구하여 업무 연속성을 지원하는 재해복구 시스템
  • 키워드: 미러 사이트, 핫/웜/콜드 사이트, RTO/RPO, 자동 Failover, HA, FT, IP-SAN, DWDM, GSLB, Snapshot, 지리적 분산, DRaaS, 다중지역 동시 가동방식
  • 분류: 연속성
  • DRS 유형 비교:
    유형복구 시간데이터 손실비용목표시간
    미러 사이트(Mirror)즉시없음매우 높음RTO=0
    핫 사이트(hot)즉시(~수 분)거의 없음매우 높음RTO≤2h
    웜 사이트(Warm)수 시간소량높음RTO≤1W
    콜드 사이트(Cold)수일많음낮음수개월
  • 기술요소:
    구분기술 요소설명
    가용성·이중화고가용성 (HA)H/W 클러스터링·Stand-By 구성
    무중단 내결함성 (FT)장애 즉시 서비스 이중화 시스템
    네트워크·전송IP-SANSAN 트래픽을 IP 패킷으로 전송
    DWDM고밀도 파장 분할 고속 데이터 전송
    GSLB트래픽 분산·장애 시 자동 절체
    데이터 복제·보호데이터 복제 (Replication)주센터→복구센터 동기·비동기 복제
    지속적 데이터 보호 (CDP)변경 사항 실시간 기록·즉시 복구
    스냅샷 (Snapshot)특정 시점 가상 이미지 저장·복원
    백업·복구백업 (Backup)별도 매체에 데이터 저장·보관
    인프라가상화 (Virtualization)복구센터 효율성·HW 독립성 확보
  • DRS 구축 시 핵심 고려사항: RTO·RPO 충족 여부, 지리적 분산, 자동 전환(Failover), 정기 훈련
  • DR센터 유형: 상호계약형, 공동이용형, 외부위탁형, 독자구축형 <상공외독>
  • DRaaS 유형: 관리형, DIY형, 지원형 <관디지>
  • 다중지역 동시 가동방식 (Multi-Region Active-Active) 재해복구시스템
    • 실제로 서비스되는 시스템이 여러 지역에 동시에 가동되어 한 지역의 시스템에 장애가 있더라도 실시간으로 서비스 연속성이 보장되는(Active-Active) 방식으로 각 업무 조직별 비상 사태에 대비한 복구계획 수립을 통한 업무 연속성을 유지할 수 있는 체계
  • 인사이트: DRS 구축 수준은 RTO/RPO 요구사항과 비용의 트레이드오프로 결정되며, 금융·공공 핵심 시스템은 미러 사이트 + 자동 Failover를 요구함. 단순 구축보다 정기적인 DR 훈련(DRT)으로 실효성 검증이 필수임.
