12. AI 보안 위협 대응 메뉴얼(KISA, 26.7.)

  • AI 안전 (AI Safety): AI의 판단으로 시스템이 동작하거나 기능을 수행할 때 사람과 환경에 위험을 줄 가능성이 완화 또는 제거된 상태
  • AI 보안 (AI Security): AI 시스템에 대한 무단 접근 및 활용을 방지하고 전통적인 정보보호의 요소인 기밀성, 무결성, 가용성을 유지하는 능력
  • AI 안전과 AI 보안 비교
    구분AI 안전 (AI Safety)AI 보안 (AI Security)
    발생요인AI 시스템 자체, 인간의 활용 방식에 의해 발생악의적 행위자에 의해 발생
    보호대상AI 시스템의 외부 (사용자, 사회, 환경)AI 시스템 내부 (모델, 데이터, 인프라)
    의도성비의도적 (Unintentional)대부분 의도적 (Intentional)
    시점개발 전·후 발생 (Pre, Post)시스템 운영 후 발생 (Post)
  • 내재적 위협(Intrinsic Threat): 외부 공격자가 부재하더라도 LLM 자체의 학습·정렬·추론 과정에서 비롯되는 결함이 안전성·신뢰성을 훼손하는 위협
  • 내재적 위협 유형: <편환정확>
    유형설명
    학습 데이터 편향 학습 데이터의 분포 편향이 모델 응답에 그대로 반영되거나, 학습 과정에서 특정 문자열이 모델에 저장되어 추론 시점에 재현되는(기억화) 위협
    환각모델이 사실과 무관한 내용을 사실인 것처럼 그럴듯하게 생성하는 위협
    정렬 실패 모델 출력이 운영자의 정책·시스템 프롬프트와 어긋나는 위협
    확률적 비결정성 동일 입력에 대해 모델 출력이 매번 달라지는 현상
  • 외재적 위협(Extrinsic Threat): LLM 시스템의 경계를 구성하는 외부 행위자에 의해 LLM 시스템이 손상되거나 악용되는 위협
  • 외재적 위협 유형: <탈모공고>
    유형설명
    탈옥 모델이 정책상 거부해야 할 요청을 무력화하는 기법.
    모델 추출추론 API에 대한 반복 질의로 원본 모델을 복제하여 유출하는 위협.
    공급망·인프라 대상 • 모델·라이브러리·의존성·학습데이터·벡터DB 등 공급망과 외부 데이터 경로 오염 • MCP 등 외부 도구 연결을 통한 악성코드 주입 • 보안 패치 미흡한 추론엔진·서빙스택 취약점 • 과도한 도구 권한 • 에이전트 하이재킹·메모리 오염 • 시스템 프롬프트 유출 및 권한 분리 실패 등 악용
    고성능 모델 위협프론티어 AI·고영향 AI·범용 AI 등을 포괄하는 "고성능 모델(High Performance Model)" 자체가 외부 공격자의 도구로 전용되는 위협
  • LLM 구성요소: <데개모애사>
    구성 요소설명
    데이터 LLM 애플리케이션 생명주기의 시작

    모델 LLM 애플리케이션으로부터 다양한 입력을 받아 자연어 처리를 수행
    LLM 애플리케이션사용자와 시스템이 실제로 상호작용하는 핵심 지점
    사용자 전체 시스템의 출발점이자 최종 목적지
