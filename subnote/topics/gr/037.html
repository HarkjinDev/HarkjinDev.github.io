1. SDD (Spec-Driven Development, 명세 주도 개발)

  • 정의: 코드보다 명세를 먼저 작성하고 그 명세를 충족하는 코드를 개발하는 방법론
  • 배경:
    • Vibe Coding의 실패모드(의도이탈·API환각·규모확장시 붕괴) 대응
    • LLM이 벤치마크상 9.8~42.1% 비율로 취약한 코드 생성
  • 핵심 원리
    • 문서/정의 우선(Documentation First), 구현의 기준(Source of Truth), 기계 가독성(Machine-Readable), AI와의 협업
  • SDD 핵심 사이클 (6단계, github spec-kit)
    단계설명핵심
    1. Constitution프로젝트 전체 불변원칙 정의원칙 사전고정
    2.SpecifyWhat·Why 관점만 요구사항 작성How는 배제
    3. Clarify애매한 부분 명시적 태그 표시AI추측 차단
    4. Plan헌법·스펙 기반 기술계획 수립게이트로 검증
    5. Tasks실행가능 작은단위로 분할병렬처리 표시
    6. Implement축적맥락 기반 코드 작성맥락기반 생성
  • Spec의 적용 범위:
    산출물Spec으로부터 생성
    코드구현 로직
    UI 컴포넌트화면 구성요소
    DB 스키마데이터 모델
    테스트 케이스·데이터검증 자산
    환경설정·배포 아티팩트인프라 구성
