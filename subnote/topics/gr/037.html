1. SDD (Spec-Driven Development, 명세 주도 개발)

  • 정의: 실행 가능한 명세를 단일 진실공급원으로 삼아 AI가 코드를 생성하는 개발방법론
  • 배경:
    • Vibe Coding의 실패모드(의도이탈·API환각·규모확장시 붕괴) 대응
    • LLM이 벤치마크상 9.8~42.1% 비율로 취약한 코드 생성
  • SDD 핵심 워크플로우:
    단계활동핵심
    1. Spec 작성시스템 동작을 구조화·버전관리된 명세로 상세 기술설계와 구현 분리
    2. Plan 도출Spec 기반 구현계획 수립계획 자동생성
    3. Task 분해계획을 원자적(atomic) 작업단위로 분할세분화된 작업
    4. 코드 생성AI 에이전트가 Task별 코드·테스트 생성AI 실행
    5. 재생성 순환요구사항 변경 시 Spec 수정→해당 코드 재생성Spec이 항상 최신
  • Spec의 적용 범위:
    산출물Spec으로부터 생성
    코드구현 로직
    UI 컴포넌트화면 구성요소
    DB 스키마데이터 모델
    테스트 케이스·데이터검증 자산
    환경설정·배포 아티팩트인프라 구성
