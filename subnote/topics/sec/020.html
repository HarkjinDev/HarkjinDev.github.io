[하위] 9-3. 키 스트레칭 (Key Stretching)

  • 정의: 해시 연산을 반복하여 무차별 대입 공격의 시간 비용을 증가시키는 보안 기법
  • 키워드: 반복 해싱, N번째 다이제스트(Iteration Count), bcrypt, PBKDF2, scrypt, Argon2, 무차별 공격 방어
  • 분류: 암호학
  • 주요 알고리즘:
    • PBKDF2: 솔트 적용 후 반복 횟수 조절 가능, NIST 권고 (최소 310,000회 반복) DIGEST = PBKDF2(Password, Salt, C, DLen) // C: 카운트, DLen: 다이제스트 길이
    • bcrypt: 작업 인수(Work Factor)로 강도 조절, 메모리 사용
    • scrypt: 메모리 집약적, GPU 공격에 강함
    • Argon2: PHC(Password Hashing Competition) 우승, 최신 권고 알고리즘
  • 인사이트: 키 스트레칭의 반복 횟수는 하드웨어 성능 발전에 맞춰 주기적으로 증가시켜야 하며, Argon2가 현재 최신 표준임
