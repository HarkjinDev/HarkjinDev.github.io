1. Secure SDLC (보안 소프트웨어 개발 생명주기)

  • 정의: 소프트웨어 개발 전 단계에 보안 활동을 내재화하는 개발 방법론
  • 키워드: MS-SDL, OWASP, CLASP, 보안 내재화
  • 분류: 시스템/SW 보안
  • 주요 방법론 비교:
    방법론개발사특징
    MS-SDLMicrosoft단계별 보안 활동, 위협 모델링 필수
    CLASPSecure Software역할 중심, 경량 프로세스
    BSIMMSynopsys업계 벤치마크 기반 성숙도 측정
  • 단계별 보안 활동:
    • 요구사항: 보안 요구사항 정의, 위협 모델링
    • 설계: 보안 아키텍처 검토, 공격 표면 분석
    • 구현: 시큐어 코딩, 코드 리뷰
    • 테스트: SAST, DAST, 침투 테스트
    • 배포: 구성 검토, 패치 관리
  • 인사이트: 보안 결함은 개발 후 발견할수록 수정 비용이 기하급수적으로 증가하므로(Shift Left 원칙), 요구사항/설계 단계에서의 위협 모델링이 가장 비용 효율적임
