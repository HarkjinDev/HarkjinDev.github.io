19. 마이크로세그멘테이션(Microsegmentation)

  • 정의: 데이터센터·클라우드 환경에서 워크로드·애플리케이션·사용자 단위로 네트워크를 세분화하여 동서(East-West) 트래픽을 정밀 통제하는 제로트러스트 핵심 기술
  • 키워드: 제로트러스트, East-West 트래픽, 정책 기반 격리, SDN, 소프트웨어 정의 경계(SDP), 측면 이동 차단, 워크로드 격리
  • 분류: 위협/공격
  • 핵심 구성요소:
    구성요소설명핵심
    정책 엔진워크로드별 세그먼트 정책 정의·배포·관리중앙 정책 관리
    소프트웨어 정의 경계 (SDP)SW로 동적 경계 생성·접근 허용동적 경계
    에이전트/센서워크로드별 트래픽 수집·정책 적용엔드포인트 적용
    가시성 엔진East-West 트래픽 전체 흐름 모니터링내부 가시성
    오케스트레이터SDN·클라우드 API 연동 정책 자동 배포자동화 배포
  • NIST SP 800-207 제로트러스트 연계:
    원칙마이크로세그멘테이션 적용핵심
    명시적 검증모든 워크로드 간 통신 ID·속성 기반 검증묵시적 신뢰 제거
    최소 권한업무 필요 통신만 허용·나머지 차단Least Privilege
    침해 가정내부 침해 전제 워크로드 단위 격리Assume Breach
    측면 이동 차단침해 후 내부 수평 이동 경로 차단피해 범위 최소화
  • 인사이트: 마이크로세그멘테이션은 제로트러스트의 핵심 구현 기술로, 워크로드 단위 East-West 트래픽 정밀 통제로 측면 이동을 차단하며, 클라우드·컨테이너·하이브리드 환경에서 소프트웨어 정의 정책 기반으로 동적 확장 가능한 차세대 내부 네트워크 보안 아키텍처의 표준으로 자리잡음
