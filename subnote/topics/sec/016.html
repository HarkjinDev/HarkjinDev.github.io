8. ECC (Elliptic Curve Cryptography, 타원곡선 암호)

  • 정의: 타원곡선 위의 점 연산(이산대수 문제)을 기반으로 하는 공개키 암호 알고리즘
  • 키워드: 타원곡선 이산대수, 짧은 키 길이, ECDH, ECDSA, 모바일·IoT
  • 분류: 암호학
  • 수학적 기반:
    • 타원곡선 방정식: y² = x³ + ax + b (mod p)
    • 타원곡선 위의 점 P에 대해 k번 덧셈(점 배가) → Q = kP 계산은 쉬움
    • Q와 P로부터 k를 역산하는 것은 계산상 불가능 (타원곡선 이산대수 문제)
  • ECC 기반 알고리즘:
    • ECDH: 타원곡선 기반 Diffie-Hellman 키 교환 → TLS 1.3 기본 방식
    • ECDHE: ECDH + 임시 키 사용 → 전방향 안전성(PFS) 제공
    • ECDSA: 타원곡선 기반 전자서명 → DSA의 ECC 버전
    • EdDSA (Ed25519): Edwards 곡선 기반, 빠른 서명·검증, SSH·TLS에 활용
  • 장점: RSA 대비 짧은 키로 동등한 보안강도 → 연산 속도 빠름, 전력 소모 적음
  • 단점: 구현 복잡도 높음, 잘못된 곡선 파라미터 선택 시 취약점 발생 가능
  • 인사이트: ECC는 모바일·IoT·TLS 환경에서 RSA를 대체하는 추세이며, 256bit(P-256) 곡선이 사실상 표준임. 단, 양자컴퓨터의 Shor 알고리즘에 취약하므로 장기적으로는 PQC(양자내성암호)로의 전환이 필요함
