7. Secure OS (보안 운영체제)

  • 정의: 일반 운영체제에 강제적 접근 제어·최소 권한·보안 커널 등 다층 보안 메커니즘을 적용하여 무결성·기밀성·가용성을 시스템 수준에서 보장하는 보안 강화 운영체제
  • 키워드: 보안 커널, MAC, SELinux, 최소 권한, TCB, 참조 모니터, 무결성, 다중 등급 보안(MLS), CC 인증
  • 분류: 시스템/SW 보안
  • 핵심 보안 메커니즘:
    구분항목설명핵심
    접근 제어MAC보안 레이블 기반 시스템 정책으로 접근 강제 통제정책 기반 강제
    DAC자원 소유자가 접근 권한 설정·관리소유자 재량
    RBAC사용자 역할에 따라 권한 부여·관리역할 기반
    보안 커널TCB보안 정책 집행하는 최소한의 신뢰 컴포넌트신뢰 기반
    참조 모니터모든 접근 요청을 중재·검증하는 핵심 보안 모듈접근 중재
    보안 커널 최소화공격 표면 최소화를 위한 커널 기능 최소 구성최소화 원칙
    무결성 보호커널 무결성 검증부팅 시 커널·드라이버 무결성 해시 검증부팅 보안
    메모리 보호ASLR·DEP·스택 카나리로 메모리 공격 방어메모리 방어
  • 참조 모니터 (Reference Monitor): 모든 주체·객체 간 접근 요청을 중재·검증하는 보안 커널 핵심 모듈
    • 필수 속성:
      항목설명핵심
      완전성 (Complete Mediation)모든 접근 요청을 예외 없이 반드시 거쳐야 함우회 불가
      격리성 (Isolation)참조 모니터 자체가 외부 변조·우회로부터 보호자기 보호
      검증 가능성 (Verifiability)소규모·단순 구조로 형식적 검증 가능해야 함수학적 검증
    • 동작방식: 주체 보안 레이블 ↔ 객체 보안 레이블 비교 후 정책 판단·허용·거부
    • 구현체: SELinux의 LSM(Linux Security Module)이 대표적 구현
  • 인사이트: Secure OS는 보안을 애플리케이션 수준이 아닌 OS 커널 수준에서 강제하는 근본적 접근으로, SELinux 기반 MAC이 현대 Linux·컨테이너 환경의 사실상 표준 보안 메커니즘으로 자리잡고 있음. Zero Trust 아키텍처와 결합하여 최소 권한·강제 접근 제어를 OS 수준에서 구현하는 방향으로 발전하고 있으며, CC EAL 인증이 국방·정부 시스템 도입의 핵심 요건으로 강화되는 추세임
