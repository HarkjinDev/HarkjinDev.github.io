4. 스푸핑 (Spoofing)

정의: 공격자가 자신의 신원(IP, MAC, DNS, 이메일, 웹 등)을 위장하여 피해 시스템을 속이거나 신뢰 관계를 악용하는 사이버 공격 기법
키워드: ARP 스푸핑, IP 스푸핑, DNS 스푸핑, 신원 위장, Cache Poisoning, MITM
분류: 네트워크 보안
스푸핑 공격 유형:
유형
개념
핵심 메커니즘
대응 방안
ARP 스푸핑
MAC 주소 위조
ARP Cache Poisoning / MAC 주소 변조
동적 ARP 검사(DAI), Static ARP, ARP Watch
IP 스푸핑
출발지 IP 위조
트러스트 관계 악용 / IP 헤더 위조
인그레스 필터링(RFC 2827), 역방향 경로 검증
DNS 스푸핑
DNS 응답 위조
DNS Cache Poisoning / 응답 ID 탈취
DNSSEC 도입, TTL 최소화, 신뢰 DNS 사용
E-mail 스푸핑
발신자 주소 위조
SMTP 취약점 / 사회공학적 기법
SPF, DKIM, DMARC 발신자 인증 강화
Web 스푸핑
가짜 사이트 위장
URL 변조·피싱 / SSL 인증서 위장
HTTPS 인증서 검증, EV 인증서, HSTS
MAC 스푸핑
MAC 주소 변조
MAC 인증 우회 / 보안 필터링 무력화
802.1X 포트 인증, MAC 필터링, 포트 보안
인사이트: 스푸핑 공격은 단독으로 사용되기보다 MITM(중간자 공격), 세션 하이재킹, 피싱 등 후속 공격의 전제 조건으로 활용되는 경우가 많아, 네트워크 계층별 인증·검증 체계의 중첩 적용이 핵심 방어 전략임
