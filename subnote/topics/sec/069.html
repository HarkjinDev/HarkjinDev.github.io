7. SSRF (Server-Side Request Forgery)

  • 정의: 공격자가 서버로 하여금 내부 네트워크·클라우드 메타데이터 등 외부에서 직접 접근 불가한 리소스에 요청을 보내도록 조작하는 웹 공격
  • 키워드: 내부망 접근, 클라우드 메타데이터, URL 파라미터 조작, SSRF 필터 우회, 방화벽 우회, 169.254.169.254
  • 분류: 시스템 / SW 보안 / 웹 보안
  • 공격 동작 원리:
    단계설명핵심
    1. URL 파라미터 조작공격자가 서버가 요청할 URL을 내부 주소로 변조내부 주소 삽입
    2. 서버 요청 발생서버가 변조된 URL로 내부 리소스에 요청 전송서버가 대리 요청
    3. 내부 응답 획득방화벽 우회하여 내부망·메타데이터 응답 수신내부 정보 노출
    4. 정보 탈취·횡적 이동내부 서비스·자격증명·클라우드 키 탈취권한 상승·확산
  • 대응방안:
    구분구성요소핵심 기술
    입력 검증URL 화이트리스트허용된 외부 도메인만 요청 가능하도록 제한
    내부 IP 차단127.0.0.1·10.x·169.254.x 등 내부 주소 요청 차단
    네트워크IMDSv2 적용AWS 메타데이터 서비스 토큰 기반 인증으로 SSRF 방어
    네트워크 분리서버의 내부망 직접 접근 차단·아웃바운드 필터링
    우회 방지DNS 리바인딩 방지DNS 재확인으로 내부 주소 우회 차단
    스킴 제한http·https 외 file·dict·gopher 스킴 차단
  • 인사이트: SSRF는 클라우드 환경에서 메타데이터 서버를 통한 IAM 자격증명 탈취로 이어져 전체 클라우드 인프라 침해로 확대될 수 있으므로, URL 화이트리스트·내부 IP 차단·IMDSv2 적용의 다층 방어가 클라우드 환경 필수 보안 요건임
