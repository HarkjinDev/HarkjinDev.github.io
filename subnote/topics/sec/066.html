4. 인젝션 (Injection)

  • 정의: 악의적 코드를 삽입하여 시스템을 비정상 동작시키는 보안 공격 기법
  • 키워드: SQL/OS/LDAP 인젝션, 입력값 검증 부재
  • 분류: 위협/공격
  • 주요 유형:
    • SQL 인젝션: DB 쿼리 조작으로 데이터 탈취·변조
    • OS 인젝션: 시스템 명령어 실행
    • LDAP 인젝션: 디렉토리 서비스 조작
    • XML 인젝션 / XPath 인젝션: XML 파서 조작
  • 대응 방안: 입력값 검증, 매개변수화 쿼리(Prepared Statement), ORM 사용, 최소 권한 DB 계정
  • SQL 인젝션 유형:
    유형핵심대응 방안
    일반 SQL 인젝션입력값에 SQL 구문 삽입·직접 실행PreparedStatement, 입력값 검증·이스케이프
    Blind SQL 인젝션응답 결과 없이 참·거짓으로 데이터 추출에러 메시지 숨김, 응답 시간 일관화
    시간 기반 블라인드SLEEP() 등으로 응답 지연을 통해 데이터 추출쿼리 실행 시간 제한, WAF 적용
    에러 기반 인젝션의도적 에러 유발로 DB 구조·데이터 노출에러 메시지 외부 노출 차단, 커스텀 에러 페이지
    UNION 기반 인젝션UNION 구문으로 다른 테이블 데이터 병합 추출컬럼 수·타입 검증, 최소 권한 계정 사용
    Out-of-Band 인젝션DNS·HTTP 요청으로 데이터를 외부 서버로 유출외부 네트워크 요청 차단, DB 서버 아웃바운드 제한
  • 인사이트: SQL 인젝션은 가장 오래된 취약점이지만 여전히 OWASP Top 3에 포함되어 있으며, Prepared Statement 사용만으로도 대부분의 SQL 인젝션을 방어할 수 있음
