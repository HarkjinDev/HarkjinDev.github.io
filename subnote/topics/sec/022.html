[하위] 10-1. HMAC (Hash-based Message Authentication Code)

  • 정의: 해시 함수와 공유 비밀키를 결합하여 메시지 인증 코드를 생성하는 기술
  • 키워드: 공유 비밀키, 해시 함수, 무결성, 인증
  • 분류: 암호학
  • 동작 원리:
    • ① 키 패딩: 비밀키 K를 해시 함수의 블록 길이에 맞게 패딩 처리
    • ② ipad XOR: 패딩된 키와 ipad(0x36 반복)를 XOR 수행 → 입력 패딩 키(I padkey) 생성
    • ③ 메시지 결합: I padkey에 메시지 m을 결합
    • ④ 내부 해시: H(I padkey || m) 계산 → 일방향 해시 함수 적용
    • ⑤ opad XOR: 패딩된 키와 opad(0x5C 반복)를 XOR 수행 → 출력 패딩 키(O padkey) 생성
    • ⑥ 해시값 결합: O padkey에 ④의 내부 해시값을 결합
    • ⑦ 최종 해시: H(O padkey || 내부해시) 계산 → 최종 HMAC 값 출력
  • 수식: HMAC(K, m) = H((K ⊕ opad) || H((K ⊕ ipad) || m))
  • 제공 속성: 무결성(Integrity) + 인증(Authentication) — 기밀성은 제공하지 않음
  • 활용: JWT 서명, API 인증, TLS MAC
  • 인사이트: HMAC은 단순 해시와 달리 키가 필요하므로, 키를 모르는 공격자가 메시지를 위조해도 HMAC 검증에서 실패함. 단, 기밀성은 제공하지 않으므로 내용 보호가 필요하면 암호화를 병용해야 함
