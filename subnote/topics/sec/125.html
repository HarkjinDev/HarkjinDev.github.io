9. ML 편향 문제 (Machine Learning Bias)

  • 정의: 머신러닝 모델이 학습 데이터·알고리즘·설계 과정의 편향으로 인해 특정 집단·속성에 대해 불공정하거나 차별적인 예측·결정을 내리는 문제
  • 키워드: 데이터 편향, 알고리즘 공정성, 대표성 부족, 역사적 편향, 확증 편향, XAI, 공정성 지표, AI 윤리
  • 분류: AI 보안
  • 편향 발생 원인:
    구분항목설명핵심
    데이터샘플링 편향특정 집단이 과소·과대 대표된 학습 데이터 수집대표성 부족
    역사적 편향과거 차별·불평등이 반영된 데이터 그대로 학습과거 차별 재현
    레이블 편향어노테이터 주관·편견이 레이블링에 반영주관적 레이블
    측정 편향집단별 데이터 수집·측정 방식의 불일치측정 불일치
    알고리즘집계 편향전체 집단 최적화 과정에서 소수 집단 성능 저하소수 집단 소외
    평가 편향편향된 기준·지표로 모델 성능 평가잘못된 평가
    피드백 루프편향된 예측이 데이터로 재입력되어 편향 증폭편향 자기 강화
    설계·운영확증 편향기존 가설을 확인하는 데이터만 선택 활용선택적 데이터
    자동화 편향AI 결과를 무비판적으로 수용하는 인간 의존맹목적 신뢰
  • 대응 방안:
    구분항목설명핵심
    데이터데이터 다양성 확보소수 집단 포함 균형 있는 학습 데이터 수집대표성 확보
    재샘플링·증강과소 표현 집단 데이터 증강·오버샘플링데이터 균형화
    데이터 감사학습 데이터 편향 여부 정기 검토·검증데이터 품질 관리
    모델편향 완화 알고리즘학습 단계에서 공정성 제약 조건 추가알고리즘 공정화
    후처리 보정모델 출력 후 집단 간 결과 균등화 보정출력 보정
    XAI 적용모델 편향 원인·특징 중요도 투명하게 설명편향 원인 탐지
    거버넌스편향 감사 체계정기적 모델 편향·공정성 독립 감사 수행주기적 검증
    다양성 팀 구성다양한 배경·관점의 AI 개발팀 구성설계 단계 예방
    규제 준수EU AI Act·공정성 가이드라인 준수법적 책임 이행
  • 인사이트: ML 편향은 기술적 문제를 넘어 사회적 차별을 AI가 자동화·증폭시키는 구조적 위험으로, 데이터·알고리즘·거버넌스 3축의 통합적 대응이 필수임. 공정성 지표 간 수학적 상충 관계(Trade-off)가 존재하여 어떤 공정성 기준을 우선할지는 기술이 아닌 사회적 가치 판단의 문제이며, AI 개발 초기부터 다양성 확보와 편향 감사를 설계에 내재화하는 것이 핵심 예방 전략임
