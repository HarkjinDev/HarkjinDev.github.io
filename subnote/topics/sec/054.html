2. 시큐어 코딩 (Secure Coding)

  • 정의: 보안 취약점을 예방하는 안전한 소프트웨어 코딩 기법
  • 키워드: 입력값 검증, SQL 인젝션 방지, XSS 방지
  • 분류: 시스템/SW 보안
  • 7대 보안 약점 분류 (행안부):
    • 입력 데이터 검증 및 표현: SQL 인젝션, XSS, 버퍼 오버플로우
    • 보안 기능: 적절한 인증, 암호화 사용
    • 시간 및 상태: TOCTOU, 경쟁 조건
    • 에러 처리: 민감 정보 노출, 부적절한 예외 처리
    • 코드 품질: 사용되지 않는 코드, 복잡도
    • 캡슐화: 민감 데이터 노출
    • API 오용: 안전하지 않은 API 사용
  • 인사이트: SAST(정적 분석)와 DAST(동적 분석)를 CI/CD 파이프라인에 통합하여 자동화된 보안 검증 체계(DevSecOps)를 구축하는 것이 현대적 시큐어 코딩의 실천 방법임
