8. SIEM (Security Information & Event Management)

  • 정의: 보안 로그를 통합 수집·분석하여 위협을 실시간 탐지하는 보안 관리 시스템
  • 키워드: 로그 통합, 실시간 상관분석, 이벤트 관리, 정규화, 위협 탐지, 컴플라이언스, UEBA, SOAR, MTTD
  • 분류: 네트워크 보안
  • 구성요소 및 기술요소:
    구분구성요소핵심 기술
    데이터 수집로그 수집방화벽·IDS·OS·애플리케이션 로그 에이전트·Syslog 수집
    정규화이기종 로그를 공통 포맷으로 변환·파싱
    인덱싱대용량 로그 고속 검색을 위한 인덱스 구성
    탐지·분석상관분석 (Correlation)복수 이벤트 연관 분석으로 공격 시나리오 탐지
    실시간 경보임계치·룰 기반 이상 이벤트 즉시 알림
    UEBA사용자·엔티티 행위 분석으로 내부자 위협·이상 탐지
    ML 기반 탐지룰 기반 한계 보완, 미지 위협·이상 패턴 자동 탐지
    운영·관리대시보드·가시성보안 이벤트·KPI·위협 현황 실시간 시각화
    컴플라이언스 보고ISMS·ISO 27001·PCI-DSS 준수 증적 자동 생성
    로그 보존법적·규제 요건에 따른 장기 로그 아카이빙
  • SIEM vs SOAR:
    • SIEM: 탐지·분석·경보 (분석가 개입 필요)
    • SOAR: SIEM + 자동 대응(Playbook 기반 자동화)
  • 인사이트: SIEM의 로그 수집 범위와 규칙(Rule) 품질이 탐지 성능을 결정하며, 현재는 AI/ML 기반 이상 탐지를 결합한 차세대 SIEM이 주류임
