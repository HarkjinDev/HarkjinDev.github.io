2. 접근제어/통제 (Access Control)

  • 정의: 인가된 주체만 자원에 접근하도록 통제하는 보안 메커니즘
  • 키워드: MAC, DAC, RBAC, ABAC
  • 분류: 접근제어
  • 접근통제 원칙: 최소 권한, 권한 분리, 최소 기능
  • 접근제어 3요소: 정책(Policy), 모델(Model), 메커니즘(Mechanism) <정모메>
  • 접근제어 메커니즘:
    구분개념특징
    ACL (접근제어 목록)객체(자원) 기준 권한 목록 관리객체 중심·자원별 권한 확인 용이·사용자 증가 시 관리 복잡
    CL (권한 목록)주체(사용자) 기준 권한 목록 관리주체 중심·사용자 전체 권한 확인 용이·권한 위임 관리 어려움
  • 접근제어 정책 유형:
구분MAC (Mandatory Access Control)DAC (Discretionary Access Control)RBAC (Role-Based Access Control)ABAC (Attribute-Based Access Control)
기본 개념보안 등급에 따라 접근을 강제적으로 제한리소스 소유자가 접근 권한을 부여역할(Role)에 따라 권한을 부여속성(Attribute) 기반으로 접근 통제
관리 주체시스템 관리자리소스 소유자보안 관리자정책에 따라 자동 결정
정책 기준보안 등급(Label) 및 규칙파일/디렉터리 소유자의 결정조직 내 역할(Role)사용자, 자원, 환경 속성 조합
유연성낮음 (엄격한 보안 정책)높음 (사용자 간 자유로운 공유)중간 (역할 변경 시 유연함)높음 (동적 정책 적용 가능)
보안성매우 높음 (사용자 임의 변경 불가)낮음 (사용자가 임의로 권한 변경 가능)중간 (역할 기반 정책 설정 필요)높음 (세부적인 정책 설정 가능)
적용 사례군사, 정부기관 (기밀 정보 보호)일반 OS 파일 시스템 (Windows, Linux 등)기업 내부 시스템 (ERP, 금융권)클라우드 환경, AI 기반 접근통제
단점관리 복잡, 유연성 부족보안 취약점 (권한 전이 가능)역할 증가 시 관리 부담정책 설계 및 실행이 복잡
분류규칙기반(MLP, CBP), 관게기반신원기반, 사용자기반, 혼합방식--
  • 인사이트: 현대 환경에서는 RBAC과 ABAC를 결합한 하이브리드 모델이 증가하며, 제로 트러스트 아키텍처에서는 ABAC가 필수 요소임
