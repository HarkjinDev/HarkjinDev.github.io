[하위] 2-2. ABAC (Attribute-Based Access Control)

  • 정의: 속성 기반으로 주체/객체/환경을 동적으로 분석하여 접근 사용자 및 시스템에 권한을 부여하는 접근통제 정책
  • 등장 배경:
    구분배경핵심
    RBAC 한계역할 폭발(Role Explosion)·세분화 부족역할 수 기하급수 증가
    클라우드 전환동적·분산 환경 정적 역할 기반 통제 한계동적 정책 필요
    제로트러스트상황 인식 기반 정밀 접근 제어 요구속성 기반 신뢰
    데이터 공유조직 간 정보 공유 시 세밀한 통제 필요외부 공유 통제

  • ABAC NIST SP 800-162 구성요소:
    구분구성요소핵심
    핵심 속성주체 속성 (Subject Attributes)사용자 역할·직위·부서·보안등급
    객체 속성 (Object Attributes)자원 민감도·소유자·분류·유형
    행위 속성 (Operation Attributes)읽기·쓰기·삭제·실행 허용 행위
    환경 조건 Environment Conditions)시간·위치·네트워크·위험도 상황
    기능 구성요소PEP (Policy Enforcement Point)접근 요청 인터셉트·결정 집행
    PDP (Policy Decision Point)속성·정책 평가 후 허용·거부 결정
    PIP (Policy Information Point)외부 속성 저장소 속성값 조회·제공
    PAP (Policy Administration Point)접근 정책 생성·관리·배포
