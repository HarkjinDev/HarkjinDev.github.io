19. 부채널 공격 (Side-Channel Attack)

  • 정의: 물리적 부가 정보를 분석하여 비밀키·민감 정보를 추출하는 공격 기법
  • 키워드: 전력 분석, 타이밍 공격, 전자기 분석, 캐시 공격, Spectre·Meltdown, 물리적 취약점, 암호 구현 취약점
  • 분류: 위협/공격
  • 공격 유형:
    구분항목설명핵심
    전력 기반SPA (단순 전력 분석)단일 전력 파형으로 연산 과정·키 추출파형 직접 분석
    DPA (차분 전력 분석)다수 전력 파형의 통계적 차이로 키 추출통계적 분석
    타이밍 기반타이밍 공격연산 수행 시간 차이로 비밀 정보 추론시간 차이 분석
    캐시 타이밍CPU 캐시 히트·미스 시간 차이로 데이터 추론캐시 상태 추론
    전자기 기반EM 분석장치에서 방출되는 전자기파 분석으로 키 추출전자기파 분석
    하드웨어 취약점SpectreCPU 분기 예측 악용, 다른 프로세스 메모리 읽기분기 예측 악용
    MeltdownCPU 투기적 실행 악용, 커널 메모리 접근투기적 실행 악용
    RowhammerDRAM 메모리 셀 반복 접근으로 인접 비트 반전메모리 비트 반전
    기타음향 공격장치 동작 중 발생하는 소리로 연산 정보 추출음향 신호 분석
    광학 공격LED 깜박임·화면 반사광으로 정보 유출광학 신호 분석
  • 대응 방안:
    구분구성요소핵심 기술
    하드웨어 방어전력 노이즈 추가연산과 무관한 랜덤 전력 소비 추가로 파형 은닉
    전자기 차폐패러데이 케이지·차폐 설계로 전자기파 유출 차단
    소프트웨어 방어일정 시간 연산입력값에 관계없이 동일 시간 소요되도록 구현
    마스킹중간 연산값에 랜덤값을 XOR하여 전력 상관관계 제거
    CPU 패치Spectre·Meltdown 패치커널 페이지 테이블 격리(KPTI)·펌웨어 업데이트
    투기적 실행 제한분기 예측·투기적 실행 범위 제한 설정
    설계 보안보안 칩 설계부채널 내성 암호 하드웨어 모듈(HSM·SE) 사용
  • 인사이트: 부채널 공격은 암호 알고리즘 자체가 아닌 구현 환경의 물리적 정보를 악용하므로 수학적으로 완벽한 암호도 구현 취약점으로 무력화될 수 있으며, Spectre·Meltdown 이후 CPU 아키텍처 설계 단계부터 부채널 내성을 고려하는 하드웨어 보안 설계가 필수 요건으로 자리잡고 있음
