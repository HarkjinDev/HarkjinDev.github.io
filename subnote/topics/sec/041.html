3. 네트워크 스니핑 (Network Sniffing)

  • 정의: 네트워크 상에서 흐르는 패킷을 무차별적으로 캡처·분석하여 민감 정보를 탈취하는 공격 기법
  • 키워드: 패킷 캡처, 무차별 모드(Promiscuous Mode), Wireshark, 수동적 공격
  • 분류: 네트워크 보안
  • 동작 원리: NIC를 무차별 모드(Promiscuous Mode)로 설정 → 자신에게 오지 않는 패킷도 모두 수신 → 평문 프로토콜(HTTP, FTP, Telnet 등)에서 ID/PW·세션 쿠키 추출
  • 스니핑 유형:
    구분설명예시
    수동적 스니핑허브 기반 네트워크에서 브로드캐스트 패킷을 그냥 수신공유 허브 환경
    능동적 스니핑스위치 환경에서 ARP 스푸핑 등으로 트래픽을 강제 우회 후 캡처ARP Poisoning + 스니핑
  • 스니핑 공격 기법:
    공격 기법설명동작 원리대응 방안
    미러링 포트포트 트래픽 복사 스니핑SPAN 포트 설정 → 트래픽 복제·캡처포트 미러링 접근 제한, 물리적 보안
    Switch JammingCAM Table 가득 채워 허브화가짜 MAC 플러딩 → CAM Overflow → 브로드캐스트Port Security, MAC 수 제한, VLAN 분리
    ARP RedirectARP 위조로 트래픽 경유 유도가짜 ARP Reply → ARP 테이블 오염 → 트래픽 경유DAI, Static ARP, ARP Watch
    ICMP RedirectICMP 위조로 경로 공격자 변경ICMP Type 5 위조 → 라우팅 테이블 변경 → 트래픽 우회ICMP Redirect 차단, 정적 라우팅
  • 주요 도구: Wireshark, tcpdump, ettercap, dsniff
  • 탐지 방법: ARP 테이블 모니터링, 무차별 모드 감지(ARP Watch), 비정상 트래픽 패턴 탐지
  • 대응 방안: 전송 구간 암호화(TLS/HTTPS/SSH), 스위치 기반 네트워크 구성, VPN 적용, 802.1X 포트 인증
  • 인사이트: 스니핑은 평문 프로토콜이 사용되는 환경에서 치명적이므로 암호화가 근본적 대응책이며, 현대 환경에서 HTTP·FTP·Telnet은 반드시 암호화 버전(HTTPS·SFTP·SSH)으로 대체해야 함
