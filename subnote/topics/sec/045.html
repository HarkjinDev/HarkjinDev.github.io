6. SSL (Secure Sockets Layer)

  • 정의: 네트워크 통신에서 데이터 암호화·인증·무결성을 제공하는 보안 프로토콜로, 현재는 보안 취약점으로 폐기되고 TLS로 대체된 프로토콜
  • 키워드: 공개키 암호화, 대칭키 암호화, 인증서, CA, 핸드셰이크, HTTPS, TLS 전신, POODLE, BEAST
  • 분류: 네트워크 보안
  • SSL 버전 변천:
    항목설명핵심
    SSL 1.0공개되지 않음, 심각한 결함으로 미출시미출시
    SSL 2.01995년 출시, DROWN 공격 취약점으로 폐기폐기
    SSL 3.01996년 출시, POODLE 공격 취약점으로 폐기폐기
    TLS 1.0~1.1SSL 3.0 후속, BEAST·POODLE 취약점으로 폐기폐기
    TLS 1.2~1.3현재 표준, SSL의 완전한 대체현행 표준
  • SSL 핸드셰이크 과정:
    단계설명핵심
    ① ClientHello클라이언트가 지원 암호화 알고리즘·난수 전송협상 시작
    ② ServerHello서버가 알고리즘 선택·인증서·공개키 전송서버 응답
    ③ 인증서 검증클라이언트가 CA 서명으로 서버 인증서 유효성 확인신원 인증
    ④ 세션키 생성공개키로 암호화한 Pre-Master Secret 교환 후 대칭키 생성키 교환
    ⑤ 암호화 통신생성된 대칭키로 이후 데이터 암호화 통신 시작암호화 시작
  • 주요 취약점:
    항목설명핵심
    POODLESSL 3.0의 CBC 모드 패딩 취약점으로 복호화 가능SSL 3.0 폐기 원인
    BEASTTLS 1.0 CBC 암호화 취약점, 세션 하이재킹TLS 1.0 폐기 원인
    DROWNSSL 2.0 서버와 공유 키 사용 시 TLS 복호화 가능레거시 서버 위협
    CRIME압축 기능 악용 세션 쿠키 탈취압축 비활성화
  • 인사이트: SSL은 HTTPS의 기반을 만든 핵심 프로토콜이나 POODLE·BEAST 등 구조적 취약점으로 전면 폐기되었으며, SSL과 TLS는 명칭이 혼용되지만 현재 SSL은 사실상 존재하지 않고 TLS 1.2·1.3만이 유효한 표준으로 레거시 SSL·TLS 1.0·1.1 사용 여부가 보안 진단의 핵심 점검 항목임
