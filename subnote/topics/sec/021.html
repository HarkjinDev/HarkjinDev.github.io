10. MAC (Message Authentication Code, 메시지 인증)

정의: 메시지의 무결성과 송신자 인증을 동시에 보장하기 위해 비밀키를 이용해 생성하는 짧은 인증 코드
키워드: 비밀키, 무결성, 인증, 해시함수, 위변조 방지, CBC-MAC, HMAC, GMAC
분류: 암호학
동작 원리:
① 송신자가 비밀키 K와 메시지 m으로 MAC 값 생성: MAC = h(K, m)
② 수신자가 동일한 비밀키 K로 MAC 재계산 후 수신된 MAC과 비교
③ 일치하면 메시지 무결성 및 출처 인증 확인
제공 속성: 무결성(Integrity) + 인증(Authentication) — 기밀성·부인방지는 제공하지 않음
유형 비교:
유형
설명
예시
CBC-MAC
블록 암호(CBC 모드)로 MAC 생성
금융 메시지 인증 (ISO 9797)
HMAC
해시 함수와 비밀키를 결합하여 MAC 생성
JWT, TLS, API 인증
GMAC
AES-GCM 기반 인증 태그 생성
TLS 1.3, IPSec
인사이트: MAC은 비밀키를 공유한 당사자 간에만 유효하므로 제3자에 대한 부인방지는 불가능함. 부인방지가 필요한 경우 전자서명을 사용해야 함
