11. CVE / CWE / CCE (보안 취약점 표준 식별 체계)

  • 정의: 사이버보안 취약점·약점·설정 오류를 표준화된 식별자로 체계화하여 보안 정보의 공유·자동화·관리를 지원하는 국제 표준 식별 체계
  • 키워드: 취약점 식별, NVD, CVSS, CWE-Top 25, 보안 설정, SCAP, 취약점 데이터베이스, MITRE
  • 분류: 위협/공격
  • 3대 식별 체계 비교:
    항목CVECWECCE
    전체명Common Vulnerabilities and ExposuresCommon Weakness EnumerationCommon Configuration Enumeration
    대상특정 제품의 개별 취약점소프트웨어 취약점 유형·근본 원인시스템·소프트웨어 보안 설정 오류
    식별 단위개별 취약점 인스턴스취약점 유형·패턴보안 설정 항목
    관리 기관MITRE / NVD(NIST)MITREMITRE
    활용 목적취약점 추적·패치 관리시큐어 코딩·설계 개선보안 설정 감사·컴플라이언스
    예시CVE-2021-44228 (Log4Shell)CWE-79 (XSS), CWE-89 (SQL 인젝션)CCE-2715 (패스워드 정책 설정)
  • 활용 분야:
    • 취약점 관리: CVE 기반 취약점 스캔·패치 우선순위 자동화
    • 시큐어 코딩: CWE Top 25 기반 정적 분석·코드 리뷰 기준
    • 컴플라이언스: CCE 기반 보안 설정 감사·ISMS·ISO 27001 준수
    • SBOM 연계: 컴포넌트별 CVE 자동 매핑으로 공급망 위험 관리
  • 인사이트: CVE·CWE·CCE는 각각 취약점 사례·근본 원인·설정 오류를 표준화하는 상호 보완적 체계로, 세 식별 체계를 통합 운용하면 취약점 발생 원인 분석(CWE)→개별 취약점 추적(CVE)→설정 강화(CCE)로 이어지는 완전한 취약점 생명주기 관리가 가능해짐
