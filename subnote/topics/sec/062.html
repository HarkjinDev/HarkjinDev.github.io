1. 버퍼 오버플로우 (Buffer Overflow)

  • 정의: 프로그램이 할당된 메모리 버퍼 경계를 초과하여 데이터를 쓸 때 인접 메모리 영역을 덮어써 시스템을 오작동시키거나 악의적 코드를 실행시키는 취약점
  • 키워드: 스택 오버플로우, 힙 오버플로우, 리턴 주소 변조, 쉘코드, NX bit, ASLR, 스택 카나리, 포맷 스트링
  • 분류: 위협/공격
  • 공격 유형:
    구분항목핵심
    스택 기반스택 오버플로우리턴 주소 변조로 공격자 코드 실행 유도
    리턴 주소 변조함수 복귀 주소를 쉘코드 주소로 덮어쓰기
    힙 기반힙 오버플로우동적 할당 영역 초과 쓰기로 힙 메타데이터 변조
    Use-After-Free해제된 메모리 재사용으로 임의 코드 실행
    기타포맷 스트링printf 포맷 문자열 취약점으로 메모리 읽기·쓰기
  • 주요 기술:
    구분구성요소핵심 기술
    방어 기법ASLR메모리 주소 무작위화로 공격 주소 예측 불가
    스택 카나리리턴 주소 앞에 카나리 값 배치, 변조 시 탐지
    NX bit / DEP스택·힙 영역 코드 실행 불가 설정
    CFI제어 흐름 무결성 검증으로 비정상 분기 차단
    개발 보안안전한 함수strcpy→strncpy, gets→fgets 등 경계 검사 함수 사용
    정적 분석컴파일 단계 취약 코드 패턴 자동 탐지
  • 인사이트: 버퍼 오버플로우는 수십 년간 가장 많이 악용된 취약점으로 ASLR·NX·카나리 등 다층 방어가 표준화되었으나, ROP(Return-Oriented Programming) 등 우회 기법이 지속 진화하여 컴파일러 보안 옵션과 정적·동적 분석의 병행 적용이 필수임
