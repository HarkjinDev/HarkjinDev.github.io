1. LDAP (Lightweight Directory Access Protocol)

  • 정의: 디렉토리 서비스에 접근하여 사용자 정보를 조회하고 인증하는 프로토콜
  • 키워드: 디렉토리 서비스, DN, X.500 경량화, AD/SSO 인증
  • 분류: 인증/신원관리
  • 구조: 트리 형태의 계층 구조 (DIT, Directory Information Tree)
    • DN(Distinguished Name): 디렉토리 항목의 고유 이름 (예: cn=홍길동,ou=IT,o=회사,c=KR)
  • LDAP 인증 흐름: 클라이언트 → 서버 연결(포트 389/636) → 바인딩(Bind) → 검색(Search) → 결과 반환 → 언바인딩
  • 활용: Active Directory(AD) 통합 인증, SSO 기반 인증, 기업 디렉토리 서비스
  • 인사이트: LDAP은 읽기 작업에 최적화되어 있어 사용자 인증 조회에 탁월하나, 쓰기 작업이 많은 환경에서는 성능 한계가 있음
