5. XSS (Cross-Site Scripting)

  • 정의: 공격자가 웹 페이지에 악성 스크립트를 삽입하여 피해자의 브라우저에서 실행시켜 세션 탈취·악성코드 배포 등을 유발하는 웹 공격
  • 키워드: 악성 스크립트, 세션 탈취, DOM, 반사형, 저장형, CSP, 입력값 검증, 이스케이프
  • 분류: 시스템 / SW 보안 / 웹 보안
  • 공격 유형:
    유형설명핵심
    반사형 (Reflected)악성 스크립트를 URL에 삽입, 서버 응답에 반사되어 즉시 실행URL 기반 즉시 실행
    저장형 (Stored)악성 스크립트를 DB에 저장 후 피해자 접근 시 지속 실행지속적 영향
    DOM 기반 (DOM-based)서버 개입 없이 클라이언트 DOM 조작으로 스크립트 실행클라이언트 측 실행
  • 주요 기술:
    구분구성요소핵심 기술
    입력 처리입력값 검증허용 문자 화이트리스트 기반 입력값 검증
    출력 이스케이프HTML 특수문자(&·<·>·"·') 인코딩 처리
    브라우저 방어CSP허용된 스크립트 출처만 실행하는 콘텐츠 보안 정책
    HttpOnly 쿠키JS에서 쿠키 접근 불가 설정으로 세션 탈취 방지
    서버 방어WAF악성 스크립트 패턴 탐지·차단
    X-XSS-Protection브라우저 내장 XSS 필터 활성화 헤더 설정
  • 인사이트: XSS는 OWASP Top 10 지속 등재 취약점으로 입력·출력 양방향 검증이 핵심이며, CSP 도입으로 스크립트 실행 출처를 제한하는 것이 가장 효과적인 심층 방어 수단임
