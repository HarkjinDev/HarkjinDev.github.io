6. CSRF (Cross-Site Request Forgery)

  • 정의: 인증된 사용자의 브라우저를 이용해 사용자 모르게 의도하지 않은 요청을 서버에 전송하여 권한 있는 행위를 강제 실행시키는 웹 공격
  • 키워드: 위조 요청, CSRF 토큰, SameSite, Referer 검증, 세션 인증, 이중 쿠키
  • 분류: 시스템 / SW 보안 / 웹 보안
  • 공격 동작 원리:
    단계설명핵심
    1. 인증피해자가 정상 사이트에 로그인하여 세션 쿠키 보유인증 상태 유지
    2. 유도공격자가 악성 페이지·이메일로 피해자를 위조 요청 페이지로 유도사회공학적 유도
    3. 요청 전송피해자 브라우저가 세션 쿠키를 포함한 위조 요청 자동 전송자동 쿠키 포함
    4. 서버 처리서버가 정상 요청으로 인식하여 송금·정보 변경 등 처리권한 행위 실행
  • 대응방안:
    구분구성요소핵심 기술
    토큰 기반CSRF 토큰요청마다 서버 발급 토큰 검증으로 위조 요청 차단
    이중 쿠키 제출쿠키값과 요청 파라미터 일치 여부 검증
    쿠키 보안SameSite 쿠키타 사이트 요청에 쿠키 전송 제한 (Strict·Lax)
    요청 검증Referer·Origin 검증요청 출처 헤더 검증으로 타 사이트 요청 차단
    재인증 요구중요 작업(송금·비번 변경) 시 추가 인증 요구
  • XSS vs CSRF 비교:
    항목XSSCSRF
    공격 대상피해자 브라우저피해자 인증 세션
    실행 위치클라이언트 브라우저서버
    필요 조건스크립트 삽입 가능피해자 인증 상태
    주요 피해세션 탈취·악성코드권한 행위 강제 실행
  • 인사이트: CSRF는 서버가 요청의 출처를 검증하지 않는 구조적 허점을 악용하므로 CSRF 토큰·SameSite 쿠키·Referer 검증의 다층 적용이 표준 방어 전략이며, SameSite=Strict 설정만으로도 대부분의 CSRF 공격을 차단할 수 있음
