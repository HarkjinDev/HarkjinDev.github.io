4. 공급망 보안 (Supply Chain Security)

  • 정의: 소프트웨어 공급망 전반(오픈소스, 서드파티, 빌드 파이프라인)의 보안 위협을 관리하는 활동
  • 키워드: SW 공급망, 무결성, 서드파티 리스크
  • 분류: 시스템/SW 보안
  • 주요 위협: 타이포스쿼팅 패키지, 오픈소스 악성 코드 삽입, 빌드 시스템 침해, 의존성 컨퓨전
  • 대응 방안:
    • SBOM: 소프트웨어 구성 요소 목록 관리
    • 의존성 스캔: SCA(Software Composition Analysis)
    • 코드 서명: 빌드 산출물 서명 검증
    • 제로 트러스트 기반 아키텍처: 공급망 전 단계 검증
  • 대표 사례: SolarWinds 공격 (2020), Log4Shell 취약점
  • 인사이트: SolarWinds 공급망 공격 이후 미국 행정명령(EO 14028)으로 연방 소프트웨어 SBOM 제공이 의무화되었으며, 한국도 SW 공급망 보안 가이드라인이 강화되고 있음
