10. APT (Advanced Persistent Threat)

  • 정의: 특정 대상을 장기간 지속적으로 공격하는 지능형 위협
  • 키워드: 정찰, 침투, 내부 이동, 데이터 유출, 사이버 킬체인, IoC, UEBA, 제로데이, 스피어피싱, C2
  • 분류: 위협/공격
  • 주요 특징: 지능성 (Advanced), 지속성 (Persistent), 목표성 (Threat), 조직적 배후
  • APT 공격 절차: <침탐수유>
    구분절차핵심
    침입1. 정찰 — 대상 조직·인물·인프라 정보 수집 (OSINT·소셜미디어)목표 정보 수집
    2. 초기 침입 — 스피어피싱·제로데이·공급망으로 최초 거점 확보초기 접근
    3. 지속성 확보 — 백도어·RAT·루트킷 설치로 재진입 경로 유지장기 거점 유지
    탐지 회피4. 권한 상승 — 관리자 권한 획득으로 내부 접근 범위 확대권한 확대
    5. 은닉·잠복 — 정상 프로세스·트래픽으로 위장하여 탐지 회피장기 잠복
    수집6. 내부 이동 — 자격증명 탈취·Pass-the-Hash로 내부 시스템 횡적 이동내부 확산
    7. 정보 수집 — 목표 데이터·자격증명·기밀 정보 탐색·수집목표 자산 수집
    유출데이터 유출 — 수집 데이터를 암호화·분할하여 C2 서버로 외부 유출목표 달성
  • 대응 방안:
    구분구성요소핵심 기술
    탐지UEBA행위 기반 분석으로 정상 위장 APT 활동 탐지
    위협 헌팅IoC·TTPs 기반 잠복 위협 능동 탐색
    NDR내부 네트워크 트래픽 이상 탐지·C2 통신 식별
    예방이메일 보안스피어피싱 차단·첨부파일 샌드박스 분석
    Zero Trust내부 이동 차단·지속적 인증·최소 권한 적용
    대응SIEM·SOAR다단계 공격 상관분석·자동화 대응
    침해 대응 (IR)침해 지표 기반 신속 격리·포렌식·복구
  • 인사이트: APT는 수개월간 정상 행위로 위장하며 잠복하므로 시그니처·IoC 기반 탐지만으로는 한계가 있으며, UEBA 기반 행위 분석·위협 헌팅·Zero Trust를 통합한 다층 방어와 함께 MITRE ATT&CK 프레임워크 기반 TTPs 분석이 APT 대응의 핵심 전략임
