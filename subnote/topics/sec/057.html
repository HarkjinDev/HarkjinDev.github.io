4. 공급망 보안 (Supply Chain Security)

  • 정의: 소프트웨어 공급망 전반(오픈소스·서드파티·빌드 파이프라인)에서 발생할 수 있는 보안 위협을 식별·관리·대응하는 보안 활동
  • 키워드: SW 공급망, SBOM, SCA, 무결성 검증, 서드파티 리스크, 코드 서명, 의존성 컨퓨전, 타이포스쿼팅, SLSA
  • 분류: 시스템/SW 보안
  • 주요 위협:
    구분항목핵심
    패키지 위협타이포스쿼팅유사한 이름의 악성 패키지를 공식 저장소에 등록하여 설치 유도
    의존성 컨퓨전내부 패키지명과 동일한 공개 패키지를 배포하여 내부 시스템 침투
    오픈소스 악성코드 삽입오픈소스 기여자로 위장하여 악성 코드를 정상 패키지에 삽입
    빌드·배포 위협빌드 시스템 침해CI/CD 파이프라인 침해로 빌드 산출물에 악성코드 삽입
    업데이트 메커니즘 악용정상 업데이트 채널을 통해 악성 코드를 배포
    서드파티 위협서드파티 라이브러리 취약점외부 라이브러리의 알려진 취약점이 내부 시스템에 전파
    협력업체 침해신뢰된 협력업체 시스템을 거점으로 내부망 침투
  • 대응 방안:
    구분구성요소핵심 기술
    가시성 확보SBOM소프트웨어 구성 요소 목록 관리·취약점 연계 모니터링
    SCA (의존성 스캔)오픈소스·서드파티 라이브러리 취약점 자동 탐지
    무결성 검증코드 서명빌드 산출물·패키지 디지털 서명으로 위변조 탐지
    SLSA 프레임워크빌드 무결성 수준을 단계별로 검증하는 공급망 보안 표준
    파이프라인 보안CI/CD 보안빌드 환경 격리·접근 제어·파이프라인 무결성 검증
    Zero Trust 적용공급망 전 단계 지속 검증·최소 권한 원칙 적용
    취약점 관리CVE·NVD 연계알려진 취약점 DB와 연계하여 사용 컴포넌트 위험 모니터링
  • 대표 사례: SolarWinds 공격 (2020), Log4Shell 취약점(2021)
  • 인사이트: SolarWinds·Log4Shell로 이어지는 공급망 공격은 신뢰된 채널을 통한 침투라는 점에서 기존 경계 보안으로 탐지가 불가능하며, SBOM 의무화·SLSA 프레임워크·SCA 자동화를 통한 소프트웨어 구성 요소의 전주기 가시성 확보가 현대 공급망 보안의 핵심 전략으로 자리잡고 있음
