6. Diffie-Hellman/디피헬만 (DH)

정의: 두 당사자가 안전하지 않은 채널에서 공유 비밀키를 안전하게 교환할 수 있는 키 교환 프로토콜
키워드: 이산대수, 키 교환 전용, 전방향 안전성(PFS), MITM 취약, ECDH
분류: 암호학
동작 원리:
① 공개 파라미터 합의: 소수 p, 원시근 g 공개
② Alice: 개인키 x 선택 → 공개값 A = gxg^xgx mod p 전송
③ Bob: 개인키 y 선택 → 공개값 B = gyg^ygy mod p 전송
④ 공유 비밀키 계산: Alice → K = BxB^xBx mod p, Bob → K = AyA^yAy mod p
⑤ 두 값이 동일한 K로 일치 → 세션키로 활용( g^xy mod p )
보안 근거: 이산대수 문제 — gxg^xgx mod p에서 x를 역산하기 계산상 불가능
DH vs ECDH 비교:
구분
DH (Diffie-Hellman)
ECDH (Elliptic Curve DH)
수학적 기반
유한체 이산대수
타원곡선 이산대수
키 길이
2048bit 이상 권고
256bit으로 동등 보안강도
성능
상대적으로 느림
빠름 (모바일·IoT 적합)
활용
TLS 키 교환 (레거시)
TLS 1.3 표준 (ECDHE)
전방향 안전성(PFS, Perfect Forward Secrecy): 세션마다 임시 키 쌍 생성(DHE/ECDHE) → 장기 개인키 유출 시에도 과거 세션 복호화 불가
취약점: 중간자 공격(MITM) 취약 → 인증서 기반 인증(TLS)과 결합 필요. 소수 p가 충분히 크지 않으면 이산대수 공격에 취약 (Logjam 공격)
인사이트: DH는 암호화·서명이 불가능한 순수 키 교환 프로토콜로, 단독 사용 시 MITM에 무방비하므로 반드시 인증 메커니즘과 결합해야 하며, TLS 1.3에서는 ECDHE가 필수 적용됨
