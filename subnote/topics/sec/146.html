11. 개인정보 전송요구권 (Right to Data Portability)

  • 정의: 정보주체가 개인정보처리자에게 자신의 개인정보를 본인 또는 다른 개인정보처리자에게 전송해 줄 것을 요구할 수 있는 권리
  • 키워드: 데이터 이동권, 마이데이터, 개인정보보호법 제35조의2, GDPR Article 20, 개인정보관리 전문기관, 전송 요구 대상 기관
  • 분류: 개인정보보호
  • 법적 근거:
    • 개인정보보호법 제35조의2 (2023년 개정·2024년 시행)
    • 신용정보법 제33조의2 (금융 마이데이터)
    • EU: GDPR Article 20 (데이터 이동권)
  • 전송요구 대상 기관 및 분야:
    분야대상 기관전송 대상 정보핵심
    금융은행·카드·보험·증권사거래내역·계좌·보험 정보금융 마이데이터
    의료병원·의원·약국진료기록·처방전·건강검진의료 마이데이터
    공공중앙부처·지자체·공공기관행정·세금·복지 정보공공 마이데이터
    전자상거래쇼핑몰·플랫폼·배달앱구매이력·리뷰·배송 정보상거래 이력
    통신이동통신사통화·데이터 이용 내역통신 이력
  • 전송요구 절차:
    단계설명핵심
    1. 전송 요구정보주체가 개인정보처리자에게 전송 요구서면·전자적 방법
    2. 본인 확인전송 요구자 신원 확인·인증본인 인증
    3. 요구 검토전송 가능 여부·예외 사유 검토15일 이내 처리
    4. 전송 수행기계가독형 표준 형식으로 전송안전한 전송
    5. 전송 기록전송 일시·내용·수신자 기록 보관감사 추적
  • 전송요구 예외 사유:
    예외 사유설명핵심
    법령 의무법령에 따른 보존 의무 있는 정보법적 보존 우선
    제3자 권리 침해전송 시 타인 권리·이익 침해 우려타인 보호
    기술적 불가기술적으로 전송 불가능한 경우기술 한계
    가명·익명 정보가명처리·익명처리된 정보재식별 불가
  • 전송요구권 활용 사례:
    분야활용 사례핵심
    금융여러 은행 계좌·카드 이용 내역 통합 자산 관리오픈뱅킹·마이데이터
    의료병원 간 진료기록 공유·원격진료 활용의료 연속성
    공공정부 보유 행정 데이터 민간 서비스 활용공공 데이터 활용
    전자상거래플랫폼 전환 시 구매이력·리뷰 이전플랫폼 Lock-in 해소
  • 보안 고려사항:
    항목설명핵심
    전송 구간 암호화TLS 1.3 이상 암호화 전송전송 중 보호
    본인 인증 강화MFA·공동인증서 기반 본인 확인무단 전송 방지
    API 보안OAuth 2.0·API 키 기반 안전한 전송 인터페이스API 접근 통제
    수신자 검증전송 대상 기관 인증·허가 여부 확인무단 수신 방지
    전송 이력 관리전송 일시·수신자·범위 감사 로그 보관이상 감지·추적
  • 인사이트: 전송요구권은 금융 마이데이터를 시작으로 의료·공공·전자상거래로 확대되며 개인이 자신의 데이터를 주도적으로 활용·이전할 수 있는 데이터 주권의 핵심 권리로, API 표준화·전문기관 지정·보안 체계 구축을 통해 데이터 경제의 핵심 인프라로 발전 중임
