4. Model DoS (모델 서비스 거부 공격)

  • 정의: AI 모델에 과도한 연산·토큰·API 요청을 유발하여 서비스를 마비시키거나 운영 비용을 폭증시키는 가용성 공격
  • 키워드: 토큰 폭탄, API 플러딩, 자원 소진, Rate Limiting, 토큰 예산, 이상 탐지
  • 분류: AI 보안
  • 공격 유형:
    구분항목설명핵심
    자원 소진토큰 폭탄극도로 긴 입력·반복 패턴으로 토큰 처리 자원 소진토큰 자원 고갈
    복잡 연산 유도복잡한 수학·코드 생성 요청으로 GPU 연산 자원 소진연산 과부하
    무한 반복 유도재귀적·순환 응답 유도 프롬프트로 출력 토큰 폭증출력 폭증
    API 공격API 플러딩대량 API 요청으로 처리 한계 초과·서비스 응답 불가요청 과부하
    비용 공격대량 토큰 소비 유도로 API 과금 비용 폭증비용 유발
  • 대응 방안:
    구분구성요소핵심 기술
    예방Rate LimitingAPI 호출 횟수·토큰 수 임계치 제한
    토큰 예산 설정입력·출력 토큰 수 상한선 강제 적용
    탐지이상 탐지비정상 요청 패턴·반복 쿼리 자동 탐지·차단
    비용 모니터링API 사용량·비용 급증 실시간 경보
    대응자동 차단임계치 초과 요청 IP·계정 자동 차단
    캐싱동일 요청 결과 캐싱으로 중복 연산 방지
  • 인사이트: Model DoS는 기존 네트워크 DoS와 달리 토큰·연산 자원을 표적으로 하여 서비스 마비뿐 아니라 막대한 API 비용을 유발하는 경제적 공격 수단으로 악용되므로, 토큰 예산·Rate Limiting·비용 모니터링의 다층 방어가 AI 서비스 운영의 필수 보안 요건임
