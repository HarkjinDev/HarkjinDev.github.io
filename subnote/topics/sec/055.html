3. OWASP (Open Web Application Security Project)

  • 정의: 웹 애플리케이션의 주요 보안 취약점을 발표하는 국제 오픈 보안 프로젝트
  • 키워드: 웹 보안 Top 10, 취약점 가이드
  • 분류: 시스템/SW 보안
  • OWASP Top 10 (2025): <접보공암/인불인무/로예>
    항목핵심대응 방안
    A01. 접근 제어 취약점 (Broken Access Control)권한 외 접근최소 권한 원칙, IDOR·SSRF 방어, 접근 제어 정책 서버 측 강제 적용
    A02. 보안 설정 오류 (Security Misconfiguration)설정 미흡기본 계정·불필요 기능 제거, 보안 설정 자동화·검증, 클라우드 설정 감사
    A03. 공급망 실패 (Software Supply Chain Failures)오픈소스 취약점SCA 도구 적용, CI/CD 파이프라인 무결성 검증, SBOM 관리
    A04. 암호화 실패 (Cryptographic Failures)민감 데이터 노출강력한 암호 알고리즘(AES-256) 적용, 키 관리 체계 수립, TLS 강제 적용
    A05. 인젝션 (Injection)악성 명령어 실행입력값 검증·이스케이프, PreparedStatement 사용, WAF 적용
    A06. 불안전한 설계 (Insecure Design)설계 단계 보안 누락위협 모델링(STRIDE), 보안 요구사항 설계 반영, SDL 프로세스 도입
    A07. 인증 실패 (Authentication Failures)인증 체계 취약MFA 적용, 강력한 패스워드 정책, 세션 타임아웃·무차별 대입 방어
    A08. 무결성 실패 (Software and Data Integrity Failures)무결성 검증 누락디지털 서명·체크섬 검증, 안전한 직렬화 라이브러리 사용, 업데이트 서명 검증
    A09. 로깅·모니터링 실패(Security Logging and Monitoring Failures)침해 탐지 불가중앙화 로그 수집(SIEM), 경보 임계치 설정, 로그 보존 정책 수립
    A10. 예외 처리 미흡 (Mishandling of Exceptional Conditions)오류 정보 노출에러 메시지 최소화, 예외 처리 표준화, 스택 트레이스 외부 노출 차단
  • 인사이트: 클라우드 네이티브 환경과 오픈소스 활용이 보편화되면서 공급망 보호가 최우선 과제로 부상, 인프라가 복잡해지면서 코드로 관리하는 인프라나 클라우드 설정 오류가 주요 공격 경로가 됨.
