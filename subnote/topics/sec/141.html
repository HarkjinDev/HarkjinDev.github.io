5. 가명처리 (Pseudonymization)

  • 정의: 개인 식별 정보를 대체하여 추가 정보 없이는 식별 불가하게 처리하는 기법
  • 키워드: 대체, 가명화, 재식별 가능, 추가 정보 분리
  • 분류: 개인정보보호
  • 법적근거: 개인정보보호법 제2조1의2
  • 가명처리 원칙: 가명 정보와 추가 정보(연결 테이블)를 분리 보관
  • 활용 범위: 통계 작성, 과학적 연구, 공익 목적 활용
  • 가명 처리 대상:
    대상설명사례
    직접식별자 (고유식별자)그 자체만으로 개인 식별 가능이름·사진·주민등록번호·전화번호·이메일·IP주소·차량번호판
    간접식별자 (준식별자)결합 시 특정 개인 식별 가능생년월일·사망일·결혼기념일·직업·성별·신용등급·주소·우편번호
    속성정보결합해도 식별 어려운 정보대출총액·예금총액·보험구좌수·고객고유번호
    특이정보 (민감정보)누구든지 쉽게 식별 가능특이질환자·초고소득자·초고령연령자·특이직업·이동동선
  • 가명 처리 주요 기법:
    구분항목설명
    삭제기술삭제원본정보에서 개인정보 단순 삭제 (부분삭제·로컬삭제·행항목 삭제)
    마스킹특정 항목 일부·전부를 공백 또는 문자로 대체
    통계도구총계처리평균값·최댓값·최솟값·최빈값·중간값 등으로 처리
    부분 총계오차 범위가 큰 항목을 평균값 등으로 대체
    일반화일반 라운딩올림·내림·반올림 기준 적용하여 집계 처리
    랜덤 라운딩수치 데이터를 임의 자리 수 기준으로 올림·내림 처리
    암호화양방향 암호화암호화·복호화 모두 가능 (대칭키·비대칭키)
    암호학적 해시함수단방향 암호화·복호화 불가 (MDC·MAC)
    동형암호화암호화 상태에서 연산 가능한 4세대 암호 기법
    다형성 암호화도메인별 서로 다른 가명처리로 부정 결합 차단
    무작위화잡음 추가개인정보에 임의의 숫자 등 잡음 추가
    치환열 항목 내 모든 값을 무작위로 순서 변경
    토큰화식별 가능 정보를 토큰으로 변환하여 식별 위험 제거
    PRNG예측 불가·패턴 없는 임의 숫자로 개인정보 대체
    기타 기술표본추출 (Sampling)전체 모집단 대신 무작위 표본 추출하여 분석 대체
    해부화 (Anatomization)데이터셋을 식별성 有·無 2개 집합으로 분리
    재현데이터 (Synthetic data)원본과 유사한 통계적 성질의 가상 데이터 생성
    동형비밀분산식별정보를 2개 이상의 쉐어(share)로 대체
    차분 프라이버시응답값에 잡음 추가로 특정 개인 존재 여부 은닉
  • 가명 처리 주요 알고리즘:
    알고리즘설명핵심
    k-익명화동일 속성값 k개 이상 유지·프라이버시 보호동일값 k개 유지
    부분그래프 익명화부분 그래프 프라이버시 노출 방지 그래프 수정그래프 구조 익명화
    차수 익명화k-차수 익명화 위해 간선 추가·삭제간선 조정 익명화
    부분그래프 + 차수 익명화부분 그래프·차수 동시 배경 지식 프라이버시 보호복합 배경지식 대응
    매크로 기법 (Suppression)민감 셀·주변 행열 값 동시 감추기셀값 감추기
    휴리스틱 익명화규칙·판단 기반 준식별자 값 가공·은닉규칙 기반 익명화
  • 가명 처리 평가 기법: <케익엘다티근>
    평가 기법취약점설명
    k-익명성연결 공격 (Linking Attack)준식별자 동일 레코드 K개 이상 존재 보장
    ℓ-다양성동질성 공격동일 집합 내 민감값 L개 이상 다양성 보장
    t-근접성분포도집합 내·전체 민감정보 분포 유사성 보장
  • 가명정보 처리 절차: <준위가적안>
    단계단계별 활동핵심
    ① 사전준비처리 목적 설정목적 선행 확정
    가명처리 대상 선정최소 범위 원칙
    활용 계획서 작성계획서 의무화
    내부규정 마련·검토내부통제 체계
    ② 위험성 검토기본 위험도 판단(신설, 26.3.)저·중·고 3단계
    사례별 위험도 조정최종 위험도 확정
    위험성 검토서 작성사후 점검 근거
    ③ 가명처리항목별 계획 수립목적 달성 최소화
    가명처리 수행결과서 의무화
    ④ 적정성 검토처리 결과 적정 검토저:담당자, 중:내부심의, 고:위원회
    ⑤ 안전한 관리재식별 가능성 모니터링재식별 방지
    가명정보 파기파기기록 보관
    활용 기간 연장재설정 허용
  • 가명정보 결합·반출 절차:
    절차활동설명
    결합신청결합 신청목적설정·결합가능정보 탐색·기관협의·신청
    결합 및 추가처리결합키 생성결합키관리기관과 협의·Salt값 수신
    모의결합(선택)일부 결합·분석으로 결합 유용성 사전 확인
    결합률 확인(선택)결합키·일련번호로 결합률 사전 확인
    가명정보 추출(선택)결합대상 정보 가명처리 후 추출·전송
    가명처리 및 검토결합전문기관이 데이터·가명처리 내역 검토
    결합결합키연계정보·일련번호·결합대상정보로 결합
    추가처리(필요시)보완 필요 시 추가 가명처리 수행
    반출 및 활용반출결합전문기관 승인 후 결과물·데이터셋 반출
    활용반출심사 시 제출 환경·목적범위 내 활용 원칙
    안전한 관리안전한 관리재식별 목적 처리 금지·기술적·관리적·물리적 조치
  • 인사이트: 가명처리 후 추가 정보가 노출되면 재식별이 가능하므로, 추가 정보는 별도 안전한 저장소에 보관하고 접근을 엄격히 통제해야 함
