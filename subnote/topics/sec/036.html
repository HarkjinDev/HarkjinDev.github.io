4. FIDO (Fast Identity Online)

  • 정의: 비밀번호 없는 생체인증 기반 온라인 인증 표준
  • 키워드: UAF, U2F, CTAP, WebAuthn
  • 분류: 인증/신원관리
  • 구성요소:
    • FIDO 서버: 인증장치에 대한 정책 설정, 사용자의 공개키 등록, 관리, 검증
    • FIDO 클라이언트: FIDO 서버의 정책에 따라 인증자를 필터링 하고 ASM과 RP 클라이언트 간의 중계 역할
    • ASM(Authenticator Specific Module): FIDO 클라이언트의 요청을 인증자로 전달하고, 인증자에서 생성된 응답 값을 FIDO 클라이언트로 전달하는 중계역할 수행
    • 인증자(Authenticator): 생체인증 등으로 사용자를 사용자 단말에서 로컬 인증하고 서버에서의 원격 인증을 위한 공개키/개인키 쌍을 생성하여 개인키를 이용해 전자서명을 수행
  • 주요기능:
    • 등록: 생성된 공개키를 검증서버에 등록하는 과정
    • 인증: 검증서버의 챌린지에 전자서명값을 생성 및 검증하는 과정
    • 거래확인: 인증과정과 동일하며 챌린지에 거래내역이 포함되어 전자서명 값을 생성 및 검증하는 과정
    • 등록 해지: 저장된 개인키를 삭제하여 등록을 해지하는 과정
  • 기술요소:
    • UAF(Universal Authentication Framework): 생체인증 기반 비밀번호 대체
    • U2F(Universal 2nd Factor): 보안키(USB/NFC) 기반 2차 인증
    • WebAuthn: W3C 표준, 브라우저에서 FIDO2 인증 지원
    • CTAP(Client to Authenticator Protocol): 인증기와 플랫폼 간 통신 프로토콜
  • 버전 비교:
    구분FIDO 1.0FIDO 2.0
    표준UAF + U2FCTAP + WebAuthn
    인증 방식생체인증(UAF), 2차 인증(U2F)패스키(Passkey) 지원
    적용 범위앱 기반웹 브라우저 표준 포함
  • 인사이트: FIDO2의 패스키(Passkey)는 비밀번호를 완전히 대체하는 차세대 인증으로, 구글·애플·마이크로소프트가 공동으로 채택하여 확산 중임
