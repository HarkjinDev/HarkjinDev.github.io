3. 프롬프트 인젝션 (Prompt Injection)

  • 정의: 악의적 프롬프트를 삽입하여 LLM의 동작을 조작하는 공격 기법
  • 키워드: 직접 인젝션, 간접 인젝션, 가드레일 우회
  • 분류: AI 보안
  • 유형:
    • 직접 인젝션: 사용자가 직접 시스템 프롬프트 덮어쓰기 시도
    • 간접 인젝션: 웹페이지·문서 등 외부 콘텐츠에 악성 명령 삽입 → LLM이 처리 시 실행
  • 프롬프트 인젝션 유형:
    구분유형설명
    직접 인젝션프롬프트 레이어링중첩 지시로 원래 시스템 프롬프트 무력화
    탈옥 (Jailbreak)역할극·가상 시나리오로 안전장치 우회
    시스템 프롬프트 추출내부 시스템 프롬프트 내용 노출 유도
    간접 인젝션웹 콘텐츠 인젝션웹페이지에 숨겨진 악성 지시 AI 에이전트 실행
    이메일 인젝션이메일 본문 숨겨진 지시로 에이전트 원격 조종
    RAG 데이터 오염지식베이스 문서에 악성 지시 삽입·검색 조작
    문서·파일 인젝션PDF·Word 파일 내 숨겨진 악성 지시 삽입
    에이전트 인젝션에이전트 목표 탈취외부 콘텐츠로 에이전트 목표·행동 경로 변조
    메모리·컨텍스트 오염에이전트 장기 메모리에 악성 지시 지속 삽입
    도구 호출 조작에이전트 도구 호출 과정에 악성 파라미터 주입
  • 대응 방안: 입력값 검증, 프롬프트 캡슐화·정규화, 최소 권한, 신뢰 경계 설정, 가드레일 LLM
  • 인사이트: 간접 프롬프트 인젝션은 LLM이 신뢰하는 외부 데이터 소스에 삽입되어 탐지가 어려우며, 에이전트 기반 AI 시스템에서 가장 위험한 공격 벡터임
