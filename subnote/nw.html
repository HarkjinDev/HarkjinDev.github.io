서브노트 — NW 네트워크

[Part 1]. 네트워크 기초

1. 정보이론 (Information Theory)

  • 정의: 정보의 정량적 측정과 효율적 전송 가능성을 수학적으로 연구하는 이론 체계
  • 키워드: 엔트로피, 정보량, 샤논, 채널 용량, 부호화, 잡음
  • 분류: 네트워크 기초
  • 핵심 개념:
    개념설명
    정보량(자기정보)I(x) = log₂(1/P(x)) = -log₂P(x) [비트]
    엔트로피(H)평균 정보량 H(X) = -Σ P(xᵢ)·log₂P(xᵢ)
    채널 용량(C)잡음 채널에서 오류 없이 전송 가능한 최대 속도
  • 샤논 3대 정리:
    • 제1정리(소스 코딩): 소스 엔트로피 이상의 부호화 필요 → 무손실 압축 이론적 한계
    • 제2정리(채널 코딩): 채널 용량 이하 전송률이면 오류 없는 전송 가능
    • 샤논-하틀리 정리: C = B·log₂(1 + S/N), B=대역폭, S/N=신호대잡음비
  • 인사이트: 정보이론은 압축(소스 코딩)과 오류 정정(채널 코딩)의 이론적 토대를 제공하며, 현대 통신·암호·머신러닝의 핵심 기반으로 시험에서 샤논-하틀리 공식과 엔트로피 식이 자주 출제됨.

2. 샤논의 정리 (Shannon’s Theorem)

  • 정의: 잡음 채널에서 오류 없는 전송의 최대 속도(채널 용량)를 수학적으로 정의하는 정리
  • 키워드: 채널 용량, 대역폭, SNR, 샤논-하틀리, 나이퀴스트 한계
  • 분류: 네트워크 기초
  • 핵심 공식:
    구분공식설명
    샤논-하틀리C = B·log₂(1 + S/N)잡음 채널 최대 전송률
    나이퀴스트C = 2B·log₂M무잡음 채널, M=신호 레벨 수
  • 의미: 아무리 강력한 부호화를 사용해도 채널 용량 C를 초과하는 오류 없는 전송은 불가능
  • 인사이트: 샤논 용량은 이론적 상한선으로, 실제 통신 시스템은 이 한계에 근접하도록 설계됨. MIMO, OFDM 등 5G/6G 기술은 S/N 개선과 대역폭 확대로 C를 극대화하는 방향으로 발전.

3. OSI 7 Layer (Open Systems Interconnection 7 Layer)

  • 정의: 이기종 네트워크 간 통신을 표준화하기 위해 ISO가 정의한 7계층 참조 모델
  • 키워드: ISO, 계층화, 캡슐화, PDU, 프로토콜, 인터페이스
  • 분류: 네트워크 기초
  • 계층 구조:
    계층명칭PDU주요 프로토콜장비
    7응용 (Application)데이터HTTP, FTP, SMTP, DNS웹서버, 방화벽(L7), WAF
    6표현 (Presentation)데이터SSL/TLS, JPEG, ASCII-
    5세션 (Session)데이터RPC, NetBIOS-
    4전송 (Transport)세그먼트TCP, UDP, SCTPL4 스위치, 로드밸런서
    3네트워크 (Network)패킷IP, ICMP, OSPF라우터, L3 스위치
    2데이터링크 (Data Link)프레임Ethernet, VLAN, HDLCL2 스위치, 브리지
    1물리 (Physical)비트RS-232, DSL, 광케이블허브, 리피터, 케이블
  • 캡슐화/역캡슐화:
    • 캡슐화: 송신 시 각 계층이 헤더를 추가
    • 역캡슐화: 수신 시 역순으로 헤더 제거
  • 인사이트: OSI 모델은 계층 독립성이 핵심으로, 각 계층은 인접 계층에만 서비스를 제공하므로 특정 계층 프로토콜 교체가 상위/하위 계층에 영향을 주지 않음. TCP/IP 4계층 모델과의 매핑 비교도 자주 출제.

4. ASN.1 (Abstract Syntax Notation One)

  • 정의: 통신 프로토콜에서 데이터 구조를 언어·플랫폼에 독립적으로 정의하는 추상 구문 표기 표준
  • 키워드: ISO/ITU-T, BER/DER/CER/PER, 직렬화, SNMP, X.509, 인코딩 규칙
  • 분류: 네트워크 기초
  • 인코딩 규칙 비교:
    규칙특징주요 용도
    BER(Basic Encoding Rules)가장 기본, 유연한 인코딩SNMP, LDAP
    DER(Distinguished Encoding Rules)BER 부분집합, 유일한 인코딩X.509 인증서
    CER(Canonical Encoding Rules)긴 데이터에 최적화보안 프로토콜
    PER(Packed Encoding Rules)비트 단위 최적화, 경량이동통신 프로토콜
  • 활용 분야: X.509 인증서, SNMP MIB, LTE/5G 무선 프로토콜, LDAP
  • 인사이트: ASN.1은 이종 시스템 간 데이터 호환성을 보장하는 핵심 표준으로, 보안 인증서(PKI)와 네트워크 관리(SNMP)에 직결되므로 SEC 도메인과 연계하여 정리할 것.

[Part 2]. 데이터링크/LAN

1. VLAN (Virtual LAN)

  • 정의: 물리적 위치에 무관하게 스위치 포트나 MAC 주소를 기준으로 논리적으로 네트워크를 분리하는 가상 LAN 기술
  • 키워드: IEEE 802.1Q, 트렁크 포트, 액세스 포트, 태깅, 브로드캐스트 도메인 분리
  • 분류: 데이터링크/LAN
  • VLAN 유형:
    • 포트 기반 VLAN: 스위치 포트에 VLAN ID 할당 (가장 일반적)
    • MAC 기반 VLAN: MAC 주소로 동적 할당
    • 프로토콜 기반 VLAN: 상위 프로토콜 타입으로 분류
  • 트렁크/액세스 포트:
    구분역할태그 처리
    액세스 포트단일 VLAN 연결(단말)태그 없음
    트렁크 포트복수 VLAN 전송(스위치 간)802.1Q 태그 포함
  • 인사이트: VLAN은 브로드캐스트 도메인 분리와 보안 구역 설정에 활용되며, 인터-VLAN 라우팅(L3 스위치 또는 라우터)이 필요함을 함께 언급해야 함.

2. VXLAN (Virtual eXtensible LAN)

  • 정의: UDP 헤더로 L2 이더넷 프레임을 캡슐화하여 L3 네트워크에서 대규모 가상 L2 네트워크를 구현하는 터널링 기술
  • 키워드: VNI, VTEP, UDP 4789, 오버레이 네트워크, 클라우드, 멀티테넌시
  • 분류: 데이터링크/LAN
  • VLAN vs VXLAN 비교:
    구분VLANVXLAN
    식별자12비트 VLAN ID (최대 4,096)24비트 VNI (최대 16,777,216)
    전송 방식L2 스위칭UDP 캡슐화(L3 전송)
    확장성제한적대규모 클라우드 환경 적합
    표준IEEE 802.1QRFC 7348
  • 동작 구성요소:
    • VTEP(VXLAN Tunnel End Point): 캡슐화/역캡슐화 수행
    • VNI: 가상 네트워크 식별자
  • 인사이트: VXLAN은 클라우드 데이터센터에서 테넌트 격리와 대규모 VM 마이그레이션을 가능하게 하는 핵심 기술로, SDN·NFV와 결합되어 현대 네트워크 가상화의 기반이 됨.

3. HDLC (High-level Data Link Control)

  • 정의: 슬라이딩 윈도우 기반 오류 제어를 수행하는 데이터링크 계층의 비트 지향 동기식 프로토콜
  • 키워드: ISO 13239, 플래그(01111110), 스터핑, NRM/ABM, 프레임 구조, WAN
  • 분류: 데이터링크/LAN
  • 프레임 구조:
    필드크기설명
    플래그8비트01111110 — 프레임 시작/끝
    주소가변수신지 주소
    제어8/16비트I/S/U 프레임 구분
    데이터가변상위 계층 데이터
    FCS16/32비트CRC 오류 검출
  • 동작 모드:
    • NRM: 정상 응답 모드, 주종 관계
    • ABM: 비동기 균형 모드, 피어 투 피어, 가장 널리 사용
  • 비트 스터핑: 플래그 패턴 방지를 위해 5개 연속 ‘1’ 뒤 ‘0’ 삽입
  • 인사이트: HDLC는 PPP, Frame Relay 등 WAN 프로토콜의 기반이 되는 프레임 구조로, 비트 스터핑과 ABM 모드의 특성을 답안에 명시하면 차별화됨.

[Part 3]. 전송 프로토콜

1. 소켓 통신 (Socket Communication)

  • 정의: IP 주소와 포트 번호를 결합한 소켓을 통해 프로세스 간 양방향 데이터를 교환하는 네트워크 통신 방식
  • 키워드: IP+Port, 소켓 API, BSD 소켓, 스트림 소켓, 데이터그램 소켓, 전이중
  • 분류: 전송 프로토콜
  • 소켓 유형 비교:
    구분스트림 소켓(TCP)데이터그램 소켓(UDP)웹소켓
    연결 방식연결 지향비연결HTTP 업그레이드 후 연결 유지
    신뢰성보장비보장보장(TCP 기반)
    주요 용도파일 전송, HTTP스트리밍, DNS실시간 웹 앱
  • 소켓 통신 흐름: //그림 표현 필요
    • 서버: socket()→bind()→listen()→accept()
    • 클라이언트: socket()→connect()
    • 통신: send()/recv()
    • 종료: close().
  • HTTP vs 웹소켓: HTTP는 요청-응답 단방향 반이중, 웹소켓은 핸드셰이크 후 전이중 유지
  • 인사이트: 소켓은 OS 커널이 제공하는 추상화 인터페이스로, 애플리케이션이 네트워크 세부 사항을 몰라도 통신 가능. 실시간 서비스에서는 WebSocket, IoT에서는 MQTT over TCP가 활용됨.

2. TCP 소켓 (TCP Socket)

  • 정의: 연결 지향 스트림 방식으로 3-way Handshake 연결 설정 후 신뢰성 있는 순서 보장 데이터 전송을 제공하는 소켓
  • 키워드: 연결 지향, 스트림, 신뢰성, 순서 보장, 흐름제어, 혼잡제어
  • 분류: 전송 프로토콜
  • TCP 소켓 특성:
    • 신뢰성: ACK 기반 재전송 보장
    • 순서 보장: 시퀀스 번호로 재조립
    • 흐름 제어: 슬라이딩 윈도우
    • 혼잡 제어: Slow Start, AIMD
  • 인사이트: TCP 소켓은 신뢰성-지연 트레이드오프 구조로, 지연에 민감한 실시간 미디어는 UDP/QUIC를 선택하며 HTTP/3는 TCP 대신 QUIC(UDP 기반)를 채택함.

3. 웹소켓 (WebSocket)

  • 정의: HTTP 프로토콜을 업그레이드하여 클라이언트-서버 간 단일 TCP 연결로 전이중 실시간 양방향 통신을 지원하는 프로토콜
  • 키워드: RFC 6455, HTTP Upgrade, ws/wss, 전이중, Upgrade 헤더, 실시간
  • 분류: 전송 프로토콜
  • 연결 수립 절차:
    1. 클라이언트 → HTTP GET + Upgrade: websocket 요청
    2. 서버 → 101 Switching Protocols 응답
    3. 이후 TCP 연결 유지, 프레임 단위 양방향 통신
  • HTTP Polling vs WebSocket 비교:
    구분HTTP PollingWebSocket
    연결 방식반복 요청-응답단일 연결 유지
    지연높음낮음
    오버헤드HTTP 헤더 매번최초 1회만
    용도단순 갱신채팅, 주식, 게임
  • 인사이트: 웹소켓은 HTTP와 호환되는 포트(80/443)를 사용하여 방화벽 우회가 용이하며, wss://는 TLS 기반 보안 웹소켓으로 도청 방지.

4. TCP 3-way Handshake

  • 정의: TCP 연결 설정 시 SYN→SYN-ACK→ACK 3단계 메시지 교환으로 양방향 연결을 수립하는 절차
  • 키워드: SYN, SYN-ACK, ACK, ISN, 연결 상태, LISTEN, SYN_SENT, ESTABLISHED
  • 분류: 전송 프로토콜
  • 절차: //그림 추가
  • 보안 취약점: SYN Flooding 공격 — 대량 SYN으로 서버 backlog 소진 → SYN Cookie로 대응
  • 인사이트: 3-way Handshake는 상호 ISN 교환을 통해 양방향 순서번호를 동기화하며, SYN Flooding DDoS 공격의 타깃이 됨. SYN Cookie는 메모리 없이 상태를 검증하는 핵심 대응 기법.

5. TCP 4-way Handshake

  • 정의: TCP 연결 종료 시 FIN→ACK→FIN→ACK 4단계 교환으로 양방향 연결을 정상 종료하는 절차
  • 키워드: FIN, ACK, TIME_WAIT, CLOSE_WAIT, FIN_WAIT, 반이중 종료
  • 분류: 전송 프로토콜
  • 절차: // 그림 추가
  • TIME_WAIT: 마지막 ACK 손실 대비 및 이전 세그먼트 소멸 대기 (2·MSL = 약 60~120초)
  • 인사이트: TIME_WAIT 상태는 서버 포트 재사용 문제를 야기할 수 있으며, SO_REUSEADDR 옵션으로 해소 가능. 3-way와 달리 FIN과 ACK가 분리되는 이유는 수동 종료측의 잔여 데이터 전송 때문.

6. 오류 제어 (Error Control)

  • 정의: 데이터 전송 중 발생하는 오류를 검출하고 재전송 또는 정정을 통해 신뢰성 있는 통신을 보장하는 기법
  • 키워드: 오류 검출, 오류 정정, ARQ, FEC, CRC, 해밍코드, 패리티
  • 분류: 전송 프로토콜
  • 오류 제어 분류:
    분류오류 검출오류 정정
    방식재전송 요청자체 정정
    기법BEC - ARQ(Stop-and-Wait, Go-Back-N, SR)FEC (해밍코드, 터보코드, LDPC)
    특징수신측이 오류 감지 후 재전송 요청수신측이 직접 오류 정정
  • 인사이트: FEC는 재전송이 불가능한 위성·방송 환경에서 필수이며, ARQ는 지상 유선망에서 효율적. 5G는 HARQ(Hybrid ARQ = ARQ + FEC 결합)를 채택하여 성능 최적화.

[하위] 6-1. ARQ (Automatic Repeat reQuest)

  • 정의: 데이터 전송 오류 발생 시 수신 측이 송신 측에 재전송을 자동으로 요청하여 신뢰성을 보장하는 오류 제어 방식
  • 키워드: Stop-and-Wait, Go-Back-N, Selective Repeat, NAK, 재전송, 시퀀스 번호
  • 분류: 전송 프로토콜

[하위] 6-2. HARQ (Hybrid Automatic Repeat reQuest)

  • 정의: ARQ의 재전송 기법과 FEC의 전진 오류 정정을 결합하여 오류 패킷을 재전송 시 이전 수신 신호와 합산·정정함으로써 전송 효율과 신뢰성을 동시에 향상시키는 오류 제어 기술
  • 키워드: ARQ, FEC, 소프트 컴바이닝, Chase Combining, IR, 터보코드, LDPC, ACK/NACK, 재전송
  • 분류: 네트워크 / 이동통신
  • 동작 원리:
    단계설명핵심
    1. 초기 전송송신측이 FEC 부호화된 패킷 전송부호화 전송
    2. 수신·복호수신측이 FEC로 오류 정정 시도자체 정정
    3. ACK·NACK정정 성공 시 ACK, 실패 시 NACK 전송결과 피드백
    4. 재전송NACK 수신 시 동일 또는 추가 패킷 재전송재전송 수행
    5. 소프트 컴바이닝이전 수신 신호와 재전송 신호를 합산하여 복호신호 합산 복호

    유형설명핵심

[하위] 6-3. FEC (Forward Error Correction, 전진 오류 정정)

  • 정의: 송신측이 데이터에 잉여 비트(Redundancy)를 추가하여 전송하고 수신측이 별도 재전송 요청 없이 스스로 오류를 검출·정정하는 채널 부호화 기술
  • 키워드: 채널 부호화, 잉여 비트, 부호율, 해밍 코드, 터보 코드, LDPC, Polar Code, BCH, 리드-솔로몬
  • 분류: 네트워크 / 통신 이론
  • 핵심 개념:
    항목설명핵심
    부호율 (Code Rate)정보 비트 수 / 전체 전송 비트 수 (낮을수록 강한 오류 정정)정정 능력 조절
    해밍 거리두 코드워드 간 다른 비트 수, 클수록 오류 정정 능력 향상정정 능력 기준
    잉여 비트 (Redundancy)오류 검출·정정을 위해 추가하는 패리티·체크 비트오류 정정 재료
    코드워드정보 비트 + 잉여 비트로 구성된 전송 단위전송 단위

  • 인사이트: FEC는 재전송 없이 오류를 자체 정정하므로 위성·방송·5G처럼 피드백 채널이 없거나 지연이 허용되지 않는 환경의 필수 기술임. 터보 코드→LDPC→Polar Code로 이어지는 발전은 Shannon 정보 이론의 한계에 점점 근접하는 과정이며, 5G에서 데이터 채널(LDPC)과 제어 채널(Polar)을 구분 적용하는 설계가 이론적 최적 성능 실현의 핵심임

[하위] 6-4. CRC (Cyclic Redundancy Check)

  • 정의: 데이터를 생성 다항식으로 나눈 나머지(체크섬)를 프레임에 부가하여 전송 오류를 검출하는 순환 중복 검사 기법
  • 키워드: 생성 다항식, 나머지, FCS, 코드워드, GF(2) 나눗셈, 버스트 오류
  • 분류: 전송 프로토콜
  • CRC 계산 절차:
    • ① 데이터 비트열 뒤에 생성 다항식 차수만큼 0 추가
    • ② GF(2)(XOR) 나눗셈 수행
    • ③ 나머지 = CRC 체크섬 → FCS 필드에 첨부
    • ④ 수신측: 동일 나눗셈 후 나머지 = 0이면 정상
  • 주요 표준:
    표준다항식적용
    CRC-16x¹⁶+x¹⁵+x²+1HDLC, USB
    CRC-3232차 다항식Ethernet, ZIP
    CRC-CCITTx¹⁶+x¹²+x⁵+1SDLC, X.25
  • 인사이트: CRC는 버스트 오류 검출에 탁월하지만 오류 정정은 불가. Ethernet 프레임의 FCS가 CRC-32를 사용하며, CRC는 검출만 가능하므로 오류 정정은 ARQ/FEC 필요.

7. 흐름 제어 (Flow Control)

  • 정의: 송신측과 수신측 간 처리 속도 차이로 인한 수신 버퍼 오버플로우를 방지하기 위해 송신 속도를 수신 능력에 맞게 동적으로 조절하는 전송 제어 기술
  • 키워드: 슬라이딩 윈도우, 수신 윈도우(rwnd), Stop-and-Wait, 버퍼 오버플로우, ACK, TCP, XON/XOFF, 어리석은 윈도우 증후군(payload(1byte) << header(40byte)), Clark’s(버퍼, MSS), Nalge(버퍼, 일괄전송)
  • 분류: 네트워크 / 전송 계층
  • 흐름 제어 vs 혼잡 제어 비교:
    항목흐름 제어혼잡 제어
    목적수신측 버퍼 오버플로우 방지네트워크 과부하 방지
    제어 대상송신자 ↔ 수신자송신자 ↔ 네트워크
    윈도우수신 윈도우 (rwnd)혼잡 윈도우 (cwnd)
    주체수신측 제어송신측 판단
    계층전송 계층 (L4)전송 계층 (L4)
  • 주요 기법:
    구분기법설명핵심
    정지 대기Stop-and-Wait패킷 1개 전송 후 ACK 수신까지 대기단순·비효율
    슬라이딩 윈도우Go-Back-N오류 이후 전체 윈도우 재전송윈도우 순차 재전송
    Selective Repeat오류 패킷만 선택 재전송선택 재전송
    버퍼 기반XON/XOFF버퍼 임계치 도달 시 XOFF 전송·중단 요청하드웨어 흐름 제어
    Credit 기반수신 가능 크레딧(버퍼 잔량)을 송신측에 통보크레딧 기반 조절
  • TCP 흐름 제어 동작:
    단계설명핵심

    2. 전송량 조절송신측이 rwnd 초과하지 않는 범위에서 데이터 전송전송량 제한
    3. ACK + 윈도우 갱신ACK 전송 시 최신 rwnd 값 함께 전달동적 조절
    4. Zero Window 처리rwnd=0 수신 시 Window Probe로 주기적 상태 확인교착 방지
    5. 윈도우 재개수신측 버퍼 여유 생기면 rwnd 갱신 ACK 전송전송 재개
  • Silly Window Syndrome (어리석은 윈도우 증후군):
    항목설명핵심
    개념소량 데이터를 반복 전송하여 헤더 오버헤드가 급증하는 현상비효율 전송
    수신측 대응Clark 해결책 — 윈도우가 충분히 커질 때까지 rwnd=0 유지수신 지연 공지
    송신측 대응Nagle 알고리즘 — 소량 데이터를 모아 한번에 전송데이터 묶음 전송
  • 인사이트: 흐름 제어는 수신측 버퍼 보호를 목적으로 하며, TCP에서 rwnd를 통한 동적 윈도우 조절이 핵심 메커니즘임. 혼잡 제어의 cwnd와 결합하여 실제 전송 크기는 min(rwnd, cwnd)로 결정되므로, 두 제어 메커니즘의 상호작용 이해가 TCP 성능 최적화의 핵심 포인트이며 Zero Window·Silly Window Syndrome이 대표적 시험 출제 포인트임

[하위] 7-1. 슬라이딩 윈도우 (Sliding Window)

  • 정의: 수신 측의 ACK를 기다리지 않고 윈도우 크기만큼 연속으로 프레임을 전송하여 채널 효율을 극대화하는 흐름 제어 기법
  • 키워드: 윈도우 크기, ACK, 흐름 제어, 버퍼, 수신 윈도우(rwnd), 송신 윈도우
  • 분류: 전송 프로토콜
  • 동작 원리:
    단계설명핵심
    1. 윈도우 설정수신측 rwnd 기반으로 송신 윈도우 크기 결정전송 허용 범위 설정
    2. 연속 전송ACK 대기 없이 윈도우 크기만큼 프레임 연속 전송연속 전송
    3. ACK 수신수신측이 정상 수신 확인 후 ACK + 최신 rwnd 전송수신 확인·갱신
    4. 윈도우 슬라이드ACK 수신 시 윈도우를 오른쪽으로 이동·새 프레임 전송 허용윈도우 이동
    5. 윈도우 축소수신 버퍼 감소 시 rwnd 축소·송신 속도 제한속도 조절
    6. Zero Windowrwnd=0 수신 시 전송 중단·Window Probe 주기 전송교착 방지
    7. 전송 재개수신 버퍼 여유 생기면 rwnd 갱신 ACK로 전송 재개전송 복구
  • 윈도우 크기 결정 요소:
    요소설명
    수신 윈도우(rwnd)수신 버퍼 여유 공간
    혼잡 윈도우(cwnd)네트워크 혼잡 기반 제한
    실효 윈도우min(rwnd, cwnd)
  • 인사이트: 슬라이딩 윈도우는 흐름 제어의 핵심 메커니즘이며, TCP에서는 수신 윈도우(rwnd)와 혼잡 윈도우(cwnd)의 최솟값으로 실효 윈도우를 결정. Zero Window Probe로 rwnd=0 교착 상태 해소.

8. 혼잡 제어 (Congestion Control)

  • 정의: 네트워크 혼잡 발생 시 전송률을 동적으로 조절하여 패킷 손실과 지연을 최소화하는 트래픽 관리 기법
  • 키워드: 혼잡 윈도우(cwnd), Slow Start, AIMD, 혼잡 회피, Fast Retransmit, ECN
  • 분류: 전송 프로토콜

    단계동작설명

1. IGP (Interior Gateway Protocol)

  • 정의: 단일 자율 시스템(AS) 내부에서 라우터 간 라우팅 정보를 교환하여 최적 경로를 결정하는 내부 게이트웨이 프로토콜
  • 키워드: AS(자율 시스템), 링크 상태, 거리 벡터, OSPF, RIP, EIGRP, 수렴
  • 분류: 라우팅/주소체계
  • IGP 분류:
    구분거리 벡터링크 상태하이브리드

  • 인사이트: IGP는 AS 내부 최적 경로 계산을 담당하며, 기업망에서는 OSPF가 표준. 수렴 속도와 확장성이 프로토콜 선택의 핵심 기준이며 OSPF는 링크 상태 변화 시 LSA(Link State Advertisement)로 빠르게 수렴.

2. EGP (Exterior Gateway Protocol)

  • 정의: 서로 다른 자율 시스템(AS) 간 라우팅 정보를 교환하여 인터넷 전체의 경로를 관리하는 외부 게이트웨이 프로토콜
  • 키워드: AS 간 라우팅, BGP-4, 정책 라우팅, AS 번호, AS-PATH, 피어링, eBGP, iBGP, NLRI
  • 분류: 라우팅 / 주소체계

3. OSPF (Open Shortest Path First)

  • 정의: 링크 상태 정보를 AS 내 모든 라우터에 플러딩하여 각 라우터가 SPF 알고리즘으로 최단 경로를 계산하는 링크 상태 기반 IGP 프로토콜
  • 키워드: RFC 2328, LSA, LSDB, SPF·Dijkstra, Area, DR·BDR, Cost, Hello, ABR, ASBR, OSPFv3
  • 분류: 라우팅 / 주소체계
  • 동작 절차:
    단계설명핵심
    1. 인접 관계 수립Hello 패킷으로 인접 라우터 발견·Neighbor 관계 형성Hello 프로토콜
    2. DR·BDR 선출멀티액세스 네트워크에서 대표 라우터 선출 * DR(지정 라우터), BDR(백업 DR)플러딩 최소화
    3. LSA 교환Link State Advertisement 교환으로 LSDB 동기화토폴로지 공유
    4. SPF 계산Dijkstra 알고리즘으로 최단 경로 트리 계산최단 경로 산출
    5. 라우팅 테이블 갱신계산된 최단 경로를 라우팅 테이블에 반영경로 적용

6. 슈퍼네팅 (Supernetting)

  • 키워드: CIDR, 경로 집약, 라우팅 테이블 축소, 애그리게이션, 연속된 네트워크, /N

    • → 슈퍼넷: 192.168.0.0/22 (4개 네트워크를 1개로 집약)
  • 서브네팅 vs 슈퍼네팅 비교:
    구분서브네팅슈퍼네팅
    방향큰 네트워크 → 작게 분할작은 네트워크 → 크게 결합
    마스크길어짐(비트 증가)짧아짐(비트 감소)
    목적주소 효율화, 트래픽 분리라우팅 테이블 축소
  • 인사이트: 슈퍼네팅(CIDR)은 BGP 경로 집약에 핵심적으로 적용되어 인터넷 라우팅 테이블 폭발을 방지함. 연속 네트워크 블록이 2의 거듭제곱 정렬이어야 집약 가능.

7. 서브네팅 (Subnetting)

  • 정의: 하나의 IP 네트워크를 서브넷 마스크를 이용하여 여러 개의 작은 서브넷으로 분할하는 IP 주소 관리 기법
  • 키워드: 서브넷 마스크, CIDR, 네트워크 주소, 브로드캐스트, VLSM, 호스트 범위
  • 분류: 라우팅/주소체계
  • 서브넷 분할 절차:
    1. 필요 서브넷 수 확인 → 빌려올 비트 수 결정 (2ⁿ ≥ 서브넷 수)
    2. 서브넷 마스크 계산 (기본 마스크 + n비트)
    3. 각 서브넷의 네트워크 주소, 브로드캐스트, 호스트 범위 도출
  • 계산 예시:
    구분값설명
    원본 네트워크192.168.1.0/24C클래스
    서브넷 마스크/26 (255.255.255.192)2비트 차용
    서브넷 수4개2²
    호스트 수62개/서브넷2⁶-2
  • VLSM(Variable Length Subnet Mask): 서브넷마다 다른 마스크 적용 → 주소 낭비 최소화
  • 인사이트: 서브네팅은 주소 효율화 + 브로드캐스트 도메인 분리가 핵심 목적이며, VLSM은 OSPF/EIGRP 같은 클래스리스 라우팅 프로토콜이 지원해야 적용 가능.

[Part 5]. QoS

1. QoS (Quality of Service)

  • 정의: 네트워크에서 트래픽 유형별로 대역폭·지연·손실을 차등 보장하여 서비스 품질을 관리하는 기술 체계
  • 키워드: 대역폭, 지연, 지터, 패킷 손실, IntServ, DiffServ, RSVP, 마킹, 큐잉
  • 분류: QoS
  • QoS 4대 요소:
    요소설명영향 서비스
    대역폭(Bandwidth)단위 시간 전송량고화질 스트리밍
    지연(Latency)패킷 전달 소요 시간VoIP, 게임
    지터(Jitter)지연 변동성실시간 미디어
    패킷 손실(Loss)패킷 폐기 비율모든 서비스
  • 인사이트: 현대 엔터프라이즈 망은 DiffServ를 기본으로 하되, 특정 핵심 경로에 RSVP 예약을 병행하는 하이브리드 방식을 채택. 5G QoS는 QFI(QoS Flow Indicator)로 슬라이스별 SLA를 보장.

2. WFQ (Weighted Fair Queuing)

  • 정의: 각 트래픽 흐름에 가중치를 부여하여 우선순위별로 공정하게 대역폭을 분배하는 스케줄링 기법
  • 키워드: 가중치, 공정 큐잉, 흐름 기반, CBWFQ, LLQ, 선점, 스케줄러
  • 분류: QoS
  • 큐잉 기법 비교:
    기법원리특징
    FIFO선착순간단, QoS 없음
    PQ(Priority Queuing)우선순위 절대 선점높은 큐 독점 위험
    WFQ가중치 비례 대역폭공정성 + 우선순위
    CBWFQ클래스 기반 WFQ정책 유연성
    LLQCBWFQ + 엄격 우선순위 큐VoIP 최적
  • WFQ 가중치: 낮은 IP Precedence → 높은 가중치(역비례), 저지연 흐름에 높은 우선순위
  • 인사이트: LLQ(Low Latency Queuing)는 VoIP 트래픽에 엄격한 우선순위를 부여하고 나머지는 CBWFQ로 처리하는 Cisco 권장 구성. WFQ는 흐름 수가 많을수록 CPU 부하가 증가.

3. DiffServ (Differentiated Services)

  • 정의: IP 패킷의 DSCP 필드를 이용하여 트래픽을 클래스별로 분류하고 차등적 포워딩 처리를 적용하는 QoS 모델
  • 키워드: RFC 2474, DSCP, PHB, EF/AF/BE, 토큰 버킷, 폴리싱, 쉐이핑, 에지 라우터
  • 분류: QoS
  • PHB(Per-Hop Behavior) 종류:
    PHBDSCP 값설명대상 트래픽
    EF(Expedited Forwarding)46최소 지연/손실 보장VoIP, 실시간
    AF(Assured Forwarding)12~444클래스×3드롭비즈니스 앱
    BE(Best Effort)0기본 처리일반 인터넷
  • DiffServ 동작: 에지 라우터에서 패킷 분류·마킹 → 코어 라우터는 DSCP 기반 PHB만 적용
  • DiffServ vs IntServ 비교:
    구분IntServDiffServ
    자원 예약RSVP per-flow없음(클래스 기반)
    확장성낮음(상태 유지)높음
    보장 수준강한 보장상대적 보장
    적용 범위소규모대규모 인터넷
  • 인사이트: DiffServ는 확장성이 높은 QoS 모델로 코어 라우터에서 per-flow 상태 유지가 불필요함. IntServ의 RSVP 시그널링 부하 없이 대규모 망에서 실용적으로 적용 가능.

4. IntServ (Integrated Services)

  • 정의: RSVP 프로토콜로 각 데이터 흐름에 필요한 네트워크 자원을 종단 간 사전 예약하여 엄격한 QoS를 보장하는 모델
  • 키워드: RFC 2210, RSVP, 자원 예약, per-flow, Guaranteed Service, CL Service, 상태 유지
  • 분류: QoS
  • IntServ 서비스 클래스:
    서비스설명대상
    Guaranteed Service최대 지연·대역폭 엄격 보장실시간 제어, 의료
    Controlled Load부하 없는 망 수준 보장멀티미디어
    Best Effort기본 전달일반 데이터
  • RSVP 동작: 수신측이 PATH 메시지 수신 → RESV 메시지로 역방향 경로 자원 예약 → 중간 라우터 상태 저장
  • 단점: 라우터당 per-flow 상태 유지 → 대규모 망 확장성 한계
  • 인사이트: IntServ는 엄격한 보장이 필요한 소규모 망(기업 내부, 실시간 제어망)에 적합하며, 인터넷 전체 적용은 DiffServ가 대안. 두 모델의 비교는 시험 단골 주제.

[Part 6]. 네트워크 가상화/SDN

1. NFV (Network Function Virtualization)

  • 정의: 방화벽·로드밸런서·IDS 등 전용 하드웨어 장비의 네트워크 기능을 범용 x86 서버에서 소프트웨어(VNF)로 구현하는 가상화 기술
  • 키워드: ETSI, VNF, NFVI, NFV-MANO, 오케스트레이션, 상용 서버, SDN 연계
  • 분류: 네트워크 가상화/SDN
  • NFV 구조:
    구성요소설명
    VNF(Virtual Network Function)소프트웨어로 구현된 네트워크 기능
    NFVI(NFV Infrastructure)컴퓨팅·스토리지·네트워크 자원 풀
    NFV-MANOVNF 생명주기 관리·오케스트레이션
    VNFM개별 VNF 관리
    NFVO(Orchestrator)서비스 체이닝, 자원 조율
  • 기존 방식 vs NFV 비교:
    구분전용 장비NFV
    확장성장비 추가 필요SW 배포로 즉시 확장
    비용고비용 전용 HW범용 서버 활용
    유연성낮음동적 서비스 체이닝
  • 인사이트: NFV와 SDN은 상호 보완 관계로, NFV는 기능 가상화, SDN은 제어 평면 분리를 담당. 5G 코어 네트워크는 NFV/SDN 기반 클라우드 네이티브 아키텍처로 구현.

2. SDN (Software Defined Networking)

  • 정의: 네트워크 장비의 제어 평면(Control Plane)을 데이터 평면(Data Plane)에서 분리하여 중앙 집중식 소프트웨어 컨트롤러로 네트워크를 프로그래밍 방식으로 제어하는 기술
  • 키워드: ONF, 제어 평면 분리, 오픈플로우, 컨트롤러, 사우스바운드/노스바운드 API, 프로그래밍
  • 분류: 네트워크 가상화/SDN
  • SDN 3계층 구조:
    계층구성요소인터페이스
    응용 계층네트워크 앱(방화벽, LB, 모니터링)노스바운드 API(REST)
    제어 계층SDN 컨트롤러(ONOS, OpenDaylight)사우스바운드 API
    인프라 계층네트워크 스위치/라우터(데이터 평면)OpenFlow
  • 기존 네트워크 vs SDN 비교:
    구분기존SDN
    제어 위치분산(장비별)중앙 집중(컨트롤러)
    설정 방식CLI, SNMPAPI, 프로그래밍
    유연성낮음높음
    가시성제한적전체 토폴로지
  • 인사이트: SDN의 핵심은 지능(제어)과 전달(포워딩)의 분리로, 네트워크를 소프트웨어로 추상화하여 자동화와 민첩성을 확보. 단일 컨트롤러 장애 시 SPOF(단일 장애점) 위험이 있으므로 컨트롤러 HA 구성 필수.

3. 오픈플로우 (OpenFlow)

  • 정의: SDN 컨트롤러와 네트워크 스위치의 데이터 평면 간 통신을 표준화한 사우스바운드 API 프로토콜
  • 키워드: ONF, 플로우 테이블, 매치-액션, 컨트롤 채널, TCP 6633/6653, 프로그래밍
  • 분류: 네트워크 가상화/SDN
  • 오픈플로우 동작:
    구성요소설명
    플로우 테이블매치 조건 + 액션(전달/폐기/수정) 저장
    매치 필드L2~L4 헤더(MAC, IP, Port 등)
    액션Output, Drop, Modify, Flood
    컨트롤 채널컨트롤러-스위치 간 TLS/TCP 연결
  • 패킷 처리: 플로우 테이블 매치 → 액션 수행 / 미매치 → Packet-In으로 컨트롤러에 문의
  • 인사이트: 오픈플로우는 SDN을 실용화한 최초의 표준 인터페이스로, 컨트롤러가 전체 네트워크 트래픽 정책을 중앙에서 프로그래밍 가능하게 함. P4 언어는 오픈플로우를 넘어 데이터 평면 자체를 프로그래밍하는 차세대 기술.

4. SDDC (Software-Defined Data Center)

  • 정의: 컴퓨팅(서버 가상화), 스토리지(SDS), 네트워킹(SDN)을 모두 소프트웨어로 정의하고 자동화하여 관리하는 차세대 데이터센터 패러다임
  • 키워드: 서버 가상화, SDS, SDN, NFV, 오케스트레이션, 자동화, 클라우드 인프라
  • 분류: 네트워크 가상화/SDN
  • SDDC 구성 요소:
    레이어기술설명
    컴퓨팅서버 가상화(VMware/KVM)vCPU, vRAM 동적 할당
    스토리지SDS(Ceph, VSAN)스토리지 풀 추상화
    네트워킹SDN + NFV가상 네트워크 구성
    관리오케스트레이션(OpenStack)통합 자동화 플랫폼
  • 인사이트: SDDC는 하이브리드 클라우드의 기반 인프라로, 온프레미스와 퍼블릭 클라우드를 동일한 소프트웨어 정의 방식으로 관리하여 일관된 운영 효율성을 제공.

[Part 7]. VPN

1. VPN (Virtual Private Network, 가상 사설망)

  • 정의: 공용 네트워크(인터넷) 위에 암호화된 터널을 생성하여 사설망과 동일한 수준의 보안·기밀성을 제공하는 가상 사설 네트워크 기술
  • 키워드: 터널링, 암호화, IPSec, SSL/TLS, L2TP, OpenVPN, 스플릿 터널링, 제로트러스트, Site-to-Site, Remote Access
  • 분류: VPN
  • VPN 유형:
    구분구성요소핵심 기술
    접속 방식Remote Access VPN개인 사용자·재택근무자 사설망 원격 접속
    Site-to-Site VPN거점 간 사설망 연결, 기업 지사·데이터센터 연결
    프로토콜IPSec VPN네트워크 계층 암호화, AH·ESP 프로토콜 사용
    SSL/TLS VPN애플리케이션 계층, 브라우저 기반 접속 용이
    L2TP/IPSecL2TP 터널링 + IPSec 암호화 결합
    OpenVPN오픈소스, SSL/TLS 기반 높은 호환성
  • 핵심 기술:
    구분구성요소핵심 기술
    터널링캡슐화원본 패킷을 암호화 후 새 패킷으로 감싸 전송
    터널 프로토콜IPSec·L2TP·GRE·WireGuard 등 터널 생성
    암호화데이터 암호화AES-256 대칭키 암호화로 데이터 기밀성 보장
    키 교환IKE·DH 알고리즘으로 세션키 안전 교환
    인증사용자 인증ID·PW·인증서·MFA 기반 접속자 신원 확인
    무결성 검증HMAC·SHA로 데이터 변조 여부 확인
  • IPSec 동작 모드:
    항목설명핵심
    전송 모드 (Transport)페이로드만 암호화, 원본 IP 헤더 유지단말 간 통신
    터널 모드 (Tunnel)전체 패킷 암호화 후 새 IP 헤더 추가게이트웨이 간 통신
  • 인사이트: VPN은 원격 접속 보안의 오랜 표준이나 내부망 묵시적 신뢰·측면 이동 위험으로 인해 Zero Trust Network Access(ZTNA)로 대체되는 전환기에 있으며, 특히 클라우드·하이브리드 환경에서 ZTNA·SDP와의 병행 운용이 현대 네트워크 보안 아키텍처의 실무 표준으로 자리잡고 있음

2. IPSec (Internet Protocol Security)

  • 정의: 네트워크 계층(L3)에서 IP 패킷의 인증·암호화·무결성을 보장하여 안전한 통신을 제공하는 표준 보안 프로토콜 스위트
  • 키워드: AH, ESP, IKE, SA, 전송 모드, 터널 모드, 암호화, 무결성, VPN 핵심 프로토콜
  • 분류: VPN
  • 핵심 구성 프로토콜:
    구분구성요소핵심 기술
    인증AH (Authentication Header)패킷 출처 인증·무결성 보장, 암호화 미제공
    암호화·인증ESP (Encapsulating Security Payload)페이로드 암호화·인증·무결성 동시 제공
    키 관리IKE (Internet Key Exchange)SA 협상·세션키 자동 교환·갱신
    SA (Security Association)통신 양단 간 보안 파라미터 협약 단방향 연결
  • 동작 모드:
    항목전송 모드 (Transport)터널 모드 (Tunnel)
    암호화 범위페이로드만 암호화전체 패킷 암호화
    IP 헤더원본 IP 헤더 유지새 IP 헤더 추가
    주요 용도단말 간 직접 통신게이트웨이 간 VPN
    적용 환경Host-to-HostSite-to-Site VPN
  • IKE 협상 단계:
    단계설명핵심
    IKE Phase 1안전한 채널 구성을 위한 IKE SA 협상·DH 키 교환보안 채널 수립
    IKE Phase 2IPSec SA 협상, 실제 데이터 암호화 파라미터 결정데이터 보호 협상
  • AH vs ESP 비교:
    항목AHESP
    암호화❌✅
    인증·무결성✅✅
    IP 헤더 보호✅ 포함❌ 미포함
    NAT 통과❌ 불가✅ NAT-T 지원
    실무 사용드묾✅ 주로 사용
  • 인사이트: IPSec은 VPN의 핵심 암호화 인프라로 네트워크 계층에서 투명하게 동작하는 강점이 있으나, NAT 환경에서의 AH 비호환·복잡한 설정 구조로 인해 경량·고성능의 WireGuard가 대안으로 부상하고 있으며 ESP+터널 모드 조합이 실무 표준으로 자리잡고 있음

3. SSL VPN (Secure Socket Layer VPN)

  • 정의: SSL/TLS 프로토콜 기반으로 웹 브라우저나 경량 클라이언트를 통해 원격에서 내부 네트워크 자원에 접근하는 응용 계층 VPN
  • 키워드: TLS, 443 포트, 클라이언트리스, 원격 접속, HTTPS, BYOD, 세션 기반
  • 분류: VPN
  • SSL VPN 접속 방식:
    방식설명특징
    클라이언트리스(Portal)웹 브라우저만 사용추가 SW 불필요
    씬 클라이언트경량 플러그인 설치특정 앱 터널링
    풀 터널VPN 클라이언트 설치전체 트래픽 터널링
  • 인사이트: SSL VPN은 방화벽 443 포트를 활용하여 어떤 환경에서도 접속 가능해 재택근무·BYOD 환경에 최적. 단, 브라우저 취약점이 공격 경로가 될 수 있으므로 MFA와 결합 필수.

4. Tor (The Onion Router)

  • 정의: 데이터를 여러 겹의 암호화 레이어로 감싸 3개 이상의 중계 노드를 경유함으로써 발신자의 신원을 은닉하는 익명 통신 네트워크
  • 키워드: 양파 라우팅, 회로, Guard/Middle/Exit 릴레이, Hidden Service, .onion, 익명성
  • 분류: VPN
  • Tor vs VPN 비교:
    구분VPNTor
    목적보안 터널 + 우회익명성 극대화
    신뢰 요구VPN 제공자 신뢰 필요신뢰 노드 불필요
    속도빠름느림(다중 홉)
    추적 가능성VPN 로그 존재 가능매우 어려움
    합법성대부분 합법국가별 차이
  • 익명성 한계: Exit 노드가 평문 트래픽 감청 가능, 트래픽 분석 공격(상관관계 분석)에 취약
  • 인사이트: Tor는 완전한 익명성을 제공하지 않으며, Exit 노드 공격과 타이밍 분석에 취약. .onion 주소의 Hidden Service는 서버와 클라이언트 모두 익명성 보장으로 다크웹에 활용.

[Part 8]. WAN/전송매체

1. 모바일 FSO 통신 (Free Space Optics)

  • 정의: 대기 중 레이저 빔을 이용하여 광섬유 없이 자유 공간에서 수 Gbps급 고속 광통신을 수행하는 무선 전송 기술
  • 키워드: 레이저, 적외선, 시야각(LOS), 기상 조건, 고속, 라스트 마일, 백홀
  • 분류: WAN/전송매체
  • 특징:
    항목내용
    전송 속도수 Gbps (최대 100 Gbps 이상)
    전송 거리통상 수백 m ~ 수 km
    주요 제약안개·비·연기 등 기상 조건, LOS 필요
    주요 용도빌딩 간 백홀, 재해복구, 라스트마일
  • 모바일 FSO: 이동 플랫폼(UAV, 위성)에 FSO를 적용하여 이동 중에도 고속 광링크 제공
  • 인사이트: FSO는 RF 간섭 없는 고보안 고속 통신이 필요한 군사·재해복구·빌딩 간 연결에 적합. 날씨 의존성이 최대 약점으로 RF 백업 채널과 병행 구성하는 하이브리드 방식이 실용적.

2. 프레임릴레이 (Frame Relay)

  • 정의: 고정 회선을 대체하여 가변 길이 프레임과 가상 회선(PVC/SVC)으로 WAN 구간을 연결하는 패킷 스위칭 프로토콜
  • 키워드: DLCI, PVC/SVC, CIR, FECN/BECN, LMI, 오류 정정 생략, 1990년대 WAN
  • 분류: WAN/전송매체
  • 주요 구성요소:
    요소설명
    DLCI(Data Link Connection Identifier)가상 회선 식별 번호
    PVC(Permanent Virtual Circuit)사전 설정 고정 가상 회선
    CIR(Committed Information Rate)보장 전송률
    FECN/BECN혼잡 알림 비트 (전방향/역방향)
  • 특징: 오류 검출만 수행(오류 정정은 상위 계층 위임), 낮은 지연, X.25 대비 단순화
  • 인사이트: 프레임릴레이는 1980~2000년대 WAN의 주류였으나 현재는 MPLS, SD-WAN으로 대체됨. DLCI와 PVC 개념은 MPLS 레이블 스위칭의 전신으로 이해할 수 있음.

[Part 9]. 이동통신/무선

1. 5G 특화망 (Private 5G Network)

  • 정의: 특정 기업·산업·구역에 전용 5G 네트워크 인프라를 구축하여 맞춤형 초저지연·고신뢰 서비스를 제공하는 지역 특화 이동통신 서비스
  • 키워드: URLLC, 국내 28GHz 대역, 산업 IoT, 스마트팩토리, 간섭 회피, 프라이빗 코어
  • 분류: 이동통신/무선
  • 안정성/신뢰성 확보 방안:
    방안설명
    전용 주파수 할당공공망과 간섭 방지
    네트워크 슬라이싱산업 서비스별 QoS 보장
    엣지 컴퓨팅 결합지연 최소화 (MEC)
    이중화 구성코어·기지국 N+1 이중화
  • 간섭 회피 방안: 빔포밍(지향성 안테나), 주파수 조정(FR2/FR1 선택), 전력 제어
  • 인사이트: 5G 특화망은 스마트팩토리·자율주행·원격의료 등 산업 디지털 전환의 핵심 인프라로, 공공 5G 대비 보안성과 SLA 보장이 강점. 국내는 4.7GHz/28GHz 대역에서 할당 운용.

2. SDR (Software Defined Radio)

  • 정의: 무선 통신의 변조·복조·주파수 처리 등 하드웨어로 구현되던 기능을 소프트웨어로 대체하여 유연하게 구현하는 기술
  • 키워드: ADC/DAC, FPGA, DSP, 재구성 가능, 인지 무선(Cognitive Radio), 다중 표준
  • 분류: 이동통신/무선
  • SDR 구성:
    구성요소역할
    안테나RF 신호 수신
    ADC/DAC아날로그-디지털 변환
    FPGA/DSP소프트웨어 기반 신호 처리
    SW 모듈변조/복조/프로토콜 처리
  • 장점: 단일 하드웨어로 다중 표준(LTE/5G/Wi-Fi) 지원, 펌웨어 업데이트로 기능 변경
  • 인지 무선(CR) 연계: SDR 기반으로 주파수 사용 환경을 감지하여 동적으로 주파수 선택
  • 인사이트: SDR은 주파수 유연성과 다중 표준 지원이 핵심으로, 군사통신·재난망·5G 기지국 가상화(vRAN)에 적용됨. GNU Radio 등 오픈소스 SDR 플랫폼이 연구·교육에 활용.

3. NOMA (Non-Orthogonal Multiple Access)

  • 정의: 동일 시간-주파수 자원에서 전력 수준의 차이를 이용해 다수 사용자를 동시에 다중 접속시키는 비직교 다중 접속 기술
  • 키워드: SIC(연속 간섭 제거), 전력 할당, 스펙트럼 효율, 5G/6G, 근거리/원거리 사용자
  • 분류: 이동통신/무선
  • NOMA vs OFDMA 비교:
    구분OFDMANOMA
    자원 분배직교(사용자마다 다른 주파수)비직교(동일 주파수 중첩)
    스펙트럼 효율상대적으로 낮음높음
    수신 복잡도낮음SIC로 높음
    공정성낮음(근거리 우선)높음(원거리 우선 전력 할당)
  • SIC(Successive Interference Cancellation): 강한 신호부터 순차 복조 후 제거하여 약한 신호 추출
  • 인사이트: NOMA는 셀 가장자리 사용자에게 더 높은 전력을 할당하여 공정성을 개선하며, 6G 후보 기술로 매시브 MIMO와 결합 시 스펙트럼 효율 극대화 가능.

4. C-V2X (Cellular Vehicle to Everything)

  • 정의: LTE/5G 셀룰러 네트워크 인프라를 기반으로 차량과 차량·인프라·보행자·네트워크 간 실시간 통신을 가능하게 하는 기술
  • 키워드: V2V/V2I/V2P/V2N, LTE-V2X, 5G NR-V2X, DSRC, PC5 인터페이스, 자율주행
  • 분류: 이동통신/무선
  • C-V2X 통신 유형:
    유형설명활용
    V2V(Vehicle to Vehicle)차량 간 직접 통신충돌 방지, 군집 주행
    V2I(Vehicle to Infrastructure)신호등·도로 인프라교통 최적화
    V2P(Vehicle to Pedestrian)보행자 안전충돌 경고
    V2N(Vehicle to Network)셀룰러 네트워크HD맵, OTA 업데이트
  • DSRC vs C-V2X: DSRC(IEEE 802.11p)는 전용 단거리 무선, C-V2X는 기존 셀룰러 인프라 활용으로 커버리지 우위
  • 인사이트: 5G NR-V2X는 1ms 이하 초저지연으로 레벨 4 이상 자율주행에 필요한 실시간 의사결정을 지원. V2X는 자율주행 인프라의 핵심 통신 기술로 국가 표준화가 진행 중.

5. WebRTC (Web Real-Time Communication)

  • 정의: 웹 브라우저에서 별도 플러그인·앱 설치 없이 JavaScript API만으로 음성·영상·데이터 실시간 통신을 지원하는 오픈 표준
  • 키워드: W3C/IETF, ICE, STUN, TURN, SDP, P2P, getUserMedia, DataChannel
  • 분류: 이동통신/무선
  • WebRTC 핵심 컴포넌트:
    컴포넌트역할
    getUserMedia마이크·카메라 접근
    RTCPeerConnectionP2P 미디어 스트림 연결
    RTCDataChannel임의 데이터 P2P 전송
    ICE/STUN/TURNNAT 통과 및 연결 설정
    SDP(Session Description Protocol)미디어 협상
  • NAT 통과: STUN(공인 IP 확인) → 실패 시 TURN(릴레이 서버) 경유
  • 인사이트: WebRTC는 브라우저 기반 화상회의·실시간 협업을 플러그인 없이 구현할 수 있어, Zoom·Google Meet 등의 기반 기술로 활용됨. DTLS-SRTP로 미디어 암호화가 의무화.

6. MODBUS (산업제어 통신 프로토콜)

  • 정의: 산업 자동화 환경에서 PLC·센서·RTU 등 장비 간 데이터를 교환하는 마스터-슬레이브 방식의 직렬 통신 프로토콜
  • 키워드: 마스터-슬레이브, RTU/ASCII/TCP, 코일, 레지스터, ICS/OT 보안, SCADA
  • 분류: 이동통신/무선
  • MODBUS 변형:
    변형전송 매체인코딩특징
    MODBUS RTURS-232/485이진고효율, 산업 표준
    MODBUS ASCIIRS-232/485ASCII가독성, 저효율
    MODBUS TCPEthernetTCP/IP현대 OT 네트워크
  • 보안 취약점: 인증·암호화 없음 → ICS/OT 보안에서 주요 위협 (Stuxnet 등 공격 사례)
  • 인사이트: MODBUS는 1979년 제정된 레거시 OT 프로토콜로 보안 기능이 전무하므로, ICS 환경에서는 네트워크 세그먼테이션·화이트리스트·암호화 게이트웨이로 보완 필수.

7. IEEE 802.11bn (Wi-Fi 8)

  • 정의: 협업 MIMO, 다중 AP 조정, 광대역 지원을 통해 실제 처리량과 채널 효율을 극대화하는 차세대 무선랜 표준 (예상 2028년)
  • 키워드: Wi-Fi 8, 협업 MIMO, 다중 AP 조정, 320MHz 채널, 멀티링크, 6GHz 대역
  • 분류: 이동통신/무선
  • Wi-Fi 세대 비교:
    세대표준최대 속도주파수주요 기술
    Wi-Fi 5802.11ac3.5 Gbps5MU-MIMO, 5GHz
    Wi-Fi 6/6E802.11ax9.6 Gbps2.4, 5, 6OFDMA, TWT, 6GHz
    Wi-Fi 7802.11be46 Gbps2.4, 5, 6320MHz, MLO, 4K QAM
    Wi-Fi 8802.11bn100+ Gbps2.4, 5, 6협업 MIMO, 다중 AP
  • Wi-Fi 7 기술요소:
    구분기술 요소설명

8. 5G 네트워크 슬라이싱 (Network Slicing)

  • 정의: 하나의 물리적 5G 네트워크 인프라를 NFV/SDN 기술로 다수의 독립된 가상 네트워크로 분리하여 서비스별 최적 QoS를 제공하는 기술
  • 키워드: NSI, NSSF, S-NSSAI, eMBB/URLLC/mMTC, E2E 슬라이싱, 자원 격리
  • 분류: 이동통신/무선

    URLLC초저지연(<1ms), 고신뢰자율주행, 원격 수술
    mMTC저전력, 대량 연결IoT 센서, 스마트미터
  • E2E 슬라이싱: RAN → 코어 네트워크 → 엣지 클라우드 전체 경로 가상화
  • 인사이트: 네트워크 슬라이싱은 하나의 인프라로 완전히 다른 SLA를 동시에 만족시키는 5G의 핵심 차별점. 슬라이스 간 자원 격리(security isolation)가 보안의 핵심 과제.

9. Ad-hoc 네트워크

  • 정의: 기지국이나 중앙 인프라 없이 이동 단말 간 자율적으로 다중 홉 무선 경로를 구성하는 인프라리스 이동 무선 네트워크
  • 키워드: 멀티홉, 자율 구성, MANET, 라우팅(AODV/DSR/DSDV), 토폴로지 변동, WSN

  • 경로 제어 방식:

10. IoT 프로토콜 (XMPP/CoAP/MQTT)

  • 정의: IoT 기기의 제한된 자원과 불안정한 네트워크 환경에서 효율적인 메시지 통신을 지원하는 경량 응용 계층 프로토콜들
  • 키워드: MQTT, CoAP, XMPP, Pub/Sub, REST, QoS, 브로커, 경량, M2M
  • 분류: 이동통신/무선

11. IPS (Indoor Positioning System)

  • 정의: GPS 신호가 도달하지 않는 실내 환경에서 Wi-Fi·BLE·초음파 등 다양한 측위 기술을 이용하여 위치를 결정하는 시스템
  • 키워드: 측위 기술, RSSI, ToA/TDoA, AoA, 핑거프린팅, BLE, UWB, Wi-Fi RTT

13. Itinerary Routing

  • 정의: 이동 에이전트가 미리 설정된 경로(Itinerary)를 따라 무선 센서 네트워크 노드를 순회하며 데이터를 수집하는 에이전트 기반 라우팅 기법
  • 키워드: 이동 에이전트, 순회 경로, WSN, 에너지 효율, 데이터 집계, 멀티홉 회피
  • 분류: 이동통신/무선
  • 특징:
    • 에이전트가 노드에서 노드로 이동하며 데이터 직접 수집
    • 멀티홉 전송 감소로 에너지 절약
    • Itinerary 순서 최적화(TSP 유사)로 경로 효율 극대화
  • 장단점:
    장점단점

    집계된 데이터 전송노드 장애 시 경로 재계획 필요
  • 인사이트: Itinerary Routing은 대규모 WSN에서 에너지 효율적인 데이터 수집이 필요한 환경에서 유효하며, 에이전트 이동 경로의 최적화가 성능의 핵심.

14. WLAN 실내 측위 (Indoor Positioning)

  • 정의: 무선 LAN(Wi-Fi) 전파의 신호 강도(RSSI), 도착 시간 등 전파 특성을 이용하여 실내 위치를 결정하는 기술
  • 키워드: RSSI, 삼각측량, 핑거프린팅, AP 인프라, Wi-Fi RTT, 802.11mc, 채널 지문
  • 분류: 이동통신/무선
  • WLAN 측위 방법 비교:

  • 전파 특성 활용: 다중 경로(Multipath), 감쇠(Attenuation), 반사(Reflection)를 역으로 위치 결정에 활용
  • 인사이트: 핑거프린팅은 환경 변화(가구 배치 등)에 민감하므로 정기적인 DB 갱신이 필요. 딥러닝 기반 CSI(Channel State Information) 지문 분석으로 측위 정확도를 크게 향상시키는 연구가 활발.

  • 키워드: IMT-2030, 테라헤르츠(THz), AI-Native, 디지털 트윈, 위성-지상 통합, 감지-통신 통합

2. SATIN (Satellite-Aerial-Terrestrial Integrated Network)

  • 정의: 위성(Satellite), 상공 플랫폼(Aerial/UAV), 지상 네트워크(Terrestrial)를 통합하여 전 지구적 연속 커버리지를 제공하는 3차원 통합 통신 네트워크
  • 키워드: LEO/MEO/GEO 위성, HAPS, UAV, NTN, 3차원 커버리지, 재난 통신
  • 분류: 차세대 네트워크

  • 인사이트: SATIN은 6G의 비지상 네트워크(NTN) 통합 핵심 개념으로, Starlink·OneWeb 같은 LEO 위성 군집과 5G/6G 기지국을 유기적으로 결합하여 어디서나 연결을 보장하는 방향으로 발전.

3. AI-Native Network (AI 네이티브 네트워크)

  • 키워드: 자율 최적화, 디지털 트윈, AI-RAN, 의도 기반 네트워킹, 폐루프 자동화, Zero-Touch
  • 분류: 차세대 네트워크

    AI 오케스트레이터자원 배치 및 제어전역적 자원 최적화 및 자가 복구 명령 하달

  • 인사이트: AI-Native는 6G의 핵심 아키텍처 패러다임으로, IBN과 결합하여 비즈니스 의도만 입력하면 AI가 전 계층을 자율 운영하는 완전 자율 네트워크를 목표로 함.

4. AI-RAN (AI-based Radio Access Network)

  • 정의: 무선접속망(RAN)에 AI/ML을 내재화하여 무선 자원 할당, 빔포밍, 간섭 관리, 셀 최적화를 실시간 자율 수행하는 차세대 RAN 기술
  • 키워드: O-RAN, xApp/rApp, Near-RT RIC, Non-RT RIC, 빔관리, 자원 스케줄링, 에너지 효율
  • 분류: 차세대 네트워크
  • AI-RAN/O-RAN 구성요소:
    구분구성요소설명

    Near-RT RIC10ms~1초 AI 추론으로 무선자원·빔·간섭 실시간 제어
    xAppNear-RT RIC 위 동작·빔포밍·간섭제어·스케줄링 자동화

  • 인사이트: AI-RAN은 O-RAN 오픈 인터페이스를 기반으로 다양한 벤더의 AI 앱을 RAN에 탑재하는 개방형 생태계를 형성. 에너지 효율과 주파수 효율을 동시에 극대화하는 6G RAN의 핵심 기술.

5. IBN (Intent-Based Networking)

  • 정의: 네트워크 관리자의 비즈니스 의도(Intent)를 자연어나 고수준 정책으로 입력받아 AI/ML이 자동으로 네트워크 구성을 적용하고 검증하는 기술
  • 키워드: 의도, 자동화, AI/ML, 정책 번역, 폐루프, 검증, Cisco DNA, 자율 네트워크
  • 분류: 차세대 네트워크
  • IBN 4단계 사이클:
    단계설명
    번역(Translation)비즈니스 의도 → 네트워크 정책 변환
    활성화(Activation)자동 구성 적용(SDN/API)
    인식(Awareness)네트워크 상태 실시간 수집
    보장(Assurance)의도 달성 여부 검증 및 자동 수정
  • 인사이트: IBN은 운영 자동화(NetOps)의 핵심 방향으로, 사람의 개입 없는 자가 치유 네트워크를 목표로 함. 6G AI-Native Network와 연계하여 자율 네트워크 생태계를 구성.

6. 자율 네트워크 (Autonomous Network)

  • 정의: AI·ML 기술을 기반으로 네트워크가 사람의 개입 없이 스스로 상태를 감시·분석·최적화·복구하는 자동화 네트워크 운영 패러다임
  • 키워드: 자율 운영, 자가 치유(Self-Healing), 자가 최적화, Intent-Based Networking, TM Forum(Level0~5), 자율화 레벨, 디지털 트윈, AIOps
  • 분류: 차세대 네트워크
  • 핵심 기능: <감최치구보>
    항목설명핵심
    자가 감시 (Self-Monitoring)네트워크 상태·트래픽·장애를 실시간 자율 감지실시간 탐지
    자가 최적화 (Self-Optimization)트래픽 부하·QoS에 따라 라우팅·자원 자율 조정성능 최적화
    자가 치유 (Self-Healing)장애 탐지 후 자동 복구·우회 경로 설정자동 복구
    자가 구성 (Self-Configuration)신규 장비·서비스 추가 시 자동 설정·배포자동 구성
    자가 보호 (Self-Protection)사이버 위협·이상 트래픽을 탐지하여 자동 차단자율 보안
  • 주요 기술:
    항목설명핵심
    IBN관리자의 의도(Intent)를 AI가 해석하여 네트워크 자동 설정의도 기반 제어
    AIOpsAI·ML로 네트워크 운영 데이터를 분석하여 장애 예측·자동 대응AI 기반 운영
    디지털 트윈물리 네트워크를 가상으로 복제하여 변경 사항 사전 시뮬레이션가상 검증
    강화학습네트워크 환경과 상호작용하며 최적 운영 정책 자율 학습정책 자율 학습
    이상 탐지비정상 트래픽·장애 패턴을 ML로 실시간 탐지실시간 감지
  • 정보 최신성(Information Freshness): 네트워크를 통해 전달·처리되는 데이터나 상태 정보가 실제 시스템·단말·서비스의 현재 상태를 얼마나 지연 없이 정확하게 반영하는지를 나타내는 특성
    • 차세대 네트워크에서 정보 최신성은 보통 AoI (Age of Information) 개념으로 측정
      • AoI(t) = t − u(t) [현재 시점과 최신 데이터 생성 시점 간 차이]
        • t : 현재 시간
        • u(t) : 마지막으로 수신한 정보 생성 시점
  • 인사이트: 자율 네트워크는 5G·6G의 초고밀도·초저지연 환경에서 사람이 수동으로 운영하는 것이 사실상 불가능해짐에 따라 필연적으로 부상하는 패러다임으로, TM Forum의 L5 완전 자율화를 목표로 통신사·장비 벤더 중심의 표준화가 빠르게 진행되고 있으며 IBN·디지털 트윈·AIOps의 통합 수준이 자율화 레벨을 결정하는 핵심 변수임
